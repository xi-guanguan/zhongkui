## Phase 1 — 骨架 (GLM5.1)

> 目标：跑通全部游戏逻辑，画面用**色块占位**即可（白色方块=铜钱，灰色圆=鬼，等等）。
> 验收标准：能玩完整个循环（投币→链判定→赢/输→没钱→打工→有钱→回来→买茶→buff 生效）。

### T1.1 项目骨架 + 状态机

**文件**: `config.js`, `state.js`, `main.js`

- [ ] 按重构策略 §二 创建文件结构（9个 JS + index.html）
- [ ] `config.js`: 搬运全部数值常量
  - `GT[]` 5种鬼参数（catchP, odds范围, midOdds）
  - `MILK_TEA[]` 10种奶茶（效果, 持续, 定价）
  - `FAVOR_LEVELS[]` 好感经验曲线（3/8/15/25）
  - `MINING` 打工参数（时长30s, 钩速, 摆幅）
  - `LY` 布局常量（320×480）
- [ ] `state.js`: GameState 响应式对象
  - stage 阶段机: `MENU → ROUND → PAT → RESULT → SHOP → MINING → SHOP_BACK`
  - 核心状态: coins, currentGhost, chains[], buffs{}, favor{level, exp}
  - ROI 滑动窗口（最近10局）
- [ ] `main.js`: requestAnimationFrame 主循环 + stage 分发

## Phase 2 — UI 美术 (Kimi2.6)

> 前提：Phase 1 全部逻辑已跑通。
> 目标：用 **fillRect 像素拼接 + PixelForge 风格 UI** 替换色块占位，达到参赛品质。
> 不引入任何外部图片资源，100% 程序化绘制。

### T2.1 像素角色绘制

**文件**: `renderer.js`

- [ ] 钟馗像素立绘（3~4色 fillRect 拼接，idle 呼吸微动）
- [ ] 5种鬼像素渲染（各3~4色，区分度高）
- [ ] 孟婆像素立绘（站在柜台后，简单 idle 动画）
- [ ] 钩链视觉（锁链段 + 钩头发光）

---

### T2.2 堆叠精灵图实现（全局应用）

**文件**: `renderer.js`（或独立 `sprite-stack.js`）

**核心渲染器**：
- [ ] `drawStackedSprite(ctx, layers, x, y)` 通用堆叠函数
- [ ] `prerenderRotations(layers, 8)` 8方向预渲染缓存
- [ ] 合成结果缓存到 OffscreenCanvas（不每帧重绘切片）

**角色堆叠**（参考美术文档 §14.5）：
- [ ] 钟馗（8~12层）：帽冠/头/躯干/腰带/腿 分段堆叠
- [ ] 5种鬼（4~8层）：每种鬼按体型高度切分，层间加 globalAlpha 渐变做飘渺感
- [ ] 左右翻转：角色只缓存朝右，朝左时 `ctx.scale(-1,1)` 翻转

**道具堆叠**：
- [ ] 铜钱（3~5层）：顶/底=方孔全貌，中间=边缘环，支持旋转（8方向缓存）
- [ ] 钩子头（3~4层）：金属厚度感
- [ ] 奶茶杯（4~6层）：杯底→杯身→杯口→盖，10种茶用同一套层形状+不同配色
- [ ] 锁链环（2~3层）：实时绘制（层少，跟随链段方向旋转）

**场景堆叠**：
- [ ] 孟婆柜台（5~8层）：木质截面，静态预渲染1次
- [ ] 灯笼（4~6层）：圆形从窄→宽→窄，带 lighter 叠光
