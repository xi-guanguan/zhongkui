## 一、从蒸馏文档提取的可迁移模式

### 1. 三级摘要方法论（节拍前夜）→ 模块路由

**原理**：L0 模块索引 → L1 文件索引 → L2 具体代码，每层做过滤器。

**迁移到我们项目**：
- 每个 JS 文件顶部写 1-2 行注释说明"它是什么"+"依赖谁"
- 主入口 `main.js` 只做胶水，不放逻辑
- `src/js/_summary.md` 作为 AI 协作索引（CodeBuddy 定位文件用）

### 2. GameJuice 六大技法（节拍前夜）→ 已有，需保留

**现状**：当前代码已经具备 5/6 大技法：
- ✅ 缓动函数（EASE 对象，8种）
- ✅ 屏幕震动（shake 系统）
- ✅ 粒子效果（ripples + parts 数组）
- ✅ Squash & Stretch（牛的形变动画）
- ✅ 微动画（呼吸、idle 摇摆）
- ⚠️ 发光效果（暂无，转钟馗后可加勾魂链发光）

**迁移决策**：这些系统拆分时必须完整保留，不能丢失任何 juice 细节。

### 3. 驾驭工程四支柱（安世界）→ AI 协作规范

| 支柱 | 我们怎么用 |
|------|-----------|
| 上下文工程 | 已有 CLAUDE.md + memory-index.md ✅ |
| 架构约束（≤15条） | **需要写**：`docs/ARCHITECTURE.md` 锁定模块边界 |
| 前置声明 | CodeBuddy 每次改代码前先发 EDIT_PLAN |
| 后置核对 | CodeBuddy 改完发 WORK_DIFF 摘要 |

### 4. 响应式状态管理（滑滑）→ 简化版 Store

**原理**：数据改了 → UI 自动更新，单向数据流。

**迁移到我们项目**：当前代码用全局变量（`st`, `sc`, `streak` 等）散落各处。重构后用一个集中的 `GameState` 对象 + 简单的观察者模式：

```javascript
// state.js — 极简响应式
const GameState = {
  stage: 'IDLE',
  score: 0,
  streak: 0,
  items: [],
  _listeners: {},
  set(key, val) {
    this[key] = val;
    (this._listeners[key] || []).forEach(fn => fn(val));
  },
  on(key, fn) {
    (this._listeners[key] = this._listeners[key] || []).push(fn);
  }
};
```

**好处**：
- 状态变化可追踪（调试友好）
- HUD 自动刷新（不用到处手动 `drawHUD()`）
- 后续接 AI 笑话系统时，状态驱动更自然

### 5. NanoVG 质感 UI 8层管线（浠涫）→ Canvas 2D 分层绘制

**原理**：L0外阴影 → L1背景渐变 → ... → L8微纹理，每层可开关。

**迁移到我们项目**：钟馗的 HUD/结算面板可以用类似分层：
```
L0: 外阴影（ctx.shadowBlur）
L1: 背景渐变（createLinearGradient）
L2: 内发光（二次 fillRect + 半透明）
L3: 描边（strokeRect）
L4: 纹理叠加（drawImage 小噪声图 + globalCompositeOperation）
```

比纯色矩形好看 100 倍，代码量增加不多。

### 6. 配置中心化（浠涫）→ 单一配置对象

**现状**：当前已有 `LY`（布局）、`CO`（颜色）、`CT`（牛类型）三个配置常量，做得不错。

**迁移决策**：保持这个模式，增加：
- `GT`（GhostTypes）替代 `CT`（CowTypes）
- `JK`（Jokes 笑话配置）
- `SK`（Skills 钟馗技能配置）

## 二、模块化拆分方案

### 文件结构

```
src/
├── index.html          # 只有 <canvas> + <script> 标签
├── js/
│   ├── _summary.md     # AI 协作索引（三级摘要 L1）
│   ├── config.js       # 配置中心：GT/CO/LY/JK/SK 常量
│   ├── state.js        # GameState 响应式状态 + 阶段机
│   ├── renderer.js     # Canvas 绘制基础层：背景、粒子、特效、HUD
│   ├── physics.js      # 钩链物理：绳索模拟、碰撞检测
│   ├── ghosts.js       # 鬼怪系统：类型、AI行为、动画
│   ├── zhongkui.js     # 钟馗控制：输入、动画、技能
│   ├── jokes.js        # 笑话系统：文案库、触发逻辑、AI接口
│   ├── audio.js        # Web Audio 音效：程序化合成
│   └── main.js         # 胶水：初始化 + requestAnimationFrame 主循环
├── css/
│   └── style.css       # 极简样式（canvas居中、背景色）
└── assets/
    └── ...             # 纹理、音效等
```

### 加载顺序（零配置，script 标签顺序即依赖）

```html
<script src="js/config.js"></script>
<script src="js/state.js"></script>
<script src="js/audio.js"></script>
<script src="js/renderer.js"></script>
<script src="js/physics.js"></script>
<script src="js/ghosts.js"></script>
<script src="js/zhongkui.js"></script>
<script src="js/jokes.js"></script>
<script src="js/main.js"></script>
```

**不用 ES Module 的理由**：
- 比赛环境可能有兼容性限制
- 全局变量模式对 AI 编辑更友好（grep 直接找到）
- 零工具链，复制即跑

## 三、核心改造点（从套牛 → 钩魂）

| 原系统 | 新系统 | 改动量 |
|--------|--------|--------|
| 5种牛 `CT[]` | 5种鬼 `GT[]` | 重写数据 + 绘制函数 |
| 套索物理 `rope()` | 钩链物理 `chain()` | 中等改造（弹簧→锁链段） |
| 牛仔跑动 | 钟馗飘行 | 重写角色动画 |
| 竞技场背景 | 冥界/街道场景 | 重写 `drawBg()` |
| 得分 HUD | 黑笑话 HUD | 中等改造 |
| DOOM Raycasting META | **删除** | -180行 |
| 道具商店 | 法器商店 | 换皮 + 调数值 |

### 删除清单（瘦身 ~250 行）

- DOOM Raycasting 全部（MAP, castRay, drawMeta, updateMeta）— 与核心玩法无关
- 牛的绘制细节（斑点、肌肉等牛特有视觉）

### 新增清单

- `jokes.js`：笑话系统（文案库 + AI 生成接口 + 触发时机）
- 钟馗像素角色绘制（替代牛仔）
- 鬼怪像素角色绘制（替代牛）
- 冥界视觉主题（配色方案全换）

## 四、架构约束（给 CodeBuddy 的 ≤15 条硬规则）

```markdown
1. 文件结构严格按 §二 的方案，不加新文件夹层级
2. 全局命名空间：每个 JS 文件暴露一个大写常量/对象（CONFIG, State, Renderer, ...）
3. 状态修改只通过 State.set()，禁止直接赋值全局变量
4. 绘制函数禁止修改状态（纯渲染，读 State 画 Canvas）
5. 物理/碰撞计算结果写入 State，不直接触发渲染
6. 主循环只在 main.js 的 requestAnimationFrame 里
7. 配置常量只在 config.js，游戏逻辑不出现魔法数字
8. 音效触发通过 Audio.play('sfxName')，不直接操作 AudioContext
9. Canvas 尺寸固定 320×480，CSS 缩放适配屏幕
10. 阶段切换通过 State.set('stage', 'XXX')，每个阶段有 enter/update/render 三函数
11. 笑话内容与游戏逻辑解耦：jokes.js 只提供 getJoke(ghostType) 接口
12. 粒子/特效生成归 renderer.js，不散落各处
13. 用户输入只在 zhongkui.js 处理，转化为意图（tap/swipe/hold）
14. 不用 class 语法（保持 ES5+ 兼容，对象字面量 + 函数）
15. 每个文件 < 300 行，超过就继续拆
```

## 五、重构路线图（阶段，非时间线）

### Phase 1: 骨架拆分
- 从 index.html 提取代码到 9 个 JS 文件
- 保持功能完全不变（还是套牛游戏）
- 验证：拆完能跑 = 成功

### Phase 2: 状态中心化
- 引入 GameState 对象
- 全局变量迁移到 State
- 阶段机重构为 enter/update/render 模式

### Phase 3: 换皮（牛 → 鬼）
- 替换 config.js 数据
- 重写角色绘制函数
- 调整配色方案

### Phase 4: 新增笑话系统
- 实现 jokes.js
- 接入腾讯云 AI 接口
- 设计触发时机和展示 UI

### Phase 5: 打磨
- 补 GameJuice（勾魂链发光、鬼怪消散粒子）
- 音效替换（阴森风格）
- 场景美术升级
