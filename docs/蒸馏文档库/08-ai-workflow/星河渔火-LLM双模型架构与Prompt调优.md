# LLM 双模型架构与 Prompt 调优

> **作者**: 星河渔火 | **难度**: L2-参考 | **来源**: TapTap 论坛 2026-05（二篇合并）

## 概述

本文合并了星河渔火关于 AI NPC 对话系统的两篇深度文章：
- **双模型架构**：让一个 AI 当嫌疑人，另一个 AI 当参谋
- **Prompt 调优**：如何让 AI 角色不崩人设

以《审讯室》项目为实战案例。

---

## 第一部分：双模型架构

### 单模型问题

让一个 LLM 同时做"角色扮演"+"出谋划策"，会导致身份混淆：

```
❌ 警官："你认识阿鬼吗？"
   嫌疑人回应正常，但"推荐话术"变成嫌疑人口吻：
   - "我只是听命行事"  ← 错！这是嫌疑人的话
   - "我需要律师"      ← 还是嫌疑人视角
```

**根因**：LLM 生成嫌疑人对话时进入第一人称状态，接着要切换到警官视角生成建议——上下文全是角色台词，它跳不出来。

### 解决方案：双模型分工

| 模型 | 职责 | System Prompt 核心 |
|------|------|-------------------|
| **模型A**（嫌疑人） | 纯角色扮演，生成对话 | 角色设定 + 当前感觉 |
| **模型B**（警官参谋） | 纯建议生成，警官视角 | 审讯顾问 + 策略规则 |

### 通信流程

```
玩家输入
→ 服务端做基础数值修正
→ 调用模型A（角色设定 + 对话历史 + 玩家输入）
→ A 返回：嫌疑人对话 + 姿态 + 情绪
→ 用 A 的返回文本调用模型B（案件背景 + A的回应 + 对话历史）
→ B 返回：3条推荐话术（纯警官视角）
→ 组合 A+B 结果发给客户端
```

**关键**：B 的输入是 A 的输出文本，B 只知道"嫌疑人说了什么"，不揣测内心。

### 模型B System Prompt 结构

```
你是一名资深审讯策略顾问。你正在旁观一场审讯。
嫌疑人：[角色简介]
弱点：[已知弱点]

你的任务：根据嫌疑人最新回应，为警官提供3个话术建议。

规则：
- 必须是警官可以直接说出的原话
- 必须贴合案情和当前心理状态
- 禁止以嫌疑人口吻（"我"）生成
- 第1条：直接回应追问
- 第2条：替代问法推进案情
- 第3条：保底推进话术
```

### 双模型收益

| 收益 | 说明 |
|------|------|
| A 表演质量提升 | 卸掉建议生成负担，专注角色 |
| 成本仅增 ~30% | B 输出短（每条≤30字），Token 消耗少 |
| 独立调优 | A 用高表演力模型(temperature=0.9)，B 用低成本快速模型(temperature=0.6) |
| 故障隔离 | A 失败不影响 B（B 基于兜底文本仍可生成建议） |

### 实施要点

1. **调用顺序**：必须先 A 后 B（B 需要 A 的输出）
2. **对话历史长度不同**：A 保留 6 轮（角色连贯性），B 只保留 3 轮（降低 Token）
3. **兜底文本**：A 失败 → "嫌疑人沉默不语"；B 失败 → 预设通用建议。不能用空字符串
4. **模型池共享**：A 和 B 共用一个模型列表 + 共用索引，额度耗尽自动跳下一个

## 第二部分：Prompt 调优——让角色不崩人设

### 典型翻车：演员念剧本

```
❌ 初版 Prompt：
"你是马克·威尔逊。性格：表面圆滑懦弱，内心狡猾。
 你的任务：扮演这个角色。"

❌ 实际输出：
"根据我的角色设定，我内心狡猾，所以我应该继续否认。"
```

### 三个致命错误

| 错误 | 原因 | 后果 |
|------|------|------|
| 性格标签清单 | LLM 逐条执行标签 | 机械切换，无统一人格 |
| 暴露"角色设定" | "你的任务是扮演" | LLM 偶尔跳出第一人称 |
| 缺少动机 | 为什么这样做？ | 空洞表演 |

### 调优演进：V1 → V3

**V1**（失败）：性格标签 + "你的任务是扮演"

**V2**（改善）：
- ✅ "你是"直接开头，去掉"你的任务是扮演"
- ✅ 用动机替代性格标签："为了活下去，你可以撒谎、装可怜"
- ✅ 加入具体情境

**V3**（最终版）：

```
你不再是 AI。你是马克·威尔逊，一名落入警方手里的黑帮会计。

【你的故事】
你知道 2 亿赃款的去向。这是你和警方博弈的唯一筹码。
船长今晚就要潜逃——如果你不说，他会带着钱消失。

【你的终极动机】
活下来。保护女儿。在夹缝中为自己争取最好的结果。

【当前感觉】
{由系统动态注入情绪描述}

【演绎铁律】
1. 说出每句话之前，先想：这句话对你有利还是有害？
2. 你可以害怕、愤怒、撒谎、沉默——但不能承认自己是 AI。
3. 你的性格不是固定的，它是你达到目的的工具。
```

### 五大调优原则

#### 原则 1：动机 > 性格

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| "你是懦弱的" | "你想活下去，所以你可以表现出懦弱" |
| "你爱女儿" | "你的女儿失踪了，你不知道她是否安全" |
| "你害怕船长" | "船长今晚就要跑了，如果你不说，他会抛弃你" |

> 性格是静态标签，动机是动态引擎。

#### 原则 2：用"感觉"沟通，别用"数值"

服务端数值（fear=65）→ 转换为情绪描述注入 Prompt：

```lua
-- 服务端转换函数
function GetFeeling(defense, fear, trust)
    -- fear=65 → "你感觉警方掌握的证据比你预想的多"
end
```

#### 原则 3：不要让 LLM 做数学题

- 数值变化 → 服务端计算
- 生成建议 → 交给模型 B

#### 原则 4：铁律约束底线，情境引导上限

- **铁律**：绝对不能做的事（跳出角色）
- **情境**：当前发生什么 + 你有什么感觉 → 自由发挥空间

#### 原则 5：给 LLM 一个"思维框架"

"说每句话之前先想是否对你有利" → 不是告诉它说什么，而是教它**怎么想**。

## 常见翻车急救

| 症状 | 原因 | 急救 |
|------|------|------|
| "根据我的角色设定…" | Prompt 有"你的任务是扮演" | 改为"你是XXX" |
| 数轮后说胡话 | 上下文窗口溢出 | 只保留最近 6 轮，其余摘要 |
| 出示证物无反应 | 证物权重不够 | user 消息用"警官出示了证物【XXX】" |
| 突然变正义 | 训练数据道德对齐 | 铁律加"你只忠于自己" |
| 回答千篇一律 | temperature 太低 | 调高到 0.8-0.9 |

---

## 总结

调优 Prompt 的本质：**把规则变成动机，把数值变成感觉，把性格标签变成活人的思维框架**。

一个好的角色 Prompt = 告诉演员：发生了什么、他想要什么、他现在感觉如何。剩下的，让他自己去演。
