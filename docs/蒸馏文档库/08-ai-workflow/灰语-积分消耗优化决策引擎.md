# 积分消耗优化决策引擎（Credit Saver）

> **这是什么**: 基于 80+ 实测数据点的 AI 工具调用成本优化策略集，在每次工具调用前自动选择最经济的方案。
> **原作者**: 灰语
> **推荐度**: L2（方法论参考，适用于所有 AI 辅助开发项目）
> **适用场景**: 需要控制 AI 积分消耗时；批量生成素材时；长会话效率下降时

---

## 核心成本模型

每轮成本 = **基线**（~15，随轮数增长） + **工具净成本**

输出 token 近乎免费；输入 token（系统提示 + 历史 + 工具返回）是主要成本。

## 工具净成本速查表

| 净成本 | 操作 | 备注 |
|--------|------|------|
| ~0 | 纯文本输出 | 长短几乎无差 |
| ~2-3 | Edit / 简单 Bash | 与文件大小无关 |
| ~6-8 | **batch 批量生成（3个）** | 性价比之王 |
| ~8-10 | Read 200行 / MCP Build | |
| ~15-20 | AI 单次生成（图片/音乐/音效） | |
| ~17-98 | Write（200行~2000行） | 固定开销 ~17 起 |
| ~30-50 | WebSearch | |
| ~170 | Task Agent | 且文件可能不持久化 |
| **342+** | **视频生成** | 最不可控 |

## 7 条决策规则（按优先级）

| # | 规则 | 核心 | 节省 |
|---|------|------|------|
| R1 | Edit > Write | 修改已有文件永远用 Edit | 80%+ |
| R2 | batch > 单次 | ≥2 个图片/音效必须用 batch | 80%+ |
| R3 | 并行 > 顺序 | 多个独立调用放同一轮 | 63% |
| R4 | 直接操作 > Task Agent | Agent 额外开销 ~50+，文件可能不持久 | 70%+ |
| R5 | 合并 Write | 1×Write 2000行 < 5×Write 共1130行 | - |
| R6 | 视频少轮询 | 提交后间隔 ≥3 分钟再 query | 每次省 24 |
| R7 | 长会话换新 | 基线超 ~100 时建议开新会话 | - |

## 快速决策流程

```
准备调用工具？
├─ 修改文件 → Edit（不是 Write）
├─ 生成 ≥2 图片 → batch_generate_images
├─ 生成 ≥2 音效 → batch_sound_effects
├─ 多个独立调用 → 并行执行
├─ 创建多文件 → 合并减少 Write 次数
├─ 视频 → 提交后等 3+ 分钟再查询
└─ 复杂任务 → 直接操作，避免 Task Agent
```
