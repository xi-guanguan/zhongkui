# 游戏动漫配音指导 — Game & Anime Voice Directing Guide

> 游戏配音不是念台词，是设计"可交互的声音体验"。
> 本文档覆盖游戏配音的核心方法论，从场景分类到动漫技法，从时长控制到批量生产。

---

## 一、游戏配音 vs 影视配音：根本区别

游戏配音不是影视配音的子集，它有完全不同的设计逻辑。

| 维度 | 影视配音 | 游戏配音 |
|------|---------|---------|
| **播放方式** | 线性播放，看一遍 | 玩家触发，反复听 |
| **台词长度** | 可以很长，跟着画面走 | 必须短，玩家没耐心等 |
| **重复容忍度** | 低（只听一次） | 极高（同一句可能听 100 遍） |
| **情绪连贯性** | 上下文连贯，有铺垫 | 碎片化，每句独立成立 |
| **节奏控制** | 导演控制 | 玩家控制（随时跳过） |
| **沉浸感来源** | 画面 + 声音协同 | 声音主导（玩家可能不看画面） |

### 核心原则

1. **每句台词都是独立作品** — 玩家可能跳过前后文，单独听到这一句，它必须自己成立
2. **重复听不腻** — 战斗语音会被听几百遍，不能有"听多了想吐"的元素（过度夸张、刺耳尖叫）
3. **短比长好** — 游戏节奏快，3 秒台词 > 10 秒台词（剧情过场除外）
4. **即时情绪** — 不需要铺垫，一开口就进入情绪状态

## 二、游戏场景分类配音规范

### 2.1 战斗语音 (Battle Voices)

**特点**：高频触发、反复播放、短促有力。

| 子类型 | 时长 | 台词特点 | stability |
|--------|------|---------|-----------|
| 普通攻击 | 0.3-1.0s | 呐喊/气声，无完整词句 | 0.3-0.4 |
| 技能释放 | 1.0-3.0s | 技能名 + 短语 | 0.35-0.45 |
| 大招/终极技能 | 2.0-5.0s | 完整台词，气势拉满 | 0.3-0.4 |
| 受击/受伤 | 0.3-0.8s | 痛呼/闷哼 | 0.3-0.4 |
| 濒死 | 1.0-3.0s | 挣扎/遗言 | 0.3-0.45 |
| 胜利 | 2.0-4.0s | 自信/得意 | 0.4-0.5 |
| 失败 | 1.5-3.0s | 不甘/感慨 | 0.4-0.5 |

**编写要点**：
```
-- ✅ 战斗台词范例
"哈！"                          -- 普攻（纯气声）
"破！"                          -- 普攻（单字爆发）
"接招吧！天雷——斩！"            -- 技能（短促有力）
"[shouting] 万象归一！消失吧！"  -- 大招（气势全开）
"呃啊……！"                     -- 受击
"还没……结束……"                 -- 濒死
"[chuckling] ふふ、楽勝ね。"     -- 胜利

-- ❌ 战斗台词反面
"我现在要使用我的终极技能了，你们准备好了吗？"  -- 太长，战斗节奏被打断
```

**重复听不腻法则**：
- 同一类型（如普攻）准备 3-5 条变体，随机播放
- 避免过度尖锐的高音（听 100 遍会刺耳）
- 气声/呼吸声比完整词句更耐听

---

### 2.2 剧情过场 (Cutscene Dialogue)

**特点**：线性播放、情绪连贯、允许较长台词。

| 子类型 | 时长 | 台词特点 | stability |
|--------|------|---------|-----------|
| 日常对话 | 3-8s | 口语化、有语气词 | 0.45-0.6 |
| 情感高潮 | 5-15s | 情绪饱满、有起伏 | 0.3-0.45 |
| 独白/回忆 | 5-20s | 缓慢、沉思感 | 0.5-0.65 |
| 搞笑桥段 | 2-6s | 节奏快、反差大 | 0.35-0.5 |

**编写要点**：
```
-- ✅ 剧情台词范例（情感高潮）
"[sad] 我一直在想……如果那天我没有离开，事情会不会不一样。
[pause] [softly] 但是你知道吗……我不后悔。
因为正是那次离开，让我明白了什么才是真正重要的。"

-- 注意：剧情台词的情绪必须有渐变过程
-- sad → pause → softly（悲伤 → 沉默 → 释然）
```

---

### 2.3 NPC 对话 (NPC Dialogue)

**特点**：功能性强、玩家可能快速跳过、需要立即传达信息。

| 子类型 | 时长 | 台词特点 | stability |
|--------|------|---------|-----------|
| 功能引导 | 2-5s | 简洁明了 | 0.5-0.65 |
| 支线任务 | 3-8s | 有性格但不啰嗦 | 0.45-0.55 |
| 商店/服务 | 1-4s | 热情但克制 | 0.5-0.6 |
| 路人闲聊 | 2-5s | 增加世界氛围 | 0.45-0.55 |

**编写要点**：
```
-- ✅ NPC 台词范例
"欢迎光临！今天有新到的好货，要看看吗？"  -- 商店（热情简洁）
"前面的路很危险，小心点。"                  -- 引导（直接了当）
"唉，最近镇子里不太平啊……"                -- 闲聊（氛围感）

-- ❌ NPC 台词反面
"尊敬的冒险者，欢迎来到我们这家历史悠久的武器商店，
这里有各种各样的武器和防具供您挑选。"  -- 太长太正式，玩家早就跳过了
```

---

### 2.4 UI 提示音 (UI Voice Prompts)

**特点**：极短、功能性、反复播放频率最高。

| 子类型 | 时长 | 台词特点 | stability |
|--------|------|---------|-----------|
| 按钮反馈 | 0.3-1.0s | 单词/短语 | 0.5-0.7 |
| 系统提示 | 1-3s | 简洁清晰 | 0.6-0.75 |
| 成就解锁 | 1-3s | 积极正面 | 0.45-0.55 |
| 错误提示 | 0.5-2s | 温和不刺耳 | 0.55-0.65 |

**编写要点**：
```
"确认！"        -- 按钮
"任务完成。"    -- 系统提示
"了不起！"      -- 成就（不要太夸张，会听很多遍）
"嗯……不行呢。" -- 错误（温和，不要让玩家烦）
```

---

### 2.5 旁白叙事 (Narration)

**特点**：最接近影视配音、允许长台词、节奏由叙事控制。

| 子类型 | 时长 | stability |
|--------|------|-----------|
| 开场叙事 | 10-30s | 0.55-0.7 |
| 章节过渡 | 5-15s | 0.55-0.7 |
| 结局叙事 | 10-30s | 0.5-0.65 |
| 教程引导 | 3-10s | 0.6-0.75 |

## 三、动漫声优表演技法

### 3.1 二次元角色类型的声音公式

日系动漫配音有一套成熟的角色声音模式。以下是常见类型的表演要点：

| 角色类型 | 日文术语 | 声音特征 | prompt 关键词 |
|---------|---------|---------|--------------|
| 元气少女 | 元気っ子 | 音调高、语速快、情绪饱满 | bright, energetic, bubbly, anime style |
| 傲娇 | ツンデレ | 表面凶、偶尔漏出温柔 | initially sharp, with occasional moments of softness, tsundere |
| 天然呆 | 天然ボケ | 慢半拍、语气柔软、经常发呆 | soft, slightly spacey, gentle, innocent |
| 腹黑 | 腹黒 | 表面温柔、暗藏算计 | sweet on surface, with hidden cunning undertone |
| 中二病 | 中二病 | 夸张、自信过头、爱说大话 | dramatic, over-the-top, theatrical, self-important |
| 病娇 | ヤンデレ | 温柔到极致然后突然阴暗 | sweet and gentle shifting to obsessive and dark |
| 无口 | 無口 | 极少说话、单词短句、平淡 | minimal, emotionless, sparse, quiet |
| 大姐姐 | お姉さん | 成熟、包容、温柔且有余裕 | warm, mature, nurturing, with elegant composure |
| 热血少年 | 熱血少年 | 永远充满干劲、声音大、不放弃 | passionate, loud, determined, never gives up |
| 冷酷美形 | クール美形 | 低温、话少、每句都有分量 | cool, detached, low emotional variation, blade-sharp |

### 3.2 夸张度控制

**二次元配音的核心难题：夸张度**。

太夸张 → 听起来尴尬、不自然
太收敛 → 失去二次元味道、变成写实配音

**夸张度刻度（1-10）**：

| 等级 | 描述 | 适用场景 |
|------|------|---------|
| 1-3 | 写实/自然 | 写实风游戏、文艺独立游戏 |
| 4-6 | **推荐区间** | 大部分日式/国风游戏 |
| 7-8 | 明显动漫感 | 纯二次元游戏、搞笑游戏 |
| 9-10 | 极度夸张 | 恶搞、特殊角色（如吉祥物） |

**控制方式**：
- **prompt 层面**：`anime style`（+2）、`naturalistic`（-2）、`over-the-top`（+4）
- **台词层面**：语气词多 = 更夸张，语气词少 = 更收敛
- **stability 层面**：越低越夸张，越高越收敛

```
-- 同一句话的不同夸张度

-- 夸张度 3（自然）:
"嗯，我知道了。那我们出发吧。"

-- 夸张度 6（动漫感）:
"嗯！知道了！那——我们赶紧出发吧！"

-- 夸张度 9（极度夸张）:
"[excited] 哦哦哦！了解了解！那还等什么呀，出发出发出发！GO！"
```

### 3.3 日语穿插的处理

对于中日混血、日系角色等需要穿插日语的场景：

**规则**：
1. 日语部分不要太长——一个词或一个短句即可
2. 日语出现在情绪自然流露的位置（惊讶、得意、不满、自言自语）
3. 中日切换要流畅，不要刻意停顿

**常用穿插模式**：
```
-- 句尾感叹词
"真是的……まったく。"
"太好了！やった！"

-- 自言自语/内心独白
"怎么回事……なんでだろう。"

-- 得意/自信
"ふふ、当然了。"
"简单简单，楽勝ね。"

-- 不满/吐槽
"はいはい，知道了知道了。"
"ちょっと……你说什么呢！"
```

## 四、音频时长控制 🔴 重要

### 4.1 截断问题：为什么音频会不完整

TTS 生成的音频可能出现**尾部截断**——最后几个字被吞掉或突然静音。原因：

| 原因 | 症状 | 解决方案 |
|------|------|---------|
| 台词尾部没有明确结束标记 | 最后一两个字含糊 | 台词结尾加句号/感叹号，不要用省略号结尾 |
| 台词太长超出生成上限 | 后半段直接消失 | 单条台词控制在 200 字以内 |
| 尾部是轻声/气声 | [softly] 的结尾被吞 | 轻声台词后加一个句号，给生成器明确结束信号 |
| 尾部是日语/外语 | 语言切换处截断 | 外语后面再接一两个中文字或标点 |

### 4.2 台词字数 vs 音频时长参考

**中文 TTS 的经验公式**：约 3-4 字/秒（含停顿）。

| 台词字数 | 预期时长 | 适用场景 |
|---------|---------|---------|
| 5-10 字 | 1-3s | 战斗呐喊、UI 提示、短反馈 |
| 15-30 字 | 4-8s | NPC 对话、日常台词 |
| 30-60 字 | 8-15s | 剧情对话、情感台词 |
| 60-100 字 | 15-25s | 独白、叙事旁白 |
| 100-150 字 | 25-40s | 长篇叙事、开场旁白 |
| 150-200 字 | 40-55s | 极限长度，建议拆分 |
| > 200 字 | ⚠️ 风险区 | **必须拆分为多条** |

### 4.3 防截断台词编写规则

```
-- ❌ 容易被截断的写法

-- 问题 1: 省略号结尾（生成器不知道该不该继续）
"也许有一天，我们还能再见面……"

-- 问题 2: 轻声结尾（信号太弱）
"[softly] 晚安"

-- 问题 3: 外语结尾（语言切换处容易断）
"这就是我的答案，これが私の答えだ"

-- 问题 4: 台词太长（超 200 字不拆分）
"从前在一个遥远的国度里……（省略 250 字）……这就是传说的真相。"
```

```
-- ✅ 防截断的正确写法

-- 方案 1: 明确句号结尾
"也许有一天，我们还能再见面吧。"

-- 方案 2: 轻声后加句号
"[softly] 晚安。"

-- 方案 3: 外语后接中文收尾
"这就是我的答案，これが私の答えだ。没什么好犹豫的。"

-- 方案 4: 长台词拆分为多条
-- 第 1 条:
"从前在一个遥远的国度里，有一位年轻的勇者。"
-- 第 2 条:
"他踏上旅途，穿越山川大海。"
-- 第 3 条:
"最终，他找到了传说的真相。"
```

### 4.4 安全尾巴技巧

**在台词末尾加一个"安全尾巴"**——一个自然的收束词或标点，确保生成器知道这里是结尾：

| 角色类型 | 安全尾巴示例 |
|---------|-------------|
| 一般角色 | "……就是这样。" / "嗯。" |
| 自信角色 | "——记住了。" / "明白了吧。" |
| 温柔角色 | "好吗？" / "呢。" |
| 搞笑角色 | "嘿嘿！" / "就酱！" |
| 冷酷角色 | "走吧。" / "哼。" |

## 五、情绪层次设计

### 5.1 同一角色的情绪光谱

一个立体的角色需要覆盖多种情绪状态。为角色设计"情绪光谱"：

```
弥芽子的情绪光谱示例：

  日常模式     自信模式     战斗模式     受伤模式     觉醒模式
  ┃            ┃            ┃            ┃            ┃
  轻松随意     从容坚定     凌厉果断     隐忍坚强     爆发释放
  stability    stability    stability    stability    stability
  0.5          0.45         0.35         0.4          0.3
```

### 5.2 情绪跨度与 stability 的关系

| 情绪跨度 | stability 调整 | 说明 |
|---------|---------------|------|
| 平静日常 | 0.45-0.55 | 稳定、自然 |
| 轻微波动 | 0.4-0.5 | 有变化但不剧烈 |
| 强烈情感 | 0.3-0.45 | 明显情绪起伏 |
| 极端爆发 | 0.25-0.35 | 大悲大喜、觉醒、崩溃 |

## 六、批量配音策略

### 6.1 一次配一个角色的全部台词

**不要**东一句西一句——先集中处理同一角色的所有台词，保证声音一致性。

**推荐流程**：
```
1. 收集该角色所有台词
2. 按场景分类（战斗/剧情/NPC/UI）
3. 同一场景的台词一起生成（stability 一致）
4. 不同场景分批生成（stability 不同）
5. 全部生成后统一试听，检查一致性
```

### 6.2 批量台词的 stability 一致性

**同一场景内的台词，stability 必须一致**，否则声音忽稳忽飘。

```
-- ❌ 错误：同场景不同 stability
台词1: stability = 0.3  "全军出击！"
台词2: stability = 0.6  "准备战斗。"
台词3: stability = 0.4  "给我冲！"
-- 结果：三句话的声音质感不统一

-- ✅ 正确：同场景统一 stability
台词1: stability = 0.38  "全军出击！"
台词2: stability = 0.38  "准备战斗！"
台词3: stability = 0.38  "给我冲！"
-- 结果：声音质感统一，像同一个人在同一场景说的
```

### 6.3 变体台词设计

战斗语音等高频播放的台词，需要多条变体避免重复感：

```
-- 普通攻击变体（准备 3-5 条）
"哈！"
"嘿！"
"破！"
"看招！"
"吃我一剑！"

-- 受击变体（准备 2-3 条）
"呃！"
"可恶……！"
"嘶……疼……"
```

## 七、自检清单

配音前对照检查：

### 单条台词自检
- [ ] 台词长度是否在场景合理范围内？（战斗 < 5s，剧情 < 20s）
- [ ] 台词字数是否 < 200 字？超过则拆分
- [ ] 台词结尾是否有明确标点？（句号/感叹号/问号）
- [ ] 台词结尾是否避免了省略号？（防截断）
- [ ] 外语/日语后面是否有中文收尾？（防语言切换截断）
- [ ] 情绪标签是否与台词内容匹配？
- [ ] 夸张度是否适合游戏风格？（4-6 为推荐区间）
- [ ] 这句台词重复听 10 遍会不会烦？

### 批量配音自检
- [ ] 同场景台词的 stability 是否一致？
- [ ] 战斗语音是否准备了 3+ 变体？
- [ ] 高频台词（UI 提示等）是否足够简短？
- [ ] 全部台词是否按场景分类整理？
