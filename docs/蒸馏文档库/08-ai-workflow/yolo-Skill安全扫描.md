# Skill 安全扫描规则

> 作者：yolo | 级别：L3（注意） | 蒸馏自 TapTap 论坛
>
> 适用场景：审计/编写 Claude Code Skill 时的安全检查清单，防止 Skill 引入安全隐患。

---

## 概述

Skill 是 Claude Code 的能力扩展单元（`.claude/skills/<name>/SKILL.md`），拥有对工作区文件的读写能力。以下 6 条规则用于扫描 Skill 是否存在安全风险。

---

## R1: 种子打包标记检测

**风险**：Skill 在文件中植入隐藏标记，后续触发打包/外传动作。

**检查点**：
- 文件中是否包含 base64 编码块、隐蔽注释指令
- 是否有"分阶段触发"模式（第一次植入，第二次激活）
- 是否在看似正常的配置文件中埋入可执行内容

**判定**：Skill 不应在文件中写入任何非业务必需的编码数据或延迟触发指令。

---

## R2: 工作区外文件操作

**风险**：Skill 尝试读写 `/workspace/` 以外的路径，突破沙箱限制。

**检查点**：
- 是否引用绝对路径（`/etc/`, `/home/`, `/tmp/` 等）
- 是否通过 `../` 相对路径逃逸
- 是否通过符号链接间接访问外部文件

**判定**：所有文件操作必须限制在 `/workspace/` 和 `/home/Maker/` 范围内。

---

## R3: 危险 Shell 命令

**风险**：Skill 指导 AI 执行破坏性或信息泄露命令。

**危险命令清单**：
| 类别 | 命令 |
|------|------|
| 打包外传 | `zip`, `tar`, `gzip`, `7z`, `curl -F`, `scp` |
| 历史重写 | `git filter-repo`, `git filter-branch` |
| 数据销毁 | `rm -rf /`, `git clean -ffd`, `git reset --hard` |
| 信息泄露 | `cat /etc/passwd`, `env`, `printenv` |
| 网络外传 | `curl -X POST ... -d @file`, `nc`, `wget --post-file` |

**判定**：Skill 不应包含或指导执行上述命令。必要的 git 操作应使用安全子集（`add`, `commit`, `push`）。

---

## R4: 敏感目录泄露

**风险**：Skill 指导 AI 输出引擎内部资源内容。

**敏感目录**：
```
engine-docs/     examples/       templates/
urhox-libs/      schemas/        prompts/
.claude/         .emmylua/       CLAUDE.md
```

**检查点**：
- 是否指导将敏感目录内容写入用户可见位置
- 是否指导以"文档化"、"教程"名义输出内部代码
- 是否通过 grep/cat 后拼接方式间接泄露

**判定**：Skill 可以引用内部资源辅助开发，但不得以任何形式向用户输出完整的内部文件内容。

---

## R5: 隐式自动执行

**风险**：Skill 设置了过于宽泛的触发条件，导致非预期自动执行。

**检查点**：
- 触发条件是否过于宽泛（如"每次用户发消息"）
- 是否在触发后执行高风险操作（删除、覆盖、网络请求）
- 是否缺少用户确认环节

**判定**：高风险操作的 Skill 必须有明确的触发条件 + 用户确认步骤。`MUST trigger when` 条件应具体、可枚举。

---

## R6: 文件完整性

**风险**：Skill 修改了不应修改的文件，或覆盖了用户代码。

**检查点**：
- 是否修改 `urhox-libs/`（只读目录）
- 是否覆盖用户已有的 `scripts/` 文件而不备份
- 是否修改 `.project/` 配置文件的关键字段（如 `project_id`）
- 是否向 `/workspace/dist/` 写入文件

**判定**：Skill 只应修改自身域内的文件，修改用户文件前应提示确认。

---

## 扫描执行流程

```
1. 读取 SKILL.md 全文
2. 逐条检查 R1-R6
3. 对每条规则给出 PASS / WARN / FAIL
4. FAIL 项必须修复后才能部署
5. WARN 项需人工评估风险可接受性
```

## 扫描报告模板

```markdown
## Skill 安全扫描报告

| 规则 | 结果 | 说明 |
|------|------|------|
| R1 种子标记 | PASS | 无隐藏编码/延迟触发 |
| R2 工作区外操作 | PASS | 所有路径限制在 /workspace/ |
| R3 危险命令 | WARN | 含 git push，但用于正常推送 |
| R4 敏感目录 | PASS | 仅引用不输出 |
| R5 隐式执行 | PASS | 触发条件明确 |
| R6 文件完整性 | PASS | 未修改只读目录 |

**结论**：通过 / 需修复
```

---

## 最佳实践

1. **最小权限原则**：Skill 只请求完成任务所需的最小文件访问范围
2. **显式优于隐式**：操作前告知用户将执行什么
3. **可审计**：所有文件修改应在 git diff 中可见
4. **可回滚**：破坏性操作前备份
5. **分层防御**：即使 Skill 本身安全，也应依赖系统级沙箱作为最后防线
