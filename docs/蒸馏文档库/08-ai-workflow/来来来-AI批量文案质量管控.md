# AI 批量文案质量管控：规范文档 + Prompt 策略 + 三阶段检查

> **这是什么**：用 AI（如 DeepSeek）批量生成游戏文案时的质量管控方法论。核心是"先把标准定清楚，再想办法执行标准"——通过一份文案规范文档 + 三阶段检查流程（写时约束 → 快速扫描 → 全面体检），将 AI 文案错误率从"一半有问题"降到"偶尔漏一两个"。不需要写代码，纯流程方案。
>
> **原作者**：来来来
> **推荐度**：L2（方法论通用且实战验证过，但模板需根据自己的项目填充具体规则才能使用）
> **适用场景**：任何需要大量文案的游戏（文字冒险、卡牌、AVG、RPG 等），尤其是用 AI 批量生产文案的团队或个人开发者

---

## 一、核心思路

**把"老编辑的经验"写成规则文档**，然后：

1. **给 AI 看**：作为 prompt 的一部分，让 AI 写的时候就遵守规范
2. **给自己看**：拿到文案后按规则逐条对照检查

没有规范，一切质检都是空谈。这是整套方案最重要的前提。

---

## 二、三阶段检查流程

### 第一阶段：写时约束（Prompt 红线）

在 prompt 末尾附一段"写作红线"，明确告诉 AI 哪些绝对不能干。

**效果**：不能 100% 防住，但能把错误率从"一半有问题"降到"偶尔漏一两个"。

**具体做法**：见下方"模板二：给 AI 的 Prompt"。

### 第二阶段：快速扫描（只抓硬伤）

AI 交稿后立即做一轮快速检查，**只看最致命的问题**：

| 检查项 | 方法 |
|--------|------|
| 选项文字是否超长 | 数字数 |
| 是否泄露了不该出现的关键词 | 全文搜索禁用词 |
| 是否违反世界观设定 | 搜索特定关键词（"新来的""刚入职"等） |

**返工关键技巧**：不要笼统说"这个不好改一下"，要具体指出哪里错、为什么错、怎么改。

```
❌ 错误返工指令：
"事件 2 有问题，改一下。"

✅ 正确返工指令：
"事件 2 的场景描述里出现了'情绪火山'这个词，这是 NPC 的类型名，
不能直接出现。请改用行为描写，比如'他突然把杯子摔在桌上'。"
```

精确的返工指令让 AI 一次改对，避免来回反复。

### 第三阶段：定期全面体检（攒一批集中查）

快速扫描抓不住的"软性问题"，攒一批后集中检查：

| 检查项 | 示例 |
|--------|------|
| 文案长度超标但不严重 | 场景描述写了 400 字，太长但不仔细看发现不了 |
| 数值不在合理范围 | 理智值 +3，但该阶段合理范围是 +6 到 +10 |
| 选项文字接近上限 | 23 字，没超 25 字硬限但小屏手机可能显示不佳 |
| 打破第四面墙 | 文案写了"这是最后一周了" |
| 多版本数值不同步 | 首次通关扣 10 点，二周目只扣 5 点 |

**可选进阶**：文案量大（100+）时，可写脚本自动化检查（关键词搜索、字数统计、数值范围校验）。

## 三、四个真实踩坑案例

### 坑 1：选项文字溢出

**现象**：AI 写了 29 字的选项文字，按钮最多放 25 字，发布后文字被截断。

**教训**：Prompt 里写明字数上限 + 拿到文案后必须数字数。

### 坑 2：AI 泄露答案

**现象**：游戏核心玩法是让玩家通过行为猜 NPC 类型，AI 在场景描述里直接写"这个秩序狂走过来说……"。

**教训**：Prompt 红线里明确列出所有不能出现的关键词（NPC 类型名等）。

### 坑 3：多版本数值不一致

**现象**：首次通关和二周目两套文案，改了一边的数值忘了另一边，玩家反馈"二周目反而更简单了"。

**教训**：改一边数值时**立刻**同步另一边，不要想着"回头再改"——你会忘的。

### 坑 4：AI 自己编设定

**现象**：没人告诉 AI 玩家角色的背景，AI 自己脑补了"作为刚入职的新人"，与实际设定矛盾。

**教训**：所有世界观设定都要在 prompt 里明确写出来，尤其是"什么不能写"。AI 不知道的事情，它会自己编。

## 四、拿走就用：完整模板

### 模板一：文案规范文档

保存为独立文件，每次让 AI 写文案时附上。将 `【】` 替换为你的项目信息。

```
=== 【你的游戏名】文案规范 ===

■ 基本信息
- 游戏类型：【如：文字冒险 / 卡牌 / AVG / 互动小说】
- 世界观一句话概括：【如：现代职场，玩家扮演 HR 观察奇葩同事】
- 玩家角色设定：【如：入职半年的普通员工，性别不限，性格冷静】

■ 文案结构
一条事件包含：
- 场景描述：【X】到【Y】字
- 选项：【N】个，每个选项文字不超过【Z】字
- 选项结果：每个【A】到【B】字

■ 硬性红线（绝对不能违反）
1. 【如：文案中不能出现 NPC 类型的名字】
2. 【如：选项文字不能超过 25 字】
3. 【如：不能出现"你是新来的"等暗示玩家身份的描述】
4. 【如：不能引用其他事件的内容（"你想起昨天……"）】
5. 【根据你的项目添加】

■ 软性建议（尽量遵守）
1. 【如：场景描述建议 80-200 字】
2. 【如：选项结果建议 30-100 字】
3. 【如：正确选项的奖励值建议在 +X 到 +Y 之间】
4. 【根据你的项目添加】

■ 禁用关键词（不能出现在文案正文中）
【列出所有不能出现的词，如 NPC 类型名、剧透关键词等】

■ 世界观约束（AI 不能自己编的设定）
- 【如：不能给玩家加背景故事】
- 【如：不能让 NPC 升职/离职/死亡（这些是系统控制的）】
- 【如：不能提到具体的时间节点（"第三周""最后一天"）】
```

### 模板二：给 AI 的 Prompt（生产文案用）

每次让 AI 写文案时，在具体需求前加上这段：

```
你是一个游戏文案写手。请严格按照以下规范撰写游戏事件文案。

【← 把上面的文案规范文档粘贴在这里】

=== 当前任务 ===
请为我撰写【N】条游戏事件，要求如下：
- 主题/场景：【如：办公室日常、团建活动、项目评审】
- 涉及的角色类型：【如：甩锅侠、秩序狂】
- 难度/阶段：【如：中期，玩家已经有一定经验】

=== 输出格式 ===
请按以下格式输出每条事件（保持格式统一，方便批量处理）：

事件 1：
- 标题：（简短标题，仅供内部标记）
- 场景描述：（正文）
- 选项 1：（选项文字）
  - 是否正确：是/否
  - 数值变化：+X 或 -X
  - 结果描述：（正文）
- 选项 2：（选项文字）
  - 是否正确：是/否
  - 数值变化：+X 或 -X
  - 结果描述：（正文）

=== 写作红线（违反任何一条都要返工）===
1. 文案正文中不得出现以下关键词：【你的禁用词列表】
2. 每个选项的文字不得超过【Z】字
3. 不得出现以下描述：【你的禁止描述列表】
4. 所有事件必须独立，不得引用其他事件的内容
```

### 模板三：返工指令

```
以下事件存在问题，请按要求修改：

事件【编号】（【标题】）：
- 问题：【具体描述，如"选项 2 文字超过 25 字"】
- 原文：【贴出原文】
- 要求：【具体修改要求，如"缩短到 25 字以内，保留原意"】

请只输出修改后的事件，格式和之前保持一致。
```

### 检查清单（每批文案拿到后过一遍）

```
□ 全文搜索禁用关键词 → 有就让 AI 改
□ 数每个选项的字数 → 超过上限就让 AI 缩短
□ 检查有没有违反世界观的描述 → 搜"新来""刚到""第一天"等关键词
□ 看数值是否在合理范围内 → 对照规范文档的数值表
□ 检查有没有引用其他事件 → 搜"昨天""上次""之前"等关键词
□ 如果有多版本，检查数值是否同步
```

## 五、总结：一句话方法论

**先把标准定清楚，再想办法执行标准。**

不管是用 AI 写文案还是自己写，不管是文字游戏还是 RPG 还是 AVG，只要游戏有大量文案，就值得花半天把规范写下来。写规范的过程本身就是在帮你理清思路。
