# 记忆文档与 Skill 全流程搭建指南

| 字段 | 值 |
|------|-----|
| 这是什么 | 大型 AI 协作项目的"跨会话记忆"完整方法论，包含三级文档架构、14 个 Skill 协作流程、文档编写原则、从零搭建步骤和 1 个月迭代经验 |
| 原作者 | 浠涫（节拍前夜 Beat Eve，42000+ 行 Lua / 81 文件 / 35 篇文档） |
| 推荐度 | L2（纯文档驱动方法论，非引擎代码） |
| 适用场景 | 长期迭代的大型项目，代码量超过数千行、需要跨会话保持上下文一致性的 AI 协作开发 |

---

## 核心问题

每次新会话 AI 都"失忆"——精心搭建的战斗系统被遗忘、修过的 BUG 再犯、核心规则被重新编造。

**解决方案**：把项目经验从人脑转移到文件，让 AI 每次开工前先"阅读简历"再上岗。

---

## 闭环工作流

```
开工前读文档(Skill自动) → 干活中写代码/修BUG → 收工后更新文档(Skill自动) → 下次开工继续读
```

---

## 三级文档架构

初期单一大文件导致文档膨胀、token 消耗过高，优化为三级轻量化架构：

| 级别 | 内容 | 行数 | 加载策略 |
|------|------|------|---------|
| Level 1 主文档 | 项目概况 + 文件结构索引 + 核心规则 + 游戏流程 | ~800 行 | 必读 |
| Level 2 架构子文档 | 架构模式 + 持久化层 + UI 系统 + 节奏系统 | ~900 行 | 按需 |
| Level 3 规范子文档 | 已知问题/TODO + 开发规范 + 迭代记录 + 路线图 | ~650 行 | 按需 |

**关键设计决策**：
1. 主文档严格控制在 1000 行以内，超量拆分至子文档
2. 文件结构索引是核心章节——每个文件一行标注文件名+行数+功能，方便 AI 定位
3. 精准记录核心数值规则，避免 AI 自行编造
4. 主文档添加导航标记，关联子文档，AI 按需读取

## 14 个 Skill 及协作关系

按开发阶段分布：

### 阶段 1：开工
- **记忆文档读取**：收到任务时自动读取主文档+按需子文档，杜绝失忆

### 阶段 2：理解需求
- **指令明晰度检查**：模糊指令时引导补充参数，减少返工
- **多任务完成验证**：2+ 个独立任务时逐项完成并生成验收报告

### 阶段 3：设计
- **任务前反思**：读完文档后预判风险、解析需求、构思方案
- **游戏设计原理框架**：新玩法/数值设计时调用专业理论

### 阶段 4：编码
- **API 可用性守卫**：三层校验，拦截不可用/废弃 API
- **Lua nil 空洞防护**：规避条件元素数组中的 nil 空洞
- **材质库指南**：提供预制材质与配置参考
- **NanoVG 分辨率模式**：高 DPI 屏幕 UI 适配引导
- **素材库放置规则**：生成素材后自动归类至指定目录

### 阶段 5：验收
- **开发验收 SOP**：编码→构建→预览→日志→功能五步验收

### 阶段 6：收工
- **记忆文档自动同步**：更新对应文档 + Git 提交 + 版本递增

### 全局辅助
- **Maker 协作指南**：多维度评估 + 流程指导
- **Skill 创建指南**：规范编写和触发逻辑

**核心闭环**：开工读取 + 收工更新，确保文档永远最新。

## 文档编写 5 原则

| 原则 | 说明 |
|------|------|
| 写"是什么"，不写"怎么做" | AI 可读代码，重点记录设计意图和数值规则 |
| 记录"教训"，不只是修复步骤 | 记录错误原因、规避方法、潜在风险 |
| 用表格记录枚举型信息 | 克制矩阵、意图映射等，表格比文本清晰度翻倍 |
| 标注版本号 + 更新日期 | 文档顶部标明版本与时间，快速定位过时信息 |
| 文件名直观易懂 | 拒绝冗余命名，一眼识别用途 |

## 从零搭建步骤

1. **创建主记忆文档**：`docs/` 下建主文档，写入项目信息、文件索引、核心规则，500-1000 行
2. **AI 生成初稿 + 人工修正**：投喂代码生成初稿，人工补充设计意图
3. **配置核心读写 Skill**：自动读取 + 自动同步两个 Skill
4. **逐步扩展 Skill**：前期基础读写闭环 → 中期质量保障 → 后期流程优化
5. **持续迭代文档**：新增/删除文件、更新规则、拆分超量主文档

---

## 1 个月迭代的经验教训

| 教训 | 说明 |
|------|------|
| 文档过长 = 浪费 token | 三级拆分后主文档 token 消耗大幅下降 |
| 格式适配 AI 解析 | 优先纯文本，用缩进/分隔线组织，避免复杂 Markdown 嵌套 |
| Skill 触发条件要精准 | 过宽浪费资源，过窄失去作用 |
| 自动更新是文档生命线 | 无自动更新 → 文档过时 → 基于错误信息开发 |
| BUG 经验库 ROI 最高 | 快速迭代中的 BUG 经验让 AI 举一反三，性价比最高 |
