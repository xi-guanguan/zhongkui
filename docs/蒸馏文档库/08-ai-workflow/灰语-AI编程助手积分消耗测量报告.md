# AI 编程助手积分消耗测量报告

> **元信息**
> - 作者：灰语
> - 来源：项目内部实验报告
> - 蒸馏等级：L1（实用方法论，可直接指导操作）
> - 主题：AI 编程助手（Claude Code）各类工具调用的积分成本测量与优化策略
> - 适用场景：所有使用积分制 AI 编程助手的开发者，控制开发成本

---

## 一句话总结

通过 80+ 数据点的控制变量实验，系统测量了 AI 编程助手中各类操作的积分消耗规律，得出核心结论：**输出 token 近乎免费、输入 token 是主要成本、并行调用和 batch 工具是最大优化杠杆**。

---

## 核心发现

### 1. 计费模型本质

| 计费维度 | 影响 | 说明 |
|---------|------|------|
| 输出 token（AI 生成的文本） | **≈0** | 900 字和 6 字回复积分几乎相同 |
| 输入 token（系统提示 + 历史 + 工具返回） | **主要成本** | 随对话轮数线性增长 |
| 基线成本 | 每轮 12-15 积分 | 与输出长度无关 |
| 历史累积 | 每 3-4 轮 +1 积分 | 隐性成本，长对话越来越贵 |

### 2. Edit 与文件大小无关

Edit 只传输 `old_string` / `new_string` 差异，不传输整个文件：

| Edit 操作 | 目标文件大小 | 积分 |
|-----------|-------------|------|
| +3 行 | ~90 行 | 21 |
| +3 行 | 2057 行 | 21 |

**结论：Edit 万行文件和十行文件一样便宜。**

### 3. 并行调用节省 63%

同一组操作（Read + Grep + Bash）：

| 方式 | 积分 | 节省 |
|------|------|------|
| 3× 并行（单轮） | 31 | 基准 |
| 3× 顺序（三轮） | 85 | — |

**原理**：顺序调用每轮重发完整历史（输入 token 重复计费）。

### 4. Batch 是性价比之王

| 工具 | 数量 | 积分 | 单次 ×3 理论 | 节省率 |
|------|------|------|-------------|--------|
| batch_generate_images | 3 张图 | 22 | 105+ | **79%** |
| batch_sound_effects | 3 个音效 | 21 | 111+ | **81%** |

Batch 调用比单次调用还便宜——LLM 只构造一次请求。

## 文件操作定价模型

### Write 定价

```
Write 积分 ≈ 17（固定开销）+ 行数 × 0.04
```

验证：17 + 2000 × 0.04 = 97 vs 实测 98，误差 < 2%。

### Read 定价

```
Read 积分 ≈ 3（固定开销）+ 行数 × 0.02
```

### Edit 定价

```
Edit 积分 ≈ 2-3（固定开销）+ 变更行数 × 少量增量
```

与目标文件大小无关。

## 视频生成成本拆解

视频是唯一的**异步 + 双层计费**工具：

```
总积分 = 41（提交）+ 24 × N（轮询等待）+ ~205（成功查询，含后端费 ~179）
```

实测 4 秒 480p 视频：41 + 24×4 + 205 = **342 积分**。

等价于：写 ~7000 行代码 / 生成 ~10 张图片 / 做 ~40 次 Edit。

## 长会话膨胀实测

| 会话阶段 | 轮数 | 典型操作积分 | 建议 |
|---------|------|-------------|------|
| 早期 | ~5 轮 | ~30 | 正常 |
| 中期 | ~15 轮 | ~72 | 留意 |
| 后期 | ~25 轮 | ~189 | 考虑换新 |
| 末期 | ~30 轮 | ~158（纯文本） | **必须换新** |

**判断标准**：单轮基线超过 100 积分时，开新会话。

## 特殊场景成本

### 上下文压缩恢复

压缩后首轮额外 34-45 积分，之后恢复正常。

### AI 生成类工具统一成本

图片/音乐/音效/配音均在 32-37 积分区间，与后端生成时间无关（0.5 秒音效 ≈ 64 秒音乐）。积分只计 LLM 调用开销。

### 中英文差异

Write 80 行：中文 20 积分 vs 英文 27 积分。中文信息密度更高。

## 黄金法则（11 条）

1. **能并行就并行** — 节省 63%
2. **能 Edit 就不 Write** — 节省 80%+，且与文件大小无关
3. **能直接操作就不用 Agent** — 节省 70%+，且更可靠
4. **合并文件减少 Write 次数** — 每次 Write 固定 17 积分开销
5. **Read 大文件放心读** — 上下文膨胀效应极弱（1000 行内 ±1 积分）
6. **长会话及时换新** — 单轮基线超 100 积分时果断开新会话
7. **视频生成要谨慎** — 342+ 积分一个 4 秒视频，少轮询多等待
8. **AI 生成工具性价比高** — 图片/音乐/音效/配音统一 32-37 积分
9. **压缩恢复有一次性开销** — 首轮额外 34-45 积分，之后正常
10. **能 batch 绝不单次** — batch 3 个素材（21-22 积分）比单次 1 个还便宜，节省 80%+
11. **search_3d_resource 放心用** — 搜索 + 下载仅 ~26 积分
