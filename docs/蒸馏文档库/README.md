## 命名规范

```
[作者]-[主题描述].md
```

- 作者名取论坛 ID 或笔名
- 主题描述用中文，简明扼要
- 子目录用于有多文件依赖的复杂 Skill（如 CV 配音专家）

## 贡献规范

### Commit Message

```
<type>: <简要描述>
```

| type | 用途 |
|------|------|
| `feat` | 新增蒸馏文档 |
| `refactor` | 重分类、重组织 |
| `fix` | 修正内容错误 |
| `docs` | README / 索引更新 |
| `chore` | 配置、工具链 |

描述不超过 50 字，中文，不加句号。

### 禁止

- 省略号、语气词、emoji
- 无意义细节（"调整了文件"）
- 英文 commit message（本仓库统一中文）

## 蒸馏工作流

详见根目录 `蒸馏工作流(meta).md`。

两阶段流程：
1. **批量导入** — 原始素材转为 Skill 格式 Markdown，平铺保存
2. **分类整理** — 同主题文档归入子目录，补充对比分析段
