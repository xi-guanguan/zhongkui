# 滑动列表后松手误触发点击事件修复

> **作者**: 汪美人 | **级别**: L3-注意 | **标签**: `触摸` `滚动` `误触` `已修复`
>
> ⚠️ **此问题已被引擎官方修复**，记录供了解原理。

---

## 问题描述

手指在列表上滑动滚动后松手，会误触发列表项的点击（onClick）事件。

**根因**：HandlePointerUp 只检查 `widget == pressedWidget`，不检查手指位移量。

---

## 原解决方案：TouchGuard 模块

```lua
-- TouchGuard.lua 核心逻辑
local startX, startY = 0, 0
local THRESHOLD = 12  -- 像素

function TouchGuard.OnTouchBegin(x, y)
    startX, startY = x, y
end

function TouchGuard.OnTouchMove(x, y)
    local dx = math.abs(x - startX)
    local dy = math.abs(y - startY)
    if dx > THRESHOLD or dy > THRESHOLD then
        scrolling = true
    end
end

function TouchGuard.IsScrolling()
    return scrolling
end

-- onClick 中检查
onClick = function(self)
    if TouchGuard.IsScrolling() then return end
    -- 正常点击逻辑...
end
```

---

## 现状

引擎已内部修复此问题，ScrollView 组件在滑动后自动屏蔽子元素的点击事件。无需手动添加 TouchGuard。
