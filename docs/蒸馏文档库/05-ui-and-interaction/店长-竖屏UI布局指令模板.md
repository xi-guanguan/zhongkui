## 一、方法论：如何描述 UI 布局

当你需要让 AI 按设计稿实现 UI 时，采用以下标准化格式描述每个元素：

### 元素描述模板

```
[序号]. 创建[元素类型] [素材文件名(如有)]
    位置: X[水平坐标] Y[垂直坐标]
    尺寸: [宽]×[高]
    样式: [颜色/圆角/不透明度/描边等]
    内容: [文本内容/格式说明]
    对齐: [居中/居左/居右]
    层级: [叠加/覆盖说明]
```

### 示例

```
1. 创建头像背景框 矩形
   位置: X239 Y139
   尺寸: 382×136
   样式: 纯黑, 不透明度40%, 圆角36px

2. 创建金币图标 icon_coin.png
   位置: X653 Y100
   尺寸: 73×73

3. 创建金币数值文本
   位置: X轴距资源栏左侧44px, Y轴与背景居中
   样式: 字号33, 纯白, 黑色描边4px
```

## 三、顶部信息区模板

常见元素及排布方式：

### 玩家信息组（左侧）

```
1. 头像背景框
   - 矩形，圆角，半透明黑底
   - 包含：头像图、等级标签

2. 玩家名称
   - 与头像右侧对齐
   - 字号30, 纯白, 黑色描边

3. 经验条
   - 背景图 + 进度条图
   - 进度条在背景中，内边距4px

4. 战力/属性文本
   - 格式: "战力:XXXX"
   - 高亮色（如 #f7fe77）
```

### 资源栏组（右侧）

```
每个资源栏 = 图标 + 背景框 + 数值文本

1. 资源背景框
   - 矩形，圆角18，黑底60%透明
   - 宽170 高47

2. 资源图标
   - 正方形，略大于背景框高度
   - 位于背景框左侧偏外

3. 资源数值
   - X轴距背景框左侧44px
   - Y轴与背景居中
   - 字号33, 纯白, 黑色描边
```

## 四、底部导航栏模板

### 结构

```
导航栏背景图（全宽，高约300px）
  └── 5个导航标签（均匀分布）
       ├── 未选中状态：小图标 + 默认背景
       └── 选中状态：放大图标 + 高亮背景 + 标签文本
```

### 状态切换描述

```
未选中：
  - 背景: tab_bg_normal.png, 194×207
  - 图标: 85×85, 在背景中心偏上13px

选中：
  - 背景: tab_bg_selected.png, 240×253, Y轴上移
  - 图标: 放大至160×160
  - 新增标签文本: 字号60, 纯白, 黑色描边6px

锁定/未开放：
  - 背景: tab_bg_locked.png
  - 图标: 锁定图标
```

## 五、卡片组合模板（战斗单位/角色卡）

### 单个卡片结构

```
卡片背景图 (198×438)
  ├── 职业/类型标签图标 (64×64, 顶部)
  ├── 角色立绘/头像 (中部)
  ├── 名称文本 (底部偏上)
  ├── 血条 (背景图 + 进度条, 内间距4px)
  ├── 血量数值文本
  ├── 攻击进度条 (背景图 + 进度条, 内间距4px)
  └── 等级文本 "等级X" (最底部)
```

### 多卡片排列规则

```
多个卡片水平排列，间隔7px，组合整体居中
  - 1个卡片: 单卡居中
  - 2个卡片: 两卡间隔7px后居中
  - 3个卡片: 三卡间隔7px后居中
  - N个卡片: N卡等间距后组合居中
```

## 七、通用样式规范

### 文本描边标准

| 场景 | 字号 | 描边大小 |
|------|------|---------|
| 标题/功能名 | 40-60 | 4-6px |
| 正文/数值 | 28-33 | 4px |
| 辅助信息 | ≤24 | 2-3px |

### 进度条标准

```
进度条 = 背景图 + 进度条图
- 进度条在背景图中间
- 内边距统一4px
- 进度条长度按比例裁剪
```

### 颜色叠加标准

```
按钮/区域需要颜色区分时：
1. 放置背景图
2. 叠加一层纯色矩形
3. 设置不透明度（通常40-60%）
```

## 八、使用建议

1. **确定设计分辨率**后再写坐标（常用竖屏: 1080×2400, 1080×1920）
2. **坐标统一以左上角为原点**，X向右，Y向下
3. **素材文件名用英文**，避免中文路径问题
4. **每个区域独立描述**，便于模块化实现
5. **多单位排列写清间距和居中规则**，而非每个单位的绝对坐标
6. **状态切换（选中/未选中/锁定）写清差异**，便于代码实现条件分支
