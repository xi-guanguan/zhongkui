# 竖屏 2D 游戏界面框架

> **来源**: 社区分享（微信群）
> **作者**: 扁子哥
> **蒸馏等级**: L1（实用精编）
> **适用场景**: 纯 UI 驱动的竖屏手游（UrhoX UI 库 + NanoVG 渲染）
> **核心价值**: 设计分辨率体系 + SHOW_ALL 适配 + NanoVG/UI 分层架构 + 字号间距数值规范

## 一、设计分辨率与初始化

| 参数 | 值 | 说明 |
|------|-----|------|
| 设计分辨率 | 450 × 1000 | 所有 px 值基于此坐标系 |
| 宽高比 | 9:20（0.45） | 主流竖屏比例 |
| 缩放模式 | `UI.Scale.DESIGN_RESOLUTION(450, 1000)` | 引擎自动等比缩放 |

**关键**：所有 fontSize、padding、width、height、gap 按 450×1000 书写，引擎自动缩放，无需手动 DPI 换算。

```lua
local UI = require("urhox-libs/UI")
UI.Init({
    fonts = { { family = "sans", weights = { normal = "Fonts/MiSans-Regular.ttf" } } },
    scale = UI.Scale.DESIGN_RESOLUTION(450, 1000),
})
```

## 二、SHOW_ALL 画面适配

用 `aspectRatio` 约束 gameFrame + 外层黑底居中，超出部分自然形成黑边：

```lua
local screenRatio = graphics:GetWidth() / graphics:GetHeight()
local gameFrameStyle = {
    aspectRatio = 9 / 20,
    overflow = "hidden",
    flexDirection = "column",
}
if screenRatio > 0.45 then
    gameFrameStyle.height = "100%"  -- 屏幕更宽（iPad）→ 左右黑边
else
    gameFrameStyle.width = "100%"   -- 屏幕更窄或刚好 → 上下黑边
end

local root = UI.Panel {
    width = "100%", height = "100%",
    backgroundColor = { 0, 0, 0, 255 },
    justifyContent = "center", alignItems = "center",
    children = { UI.Panel(gameFrameStyle) },
}
UI.SetRoot(root)
```

| 设备比例 | 行为 | 黑边 |
|---------|------|------|
| 9:20 | 铺满 | 无 |
| 9:16（更宽） | 高度100%，宽度自动 | 左右 |
| 9:21（更窄） | 宽度100%，高度自动 | 上下 |

## 三、纵向区域划分

```
gameFrame (450×1000, column)
┌──────────────────────┐
│  顶部 HUD            │  ← 固定 height（如 46px）
├──────────────────────┤
│  主游戏区域           │  ← flexGrow=1，自动填满
├──────────────────────┤
│  底部面板/Tab         │  ← 固定 height（如 320px）
└──────────────────────┘
```

**规则**：
- 固定区域用具体 px（`height = 46`），不用百分比
- 弹性区域用 `flexGrow=1, flexShrink=1, flexBasis=0`
- 所有固定高度之和不超过 1000px

## 四、NanoVG 与 UI 层共存

主游戏区域内分三层叠加：

```lua
UI.Panel {
    width = "100%", flexGrow = 1, flexShrink = 1, flexBasis = 0,
    position = "relative",
    children = {
        -- 层1：NanoVG 渲染层（底层）
        UI.Custom {
            position = "absolute", top = 0, left = 0, right = 0, bottom = 0,
            pointerEvents = "none",
            onRender = function(self, vg, x, y, w, h)
                -- NanoVG 绘制游戏画面
            end,
        },
        -- 层2：UI 控件层（中层，正常流式布局）
        UI.Panel {
            width = "100%", height = "100%",
            pointerEvents = "none",  -- 容器穿透
            children = {
                UI.Button { text = "暂停", pointerEvents = "auto" },  -- 交互控件保持 auto
            },
        },
        -- 层3：浮动层（最上层）
        UI.Custom {
            position = "absolute", top = 0, left = 0, right = 0, bottom = 0,
            pointerEvents = "none",
            onRender = function(self, vg, x, y, w, h) end,
        },
    },
}
```

**触摸穿透原则**：只有需要点击/拖拽的控件保留 `"auto"`，其余一律 `"none"`。

## 五、字号层级（基于 450×1000）

| 层级 | 字号 | 用途 |
|------|------|------|
| 特大标题 | 24-28 | 结算核心数值 |
| 大标题 | 18-20 | 弹窗标题、结算标题 |
| **标准** | **15** | **HUD 数值、面板标题、Tab 文字、核心按钮** |
| 正文 | **13** | 列表项名称、功能按钮、排行榜条目 |
| 辅助 | **11** | 等级标注、属性描述、次要信息 |
| 最小 | 8-9 | 版本号（慎用） |

**"15 为基准"原则**：用户第一眼需看到的信息用 15，其余围绕 15 递减。同一界面不超过 3 个字号层级。

## 六、控件尺寸与间距参考

### 控件尺寸

| 控件 | 尺寸 | 说明 |
|------|------|------|
| HUD 横条 | 高 **46**，paddingH 12 | fontSize=15 |
| 主按钮 | 宽 120-200，高 40-48 | 开始/确认/继续 |
| 功能按钮 | 宽 **64**，高 **30** | fontSize=13 |
| 列表行 | 高 **46**，paddingV **6**，paddingH **6** | |
| Tab 栏 | 高 **48** | fontSize=15 |
| 底部面板 | 高 **320**（含 Tab） | |

### 间距

| 属性 | 推荐值 |
|------|--------|
| paddingHorizontal | 6-14 |
| paddingVertical | 6-10 |
| gap | 4-8（列表行间推荐 5-6） |
| borderRadius | 4-6（小控件），8-10（卡片） |

### 行高与字号配合

```
fontSize ≤ 10  →  行高 34-36, paddingVertical 4
fontSize 11-13 →  行高 42-46, paddingVertical 6
fontSize ≥ 15  →  行高 48-52, paddingVertical 8
```

## 七、配置集中化

将布局常量、颜色集中到 `Config.lua`，避免魔法数字：

```lua
local Config = {}
Config.DESIGN_RATIO = 9 / 20
Config.TOP_HUD_HEIGHT = 46
Config.BOTTOM_PANEL_HEIGHT = 320
Config.TAB_HEIGHT = 48
Config.COLORS = {
    bg_dark        = { 25, 25, 30, 255 },
    text_primary   = { 240, 240, 240, 255 },
    text_secondary = { 160, 160, 170, 255 },
    text_dim       = { 100, 100, 110, 200 },
    accent         = { 80, 180, 255, 255 },
    success        = { 80, 200, 120, 255 },
    fail           = { 220, 80, 80, 255 },
}
return Config
```

## 九、适配变体

| 场景 | 调整 |
|------|------|
| **横屏** | 分辨率改 1000×450，aspectRatio 改 20/9，宽窄判断取反 |
| **非 9:20 竖屏** | 修改 DESIGN_RESOLUTION 和 aspectRatio |
| **无 NanoVG** | 移除渲染层，主区域直接用 UI 控件 |
| **多弹性区域** | 不同 flexGrow 权重分配（如主区域 3、聊天 1） |
