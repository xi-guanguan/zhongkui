# 游戏字体系统：多字族注册 + 文本角色管理 + 实时编辑

> **这是什么**：在 TapTap Maker（UrhoX）中搭建一套完整的游戏字体管理方案——8 个字体族覆盖不同排版需求，160+ 文本角色统一管理，支持游戏内实时编辑和云端同步。
>
> **原作者**：浠涫（《节拍前夜 Beat Eve》制作人）
> **推荐度**：L2（思路和字体选型有很好的参考价值，但作者自述系统仍有多字重缺失、字体未子集化、双轨注册等不足，需要根据项目情况调整）
> **适用场景**：游戏中有大量不同类型文本（标题、正文、数字、叙事等），需要统一管理字体样式并支持设计师实时调整时

---

## 一、为什么需要字体系统

默认字体（引擎内置 MiSans 或阿里巴巴普惠体）在技术上可用，但实际开发中会遇到：

- **视觉层次缺失**：标题、正文、数字全用同一字体，画面缺乏节奏感
- **与游戏风格脱节**：通用字体无法传达游戏的主题氛围（如音乐游戏需要有韵律感的字体）
- **修改效率低下**：改字体要改代码、重新构建，设计师无法自主调整
- **多设备不同步**：一台设备调好的字体配置无法自动同步到其他设备

## 二、字体族选型方案

从 29 个备选字体中精选 8 个，按用途覆盖游戏常见排版需求：

| 字族名 | 字体文件 | 用途 | 大小 | 选择理由 |
|--------|---------|------|------|---------|
| **sans** | AlibabaPuHuiTi-Regular.ttf | 通用正文 | 838K | 免费开源、易读，覆盖 90% 通用文本 |
| **heiti** | SourceHanSansSC-Regular.ttf | 强调正文 | 1.9MB | 比普惠体更有力度的黑体 |
| **display** | SmileySans-Oblique.ttf | 展示/标题 | 2.6MB | 斜体设计，适合有动感的游戏 |
| **impact** | YouSheBiaoTiHei.ttf | 冲击力标题/数字 | 1.4MB | 粗体醒目，Combo/伤害数字一眼可见 |
| **fun** | ZhanKuKuaiLeTi.ttf | 趣味文本 | 5.4MB | 活泼可爱，用于奖励、彩蛋场景 |
| **wenkai** | LXGWWenKai-Regular.ttf | 文学/叙事 | 19MB | 文学感强，战斗日志看起来像小说 |
| **serif** | SourceHanSerifSC-Regular.otf | 正式文本 | 12MB | 适合图鉴、说明等严肃场景 |
| **fangsong** | ZhuqueFangsong-Regular.ttf | 仿宋文本 | 8.5MB | 传统中文排版感 |

> 以上字体均为免费/开源字体，可商用。8 个字体总计约 50MB（不含 Twemoji）。

## 三、注册字体到引擎

### 3.1 下载字体

让 AI 将字体文件下载到 `assets/Fonts/` 目录。

### 3.2 在 UI.Init 中注册

```lua
UI.Init({
    fonts = {
        { family = "sans",     weights = { normal = "Fonts/AlibabaPuHuiTi-Regular.ttf" } },
        { family = "heiti",    weights = { normal = "Fonts/SourceHanSansSC-Regular.ttf" } },
        { family = "display",  weights = { normal = "Fonts/SmileySans-Oblique.ttf" } },
        { family = "impact",   weights = { normal = "Fonts/YouSheBiaoTiHei.ttf" } },
        { family = "fun",      weights = { normal = "Fonts/ZhanKuKuaiLeTi.ttf" } },
        { family = "wenkai",   weights = { normal = "Fonts/LXGWWenKai-Regular.ttf" } },
        { family = "serif",    weights = { normal = "Fonts/SourceHanSerifSC-Regular.otf" } },
        { family = "fangsong", weights = { normal = "Fonts/ZhuqueFangsong-Regular.ttf" } },
    },
    designSize = 1080,
})
```

### 3.3 基础使用

```lua
-- UI 组件中使用
UI.Label({ text = "玩家名称", fontFamily = "wenkai", fontSize = 16 })
UI.Button({ text = "开始游戏", fontFamily = "impact", fontSize = 24 })

-- NanoVG 中使用
nvgFontFace(nvg, "impact")
nvgFontSize(nvg, 32)
nvgText(nvg, 100, 100, "PERFECT\!")
```

## 四、FontStyleConfig：文本角色管理模块

当游戏中有大量文本（100+），逐个硬编码 fontFamily 不可维护。FontStyleConfig 模块解决以下问题：

- 统一管理所有文本角色的字体+字号
- 同类文本字体联动（改一个全部跟着变）
- 支持游戏内实时编辑 + 云端同步

### 4.1 批量注册文本角色

按 `screen.element` 命名，保持一致性：

```lua
FontStyleConfig.RegisterBatch({
    -- 战斗屏幕
    ["battle.player_name"]   = { "sans", 16 },
    ["battle.enemy_name"]    = { "sans", 16 },
    ["battle.hp"]            = { "sans", 14 },
    ["battle.judgment"]      = { "impact", 28 },
    ["battle.damage"]        = { "impact", 24 },
    ["battle.log"]           = { "wenkai", 14 },
    ["battle.log_headline"]  = { "display", 20 },
    -- 商店屏幕
    ["shop.item_name"]       = { "sans", 16 },
    ["shop.item_price"]      = { "impact", 18 },
    -- ... 更多文本角色
})
```

### 4.2 智能联动

```lua
-- 所有按钮字体联动：改一个，全部跟着变
FontStyleConfig.LinkFontFamily({
    "battle.skill_btn",
    "bpause.btn",
    "bpause.btn_pri",
})

-- 字号比例联动：基准角色字号变化时，关联角色按比例缩放
FontStyleConfig.LinkFontSize("reward.card_name", {
    "reward.card_name",   -- 基准
    "reward.card_tier",   -- 自动按比例计算
    "reward.card_desc",   -- 自动按比例计算
})
```

### 4.3 创建带绑定的组件

```lua
-- 自动绑定：字体变更时组件自动更新
local label = FontStyleConfig.Label("battle.player_name", {
    text = "玩家名称",
    fontColor = { 255, 255, 255, 255 },
})

local button = FontStyleConfig.Button("battle.skill_btn", {
    text = "技艺",
    onClick = function() end,
})
```

### 4.4 监听字体变更

```lua
local listenerId = FontStyleConfig.Listen(
    { "battle.damage", "battle.judgment" },
    function(roleKey, style)
        if roleKey == "battle.damage" then
            RecalculateDamageTextPosition()
        end
    end
)

-- 切屏时清理监听器，防止内存泄漏
FontStyleConfig.Unlisten(listenerId)
```

## 五、实时编辑器

在游戏内集成字体编辑 Tab（挂载在 MaterialEditOverlay 上），支持：

- 按屏幕分类管理文本角色
- 实时预览字体效果
- 一键重置功能
- 修改自动保存到云端

修改需要持久化时：查看控制台日志 → 复制修改记录 → 让 AI 更新代码中的默认值。

## 六、推荐默认字号

| 场景 | 推荐字族 | 推荐字号 |
|------|---------|---------|
| 正文 | sans | 14-16px |
| 标题 | display | 20-28px |
| 数字（伤害/Combo） | impact | 18-24px |
| 叙事（日志/剧情） | wenkai | 14-16px |
| 趣味（奖励/彩蛋） | fun | 16-20px |

## 七、已知不足

| 问题 | 说明 | 建议 |
|------|------|------|
| 无多字重 | 8 个字族全部只注册 normal，粗体靠字号+2 模拟 | 需要粗体时应注册 bold 字重 |
| 无 fallback 链 | 中文字体之间没有 fallback，生僻字可能显示方块 | 古风游戏等需考虑 fallback |
| 字体未子集化 | 8 个字体共约 50MB+，体积较大 | 生产环境建议用工具做字体子集化 |
| 双轨注册 | UI.Init 和 nvgCreateFont 各注册一次 | 尽量统一到一套注册机制 |
