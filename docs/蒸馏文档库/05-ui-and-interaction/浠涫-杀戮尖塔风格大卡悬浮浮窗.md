# 杀戮尖塔风格：大卡悬浮详情浮窗

> **这是什么**：实现类似《杀戮尖塔》的卡牌详情交互——鼠标悬停在手牌上时，卡牌抬升放大，旁边弹出详情浮窗展示大图、名称、费用、描述文本和关键词解释。鼠标移开则反向收回。完整覆盖布局计算、关键词提取、响应式适配。
>
> **原作者**：浠涫（《节拍前夜 Beat Eve》制作人）
> **推荐度**：L2（方案完整、经过实战，但与具体卡牌数据结构和 UI 框架耦合较深，需要根据自己的项目调整）
> **适用场景**：卡牌游戏（杀戮尖塔、炉石传说等风格）需要鼠标/手指悬停显示卡牌详细信息的交互

---

## 一、交互设计

### 1.1 交互流程

```
鼠标进入手牌区的某张卡
  ↓
[卡牌效果] 卡牌 Y 轴上移 + 缩放放大（如 1.0 → 1.3）
  ↓
[浮窗效果] 卡牌旁边弹出详情面板（大图 + 名称 + 费用 + 描述 + 关键词）
  ↓
鼠标移开
  ↓
[逆向动画] 卡牌缩回原位 + 浮窗消失
```

### 1.2 浮窗内容结构

```
┌──────────────────────┐
│  [大卡图片]           │
│                      │
│  卡牌名称    费用: 3  │
│                      │
│  "造成 12 点伤害。    │
│   如果有【易伤】，    │
│   改为造成 18 点。"   │
│                      │
│  ─────────────────── │
│  【易伤】：受到攻击   │
│   伤害 +50%          │
│  【格挡】：本回合内   │
│   抵消等量伤害       │
└──────────────────────┘
```

### 2.2 悬停事件处理

```lua
-- 卡牌的 mouseEnter 回调
function onCardMouseEnter(cardData, cardElement)
    -- 1. 卡牌抬升动画（使用逐帧插值，见"逐帧入场动画"文档）
    startCardLiftAnimation(cardElement, {
        targetOffsetY = -30,    -- 上移 30px
        targetScale = 1.3,      -- 放大 30%
        duration = 0.15,
        easing = easeOutCubic,
    })
    
    -- 2. 更新浮窗内容
    UI.FindById("card-detail-image").texture = cardData.image
    UI.FindById("card-detail-name").text = cardData.name
    UI.FindById("card-detail-cost").text = "费用: " .. cardData.cost
    UI.FindById("card-detail-desc").text = cardData.description
    
    -- 3. 提取并显示关键词
    updateKeywords(cardData.description, cardData.keywords)
    
    -- 4. 计算浮窗位置（避免超出屏幕）
    local modalPos = calculateModalPosition(cardElement)
    local modal = UI.FindById("card-detail-modal")
    modal.left = modalPos.x
    modal.top = modalPos.y
    modal.visible = true
end

-- 卡牌的 mouseLeave 回调
function onCardMouseLeave(cardElement)
    -- 1. 卡牌恢复动画
    startCardLiftAnimation(cardElement, {
        targetOffsetY = 0,
        targetScale = 1.0,
        duration = 0.12,
        easing = easeOutCubic,
    })
    
    -- 2. 隐藏浮窗
    UI.FindById("card-detail-modal").visible = false
end
```

### 2.3 位置计算（防溢出）

```lua
function calculateModalPosition(cardElement)
    local screenW = graphics:GetWidth() / graphics:GetDPR()
    local screenH = graphics:GetHeight() / graphics:GetDPR()
    local modalW = 280    -- 浮窗宽度
    local modalH = 400    -- 浮窗预估高度
    local gap = 16        -- 卡牌与浮窗之间的间距
    
    -- 获取卡牌在屏幕上的位置和尺寸
    local cardRect = cardElement:GetScreenRect()  -- {x, y, width, height}
    
    local x, y
    
    -- X 轴：优先放右边，放不下则放左边
    if cardRect.x + cardRect.width + gap + modalW <= screenW then
        x = cardRect.x + cardRect.width + gap  -- 右侧
    else
        x = cardRect.x - modalW - gap           -- 左侧
    end
    
    -- Y 轴：与卡牌顶部对齐，但不超出屏幕
    y = cardRect.y
    if y + modalH > screenH then
        y = screenH - modalH - 8  -- 留 8px 底部边距
    end
    if y < 8 then
        y = 8  -- 留 8px 顶部边距
    end
    
    return { x = x, y = y }
end
```

### 2.4 关键词提取与显示

```lua
-- 关键词库（游戏全局定义）
local KEYWORD_DICT = {
    ["易伤"]   = "受到攻击伤害 +50%，持续到下回合开始",
    ["格挡"]   = "本回合内抵消等量伤害",
    ["中毒"]   = "每回合开始失去等量生命值，然后中毒层数 -1",
    ["虚弱"]   = "造成攻击伤害 -25%",
    ["力量"]   = "每点力量增加 1 点攻击伤害",
    -- ... 更多关键词
}

-- 从卡牌描述中提取【】包裹的关键词
function extractKeywords(description)
    local keywords = {}
    for keyword in description:gmatch("【(.-)】") do
        if KEYWORD_DICT[keyword] and not keywords[keyword] then
            keywords[keyword] = KEYWORD_DICT[keyword]
            keywords[#keywords + 1] = keyword  -- 保持顺序
        end
    end
    return keywords
end

-- 更新关键词显示区域
function updateKeywords(description, cardKeywords)
    local keywords = extractKeywords(description)
    local container = UI.FindById("card-detail-keywords")
    
    -- 清空现有内容
    container:ClearChildren()
    
    if #keywords == 0 then
        container.visible = false
        return
    end
    
    container.visible = true
    for _, keyword in ipairs(keywords) do
        container:AddChild(UI.Label {
            text = "【" .. keyword .. "】" .. KEYWORD_DICT[keyword],
            fontSize = 12,
            color = "#AAAAAA",
            marginTop = 4,
            lineHeight = 1.4,
        })
    end
end
```

## 三、响应式适配

### 移动端适配

```lua
-- 移动端没有鼠标悬停，替换为长按
if PlatformUtils.IsMobile() then
    -- 长按 0.5 秒显示详情
    cardElement.onLongPress = function(self)
        onCardMouseEnter(cardData, cardElement)
    end
    
    -- 手指抬起关闭详情
    cardElement.onTouchEnd = function(self)
        onCardMouseLeave(cardElement)
    end
    
    -- 移动端浮窗改为全屏居中模态框
    -- 因为手指会遮挡旁边的浮窗
end
```

### 浮窗尺寸适配

```lua
-- 根据屏幕大小调整浮窗宽度
local function getModalWidth()
    local screenW = graphics:GetWidth() / graphics:GetDPR()
    if screenW >= 1920 then return 320 end
    if screenW >= 1280 then return 280 end
    return 240
end
```

## 四、注意事项

1. **浮窗只创建一次**：不要在每次 mouseEnter 时重建浮窗 UI，只更新内容。重建会导致闪烁和性能问题
2. **z-order 管理**：浮窗必须在 UI 树的最末尾（或使用 QueueOverlay 渲染），否则会被其他 UI 遮挡
3. **入场动画防闪烁**：卡牌抬升和浮窗出现都应使用逐帧手动插值（参考"逐帧入场动画防闪烁"文档），避免首帧闪烁
4. **关键词高亮**：如果 UI 框架支持富文本，可以在描述文本中用不同颜色高亮【关键词】
5. **快速滑过防抖**：鼠标快速扫过多张卡时，加一个 50~100ms 的延迟再显示浮窗，避免浮窗疯狂闪烁
6. **卡牌排列层级**：悬停卡牌抬升时，z-order 要高于相邻卡牌，否则会被遮挡
