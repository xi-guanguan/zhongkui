# NanoVG 质感 UI 与游戏内实时编辑器

- **这是什么**: 用 NanoVG 纯代码渲染替代"雷霆按钮"（纯色矩形+白字）的 UI 美化方案，包含 8+1 层渲染管线、游戏内实时参数编辑器、配置中心化管理、九宫格纹理面板等完整工具链
- **原作者**: 浠涫（《节拍前夜 Beat Eve》开发者）
- **推荐度**: L2（参考学习——思路完整、架构清晰，但代码量较大且深度绑定作者项目，需根据自身项目裁剪适配）
- **适用场景**: 非美术出身的独立开发者，想让 UI 从"临时占位"升级为有质感的游戏界面

---

## 核心认知：NanoVG 的三个能力层级

| 层级 | 核心功能 | 效果 |
|------|---------|------|
| 层级1 基础几何 | 圆角矩形、圆形、线条 | "雷霆按钮"级别，无质感 |
| 层级2 光影渐变 | nvgLinearGradient、nvgBoxGradient | 凸起/凹陷/发光的立体效果 |
| **层级3 纹理叠加** | nvgImagePattern 纹理平铺、染色 | **织物感、磨砂感、大理石感——质感跃升的关键** |

**关键认知**：`nvgImagePattern` 可以在任意圆角矩形上铺贴一张 64x64 小纹理贴图。一张"细颗粒噪声"小图叠加在暗紫色渐变上，立刻产生细腻的织物质感。原理与 CSS `background-image + background-color` 叠加一致。

## 8+1 层程序化渲染管线

由下到上叠加渲染，每层有独立开关和可调参数，按需组合：

| 层 | 名称 | 实现 | 作用 |
|----|------|------|------|
| L0 | 外阴影 | nvgBoxGradient | 面板脱离背景，浮起立体感 |
| L1 | 背景渐变 | nvgLinearGradient | 上亮下暗，奠定面板基调 |
| L9 | 色调微变 | 水平色相渐变 | 左暖右冷，增加色彩层次（非必需） |
| L2 | 中心暗区 | 反向 nvgBoxGradient | 中心压暗，制造厚重感 |
| L3 | 顶部高光带 | nvgLinearGradient + Scissor | 模拟顶部受光 |
| L4 | 底部反光 | 同上 | 模拟底部环境光反射 |
| L5 | 内发光 | nvgBoxGradient | 边缘向内渐变，增加精致感 |
| L6 | 内描边 | nvgStroke | 紧贴内边缘，强化轮廓 |
| L7 | 外描边 | nvgStroke | 外缘描边，提升辨识度 |
| L8 | 微纹理 | nvgImagePattern | 叠加小纹理图，告别塑料感 |

**最小组合示例**：暗色对话框只需 L0阴影 + L1渐变 + L5内发光 + L7外描边 + L8纹理 = 5 层即可远超纯色矩形。

核心管线代码约 200 行，难的是调参——这也是必须做编辑器的原因。

## 游戏内实时编辑器（效率提升 100 倍）

### 为什么必须做

硬编码参数 → 改一个数值要经历"切回代码→修改→构建→预览"完整流程，单次至少 30 秒。一个按钮可能调 20 次 = 10 分钟。几十个 UI 元素 × 十几个屏幕 = 手动调参完全不现实。

### 核心功能

- 可拖拽浮动面板，不遮挡游戏画面
- 列出当前屏幕所有 UI 元素，点击切换编辑
- 每个参数对应滑块，拖动立刻生效，无需重新构建
- 预设切换器，快速浏览不同风格
- 保存按钮，写入持久化存储
- 重置按钮，一键恢复默认参数

### 性能关键

**滑块拖动时只更新内存参数，不写磁盘。只有点击"保存"才做一次持久化。** 否则每帧拖动触发磁盘 I/O 会严重卡顿。

## 配置中心化管理

### 问题

每个屏幕代码文件存自己的 UI 参数 → 统一调整风格要去十几个文件挨个找。

### 方案：`屏幕.元素` 命名索引

```
map.topBar       → 地图界面顶部栏
shop.itemCard    → 商店界面商品卡
battle.infoBar   → 战斗界面律动条区域
reward.skillCard → 奖励界面技能卡
```

每条配置存 3 类信息：
1. `preset`：基础预设名（如"哥特-暗金"、"毛毯-紫"）
2. 通用覆盖参数：整体透明度、圆角半径等
3. `dmOpts`：精细化参数覆盖

### 运行时覆盖机制

```
代码默认值 ← 运行时覆盖层（编辑器修改） → 最终渲染参数
```

"重置"= 清空覆盖层 → 恢复默认值。

## 双渲染管线：程序化 + 纹理面板

| 管线 | 实现 | 适用场景 |
|------|------|---------|
| 管线A 程序化渲染 | L0-L9 NanoVG 分层代码 | 精细调色、动态变色、无现成素材 |
| 管线B 纹理面板渲染 | 加载 PNG + 九宫格/平铺/拉伸 | 有现成素材，风格固定 |

两条管线共享阴影、花纹边框等通用能力。开发流程：先用程序化管线快速搭出可用 UI → 找到合适素材 → 一键切到纹理管线。

---

## 九宫格（9-slice）核心技术

让小尺寸边框素材适配任意大小面板：

- **四个角**：固定尺寸，绝不拉伸，保证花纹不变形
- **四条边**：单向拉伸，填充边缘
- **中心区域**：双向拉伸，填充主体

NanoVG 实现：给每个区块计算缩放和偏移，`nvgImagePattern` + `nvgIntersectScissor` 逐块绘制，核心代码约 50 行。

---

## 花纹边框层（从合格到精致的临门一脚）

面板渲染完成后向外扩展一圈，用九宫格渲染装饰性花纹边框素材，只渲染 8 个边缘区块（跳过中心格），支持染色。

作者准备了 10 种风格：干净线框、钻石双线、圆点几何、编织结纹、霓虹发光、藤蔓自然等。

## 核心模块架构

| 模块 | 职责 |
|------|------|
| DrawingMachine | 画图机核心：8+1 层程序化管线 + 纹理面板管线 |
| UIMaterialConfig | 配置中心：屏幕.元素 → 材质参数映射 + 默认值与覆盖层合并 |
| MaterialEditOverlay | 实时编辑器：浮动面板、滑块、预设切换、保存重置 |
| DMPanel | 桥接组件：画图机输出接入引擎 UI 框架，监听参数变化重绘 |
| UserSchemeManager | 用户预设方案管理：保存/加载/切换 |
| DataOverride | 数据覆盖层：统一管理运行时修改的参数 |

**核心数据流**：
```
编辑器滑块拖动 → 写入内存覆盖层 → 参数版本号+1
→ 桥接组件检测版本变化 → 触发重新渲染 → 画面实时生效

点击保存 → 本地文件 + 云端 + 控制台日志（三重兜底）
```

## 踩坑经验

### AI 生图 vs 程序化渲染的选择

| 场景 | 建议方案 |
|------|---------|
| 项目还没开始做 UI | **AI 生图先行**，效率最高 |
| 项目已有 UI 框架想逐步美化 | 画图机 + 编辑器方案 |
| 需要动态变色、状态联动的 UI | 程序化渲染最优解 |
| 兜底手段 | AI 生图不满意时回退到程序化 |

**绝对不能踩的坑**：AI 生图必须在项目初期定好，顺序不能反。
- 正确：AI 生成风格图 → 按钮/面板尺寸适配图片 → 再写逻辑
- 错误：先定死按钮宽200高48 → 再让 AI 生图美化 → 永远适配不了

### 开发节奏建议

1. 第 1-2 周：全堆"雷霆按钮"，全力做核心玩法
2. 第 3 周：搭渲染管线（注意：UI 风格不要太早定型）
3. 第 4 周：完成编辑器 + 配置中心，逐屏替换

### 持久化三重兜底

本地文件（Web 平台不可用）+ 云端存储 + 控制台日志输出。
