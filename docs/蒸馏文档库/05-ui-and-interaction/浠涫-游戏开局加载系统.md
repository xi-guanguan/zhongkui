# 游戏开局加载系统：分组预加载 + 进度条 + 容错

> **这是什么**：一套游戏启动时的资源加载方案——将资源按优先级分组（核心 → 重要 → 次要），分阶段加载并显示进度，支持加载失败重试和设备性能适配。解决"游戏开局长时间黑屏/白屏"和"一次性加载导致内存峰值过高"的问题。
>
> **原作者**：浠涫（《节拍前夜 Beat Eve》制作人）
> **推荐度**：L2（思路和架构有很好的参考价值，但具体的资源分组和超时阈值需要根据项目实际情况调整）
> **适用场景**：资源较多的游戏（RPG、卡牌、音游等），启动时需要预加载大量纹理/音频/配置文件，且需要向玩家展示加载进度

---

## 一、问题背景

### 不做加载系统的后果

1. **首屏黑屏/白屏**：所有资源同步加载，UI 线程被阻塞
2. **内存峰值过高**：一次性加载全部资源，低端设备可能崩溃
3. **用户流失**：无进度反馈时，等待 3 秒以上用户就可能退出
4. **加载失败无感知**：某个资源加载失败了，没有任何提示，后续表现为黑图/无声

## 二、架构设计

### 2.1 资源优先级分组

将所有需要预加载的资源分为三个优先级组，按顺序加载：

| 优先级 | 组名 | 包含内容 | 加载策略 |
|--------|------|---------|---------|
| P0 核心 | `core` | 启动 Logo、加载界面自身的背景/进度条素材、基础字体、全局配置 JSON | **同步加载**，这些资源加载完才能显示加载界面本身 |
| P1 重要 | `important` | 主界面 UI 纹理、核心游戏角色模型/精灵、BGM、音效包 | **异步加载**，边加载边更新进度条 |
| P2 次要 | `secondary` | 成就图标、设置界面资源、非首屏的场景资源、教程素材 | **延迟加载**，进入主界面后在后台静默加载 |

### 2.2 加载流程

```
游戏启动
  ↓
[P0 同步加载] → 加载界面自身的最小资源（Logo + 进度条素材）
  ↓
显示加载界面（此时进度 0%）
  ↓
[P1 异步加载] → 逐个加载重要资源，每完成一个更新进度
  ↓
进度 100% → 隐藏加载界面 → 进入主菜单
  ↓
[P2 后台加载] → 用户浏览主菜单时，后台静默加载次要资源
```

### 3.2 加载器核心逻辑

```lua
local ResourceLoader = {}

function ResourceLoader:new()
    local loader = {
        manifest = RESOURCE_MANIFEST,
        currentGroup = nil,       -- 当前加载组
        currentIndex = 0,         -- 当前加载索引
        totalItems = 0,           -- 当前组总数
        loadedItems = 0,          -- 已加载数
        failedItems = {},         -- 加载失败的资源
        retryCount = {},          -- 重试次数记录
        maxRetries = 3,           -- 最大重试次数
        onProgress = nil,         -- 进度回调 function(progress, resourceName)
        onGroupComplete = nil,    -- 组完成回调 function(groupName)
        onError = nil,            -- 错误回调 function(resourcePath, errorMsg)
        batchSize = 1,            -- 每帧加载几个资源（性能适配）
    }
    setmetatable(loader, { __index = ResourceLoader })
    return loader
end

-- 加载指定组
function ResourceLoader:loadGroup(groupName)
    local group = self.manifest[groupName]
    if not group then return end
    
    self.currentGroup = groupName
    self.currentIndex = 1
    self.totalItems = #group
    self.loadedItems = 0
end

-- 每帧调用，加载下一批资源
function ResourceLoader:update()
    if not self.currentGroup then return true end -- 无任务
    
    local group = self.manifest[self.currentGroup]
    local loaded = 0
    
    while loaded < self.batchSize and self.currentIndex <= self.totalItems do
        local item = group[self.currentIndex]
        local success = self:loadResource(item)
        
        if success then
            self.loadedItems = self.loadedItems + 1
        else
            -- 记录失败，稍后重试
            table.insert(self.failedItems, item)
        end
        
        self.currentIndex = self.currentIndex + 1
        loaded = loaded + 1
        
        -- 回调进度
        if self.onProgress then
            local progress = self.loadedItems / self.totalItems
            self.onProgress(progress, item.path)
        end
    end
    
    -- 当前组全部处理完
    if self.currentIndex > self.totalItems then
        self:retryFailed()
        if self.onGroupComplete then
            self.onGroupComplete(self.currentGroup)
        end
        self.currentGroup = nil
        return true -- 组加载完成
    end
    
    return false -- 还在加载中
end

-- 加载单个资源
function ResourceLoader:loadResource(item)
    local resource = cache:GetResource(item.type, item.path)
    if resource then
        return true
    else
        if self.onError then
            self.onError(item.path, "Resource not found")
        end
        return false
    end
end

-- 重试失败的资源
function ResourceLoader:retryFailed()
    local stillFailed = {}
    for _, item in ipairs(self.failedItems) do
        local key = item.path
        self.retryCount[key] = (self.retryCount[key] or 0) + 1
        
        if self.retryCount[key] <= self.maxRetries then
            local success = self:loadResource(item)
            if not success then
                table.insert(stillFailed, item)
            end
        else
            -- 超过重试次数，记录但跳过
            log:Write(LOG_WARNING, "资源加载失败（已放弃）: " .. key)
        end
    end
    self.failedItems = stillFailed
end

return ResourceLoader
```

local function calculateProgress(groupProgress)
    -- 可以加伪进度平滑：避免某个大资源导致进度条卡住

## 四、设备性能适配

```lua
-- 根据设备性能调整每帧加载数量
local function detectBatchSize()
    local physW = graphics:GetWidth()
    local physH = graphics:GetHeight()
    local totalPixels = physW * physH
    
    if totalPixels >= 2560 * 1440 then
        return 3  -- 高端设备：每帧加载 3 个
    elseif totalPixels >= 1920 * 1080 then
        return 2  -- 中端设备：每帧加载 2 个
    else
        return 1  -- 低端设备：每帧加载 1 个，保证不卡
    end
end
```

## 五、注意事项

1. **P0 组要极小**：只放加载界面自身需要的 2~3 个资源，否则用户看到的第一个画面就会延迟
2. **进度平滑**：直接用 `loadedItems / totalItems` 会导致进度条跳跃（大文件加载时卡住，加载完突然跳一大截），建议加最低速度平滑
3. **加载提示文字**：在进度条下方显示当前正在加载的资源名称（"加载音乐..."、"加载角色..."），让用户感知到进度在动
4. **超时兜底**：设置一个总超时（如 30 秒），超过则跳过所有未加载资源直接进入游戏，避免因单个资源加载失败导致永久卡在加载界面
5. **内存监控**：大量纹理加载时注意内存峰值，必要时在 P1 组之间插入 `collectgarbage("collect")` 释放临时内存
