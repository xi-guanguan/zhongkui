# 安全区域适配指南（刘海/灵动岛/底部指示器）

> **来源**: 社区分享（微信群）
> **作者**: 滑滑
> **蒸馏等级**: L1（实用精编）
> **适用场景**: 竖屏/横屏手游需要适配刘海屏、灵动岛、底部 Home 指示器
> **核心价值**: 三种安全区域适配方案 + 硬编码常量速查 + 动态 API + 组件化封装

---

## 一、问题背景

现代手机屏幕存在三类遮挡区域：

| 区域 | 遮挡物 | 影响 |
|------|--------|------|
| 顶部 | 刘海/灵动岛/状态栏 | UI 被遮挡或难以点击 |
| 底部 | Home 指示器（横条） | 底部按钮误触、被遮挡 |
| 左右 | 弧形屏幕边缘 | 边缘 UI 变形/难点击 |

**核心原则**：游戏内容和可交互元素必须放在安全区域内。

---

## 二、方案选择

| 方案 | 适用场景 | 复杂度 | 精确度 |
|------|---------|--------|--------|
| **A. 硬编码常量** | 快速原型/竖屏为主 | 低 | 中 |
| **B. 动态 API** | 需精确适配各机型 | 中 | 高 |
| **C. SafeAreaView 组件** | UI 库项目 | 低 | 高 |

---

## 三、方案 A：硬编码常量（快速方案）

### 竖屏安全区域常量

```lua
local SAFE_AREA = {
    top    = 44,   -- 刘海/灵动岛高度（逻辑像素）
    bottom = 34,   -- Home 指示器高度
    left   = 0,
    right  = 0,
}
```

### 横屏安全区域常量

```lua
local SAFE_AREA = {
    top    = 0,
    bottom = 21,   -- Home 指示器
    left   = 44,   -- 刘海侧
    right  = 44,   -- 刘海侧（两侧对称处理）
}
```

### 使用方式

```lua
-- 顶部 HUD：整体下移安全区域高度
local hudY = SAFE_AREA.top
-- 底部按钮：整体上移安全区域高度
local btnY = screenHeight - SAFE_AREA.bottom - buttonHeight
-- 游戏内容容器：四边内缩
local contentRect = {
    x = SAFE_AREA.left,
    y = SAFE_AREA.top,
    w = screenWidth - SAFE_AREA.left - SAFE_AREA.right,
    h = screenHeight - SAFE_AREA.top - SAFE_AREA.bottom,
}
```

**局限**：不同机型的刘海/灵动岛高度不完全一致（44px 覆盖大多数场景）。

## 四、方案 B：动态 API

```lua
local PlatformUtils = require("urhox-libs.Platform.PlatformUtils")
local insets = PlatformUtils.GetSafeAreaInsets()
-- insets = { top = 47, bottom = 34, left = 0, right = 0 }
```

**返回值**：逻辑像素（已除 DPR），可直接用于 UI 布局。

---

## 五、方案 C：SafeAreaView 组件

```lua
local UI = require("urhox-libs/UI")

-- 整个界面包裹在 SafeAreaView 中
local root = UI.SafeAreaView {
    edges = { "top", "bottom" },  -- 需要避让的边
    backgroundColor = { 0, 0, 0, 255 },
    children = {
        -- 游戏内容
    },
}
UI.SetRoot(root)
```

**参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `edges` | table | 需要避让的边，可选 `"top"`, `"bottom"`, `"left"`, `"right"` |
| `backgroundColor` | table | 安全区域外的填充色（通常为黑色） |

## 六、ScreenRouter 统一入口（推荐）

将安全区域与屏幕方向检测合并，在游戏启动时一次性确定布局参数：

```lua
local function GetScreenLayout()
    local w = graphics:GetWidth()
    local h = graphics:GetHeight()
    local dpr = graphics:GetDPR()
    local logW, logH = w / dpr, h / dpr
    local isPortrait = logH > logW

    local safeArea
    -- 优先尝试动态 API
    local ok, insets = pcall(function()
        local PU = require("urhox-libs.Platform.PlatformUtils")
        return PU.GetSafeAreaInsets()
    end)
    if ok and insets then
        safeArea = insets
    else
        -- 回退到硬编码
        if isPortrait then
            safeArea = { top = 44, bottom = 34, left = 0, right = 0 }
        else
            safeArea = { top = 0, bottom = 21, left = 44, right = 44 }
        end
    end

    return {
        logicalWidth  = logW,
        logicalHeight = logH,
        isPortrait    = isPortrait,
        safeArea      = safeArea,
    }
end
```

## 七、常见机型参考值

| 机型类别 | 顶部安全区(px) | 底部安全区(px) |
|---------|---------------|---------------|
| iPhone X/XS/11 Pro | 44 | 34 |
| iPhone 12/13/14 | 47 | 34 |
| iPhone 14 Pro（灵动岛） | 59 | 34 |
| iPhone 15 Pro | 59 | 34 |
| Android 刘海屏（主流） | 30-48 | 0-24 |
| 无刘海设备 | 0 | 0 |

**建议**：竖屏硬编码取 `top=44, bottom=34` 可覆盖绝大多数设备。追求精确用动态 API。
