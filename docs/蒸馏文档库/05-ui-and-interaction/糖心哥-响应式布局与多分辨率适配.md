# 响应式布局与多分辨率适配

> **这是什么**：让游戏 UI 在手机、平板、电脑等不同分辨率设备上都正常显示的适配方案，涵盖缩放模式选择、Flexbox 响应式技巧和常见坑。
>
> **原作者**：糖心哥
> **推荐度**：L3（用户标注"适配很危险"——方案依赖引擎 UI 库特定行为，Yoga 与 CSS 有关键差异，实战踩坑概率高）
> **适用场景**：需要多设备适配的游戏 UI；使用 urhox-libs/UI 组件库的项目

---

## 前置概念：DPR（设备像素比）

| 设备 | 物理分辨率 | DPR | 逻辑分辨率 |
|------|-----------|-----|-----------|
| 电脑 | 1920×1080 | 1.0 | 1920×1080 |
| iPhone 15 | 2556×1179 | 3.0 | 852×393 |
| 安卓千元机 | 1520×720 | 2.0 | 760×360 |
| iPad Pro | 2732×2048 | 2.0 | 1366×1024 |

```lua
local dpr = graphics:GetDPR()  -- 1.0 / 2.0 / 3.0
```

同样写 `fontSize=16`，在不同 DPR 设备上视觉大小一致（高 DPR 设备用更多物理像素渲染）。

## 三种缩放模式

在 `UI.Init` 的 `scale` 参数中选择：

| 模式 | 设置方式 | 效果 | 适用场景 |
|------|---------|------|---------|
| **DEFAULT**（推荐） | `scale = UI.Scale.DEFAULT` | 1 基准像素 = 1 CSS 像素，小屏自动缩放 | 大多数游戏，不想操心适配 |
| **DPR** | `scale = UI.Scale.DPR` | 严格一致的物理尺寸，不做小屏修正 | 视觉一致性要求极高 |
| **DESIGN_RESOLUTION** | `scale = UI.Scale.DESIGN_RESOLUTION(1920, 1080)` | 按设计稿尺寸写，引擎自动缩放 | 有 UI 设计稿，有精确像素标注 |

**一句话选择**：没设计稿 → DEFAULT；有设计稿 → DESIGN_RESOLUTION；要求严格一致 → DPR。

## 响应式布局六大技巧

### 1. 百分比宽度

```lua
-- ❌ 写死像素，小屏溢出
width = 600

-- ✅ 百分比 + maxWidth 约束
width = "90%", maxWidth = 400
```

### 2. flexGrow 弹性填充

经典上中下三栏：顶栏/底栏固定高度，中间 `flexGrow=1` 自动填满。

```lua
-- 中间内容区
UI.Panel { flexGrow = 1, flexBasis = 0, ... }
```

⚠️ **父容器必须有确定的 `height`**（如 `height="100%"`），不能用 `maxHeight`，否则 flexGrow 算出来是 0。

### 3. flexWrap 自动换行

```lua
UI.Panel {
    flexDirection = "row",
    flexWrap = "wrap",
    children = { ... }  -- 放不下自动换行
}
```

或直接用 `UI.ButtonGroup`，自带换行。

### 4. SimpleGrid 自适应网格

```lua
-- 固定列数
UI.SimpleGrid { columns = 4, gap = 8, ... }

-- 自适应列数（推荐）
UI.SimpleGrid { minColumnWidth = 80, gap = 8, ... }
```

屏幕越宽列越多，每列最少 80 像素。

### 5. SafeAreaView 安全区域

手机刘海、圆角、底部横条会遮挡 UI：

```lua
UI.SafeAreaView {
    width = "100%", height = "100%",
    children = { ... }  -- 自动避开遮挡区域
}
```

### 6. margin="auto" 居中

```lua
UI.Panel {
    width = "90%", maxWidth = 360,
    margin = "auto",  -- 大屏居中，小屏撑满
    ...
}
```

## 实战模式

**适配所有设备的主菜单**核心结构：

```
SafeAreaView (100% × 100%)
  └─ Panel (justifyContent="center", alignItems="center")
       └─ 卡片 Panel (width="90%", maxWidth=360)
            ├─ 标题
            └─ 按钮组
```

**上中下三栏 + 内容滚动**：

```
SafeAreaView
  ├─ 顶栏 Panel (height=60)
  ├─ ScrollView (flexGrow=1, flexBasis=0)
  └─ 底栏 Panel (height=50, flexDirection="row")
```

## ⚠️ 常见坑（L3 标注原因）

| 坑 | 原因 | 解决 |
|----|------|------|
| flexGrow 子元素高度为 0 | 父容器用了 `maxHeight` 而非 `height` | 改为 `height` |
| 子元素溢出容器 | Yoga 默认 `flexShrink=0`（与 CSS 不同） | 加 `flexShrink=1` |
| 横排内容挤成一团 | `flexDirection` 默认 `column` | 显式设 `flexDirection="row"` |
| 手机上 UI 太小 | 没设 scale，默认用物理像素 | 确保 `scale = UI.Scale.DEFAULT` |
| ScrollView 不能滚动 | 没设 `flexGrow=1` + `flexBasis=0`，或外层无确定高度 | 补全属性 |

---

## Yoga vs CSS 关键差异

| 属性 | CSS 默认 | Yoga 默认 | 影响 |
|------|---------|----------|------|
| flexShrink | 1（自动收缩） | **0（不收缩，可能溢出）** | 经常需手动加 `flexShrink=1` |
| flexDirection | row（横向） | **column（纵向）** | 想横排必须显式设 |
| 盒模型 | content-box | **border-box** | 这个反而更直觉 |

**记住前两条**，第三条是好消息。

---

## 三条铁律

1. **`scale = UI.Scale.DEFAULT`** — 不设的话手机 UI 大小乱套
2. **不要写死像素值** — 用 `width="90%" + maxWidth=400` 代替 `width=400`
3. **flexGrow 的父容器要有确定高度** — 用 `height`，不用 `maxHeight`
