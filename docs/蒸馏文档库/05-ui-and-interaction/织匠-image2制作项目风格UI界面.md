# image2 制作项目风格 UI 界面

> **作者**: 织匠 | **难度**: L2-参考 | **来源**: TapTap 论坛 2026-05

## 核心价值

用 image2 为项目生成**风格统一**的多屏 UI 效果图，再在嗒啦啦中还原。关键在于用第一张 UI 效果图作为后续界面的风格锚点。

---

## 从零生成 UI 效果图

### 步骤 1：准备风格参考图

最佳选择：项目中已有的角色图或场景氛围图。有角色图时生成结果在美术风格上更统一（非必须）。

### 步骤 2：生成第一张界面

上传参考图到 image2，输入结构化提示词：

```
图一是一款[平台]端[游戏类型]游戏的[图片类型]，请在该插图基础上，
设计一套风格匹配的[界面类型]界面，要求：
1.布局合理；
2.[具体布局需求]；
3.[功能模块需求]
```

**提示词拆解**：

| 部分 | 作用 | 示例 |
|------|------|------|
| 平台 + 游戏类型 | 定义视觉风格基调 | PC端RPG → 宽屏复杂布局 |
| 图片类型 | 让 AI 理解参考图的用途 | "游戏插图"/"角色立绘" |
| 界面类型 | 指定要设计的页面 | "角色详情界面" |
| 布局需求 | 已明确的功能位置 | "左侧角色列表" |
| 功能需求 | 已确定的信息架构 | "属性包含力量、体魄…" |

> 如果还没明确布局思路，可只保留"布局合理"，多生成几次看效果找灵感。

### 步骤 3：以第一张为锚，派生其他界面

将已选中的效果图**替换原参考图**重新上传，提示词改为：

```
图一是一款PC端rpg游戏中的[已有界面名称]，
请根据该界面风格，设计一套风格匹配的[新界面名称]，要求：
1.[布局需求]；2.[功能需求]
```

**原理**：image2 会从上传图的配色、质感、字体风格中提取视觉特征，保证新界面与已有界面一脉相承。

---

## 已有 UI 的优化迭代

对已在开发中的界面进行美化升级：

```
分析拆解图一的界面布局，对布局进行精简优化，增强UI的设计感，
生成对应布局的[屏幕方向]游戏截图，[画风]风格，从角色到UI风格一致；
布局要求：
1.保留[需保留的部分]；
2.优化[需改进的部分]
```

**关键**：明确哪些布局和信息**保留**、哪些**需要优化**。

---

## 平台差异注意

| 提示词中的平台标签 | 效果 |
|-------------------|------|
| PC端 | 宽屏布局，功能丰富，偏端游风 |
| 移动端 | 竖屏布局，会自动加充值/稀有度图标 |
| 不指定平台，指定画幅比例 | 更灵活控制横屏/竖屏 |

---

## 完整工作流

```
角色图/场景图 → image2生成第一张UI效果图
                ↓ 选中满意的一张
         用该效果图作为输入 → 生成其他界面（编队/战斗/商店…）
                ↓
        全套效果图确定 → 嗒啦啦还原布局（见"省积分精准还原"教程）
```
