# 广告系统接入实战指南

> **这是什么**: UrhoX 游戏中接入激励视频和插屏广告的完整经验，涵盖广告类型选择、展示时机设计、频率控制、降级策略和数据埋点。
> **原作者**: 附中（猛攻兄弟工作室）
> **推荐度**: L2（面向服务端场景的广告接入经验，单机项目稍作调整同样适用）
> **适用场景**: 休闲/跑酷等需要广告变现的游戏项目

---

## 广告类型与策略选择

| 广告类型 | 适用场景 | 用户体验 | 收益 |
|---------|---------|---------|------|
| **激励视频** | 复活、双倍奖励、解锁道具 | ⭐⭐⭐⭐⭐ 用户主动选择 | 高 |
| **插屏广告** | 关卡过渡、返回主菜单 | ⭐⭐⭐ 被动展示 | 中 |
| Banner | 游戏界面底部 | ⭐⭐ 持续展示 | 低 |

**推荐策略**: 激励视频为主 + 插屏为辅。Banner 收益太低且影响体验，不推荐。

## UrhoX 广告桥接机制

UrhoX 通过 `AdBridge` API 桥接到各平台（Web/iOS/Android）的广告系统，不是直接集成广告 SDK。

```lua
local AdBridge = require "urhox-libs.Ads.AdBridge"

-- 初始化
AdBridge.Init({
    rewardedVideoId = "your_ad_unit_id",
    interstitialId = "your_interstitial_id",
    testMode = true,  -- 开发阶段开启测试模式
})

-- 预加载激励视频
AdBridge.PreloadRewardedVideo()

-- 展示激励视频
AdBridge.ShowRewardedVideo({
    onRewarded = function()
        -- 用户看完广告，发放奖励
        player.lives = player.lives + 1
        ShowToast("复活成功！")
    end,
    onClosed = function(watched)
        if not watched then
            ShowToast("看完广告才能获得奖励哦~")
        end
    end,
    onError = function(err)
        log:Error("Ad error: " .. err)
    end,
})
```

## 广告展示时机设计（核心）

### 激励视频时机（用户主动触发）

| 时机 | 转化率 |
|------|--------|
| 死亡复活 | 60-70% |
| 双倍金币（关卡结算） | 40-50% |
| 解锁皮肤（商店） | 20-30% |
| 每日宝箱（登录奖励） | ~80% |

### 插屏广告时机（系统触发）

1. 每 3 个关卡展示一次（不要每关都弹）
2. 返回主菜单时（非首次）
3. 游戏暂停恢复时（慎用）

### ❌ 绝对禁止弹广告的时机

- 游戏进行中（误操作 → 差评）
- 首次启动游戏时
- 用户刚完成付费购买后
- 连续展示两个广告

## 频率控制

```lua
local AdCooldown = {
    lastInterstitial = 0,
    interstitialInterval = 120,  -- 插屏间隔至少 120 秒
    levelsSinceLastAd = 0,
    levelsPerAd = 3,             -- 每 3 关一次插屏
}

function AdCooldown:CanShowInterstitial()
    local now = os.time()
    if now - self.lastInterstitial < self.interstitialInterval then
        return false
    end
    if self.levelsSinceLastAd < self.levelsPerAd then
        return false
    end
    return true
end

function AdCooldown:OnInterstitialShown()
    self.lastInterstitial = os.time()
    self.levelsSinceLastAd = 0
end

function AdCooldown:OnLevelComplete()
    self.levelsSinceLastAd = self.levelsSinceLastAd + 1
end
```

## 加载失败的降级策略

广告不是 100% 能加载成功的，必须有降级方案：

```lua
local function TryShowRewardedVideo(onSuccess, onFail)
    if not AdBridge.IsRewardedVideoReady() then
        -- 广告没准备好，直接给奖励（宁可少赚也不要影响体验）
        onSuccess()
        AdBridge.PreloadRewardedVideo()  -- 后台重新预加载
        return
    end

    AdBridge.ShowRewardedVideo({
        onRewarded = onSuccess,
        onError = function(err)
            onFail(err)
            AdBridge.PreloadRewardedVideo()
        end,
    })
end
```

**核心原则**: 出错时宁可亏钱也不影响体验。

## 踩过的坑（4 条）

| # | 坑 | 后果 | 解决 |
|---|---|------|------|
| 1 | 测试模式忘关 | 跑了两天才发现没收入 | 上线前检查 `testMode = false` |
| 2 | iOS 审核被拒 | 广告弹出频率太高 | 改成每 3 关一次才过审 |
| 3 | 预加载时机太晚 | 用户死亡时广告还没加载好 | 进入游戏就预加载，看完一个立即预加载下一个 |
| 4 | 在 `onClosed` 里发奖励 | 用户反馈没收到奖励 | 改成在 `onRewarded` 里发，`onClosed` 只做 UI 恢复 |

## 一句话总结

广告变现的核心是**尊重用户**：激励视频让用户有选择权，插屏控制频率，出错时宁可亏钱也不影响体验——做好这几点，广告收入和留存率可以兼得。
