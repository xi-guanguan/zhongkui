# 激励视频广告变现设计规范

> **这是什么**：激励视频广告的完整设计方法论 — 从入口放置、奖励反馈、经济占比控制、频次管理到用户分层策略。解决"广告放哪里"和"看完给什么"这两个开发者能控制的核心问题。
>
> **原作者**：综合（多来源行业方法论整合）
> **推荐度**：L1（体系完整、数值有锚点、自检清单可直接执行）
> **适用场景**：任何需要激励视频广告变现的游戏项目，尤其是混合变现（IAA + IAP）场景
>
> **与本目录其他文档的关系**：
> - 本文 = **设计层**（放哪里、给什么、给多少）
> - 附中-广告系统接入实战指南 = **接入层**（SDK 怎么调、时机怎么控）
> - 节拍前夜-SDK回调三层防护 = **防护层**（回调丢失、崩溃恢复）
> - 石头-广告奖励防刷方案 = **安全层**（防刷量、防经济崩溃）
> - 附中-GM广告监控仪表盘 = **运营层**（数据监测、异常定位）

## 1. 权责边界：开发者能控制什么

开发者**不能**控制广告内容（由平台分发），只能控制两件事：

| 开发者能控制 | 平台控制 |
|-------------|---------|
| 广告入口放在哪里 | 广告内容 |
| 什么时机弹出入口 | 广告时长 |
| 入口按钮样式和文案 | 广告填充来源 |
| 看完后给什么奖励 | eCPM 定价 |
| 奖励在经济系统中的占比 | 广告投放策略 |
| 每天最多看几次 | 素材审核 |
| 不同用户看到不同入口 | — |

**激励视频的本质**：不是"广告"，是游戏经济系统中的**用时间换资源的转换器**。

```
玩家的时间（15-30 秒）──→ [ 激励视频 ] ──→ 游戏资源/奖励
```

三赢结构：玩家得资源、开发者得收入、广告主得高完播率曝光。

## 2. 广告奖励在经济系统中的定位

### 2.1 广告奖励 = 一个额外的 Source

在 Source-Pool-Sink 模型中，激励视频是一个独立的 Source 节点：

```
  关卡掉落 ───────→ ┌─────────┐ ───→ 升级消耗
  任务奖励 ───────→ │ 主货币池 │ ───→ 商店购买
  签到     ───────→ │         │ ───→ 强化装备
  ★ 激励视频奖励 ─→ └─────────┘ ───→ 合成材料
```

**核心问题**：★ 这条线占所有 Source 的多少比重？

### 2.2 黄金法则

```
激励视频奖励 = "免费试吃" ≠ "免费自助餐"
让玩家尝到付费的味道，但不能吃饱。
```

| 约束 | 规则 | 原因 |
|------|------|------|
| 价值上限 | 广告奖励 < 最低档 IAP 的等价值 | 否则没人付费 |
| 价值下限 | 广告奖励 > 0 资源获取的替代路径 | 否则没人看广告 |
| 经济占比 | 广告 Source 占总 Source 的 10-25% | 见第 5 章详细分析 |

## 3. 广告入口设计规范

### 3.1 七种标准入口类型

| 入口类型 | 放置界面 | 触发时机 | 按钮文案示例 | 适用游戏 |
|---------|---------|---------|-------------|---------|
| **奖励翻倍** | 关卡结算界面 | 关卡通关后 | "看视频 奖励x2" | 所有类型 |
| **额外生命/续命** | 失败界面 | 玩家死亡/失败时 | "看视频 复活一次" | 闯关/动作 |
| **加速等待** | 倒计时界面 | 存在等待计时器时 | "看视频 立即完成" | 经营/放置 |
| **免费试用** | 商店/道具界面 | 浏览付费内容时 | "看视频 免费用30分钟" | 所有类型 |
| **抽奖/转盘** | 抽奖界面 | 免费次数用完后 | "看视频 再抽一次" | 所有类型 |
| **每日礼包** | 登录/主界面 | 每日首次登录 | "看视频 领今日宝箱" | 所有类型 |
| **临时增益** | 关卡开始前 | 即将进入关卡时 | "看视频 获得双倍攻击" | 战斗/闯关 |

### 3.2 入口设计铁律

| 规则 | 说明 |
|------|------|
| 必须明示奖励 | 玩家点击前就知道能获得什么、获得多少 |
| 必须可拒绝 | 提供"跳过"或"关闭"选项，不能强制 |
| 不能伪装 | 按钮不能伪装成游戏功能元素来诱导点击 |
| 位置要醒目 | 放在视线自然落点，但不阻挡核心操作 |
| 建议加动效 | 轻微呼吸光效、图标晃动等吸引注意力 |

### 3.3 入口出现条件控制

```lua
-- 伪代码：入口出现条件判断
function ShouldShowAdEntry(entryType)
    if todayAdCount >= MAX_DAILY_ADS then return false end      -- 今日上限
    if timeSinceLastAd < MIN_INTERVAL then return false end     -- 最小间隔
    if playerLevel < MIN_LEVEL_FOR_ADS then return false end    -- 新用户保护
    if player.hasAdFreePass then return false end                -- 去广告用户
    return true
end
```

### 3.4 UI 层级优先级

广告入口永远不能在视觉层级上压过游戏核心操作：

```
1. 核心操作按钮（"开始关卡"、"继续"）
2. 主要功能按钮（"分享"、"排行榜"）
3. ★ 广告入口按钮（"看视频得奖励"）
4. 次要信息（版本号、设置）
```

## 4. 奖励反馈设计规范

### 4.1 反馈流程

```
广告播放完毕 → 玩家回到游戏
  → 展示奖励动画/特效（1-3 秒）
  → 资源入账 + 数字跳动
  → 文字确认（"已获得 50 金币"）
  → 自动关闭或手动确认
```

### 4.2 四要素

| 要素 | 说明 | 示例 |
|------|------|------|
| **视觉反馈** | 奖励出现时有视觉效果 | 金币飞入钱包动画、光效、粒子 |
| **数值反馈** | 明确显示获得多少 | "+50 金币"、"x2 已生效" |
| **音效反馈** | 播放正向音效 | 叮铃声、收获声 |
| **状态变化** | 让玩家看到资源确实增加了 | 金币数从 100 跳到 150 |

**规则**：奖励反馈的品质决定了玩家下次是否还愿意看广告。反馈越爽快、越有仪式感，观看意愿越高。

### 4.3 各入口类型的奖励反馈

| 入口类型 | 反馈设计 |
|---------|---------|
| 奖励翻倍 | 原始数字 → 放大 → "x2" 飞入 → 数字翻倍跳动 → 金币飞入 |
| 额外生命 | 角色倒地 → 光圈环绕 → 角色站起 → "复活!" → 继续游戏 |
| 加速等待 | 倒计时快进动画 → 时间条瞬间填满 → "完成!" → 领取产出 |
| 免费试用 | 道具飞入背包 → "限时 30 分钟" 标记 → 属性面板更新 |
| 抽奖转盘 | 转盘再次旋转 → 指针停止 → 奖品弹出 → 入库动画 |
| 每日礼包 | 宝箱开启动画 → 物品逐个弹出 → 全部入库 |
| 临时增益 | Buff 图标飞入状态栏 → 角色发光 → "攻击力 x2" |

### 4.4 获奖预期设计

让玩家在点击之前就产生期待感：

| 方法 | 说明 |
|------|------|
| 预览奖励内容 | 按钮旁展示奖励物品图标和数量 |
| 概率暗示 | "再看 2 次必得稀有道具"（保底机制） |
| 进度条 | "看 3 次集满进度条，获得大奖" |
| 限时提示 | "限时 10 分钟内有效" 制造紧迫感 |

## 5. 广告奖励的经济占比控制

### 5.1 占总 Source 的比重

| 广告 Source 占比 | 经济影响 | 适用场景 |
|----------------|---------|---------|
| 5-10% | 微弱辅助 | 重度 IAP 游戏 |
| **10-20%** | **推荐区间** | **大多数混合变现游戏** |
| 20-30% | 显著补充，需密切监控 | 以广告为主要收入的休闲游戏 |
| 30%+ | 危险区域 | 仅限纯 IAA 超休闲游戏 |

### 5.2 按资源稀缺度分别设定

越稀缺的资源，广告给的越少：

| 资源类型 | 广告产出占该资源总 Source 比重 |
|---------|---------------------------|
| 主货币（金币） | 20-25% |
| 次要材料（铁矿） | 15-20% |
| 高级货币（钻石） | 5-10% |
| 限定道具/角色碎片 | 3-5% |
| 付费专属内容 | **0%**（绝不通过广告给出） |

### 5.3 单次广告奖励定价公式

```
单次广告奖励 ≤ (最低档 IAP 价值 × 0.25) / 每日广告上限次数

示例：¥6 = 300 钻石，每日上限 5 次
  → 日上限 = 300 × 25% = 75 钻石
  → 单次 = 75 / 5 = 15 钻石
```

### 5.4 广告奖励 vs 正常产出对比表

每个广告入口必须制作此表，确保经济不被破坏：

| 入口位置 | 广告奖励 | 正常单次产出 | 广告/正常比值 | 是否合理 |
|---------|---------|------------|-------------|---------|
| 关卡结算翻倍 | +50 金币（翻倍） | 50 金币 | 1.0x 额外 | ✅ |
| 每日宝箱 | 30 钻石 | 日常任务 = 40 钻石 | 0.75x | ✅ |
| 免费获得 SSR | 获得 SSR | 正常需抽 50 次 | 50x | ❌ 过量 |

**规则**：比值不应超过 **1.5x**。超过 2x 需特殊论证，超过 3x 必须拒绝。

### 5.5 经济健康检查流程

```
步骤 1: 不含广告的日均 Source → 例：350 金币/日
步骤 2: 广告额外 Source → 例：150 金币/日
步骤 3: 总 Source = 500 金币/日
步骤 4: 对比 Sink = 450 金币/日
步骤 5: Source(500) > Sink(450) → 轻微通胀
        广告占比 150/500 = 30% → 偏高，需降低
```

### 5.6 通胀预警阈值

| 指标 | 安全 | 预警 | 危险 |
|------|------|------|------|
| Source/Sink 比 | 0.9-1.1 | 1.1-1.3 | >1.3 |
| 广告 Source 占总 Source | <20% | 20-30% | >30% |
| 玩家主货币周均增长率 | <5% | 5-15% | >15% |

## 6. 频次控制与疲劳管理

### 6.1 核心矛盾

广告次数与 eCPM 呈负相关。存在使 ARPU 最大化的"最优次数"。

```
ARPU = 人均广告次数 × (eCPM / 1000)
```

实战案例：某游戏 27 次/天 → eCPM 从 400+ 跌到 260（ARPU=6）。降到合理次数后 eCPM 回升 450（ARPU=8.5）。

### 6.2 频次参数参考

| 参数 | 推荐值 | 说明 |
|------|-------|------|
| 每日总上限 | 5-8 次 | 所有入口总和 |
| 单个入口每日上限 | 1-3 次 | 每种入口分别限制 |
| 两次最小间隔 | 120 秒 | 避免连续触发 |
| 新玩家保护期 | 前 3-5 个关卡无广告 | 让玩家先爱上游戏 |
| 首次广告延迟 | 进入游戏 3 分钟后 | 首次体验不被打扰 |

### 6.3 广告疲劳信号与应对

| 信号 | 判断方法 |
|------|---------|
| 观看率持续下降 | 周环比 > 10% |
| eCPM 持续下降 | 周环比 > 15% |
| 留存率下降 | D7 留存降 > 3 个百分点 |
| 玩家差评增多 | "广告太多" |

**应对**：降频次、提单次价值、引入进度条机制、轮换奖励内容。

### 6.4 用户生命周期策略

| 阶段 | 频次 | 奖励价值 | 开放入口 |
|------|------|---------|---------|
| 新用户（D1-D7） | 低（2-3 次/日） | 高（给好感） | 仅结算和每日礼包 |
| 老用户（D7+） | 中（5-6 次/日） | 标准 | 全部开放 |
| 忠实用户（D30+） | 可提高（6-8 次/日） | 含稀有奖励 | 全部 + 专属入口 |

## 7. 心流嵌入与时机设计

广告入口应出现在玩家情绪的**波谷**或**自然间歇**，不能打断心流波峰。

### 7.1 各场景适合度

| 游戏场景 | 适合度 | 入口类型 | 玩家心理 |
|---------|-------|---------|---------|
| 关卡开始**前** | ★★★★★ | 临时增益 | 期待、愿意获得帮助 |
| 关卡进行**中** | ❌ 禁止 | — | 沉浸中，打断即伤害 |
| 关卡通关**后** | ★★★★★ | 奖励翻倍 | 成就感、乐意锦上添花 |
| 关卡失败**后** | ★★★★☆ | 复活/续命 | 沮丧但不甘心 |
| 返回主界面 | ★★★☆☆ | 每日礼包 | 浏览状态 |
| 等待/倒计时中 | ★★★★☆ | 加速跳过 | 无聊，愿意换时间 |
| 商店浏览中 | ★★★☆☆ | 免费试用 | 有购买意愿但犹豫 |
| 加载画面 | ❌ 不适合 | — | 等待中，弹入口违和 |

### 7.2 最佳时间窗清单

1. **结算界面** — 关卡刚结束，玩家在看收益
2. **关卡选择界面** — 选下一关之前
3. **签到/登录界面** — 每日首次登录
4. **建造/生产完成提示** — 经营类收获时
5. **背包/仓库界面** — 整理物品时
6. **体力耗尽界面** — 无法继续游戏时

## 8. 用户分层广告策略

### 8.1 分层配置

| 用户层级 | 占比 | 每日频次 | 入口数量 | 奖励策略 |
|---------|------|---------|---------|---------|
| 免费玩家 | 70-80% | 6-8 次 | 全部开放 | 标准奖励，覆盖面广 |
| 低付费玩家 | 15-25% | 3-5 次 | 高价值入口 | 减翻倍、增试用（引导付费） |
| 高付费玩家 | 1-5% | 0-2 次 | 仅自愿入口 | 可提供"去广告"选项 |

### 8.2 激励视频作为 IAP 桥梁

```
从未体验过付费内容
  → 看激励视频 → 免费试用高级道具 30 分钟
  → 体验到付费内容的好处
  → 试用结束 → 产生购买欲望
  → 一部分转化为付费用户
```

关键数据：
- 看过激励视频的玩家 IAP 概率是普通玩家的 **4.5 倍**
- 78% 的玩家愿意观看激励视频获取奖励
- 86% 的开发者加入广告后 IAP 收入**不降反升**

### 8.3 去广告产品设计

```
去广告方案：
├── 永久移除所有被动广告入口
├── 保留激励视频入口（玩家自愿点击的）
├── 激励视频奖励改为直接发放（无需观看）
└── 定价：最低档 IAP 的 1-2 倍
```

## 10. 关键指标与数据监测

### 10.1 核心指标

| 指标 | 公式 | 健康值 |
|------|------|-------|
| 广告观看率 | 点击人数 / 看到入口人数 | ≥ 60% |
| 广告完播率 | 看完人数 / 开始看人数 | ≥ 95% |
| 人均日观看次数 | 总观看 / DAU | 3-6 次 |
| 广告留存影响 | 开广告后 D7 留存 vs 前 | 差值 < 2 个百分点 |

### 10.2 经济健康监测

| 指标 | 计算方法 | 预警线 |
|------|---------|-------|
| 广告 Source 占比 | 广告产出 / 总 Source | > 30% |
| 主货币周增长率 | (本周 - 上周) / 上周 | > 10% |
| Source/Sink 比 | 日均 Source / 日均 Sink | > 1.3 |
| 广告奖励/正常产出比 | 单次广告 / 单次正常 | > 2.0 |

### 10.3 ROI 公式

```
LTV = 人均日广告次数 × (eCPM / 1000) × Σ(D1~D180 留存)
ROI = LTV / CPI
```

四个优化杠杆：降 CPI、提 eCPM、提人均次数（注意与 eCPM 负相关）、提留存。

## 11. 广告变现自检清单

### 设计阶段

- [ ] 广告变现是否在设计之初就纳入经济系统？
- [ ] 是否画了包含广告 Source 的资源流图？
- [ ] 广告 Source 占总 Source 的比重是否在 10-25%？
- [ ] 每个入口是否有明确的奖励设计和数值？
- [ ] 单次广告奖励 < 最低档 IAP 等价值的 25%？
- [ ] 广告奖励/正常产出比值 ≤ 1.5x？
- [ ] 是否设计了频次上限和最小间隔？
- [ ] 新用户保护期（前 3 个关卡无广告）？
- [ ] 是否为不同用户层级设计了差异化策略？

### 入口设计

- [ ] 每个入口明确告知了奖励内容和数量？
- [ ] 每个入口提供了"跳过/拒绝"选项？
- [ ] 入口没有伪装成游戏功能元素？
- [ ] 入口放在心流波谷而非波峰？
- [ ] 入口在 UI 层级中低于核心操作按钮？

### 奖励反馈

- [ ] 看完广告后有视觉反馈（动画/特效）？
- [ ] 明确显示了获得的资源数量？
- [ ] 有音效反馈？
- [ ] 玩家能看到资源余额确实增加了？

### 上线后监测

- [ ] 广告观看率 ≥ 60%？
- [ ] D7 留存因广告下降 < 2 个百分点？
- [ ] 主货币周增长 < 10%？
- [ ] eCPM 未持续下降（周降 < 15%）？
- [ ] Source/Sink 比 < 1.3？

## 附录：广告奖励定价速查表

假设最低档 IAP = ¥6 = 300 钻石，每日上限 = 5 次：

| 入口类型 | 单次上限 | 日累计上限 | 占 IAP 比 |
|---------|---------|----------|----------|
| 奖励翻倍 | 等于 1 次正常产出 | 5 次正常产出 | — |
| 钻石奖励 | 15 钻石 | 75 钻石 | 25% |
| 金币奖励 | 关卡产出的 100% | 5 倍关卡产出 | — |
| 体力恢复 | 20 点 | 40 点（2 次） | — |
| 加速跳过 | 30 分钟 | 150 分钟 | — |
| 免费试用 | 30 分钟 | 1 次/日 | — |

**定价公式**：单次广告奖励 ≤ (最低档 IAP 价值 × 0.25) / 每日广告上限次数
