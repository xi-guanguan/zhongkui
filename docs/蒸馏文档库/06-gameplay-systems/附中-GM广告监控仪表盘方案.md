# GM 广告监控仪表盘方案

> **这是什么**: 面向运营/GM 的广告数据实时监控面板设计方案，覆盖数据模型、统计维度、UI 架构和交互模式。用于观测广告健康度、定位异常玩家、调整个人广告配置。
>
> **原作者**: 附中
>
> **推荐度**: L2（方案级参考，含完整数据流与 UI 设计思路，需按项目实际对接 serverCloud 排行榜）
>
> **适用场景**: 需要在运营后台观测广告 KPI、排查异常玩家广告行为的游戏项目

---

## 核心数据流

```
客户端广告行为 → RecordXxx → RemoteEvent → 服务端
  → serverCloud 排行榜写入（7日周循环 + 生涯）

GM 打开仪表盘 → 查询请求(tab+page) → 服务端查排行榜
  → 返回玩家列表+汇总统计 → NanoVG 渲染
  → 批量查询昵称（异步填充）
```

## 六大统计维度

| 维度 | action | 含义 | 计入限流 |
|------|--------|------|----------|
| 点击 | `click` | 玩家点击广告按钮 | — |
| 成功 | `success` | SDK 确认播放完成 | watchCount + rewardCount |
| 免费跳过 | `free_skip` | 超日限自动跳过，仍发奖 | rewardCount |
| 主动跳过 | `manual_skip` | 玩家中途关闭 | — |
| 超限跳过 | `over_limit` | 总奖励超上限，被拒 | — |
| 救活 | `involuntary` | 崩溃恢复补发奖励 | watchCount + rewardCount |

**成功率公式**:
```
有效成功 = 成功 - 救活（进程被杀，不确定是否真看了）
有效点击 = 点击 - 主动跳过（玩家主动放弃，不拉低成功率）
成功率   = 有效成功 / 有效点击 × 100%
```

## 时间维度: 7日周循环

- **8 个标签页**: 周一~周日 + 生涯
- 使用 UTC+8 判断星期几
- 每周自动覆盖（周一数据到下个周一被替换）
- serverCloud 排行榜 key 按星期几命名，无需手动清理

## UI 布局（三层结构）

| 层 | 内容 |
|----|------|
| 汇总行 | 全量聚合统计（玩家数/点击/成功/救活/成功率） |
| 排行榜 | 每行一个玩家，按观看量降序，每页50条 |
| 操作栏 | 分页控件 + 关闭提示 + UID 复制 toast |

## 颜色编码

| 维度 | 颜色 | 含义 |
|------|------|------|
| 基础列 | 浅灰白 | 正常数据 |
| 救活 | 紫色 `(200,120,255)` | 崩溃恢复，关注频率 |
| 免费跳过 | 青色 `(80,200,255)` | 超日限仍发奖，高=活跃 |
| 主动跳过 | 橙色 `(255,180,60)` | 高=广告体验差 |
| 超限跳过 | 红色 `(255,100,80)` | 高=疑似刷量 |
| 当日新进 | 金色 UID | 首登=今天 |
| 零值 | 低透明度 | 不干扰视觉 |

## 关键运营指标阈值

| 指标 | 健康 | 异常信号 |
|------|------|----------|
| 成功率 | >85% | <70% → SDK/网络问题 |
| 救活占比 | <5% | >10% → SDK崩溃率高 |
| 主动跳过率 | <15% | >30% → 广告质量差/时长太长 |
| 超限跳过 | 接近0 | 大量 → 刷量行为 |

## 交互模式

- **标签页切换**: 清空滚动 → 发查询 → loading → 渲染
- **分页**: 每页50条，翻页后重置滚动
- **UID复制**: 点击 → 复制到变量 → toast → 带入广告调整面板
- **缓存策略**: Tab缓存 + 昵称全局缓存跨标签共享
- **超时保护**: 60s 倒计时，超时显示错误提示
