# 云存档加载失败防护指南

> **这是什么**: clientCloud 云存档加载失败时的正确处理方案，防止"加载失败 → 空数据 → 覆盖存档"的致命 BUG。  
> **原作者**: 糖心哥（项目：《深渊炼狱：堕落之塔》）  
> **推荐度**: L1（拿来就能用 — 血泪教训总结，所有用 clientCloud 存档的游戏必读）  
> **适用场景**: 任何使用 clientCloud 进行云存档的游戏

---

## 致命 BUG 复现

### 症状

玩家反馈"进度全没了"，排查发现：

1. 玩家打开游戏 → clientCloud 加载存档
2. **加载失败**（网络波动/服务器超时）
3. 返回**空数据**
4. 代码把空数据当成"新玩家" → 初始化新存档
5. 新存档**覆盖了云端真实数据**
6. 玩家数十小时进度归零，**不可恢复**

### 根因

代码没有区分"加载失败返回空"和"真的是新玩家"：

```lua
-- ❌ 致命代码（无法区分失败和空）
clientCloud:Load(function(data)
    if data == nil or data == "" then
        -- 这里把"加载失败"和"新玩家"混为一谈了！
        InitNewPlayer()   -- 初始化新角色
        SaveToCloud()     -- 覆盖云端 → 存档丢失！
    else
        LoadPlayerData(data)
    end
end)
```

---

## 5 步修复方案

### 第 1 步：区分「加载失败」和「数据为空」

关键在于利用回调的**错误参数**：

```lua
clientCloud:Load(function(success, data)
    if success then
        -- 真正成功了
        if data and data ~= "" then
            LoadPlayerData(data)       -- 有数据 → 加载
        else
            InitNewPlayer()            -- 数据为空 → 新玩家
        end
    else
        -- 加载失败 ≠ 数据为空！
        HandleLoadFailure()            -- 走失败处理流程
    end
end)
```

### 第 2 步：失败时锁定写入

加载失败后，**绝对不能写入云端**，否则就是用空数据覆盖真实存档：

```lua
local saveConfirmed = false  -- 存档安全锁

function HandleLoadFailure()
    saveConfirmed = false  -- 锁定！禁止任何云端写入
    -- 后续所有 Save 操作都要检查这个标志
end

function SaveToCloud(data)
    if not saveConfirmed then
        print("[存档] 写入被锁定，拒绝保存")
        return false
    end
    -- 正常保存逻辑...
end
```

### 第 3 步：本地备份兜底

每次成功加载后，同步一份到本地：

```lua
function OnLoadSuccess(data)
    saveConfirmed = true
    LoadPlayerData(data)
    -- 同步备份到本地
    SaveLocalBackup(data)
end

function HandleLoadFailure()
    saveConfirmed = false
    -- 尝试从本地备份恢复
    local backup = LoadLocalBackup()
    if backup then
        LoadPlayerData(backup)
        ShowNotice("已从本地备份恢复，云端同步将在网络恢复后进行")
    else
        ShowRetryDialog()  -- 无备份，只能重试
    end
end
```

### 第 4 步：显示重试弹窗

不要让玩家"卡死"在加载失败界面，给一个明确的重试按钮：

```lua
function ShowRetryDialog()
    -- 弹窗内容：
    -- "云存档加载失败，请检查网络后重试"
    -- [重试] [离线模式]
    --
    -- 重试：重新调用 clientCloud:Load
    -- 离线模式：使用本地备份（如果有），但锁定云端写入
end
```

### 第 5 步：自动重试机制

加载失败后自动重试 2-3 次，间隔递增：

```lua
local MAX_RETRY = 3
local retryCount = 0
local RETRY_DELAYS = { 2, 5, 10 }  -- 秒

function LoadWithRetry()
    retryCount = retryCount + 1
    clientCloud:Load(function(success, data)
        if success then
            retryCount = 0
            OnLoadSuccess(data)
        elseif retryCount < MAX_RETRY then
            local delay = RETRY_DELAYS[retryCount] or 10
            print(string.format("[存档] 第 %d 次重试，%d 秒后执行", retryCount, delay))
            SetTimeout(delay, LoadWithRetry)
        else
            -- 重试耗尽
            retryCount = 0
            HandleLoadFailure()
        end
    end)
end
```

## 完整流程图

```
游戏启动
  ↓
clientCloud:Load()
  ↓
┌─ 成功 ──→ 有数据？─── 是 ──→ 加载角色 + 本地备份 ──→ saveConfirmed = true
│                    └── 否 ──→ 创建新角色 ──→ saveConfirmed = true
│
└─ 失败 ──→ saveConfirmed = false（锁定写入！）
            ↓
            重试次数 < 3？
            ├─ 是 → 等待 N 秒 → 重试 Load
            └─ 否 → 有本地备份？
                    ├─ 是 → 从备份恢复（只读模式）
                    └─ 否 → 显示重试弹窗
```

## 自查清单

在你的存档代码中检查以下项目：

- [ ] `clientCloud:Load` 的回调是否区分了"失败"和"数据为空"？
- [ ] 加载失败时是否锁定了云端写入（`saveConfirmed` 机制）？
- [ ] 是否有本地备份作为兜底？
- [ ] 是否有自动重试机制（至少 2-3 次）？
- [ ] 重试全部失败后，是否有 UI 引导玩家手动重试？

---

## 关键教训

> "空数据和加载失败是两个完全不同的事情。一个意味着新玩家，一个意味着出了问题。把它们混为一谈，就是在玩家存档上埋地雷。"

**一句话总结**：加载失败 → 锁写入 → 重试 → 本地备份兜底。永远不要把加载失败当成"新玩家"处理。
