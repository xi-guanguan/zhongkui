## 1. 核心思想

**一句话**：给每个属性定一个「价值系数」，所有系统（装备/天赋/宝石/称号）按统一价值预算投放数值，杜绝数值混乱。

```
属性价值 = 1点该属性值多少「强度」（or 战力）

例：暴击率 价值=1.0，物理攻击力 价值=0.5
→ 1% 暴击率 = 2 物理攻击力（等价交换）
```

**好处**：
- 新系统投放时 AI 只需知道「总预算」即可自动分配
- 属性之间的强度对比一目了然
- 修改某属性价值后全局联动（如果用公式而非写死数字）

## 2. 确定属性价值模型

### 步骤 1：列出所有属性

将游戏内所有属性分类列出，标注数值类型：

| 类别 | 属性 | 数值类型 | 说明 |
|------|------|---------|------|
| 攻击 | 物理攻击力 | 整数 | 基础伤害加成 |
| 攻击 | 暴击率 | 百分比 | 触发暴击概率 |
| 攻击 | 暴击伤害 | 百分比 | 暴击时伤害倍率 |
| 攻击 | 攻击速度 | 百分比 | 出手频率 |
| 防御 | 生命值 | 整数 | 生存资源 |
| 防御 | 物理护甲 | 整数 | 减少物理伤害 |
| 防御 | 闪避率 | 百分比 | 回避攻击概率 |

### 步骤 2：赋予属性价值

根据伤害公式和属性作用，为每个属性赋值：

```
攻击类属性价值示例：
┌──────────────┬──────┬──────────────────────────┐
│ 属性         │ 价值 │ 换算关系                  │
├──────────────┼──────┼──────────────────────────┤
│ 物理攻击力   │ 0.5  │ 基准属性                  │
│ 暴击率       │ 1.0  │ 1%暴击 = 2物攻           │
│ 暴击伤害     │ 0.8  │ 1%暴伤 ≈ 1.6物攻         │
│ 攻击速度     │ 1.2  │ 1%攻速 ≈ 2.4物攻         │
│ 命中率       │ 0.6  │ 1%命中 ≈ 1.2物攻         │
└──────────────┴──────┴──────────────────────────┘
```

**赋值方法**：
- **旧项目**：基于已有伤害公式 + 现有系统数值反推价值
- **新项目**：先确定伤害公式 → 写清属性作用 → 计算边际收益 → 定价值

### 步骤 3（可选）：设计基础属性派生

```
基础属性（价值各 1.5）→ 派生为多个战斗属性

力量 → 物理攻击力 + 物理护甲
敏捷 → 攻击速度 + 闪避率 + 命中率
智慧 → 魔法攻击力 + 魔法护甲
体质 → 生命值 + 双甲
运气 → 暴击率 + 暴击伤害 + 最大伤害加成
精神 → 魔法攻击力 + 治疗量 + 魔法护甲
```

每点基础属性派生多少战斗属性 = 基础属性价值 ÷ 各派生属性价值之和（按权重分配）。

## 3. 属性数值投放

### 投放流程

```
确定系统（装备/天赋/宝石/...）
  ↓
定每个等级/品质的「价值预算」
  例：1级装备总价值=1，2级=1.5，3级=2.5
  ↓
设计属性倾向
  例：暴击装（70%预算给暴击相关）、破甲装（70%给穿透）
  ↓
按价值模型自动换算具体数值
  例：暴击装1级，总预算1.0，70%给暴击率
      → 暴击率额度 = 1.0 × 0.7 = 0.7
      → 暴击率数值 = 0.7 ÷ 暴击率价值(1.0) = 0.7%
      → 剩余额度分配给其他属性
```

### 装备投放示例

```
装备等级价值表：
  1级 = 1.0    2级 = 1.5    3级 = 2.5    4级 = 4.0    5级 = 6.0

「烈焰之刃」3级攻击装（属性倾向：物攻70% 暴击20% 攻速10%）
  总预算 = 2.5
  物攻额度 = 2.5 × 0.7 = 1.75 → 数值 = 1.75 ÷ 0.5 = +3.5 物攻（取整+4）
  暴击额度 = 2.5 × 0.2 = 0.50 → 数值 = 0.50 ÷ 1.0 = +0.5% 暴击
  攻速额度 = 2.5 × 0.1 = 0.25 → 数值 = 0.25 ÷ 1.2 = +0.21% 攻速（取整+0.2%）
```

### 动态公式法（推荐）

```lua
-- 属性价值配置
local ATTR_VALUE = {
    PhysicalAttack = 0.5,
    CritRate       = 1.0,
    CritDamage     = 0.8,
    AttackSpeed    = 1.2,
    MaxHp          = 0.3,
    PhysicalArmor  = 0.4,
}

-- 根据装备价值预算 + 属性倾向 → 计算最终数值
function CalcEquipStats(totalBudget, attrWeights)
    local result = {}
    for attr, weight in pairs(attrWeights) do
        local budget = totalBudget * weight
        local value = budget / ATTR_VALUE[attr]
        result[attr] = math.floor(value + 0.5)  -- 四舍五入
    end
    return result
end

-- 使用示例
local stats = CalcEquipStats(2.5, {
    PhysicalAttack = 0.7,
    CritRate = 0.2,
    AttackSpeed = 0.1,
})
-- → { PhysicalAttack = 4, CritRate = 1, AttackSpeed = 0 }
```

**优势**：修改 `ATTR_VALUE` 表后，所有装备数值自动联动更新，无需逐个重算。

## 4. 进阶：阶段价值与动态公式

### 阶段投放（长线养成游戏）

对于数值跨度大的游戏，不同阶段同一属性的价值不同：

```
阶段价值表：
┌──────────┬────────────┬──────────┬──────────┐
│ 属性     │ 前期(1-30) │ 中期(31-60) │ 后期(61+) │
├──────────┼────────────┼──────────┼──────────┤
│ 物理攻击 │ 0.5        │ 0.3      │ 0.15     │
│ 暴击率   │ 1.0        │ 1.0      │ 1.2      │
│ 生命值   │ 0.3        │ 0.2      │ 0.1      │
└──────────┴────────────┴──────────┴──────────┘
```

后期物攻价值降低 = 同样预算给更多物攻 = 数值膨胀感。
后期暴击价值升高 = 同样预算给更少暴击 = 暴击稀缺化。

### 战力计算（副产品）

```lua
-- 有了价值模型，战力计算水到渠成
function CalcCombatPower(stats)
    local power = 0
    for attr, value in pairs(stats) do
        power = power + value * ATTR_VALUE[attr]
    end
    return math.floor(power)
end
```

可直接用于：一键穿戴（选战力最高组合）、装备对比、战力排行榜。

## 5. AI 协作要点

### 让 AI 帮你做数值的正确姿势

**给 AI 的输入**：
1. 完整属性列表 + 数值类型
2. 伤害公式（或战斗机制说明）
3. 各属性的作用描述
4. 希望的基准属性和对标关系

**AI 可以帮你做的**：
- 根据伤害公式计算各属性边际收益 → 推荐价值系数
- 批量生成装备/天赋/宝石的具体数值（给定预算+倾向）
- 检查现有数值是否偏离价值模型（诊断不平衡）
- 调整阶段价值曲线

**AI 不该做的**：
- 不要让 AI 凭空定价值（必须基于你的伤害公式）
- 不要跳过测试直接上线（价值模型只是理论，实战需验证）

### 隐藏技巧：公式化存储

```
装备不存「最终数值」，存「价值预算 + 属性倾向」
→ 每次打开背包 / 掉落时实时计算
→ 改了价值模型全局生效，无需逐条改数据
```

## 总结

| 步骤 | 做什么 | 产出 |
|------|--------|------|
| 1 | 列出全部属性 + 分类 | 属性清单 |
| 2 | 定伤害公式 → 算价值系数 | 价值模型表 |
| 3 | 定系统等级预算 + 属性倾向 | 投放规则 |
| 4 | 按公式批量生成数值 | 具体配表 |
| 5 | 测试 → 调整价值 → 全局联动 | 平衡迭代 |

**核心公式**：`属性数值 = 价值预算 × 属性权重 ÷ 属性价值系数`
