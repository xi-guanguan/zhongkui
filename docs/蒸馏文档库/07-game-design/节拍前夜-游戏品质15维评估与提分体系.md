# 游戏品质 15 维评估与提分体系

> **这是什么**: 一套 15 维度加权评分体系，配合功能难度分级和多轮迭代模式，系统化评估游戏品质并找到提分路径
> **原作者**: 节拍前夜
> **推荐度**: L2（有参考学习价值）
> **适用场景**: 阶段性验收（原型/开发/打磨/发布前）、品质冲刺定位短板、多轮迭代打磨、模糊优化请求的落地

---

## 1. 核心原则

1. **复杂的先评后改** -- 大改动先出方案让用户确认，避免返工。
2. **给具体参数** -- 数值/参照物/截图 优于 "好一点"/"大一点"。
3. **定期全面审视** -- 每个阶段完成后做一次多维评估。

---

## 2. 品质评估体系（15 维度）

| # | 维度 | 权重 | 含义 |
|---|------|------|------|
| 1 | 核心玩法趣味性 | 10 | 机制是否有趣、让人想继续玩 |
| 2 | 操作手感与流畅度 | 9.5 | 输入响应、动画过渡、帧率稳定 |
| 3 | Bug 与稳定性 | 9.5 | 崩溃、卡死、数据丢失 |
| 4 | 游戏节奏与心流 | 9 | 难度曲线、正反馈循环、成就感 |
| 5 | 数值平衡 | 9 | 伤害/血量/成长/经济体系 |
| 6 | UI/UX 设计 | 8.5 | 信息层级、操作便捷、视觉一致 |
| 7 | 新手引导 | 8.5 | 前 5 分钟能否抓人 |
| 8 | 视觉风格 | 8 | 配色、动画、风格统一 |
| 9 | 音频体验 | 7.5 | BGM 契合、音效丰富、音量平衡 |
| 10 | 内容丰富度 | 7.5 | 关卡数量、多样性、重复可玩性 |
| 11 | 性能优化 | 7 | 帧率、加载速度、内存 |
| 12 | 代码质量 | 7 | 模块化、命名、可扩展 |
| 13 | 叙事与世界观 | 6.5 | 故事、角色、文案 |
| 14 | 社交与分享性 | 6 | 排行榜、分享、话题性 |
| 15 | 商业化与留存 | 5.5 | 留存钩子、成长线 |

### 一键全维度体检

触发词："全面评估" / "体检" / "打分"。

执行步骤：

1. 逐维度 1-10 打分。
2. 输出格式：`维度名 | 得分 | 一句话评价 | 最关键改进点`。
3. 按得分从低到高排序。
4. 给出加权总分和优先优化的 3 个方向。

## 3. 品质分数参考与提分路线

### 分数等级对照

| 分数 | 等级 | 玩家感受 |
|------|------|---------|
| 9.5-10 | 精品级 | 主动推荐给朋友 |
| 9.0-9.4 | 优秀 | 愿意继续玩 |
| 8.0-8.9 | 良好 | 有时想玩 |
| 7.0-7.9 | 及格 | 不太会推荐 |
| <7.0 | 待改进 | 可能弃坑 |

### 提分路线

| 目标 | 关键动作 |
|------|---------|
| 8 -> 8.5 | 修复所有已知 Bug、补全缺失音效、统一视觉风格 |
| 8.5 -> 9.0 | 优化动画细节、打磨操作手感、完善新手引导 |
| 9.0 -> 9.5 | 精调数值平衡、增加惊喜细节（彩蛋/微交互/粒子特效）、完善文案 |

## 4. 功能难度分级

| 难度 | 特征 | 推荐方式 |
|------|------|---------|
| S 微调 | 单文件 <=10 行 | 直接说改什么 |
| M 常规 | 1-3 文件 | 直接实现即可 |
| L 复杂 | 3-8 文件 | 先出方案 -> 确认 -> 分步执行 |
| XL 架构级 | 8+ 文件 | 先出方案 -> 分阶段评审 |

---

## 5. 多轮迭代模式

### 模式一：评估 -> 筛选 -> 执行 -> 验证

1. 评估：对目标维度 1-10 打分，列出所有问题，不修改。
2. 筛选：用户选择本轮要执行的条目。
3. 执行：按选定条目逐条实施。
4. 验证：重新打分，未达标则继续迭代。

### 模式二：分层递进

1. 骨架（核心玩法可运行）
2. 功能完善（系统接入）
3. 体验打磨（手感/UI/音频）
4. 品质冲刺（数值/彩蛋/细节）

### 何时停止迭代

- 评分达标。
- 剩余问题属于锦上添花级别。
- 投入产出比明显下降。
- 需要新素材/新设计，当前轮次无法解决。

## 6. 开发全流程（7 阶段）

| 阶段 | 内容 | 关键门槛 |
|------|------|---------|
| 1. 策划立项 | 明确方向、核心循环、MVP 清单 | -- |
| 2. 原型开发 | 验证核心玩法 | 核心玩法 >= 7 分才继续 |
| 3. 功能开发 | 按优先级实现，复杂功能先出方案 | -- |
| 4. 美术音频打磨 | 视觉 + 音频到专业水准 | -- |
| 5. 数值平衡调优 | 难度曲线、新手引导 | -- |
| 6. 测试修复 | 全面测试、性能优化 | -- |
| 7. 发布 | 15 维体检、材料生成、上线 | 加权总分达标 |

---

## 7. 高效沟通技巧

| 技巧 | 示例 |
|------|------|
| 给具体数值 | "按钮高度从 52 改到 70" 而非 "大一点" |
| 给参照物 | "改成和开始游戏按钮一样的风格" |
| 贴截图 | "（截图）应该 xxx 但实际 yyy" |
| 明确范围 | "只优化商店界面的按钮部分" |
| 先评后改 | "先打分 + 出方案，我确认后再改" |
