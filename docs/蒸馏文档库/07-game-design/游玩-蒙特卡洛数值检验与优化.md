# 数值调优：蒙特卡洛检验与优化

> **作者**: 游玩 | **级别**: L1-实操 | **标签**: `数值平衡` `蒙特卡洛` `概率验证` `自动优化`

---

## 为什么需要蒙特卡洛

游戏中的随机系统（掉落、匹配、战斗）靠人脑推演容易遗漏极端情况。跑 10 万次模拟比玩 100 局测试可靠得多。

---

## 六步验证流程

```
1. 定义成功指标（胜率、通关率、平均时长）
2. 编写模拟函数（纯逻辑，无渲染）
3. 设置样本量（见下方指南）
4. 运行模拟 → 收集数据
5. 统计分析（均值、方差、分位数）
6. 与设计预期对比 → 调参或确认
```

---

## 四类可检测问题

| 类型 | 示例 | 检测方法 |
|------|------|----------|
| 隐藏 BUG | 某条件组合必定失败 | 失败率异常偏离预期 |
| 极端概率 | 连续50次不出货 | P99/P999 分位数 |
| 设计不匹配 | 实际胜率30%（设计50%） | 均值偏离 |
| 长期退化 | 经济系统通胀 | 时序分析曲线 |

---

## 自动优化循环

```
初始参数 → 蒙特卡洛模拟 → 评估指标
                ↑                ↓
          调整参数 ← 不满足预期？
                         ↓ 满足
                    输出最终参数
```

**搜索策略**：
- 单参数：二分法/黄金分割
- 多参数：网格搜索 → 梯度下降
- 离散参数：遗传算法/模拟退火

## 样本量指南

| 目标精度 | 样本量 | 适用场景 |
|----------|--------|----------|
| 粗估（±5%） | 1,000 | 快速验证方向 |
| 标准（±1%） | 10,000 | 日常调参 |
| 精确（±0.1%） | 1,000,000 | 低概率事件（SSR掉率） |
| 极端（±0.01%） | 100,000,000 | 保底机制验证 |

## 高级技巧

### 分层搜索

先粗粒度扫描参数空间，找到"好区域"后细粒度搜索：

```
第一轮：步长 10，范围 [0, 100] → 找到峰值 ~40
第二轮：步长 1，范围 [30, 50]  → 找到最优 42
第三轮：步长 0.1，范围 [41, 43] → 最终 42.3
```

### 并行评估

多组参数同时跑模拟（利用 Lua 协程或多次调用），对比选最优。

### 多算法融合

先用遗传算法找大方向，再用梯度下降精调。

---

## 工程建议

1. **模拟函数与游戏逻辑共用同一份代码**，避免"模拟对了，实际不一样"
2. **固定随机种子**可复现问题局
3. **记录每次调参的模拟结果**，方便回溯决策
4. **设置合理中止条件**（收敛阈值），避免无限跑

---

## 常见坑

| 错误 | 后果 |
|------|------|
| 模拟逻辑与实际逻辑不同步 | 调出来的参数在游戏里不生效 |
| 样本量不够就下结论 | 被随机波动误导 |
| 只看均值不看方差 | 忽略"运气极差玩家"的体验 |
| 忘记极端情况（空背包、满级） | 线上炸服 |
