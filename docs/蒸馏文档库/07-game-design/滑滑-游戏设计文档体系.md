## 二、三级文档层次

| 层级 | 文件 | 职责 | 长度 |
|------|------|------|------|
| **索引** | `index.md` | 项目全貌，链接所有文档 | 1 页 |
| **概述** | `*/overview.md` | 该领域的设计要点 | 2-5 页 |
| **内容** | `*/xxx.md` | 具体设计细节 | 按需 |

### index.md 模板

```markdown
# [游戏名称] 设计文档索引

## 项目概要
- **类型**: [游戏类型]
- **平台**: [目标平台]
- **核心体验**: [一句话描述]
```

## 三、四大设计框架

在撰写设计文档时，可以融合以下框架进行结构化思考：

### 3.1 MDA 框架（Mechanics-Dynamics-Aesthetics）

```
Mechanics（机制）→ Dynamics（动态行为）→ Aesthetics（体验）

设计方向：从体验出发 → 反推需要什么行为 → 设计支撑的机制
分析方向：从机制出发 → 预测产生什么行为 → 评估带来什么体验
```

### 3.2 核心循环（Core Loop）

```
行为 → 奖励 → 成长 → 新内容 → 行为（循环）

示例（Roguelike）：
战斗 → 获得金币/装备 → 角色变强 → 解锁新关卡 → 更难的战斗
```

### 3.3 设计支柱（Design Pillars）

3-5 个最重要的设计原则，所有设计决策必须服务于这些支柱：

```markdown
## 设计支柱
1. **节奏感至上** — 一切玩法服务于音乐节奏体验
2. **随机不随意** — Roguelike 随机性要有策略深度
3. **上手即享受** — 新手 30 秒内体验到核心乐趣
```

### 3.4 体验目标（Experience Goals）

描述玩家在不同阶段的目标体验：

| 阶段 | 时长 | 体验目标 |
|------|------|---------|
| 首次启动 | 0-30秒 | 被视觉和音乐吸引 |
| 新手教学 | 30秒-3分钟 | 理解操作，感受节奏 |
| 核心体验 | 3-15分钟 | 进入心流，沉浸在节奏中 |
| 长期目标 | 多次游玩 | 解锁新内容，挑战高分 |

## 四、分角色撰写指南

### 策划（Gameplay Designer）

重点文档：`gameplay/` 全部
- 核心循环定义清晰，用流程图描述
- 数值设计有公式和参考表
- 每个系统标注优先级（P0/P1/P2）

### 美术（Art Director）

重点文档：`art/style-guide.md`
- 色板、参考图、风格关键词
- 角色/UI/场景的尺寸规范
- 资源命名规则和格式要求

### 音频（Sound Designer）

重点文档：`audio/`
- 列出每个音效的触发条件和情绪
- 音乐分场景列表（菜单/战斗/胜利/失败）
- 音量层级规范

### 技术（Tech Lead）

重点文档：`tech/architecture.md`
- 模块划分和数据流图
- 性能预算（帧率/内存/加载时间）
- 第三方依赖和技术风险

## 五、AI 辅助撰写建议

1. **先用创意指导 Skill** 确定核心方向，输出设计概要
2. **以设计概要为输入**，让 AI 生成 index.md 骨架
3. **逐文档展开**：每次聚焦一个文档，提供足够上下文
4. **版本迭代**：文档随开发迭代，每个版本标注修改日期
