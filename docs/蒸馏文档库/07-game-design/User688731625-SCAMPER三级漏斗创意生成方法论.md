| 字段 | 值 |

## 核心思路

**问题**：直接问 AI "帮我想个游戏创意" → 产出泛泛、缺乏验证、容易踩已知坑。

**解决**：用 SCAMPER 创意工具 + 三级漏斗（生成→筛选→对抗）+ 失败模式累积，让 AI 在结构化流程中自主探索，并用对抗性挑战杀死弱方案。

### 流程总览

```
用户模糊描述
  │
  ├─ ① 信号提取 ──→ 显式/隐式/矛盾信号 + 约束边界
  │
  ├─ ② SCAMPER 拆解 ──→ 7 透镜 × 核心信号 → 碎片池（8~20 个）
  │
  ├─ ③ 种子生成 ──→ 碎片组合 → 5 个创意种子（约束检查）
  │
  ├─ ④ 筛选 ──→ 5 维排序淘汰 → 3 个存活种子        ← 第一级漏斗
  │
  ├─ ⑤ 对抗性挑战 ──→ 每个种子找 3 个致命缺陷       ← 第二级漏斗
  │     │                能化解→保留，FATAL→淘汰
  │     │
  │     ├─ 有存活 → 输出最终方案
  │     └─ 全淘汰 → 失败模式入表 → 回到 ②          ← 第三级漏斗（迭代）
  │
  └─ 最多 3 轮全淘汰 → 停止，输出失败模式汇总 + 建议调整约束
```

---

## 各步骤详解

### 步骤 1：信号提取

从用户描述中拆出三类信号：

| 类型 | 来源 | 示例 |
|------|------|------|
| **显式信号** | 用户直接说的 | "我想做一个 roguelike" |
| **隐式信号** | 从措辞/上下文推断 | 用户说"小而精" → 范围恐惧 |
| **矛盾信号** | 两条信号互相冲突 | "简单操作" vs "深度策略" → 需化解 |

然后输出 **约束/开放空间表**：

```
约束条件（不可动）              | 开放空间（可自由发挥）
─────────────────────────────┼─────────────────────────
从显式+隐式信号中提取的硬限制   | 用户未限定的所有维度
```

如果有前轮积累的**失败模式表**，同步加载为附加硬约束。

### 步骤 2：SCAMPER 7 透镜拆解

选取最核心的 2~4 个信号，每个过 7 个透镜：

| 透镜 | 全称 | 思考方向 | 碎片格式 |
|------|------|---------|---------|
| **S** | Substitute | 把 X 换成 Y | "把 __ 换成 __" |
| **C** | Combine | X 与其他领域结合 | "X 与领域 Z 的 W 结合" |
| **A** | Adapt/类比 | 从其他领域映射结构 | "领域 D 的结构 E 可映射" |
| **M** | Magnify/放大 | 推到极端 | "如果只能 __" |
| **P** | Put to other use | 改变用途 | "本来用于 A，改为用于 B" |
| **E** | Eliminate | 去掉后反而更好 | "去掉 X 后，反而 __" |
| **R** | Reverse | 反转方向 | "不是 A→B，而是 B→A" |

**每个透镜输出 0~2 个碎片**（不强求每个都产出）。

**碎片合格标准**：能用一句话说清"改变了什么，带来了什么新可能"。做不到 → 丢弃。

**硬门槛**：碎片池 < 6 个 → 回到本步骤追加信号或换透镜角度，**禁止进入下一步**。

### 步骤 3：种子生成

从碎片池组合出 **5 个创意种子**（每个种子 = 1~3 个碎片的组合）：

```
种子 #N: [一句话方向描述]
来源透镜: [S+E]
覆盖维度: [解决了核心问题的哪个方面]
```

**生成后立即做约束检查**：逐一对照约束条件表 + 失败模式表，违反任一条 → 直接替换，不进入筛选。

### 步骤 4：筛选（5→3）

用 5 个维度做**相对排序**（不打分，只排序）：

| 维度 | 判断问题 |
|------|---------|
| 核心循环清晰度 | 能否一句话说清"做什么→得到什么→为什么继续"？ |
| 技术可行性 | 目标技术栈和资源下能否实现？ |
| 差异化 | 是否与已知方案高度同质？ |
| 情感匹配 | 传递的体验是否符合用户目标情感？ |
| 范围可控性 | 执行者能否在合理周期内完成？ |

**淘汰规则**：
- 任一维度排末位且无法补救 → 淘汰
- 同分时优先淘汰**差异化最弱**的种子
- 恰好淘汰 2 个，保留 3 个

### 步骤 5：对抗性挑战（最关键）

**对每个存活种子，找 3 个致命缺陷**：

```
种子 #N: [名称]
缺陷 1: [描述]
  具体环节: ____
  后果: ____
  化解方案: [能化解 → 写方案 / 不能化解 → 标记 FATAL]
缺陷 2: ...
缺陷 3: ...
结论: [存活 / 淘汰]
```

**判定规则**：
- 3 个缺陷全部化解 → 存活
- 任意 1 个标记 FATAL → 淘汰
- 淘汰时，将失败原因提炼为"失败模式"追加到表中

**关键禁令**：不能因为偏爱某个种子就给它找容易化解的缺陷。

### 步骤 6：迭代控制

| 情况 | 处理 |
|------|------|
| **有种子存活** | 如果化解方案修改了原设计 → 修正版再过一轮对抗性挑战；通过后输出最终方案 |
| **全部淘汰** | 汇总新增失败模式 → 追加到表 → 回到步骤 2 重新拆解 |
| **累计 3 轮全淘汰** | 停止迭代 → 输出失败模式汇总 → 建议用户调整约束条件 |

---

## 失败模式表

跨轮次积累的资产，格式固定：

```
失败模式（简短命名）              | 避免措施
──────────────────────────────┼──────────────────
描述：当 X 时会导致 Y            | 具体的设计规避策略
```

**维护规则**：
1. 每次淘汰种子时必须提炼失败模式并追加
2. 不删除已有条目
3. 两条可合并为更通用表述时 → 合并后保留通用版本
4. 步骤 2 生成碎片时必须逐条检查失败模式表

## 最佳适用场景

| 适合 | 不适合 |
|------|--------|
| AI 自主生成创意（最佳适配） | 关卡设计、操作性设计（AI 目前不擅长） |
| 模糊创意 → 可执行方案 | 已有明确方案只需实现 |
| 已有核心创意，充实其他板块 | 纯技术实现问题 |
| 希望大幅解放手脑 | 需要精确数值调参 |

## 使用方式

将上述完整流程（步骤 0~9）直接粘贴给 AI 助手，然后用一段话描述你的模糊需求即可启动。AI 会自动执行信号提取 → SCAMPER → 种子 → 筛选 → 对抗 → 迭代的完整流程。

## 踩坑备忘

| 问题 | 原因 | 解决 |
|------|------|------|
| AI 跳过对抗性挑战 | 流程提示词中未强调"不可跳步" | 在禁令中明确写死 |
| 碎片池太少导致种子同质 | 只用了 1~2 个信号过透镜 | 强制要求碎片 ≥ 6 个 |
| 种子全部存活（对抗太水） | AI 带入选择偏见 | 要求子代理独立挑战，不给选择理由 |
| 3 轮全淘汰后用户无从下手 | 没有输出失败模式汇总 | 按格式输出 + 建议调整约束 |
