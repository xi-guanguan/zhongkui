# 素材蒸馏工作流

> 把散落各处的经验素材（文字、文档、截图、文件夹等）蒸馏成 **Skill 文档**，供其他 AI 助手直接阅读和执行。
>
> **蒸馏 ≠ 精简**。蒸馏的目标不是缩短篇幅，而是将零散素材重新组织为**AI 可读、可执行的 Skill 文档格式**——结构完整、指令明确、上下文自足，让任何 AI 拿到就能用。

---

## 一、工作流总览

**分两个阶段，先全部导入，再统一整理**：

```
阶段一：批量导入（当前阶段）
─────────────────────────
用户发素材（文字/文档/截图/文件夹/链接…）
  ↓
【暂存】立即将原文存入 docs/蒸馏文档/原文暂存/（防止 context 压缩丢失原文）
  ↓
AI 阅读、理解、提炼
  ↓
蒸馏为 Skill 格式的 Markdown
  ↓
统一平铺保存到 docs/蒸馏文档/（不分文件夹、不写同类对比）
  ↓
继续导入下一批，直到全部导入完成

阶段二：分类整理（全部导入完成后）
─────────────────────────
审视所有已蒸馏文档
  ↓
识别同主题文档 → 建文件夹归类
  ↓
给同主题文档补写"同类对比"（特色/优点/缺点）
  ↓
用户确认后推送到 cnb.cool 仓库 → 同事拉取建站
```

---

## 二、输入：素材类型与处理方式

| 素材类型 | 来源举例 | AI 处理方式 |
|---------|---------|------------|
| 纯文字 | 对话框直接打字、复制粘贴 | 直接提炼结构化 |
| 项目文档 | 打开 .md/.txt 文件 | 读取 → 提炼核心内容，去掉项目特定细节 |
| 文件夹 | 打开整个目录 | 遍历 → 按主题归类蒸馏 |
| 截图 | 对话框贴图 | 识别内容 → 转为文字描述 + 要点提炼 |
| 网页/链接 | 发 URL | 抓取 → 提炼关键信息 |
| 代码片段 | 直接贴代码或指定文件 | 提取模式、技巧、最佳实践 |

## 三、输出：蒸馏文档规范

### 3.1 文件命名规范

**格式**：`[主题名]-by[作者名]-L[推荐度].md`

```
示例：
  记忆系统部署-by嗒啦啦-L1.md
  游戏共创协议-by嗒啦啦-L2.md
  网络同步规范-by天一-L3.md
```

**命名字段说明**：

| 字段 | 说明 |
|------|------|
| 主题名 | 文档核心内容的简要描述 |
| 作者名 | 素材原作者（同类文档可能有多个作者版本，效果差异大，必须区分） |
| 推荐度 | 1 / 2 / 3 三级评定（见下表） |

**推荐度分级**：

| 等级 | 含义 | 说明 |
|------|------|------|
| **L1** | 可以直接用 | 经过实战验证，部署即生效，效果稳定 |
| **L2** | 可以参考学习 | 思路有价值，但可能需要根据项目调整才能用 |
| **L3** | 谨慎使用 | 未经充分测试 / 实战中有已知缺点 / 效果不稳定 |

### 3.2 文档结构模板

每篇蒸馏文档**必须**以元信息头开始：

```markdown
# [标题]

> **这是什么**：一句话说明这篇文档蒸馏的是什么内容、解决什么问题。
>
> **原作者**：xxx
> **推荐度**：L1 / L2 / L3（附简要理由）
> **适用场景**：什么情况下应该读这篇文档
>
> **同类对比**（同主题有多个版本时必填）：
> - **特色**：这个版本和其他版本的核心区别是什么
> - **优点**：相比其他版本好在哪里
> - **缺点**：相比其他版本差在哪里 / 有什么局限

---

（正文内容：完整的 Skill 文档，AI 读完即可执行）

---

*素材来源：[来源说明]*
*蒸馏日期：YYYY-MM-DD*
```

> **关于同类对比**：同一主题（如"记忆系统"）可能有多个作者的不同版本，效果天差地别。同类对比帮助读者快速判断该用哪个版本，而不是逐篇读完再做选择。只有同主题存在多个版本时才需要填写。
>
> **⚠️ 阶段一（批量导入）时不写同类对比**，元信息只填：这是什么、原作者、推荐度、适用场景。等全部导入完成后（阶段二）再统一识别同主题、建文件夹、补写对比。

### 3.3 存放位置

```
docs/蒸馏文档/
├── 蒸馏工作流.md                          # 本文档
├── [主题]-by[作者]-L[等级].md             # 独立主题的文档
└── [主题名]/                              # 同主题多版本 → 放同一文件夹
    ├── [主题]-by[作者A]-L[等级].md
    └── [主题]-by[作者B]-L[等级].md
```

**文件夹规则**：同一主题有多个作者/版本的文档时，**必须建文件夹归类**，不要散落在根目录。
例如都是讲记忆系统的 → 放进 `记忆系统/` 文件夹。

> **⚠️ 阶段一先全部平铺在根目录**，阶段二再建文件夹归类。

### 3.4 蒸馏原则

**蒸馏 ≠ 精简删减，蒸馏 = 重组为 AI 可执行的 Skill 文档**。

1. **AI 可读** — 文档是写给 AI 的，结构、指令、条件判断要明确，AI 拿到就能按步骤执行
2. **上下文自足** — 每篇文档独立完整，不依赖外部隐含知识，AI 无需额外搜索就能理解
3. **去项目化** — 去掉特定项目的变量名、文件路径等，提炼为通用方案
4. **保留关键细节** — 踩坑经验、边界条件、反模式等实战细节必须保留，这些是价值核心
5. **标注来源** — 每篇底部注明素材来源，方便回溯和对比不同作者版本
