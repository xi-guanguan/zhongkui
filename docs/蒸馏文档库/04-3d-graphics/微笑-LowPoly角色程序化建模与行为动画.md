# Low Poly 角色程序化建模与行为动画

> **这是什么**: 用基础几何体（Box/Sphere/Cylinder）拼装 Low Poly 角色模型，配合状态机驱动程序化动画的完整方法论。以狗狗角色为案例，含材质配色、模型结构、行为状态机、动画参数。
>
> **原作者**: 微笑
> **推荐度**: L2（方法论通用，原始代码为 Three.js，移植到 UrhoX 需适配几何体 API）
> **适用场景**: 需要程序化生成 Low Poly 风格角色（动物、NPC）并赋予行为动画时

---

## 一、核心思路

不依赖外部模型文件，用引擎内置几何体（Box、Sphere、Cylinder）拼装角色，再用 sin/cos 驱动各部位的程序化动画。

**优势**: 零美术依赖、运行时生成、参数化可调、配色方案可热切换。

## 二、材质配色系统

### 基础配色表

为角色定义一组材质 key，每个 key 对应一个颜色值和用途：

| 材质 key | 颜色 | 用途 |
|----------|------|------|
| `body` | 金棕 `#C8A26A` | 身体、头部、腿 |
| `belly` | 浅米 `#FFF3E0` | 肚子、口鼻部 |
| `ear` | 深棕 `#8D6E63` | 耳朵 |
| `nose` | 黑色 `#212121` | 鼻子 |
| `eye` | 黑色 `#212121` | 眼睛 |
| `tongue` | 粉色 `#F48FB1` | 舌头 |
| `tail` | 金棕 `#C8A26A` | 尾巴 |
| `collar` | 红色 `#E53935` | 项圈（装饰） |

### 配色变体（四季）

同一套材质 key，替换颜色值即可实现风格切换：

| key | 春 | 夏 | 秋 | 冬 |
|-----|----|----|----|----|
| body | `#C8A26A` | `#B8924A` | `#A07840` | `#BCAAA4` |
| belly | `#FFF3E0` | `#FFE0B2` | `#FFD9A0` | `#F5F5F5` |
| tongue | `#F48FB1` | `#EF5350` | `#E57373` | `#FFCDD2` |
| collar | `#E53935` | `#C62828` | `#B71C1C` | `#E57373` |

**设计模式**: 材质 key 不变 → 颜色值查表替换 → 同一模型多套皮肤。

## 三、模型结构树

用"部位表"描述每个部件的几何体、尺寸、位置、材质映射：

```
角色模型 (Group)
├── 身体   Box(0.3, 0.25, 0.55)    y=0.22              → body
├── 肚子   Box(0.22, 0.12, 0.4)    y=0.12              → belly
├── 头部   Box(0.3, 0.26, 0.28)    y=0.35, z=0.35      → body
├── 口鼻   Box(0.18, 0.14, 0.16)   y=0.3, z=0.52       → belly
├── 鼻子   Box(0.08, 0.06, 0.05)   y=0.33, z=0.6       → nose
├── 左眼   Sphere(r=0.035)         x=-0.08, y=0.38, z=0.48  → eye
├── 右眼   Sphere(r=0.035)         x=+0.08, y=0.38, z=0.48  → eye
├── 左耳   Box(0.1, 0.18, 0.08)    x=-0.16, rotZ=0.3   → ear
├── 右耳   Box(0.1, 0.18, 0.08)    x=+0.16, rotZ=-0.3  → ear
├── 舌头   Box(0.06, 0.02, 0.08)   y=0.24, z=0.56      → tongue
├── 项圈   Box(0.32, 0.05, 0.32)   y=0.32, z=0.22      → collar
├── 前左腿 Cylinder(r=0.045, h=0.16) x=-0.1, z=0.18    → body
├── 前右腿 Cylinder(r=0.045, h=0.16) x=+0.1, z=0.18    → body
├── 后左腿 Cylinder(r=0.045, h=0.16) x=-0.1, z=-0.18   → body
├── 后右腿 Cylinder(r=0.045, h=0.16) x=+0.1, z=-0.18   → body
├── 尾巴1  Cylinder(0.04→0.03, h=0.2)  rotX=-0.8       → tail
└── 尾巴2  Cylinder(0.03→0.02, h=0.12) rotX=-1.2       → tail
```

**要点**:
- 对称部件（眼/耳/腿）用相同几何体 + 镜像 X 坐标
- 锥形部件（尾巴）用上下半径不等的 Cylinder
- 垂耳效果通过 rotation.z 实现倾斜

## 四、行为状态机

### 状态数据结构

```
dogData = {
    originX/Z     -- 放置原点
    targetX/Z     -- 移动目标
    speed         -- 移动速度（0.007 ~ 0.013）
    state         -- 当前状态: idle / walking / sitting / running
    stateTimer    -- 状态计时器
    stateDuration -- 当前状态持续时间
    walkPhase     -- 走路动画相位
    wanderRadius  -- 活动范围半径（3.5）
    tailWagSpeed  -- 尾巴摇摆速度（8）
}
```

### 状态切换概率

| 状态 | 概率 | 持续时间 |
|------|------|---------|
| walking | 35% | 2~5 秒 |
| idle | 25% | 1~3.5 秒 |
| running | 20% | 1.5~3.5 秒 |
| sitting | 20% | 2~5 秒 |

切换时机: `stateTimer >= stateDuration` 时随机选下一状态。walking/running 到达目标点也会触发切换。

### 状态转换图

```
    ┌── idle (25%) ──┐
    │        ↕       │
walking (35%) ↔ running (20%)
    │        ↕       │
    └─ sitting (20%) ┘
```

所有状态可互相转换，由概率决定。

## 五、程序化动画参数

### walking（行走）

| 部位 | 公式 | 说明 |
|------|------|------|
| 前左腿/后右腿 | `sin(walkPhase) * 0.35` | 对角线交替 |
| 前右腿/后左腿 | `-sin(walkPhase) * 0.35` | 反相 |
| 尾巴1 | `sin(time * wagSpeed) * 0.4` | 摇摆 |
| 尾巴2 | `sin(time * wagSpeed + 0.5) * 0.5` | 延迟跟随 |
| 耳朵 | `±sin(walkPhase * 0.5) * 0.1` | 轻微晃动 |
| 身体 Y | `0.2 + |sin(walkPhase)| * 0.015` | 微弹跳 |

### running（奔跑）

在 walking 基础上放大：
- 移动速度 × 2.5
- 腿部摆幅 0.35 → **0.6**
- 尾巴摇摆速度 × 1.5，幅度 0.4 → **0.5/0.6**
- 舌头伸出: `scale.y = 1.5`
- 耳朵后飘: `rotation.x = 0.3`
- 身体弹跳 0.015 → **0.04**

### idle（发呆）

| 部位 | 公式 | 说明 |
|------|------|------|
| 身体 Y | `0.2 + sin(time * 2) * 0.005` | 呼吸起伏 |
| 腿部 | `rotation.x *= 0.9` | 缓慢归零 |
| 尾巴 | `sin(time * 3) * 0.25` | 慢速摇（始终摇） |
| 舌头 | `scale.y = lerp(scale.y, 1.0, 0.05)` | 缓慢收回 |

### sitting（坐下）

| 部位 | 公式 | 说明 |
|------|------|------|
| 身体 Y | `0.16 + sin(time * 1.5) * 0.003` | 降低 + 微呼吸 |
| 后腿 | `rotation.x → 0.5`（缓动） | 弯曲 |
| 前腿 | `rotation.x *= 0.9` | 恢复 |
| 尾巴 | `sin(time * 2) * 0.15` + 偏移 0.2 | 轻摇 |

## 六、角色差异化设计模式

以猫 vs 狗为例，展示如何复用同一套系统产出不同角色：

| 属性 | 猫 | 狗 |
|------|----|----|
| 移动速度 | `0.005~0.01` | `0.007~0.013`（更快） |
| 活动范围 | 2.5 | 3.5（更大） |
| 行为状态 | idle/walking/sitting | + **running**（独有） |
| 特有部件 | 三角耳（Cone）、无项圈 | 垂耳（Box+旋转）、舌头、项圈 |
| 尾巴特征 | 慢速 | 快速（wagSpeed=8），发呆也摇 |
| 奔跑特效 | 无 | 舌头伸出 + 耳朵后飘 |

**复用策略**: 同一套「材质表 + 结构树 + 状态机」框架，通过替换几何参数、增减部件、调整动画系数来派生新角色。

## 七、生命周期管理

```
材质定义（初始化时，定义颜色映射表）
  ↓
模型创建（几何体拼装 + 行为数据初始化）
  ↓
放置到场景（检查合法性 → 设置原点 → 入场缩放动画 0→1）
  ↓
每帧更新（状态机驱动：切换状态 → 更新位置 → 驱动各部位动画）
  ↓
清除回收（从场景移除 → 清空引用数组）
```

**要点**: 角色存入独立数组（不占场景格子的 object 槽），可与其他物体共存于同一位置。
