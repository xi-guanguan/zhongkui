# 大场景优化技术指南

> **这是什么**：体素世界、大地图等大场景在手机端流畅运行的完整优化方案，涵盖架构、渲染、Lua 三层优化。
>
> **原作者**：糖心哥

> **适用场景**：开发 Minecraft 风格体素世界、大地图 RPG、开放世界等大场景游戏时

## 一、性能瓶颈三分法

优化前先定位瓶颈：

| 瓶颈 | 表现 | 判断方法 |
|------|------|---------|
| **Draw Call 过多** | 节点多、每个都有独立模型组件 | 场景中数千个独立节点 |
| **顶点/面数爆炸** | 精细模型×大量实例 | 单模型2万面×100个=200万面 |
| **Lua 逻辑过重** | 即使物体少帧率也低 | 每帧遍历大量数据、大量计算 |

> 手机 GPU 参考上限：Draw Call 几百个、面数几十万。

## 二、架构层：区块化（Chunking）

### 2.1 核心概念

将世界切成小区域独立管理，修改方块时只重建所在区块：

```
世界 192×80×192 → 切成 16×80×16 区块 → 12×12 = 144 个区块
挖一个方块 → 只重建该区块网格（非整个世界）
```

### 2.2 区块大小推荐

| 场景类型 | 推荐区块大小 |
|---------|-------------|
| 体素世界 | 16×16（Minecraft 标准） |
| 2D 瓦片地图 | 16×16 或 32×32 |
| 3D 开放世界 | 32×32 ～ 64×64 |

### 2.3 帧预算控制

多个区块同时标脏时，**每帧最多重建 1-2 个**，其余排队：

```lua
-- 伪代码
local MAX_REBUILDS_PER_FRAME = 1
local rebuilt = 0
for _, chunk in ipairs(dirtyChunks) do
    if rebuilt >= MAX_REBUILDS_PER_FRAME then break end
    chunk:RebuildMesh()
    rebuilt = rebuilt + 1
end
```

## 三、渲染层：面剔除 + 合批 + 视距

### 3.1 相邻面剔除

体素世界最大的优化项——一个 16×80×16 区块的面数从 **122,880 降到几千**：

```
规则：遍历方块的 6 个方向，如果相邻方块是不透明实心 → 跳过该面
只有面朝空气或透明方块的面才生成顶点
```

**透明方块特殊处理**：
- 不透明旁边是水 → 不透明方块的面要画
- 水旁边是水 → 同类相邻不画
- 透明/不透明分两个 CustomGeometry，用不同材质

### 3.2 CustomGeometry 合批

把整个区块合成一个 CustomGeometry → **1 个区块 = 1 个 Draw Call**：

```lua
local geom = chunkNode:CreateComponent("CustomGeometry")
geom:BeginGeometry(0, TRIANGLE_LIST)

-- 遍历区块所有方块，做面剔除后添加可见面顶点
for x, y, z in traverseChunk() do
    for _, face in ipairs(visibleFaces) do
        geom:DefineVertex(face.v1)
        geom:DefineNormal(face.normal)
        geom:DefineTexCoord(face.uv1)
        -- ... 其余顶点
    end
end

geom:Commit()
geom:SetMaterial(atlasMaterial)
```

> 144 个区块 = 144 个 Draw Call（透明分开则 ×2），手机轻松承受。

### 3.3 纹理图集（Texture Atlas）

合批前提：同一批次必须同一材质。把所有方块贴图拼成一张大图：

```
16×16 图集 → 每格 UV = 1/16 = 0.0625
泥土在第2行第3列 → UV = (2/16, 1/16) ~ (3/16, 2/16)
每种方块可分 top/side/bottom 不同 UV
```

### 3.4 视距控制 + 雾效

```lua
-- 只渲染玩家周围 N 个区块
local RENDER_DISTANCE = 6  -- 区块单位

-- 雾效平滑过渡（避免区块突然消失）
zone.fogStart = 50   -- 50 米开始变浓
zone.fogEnd = 300    -- 300 米完全不可见

-- 给装饰物设 drawDistance（引擎内部跳过，比 Lua 手动隐藏高效）
treeModel.drawDistance = 200
```

## 四、Lua 层：性能技巧

### 4.1 常量本地化

热循环中把全局/表字段提取为局部变量：

```lua
-- 文件顶部
local CHUNK_SIZE = Config.World.CHUNK_SIZE
local BLOCK_SIZE = Config.World.BLOCK_SIZE
local floor = math.floor
local max = math.max

-- 循环中直接用局部变量，不再查表
for i = 1, CHUNK_SIZE do
    local idx = floor(pos / BLOCK_SIZE)
    -- ...
end
```

> 热路径中性能差距可达 **30%+**。

### 4.2 避免热路径创建对象

```lua
-- ❌ 每帧循环中创建临时 Vector3 → GC 压力巨大
for i = 1, 1000 do
    node.position = Vector3(x, y, z)  -- 每次 new
end

-- ✅ 预分配 + 复用（用完即走场景）
local tempVec = Vector3()
for i = 1, 1000 do
    tempVec.x = x; tempVec.y = y; tempVec.z = z
    node.position = tempVec
end
```

> 注意：tempVec 会被覆盖，需要保存值时仍需创建新 Vector3。

### 4.3 数值键代替字符串键

```lua
-- ❌ 慢：字符串拼接 + 字符串哈希
local key = chunkX .. "," .. chunkZ

-- ✅ 快：一次乘法 + 一次加法
local key = chunkX * 65536 + chunkZ
```

### 4.4 高度范围记录

记录每个区块方块的 minY/maxY，只遍历有方块的高度范围：

```lua
-- 设置方块时更新范围
function chunk:SetBlock(x, y, z, blockType)
    self.data[x][y][z] = blockType
    if y < self.minY then self.minY = y end
    if y > self.maxY then self.maxY = y end
    self.dirty = true
end

-- 重建时只遍历有效范围
for y = self.minY, self.maxY do
    -- ...
end
```

### 4.5 对象池

频繁创建/销毁的节点用池化复用，避免 GC 卡顿：

```lua
local pool = {}
local MAX_POOL = 50

function acquire()
    if #pool > 0 then
        local node = table.remove(pool)
        node.enabled = true
        return node
    end
    return scene_:CreateChild("Particle")
end

function release(node)
    node.enabled = false
    if #pool < MAX_POOL then
        pool[#pool + 1] = node
    else
        node:Remove()
    end
end
```

> 适用：粒子、子弹、敌人、临时向量、UI 列表项等。

## 五、世界大小预设

| 预设 | 渲染距离 | 世界尺寸 | 区块数 | 适用 |
|------|---------|---------|-------|------|
| tiny | 2 | ~64×64 | ~25 | 测试 |
| small | 4 | ~128×128 | ~81 | 手机流畅 |
| **medium** | **6** | **~192×192** | **~169** | **推荐默认** |
| large | 8 | ~256×256 | ~289 | 仅电脑 |

> 再大内存可能超 1GB，手机/网页端危险。

## 六、常见坑与对策

| 坑 | 原因 | 解决 |
|----|------|------|
| 区块重建卡顿 | 一帧重建太多 | 帧预算控制，每帧最多1个 |
| 远处突然冒出物体 | 无雾效过渡 | 加 fog，渲染距离 < fogEnd |
| 帧率忽高忽低 | GC 周期触发 | 对象池 + 避免热路径 new 对象 |
| 合批了还卡 | 总面数仍然高 | 检查面剔除是否生效 |
| 透明/不透明闪烁 | 混在同一 CustomGeometry | 分两个 CG，透明用 Alpha 材质 |
| 世界生成时卡数秒 | 一次性生成所有区块 | 分帧生成 + 进度条 |
