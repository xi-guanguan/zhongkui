## 1. 三层数据模型

```
Tier 1: 客户端内存     ← HeroData.state / 本地变量
Tier 2: 服务端内存     ← PlayerSession
Tier 3: 云数据库       ← serverCloud score / BatchSet
```

**铁律: 所有写操作必须贯穿三层**
- ❌ 只写 Tier 1 → 重连后丢失
- ❌ 只写 Tier 1+2 → 服务器重启后丢失
- ✅ 写 Tier 1+2+3 → 安全持久化

**货币例外**: 使用 `score.money:Add/Cost` 原子操作，不走 BatchSet（并发安全）。

## 2. 连接生命周期

### 握手流程

```
客户端连接 → Client 发 ClientIdentity
  → Server 缓存身份, 发 AssignPlayer
  → Client 收到后发 ClientReady
  → Server 收到后加载云数据(BatchGet)
  → Server 加载完成发 SyncPlayerData
  → Client 执行 ApplyServerSync
```

### 超时陷阱

| 问题 | 原因 | 方案 |
|------|------|------|
| 32s 连接超时 | BatchGet 可能耗时 27s+ | 先发 AssignPlayer，再异步加载 |
| ClientReady 丢失 | 网络抖动 | 客户端 8s 间隔重试，最多 3 次 |
| SyncPlayerData 丢失 | 服务器还在加载 | 客户端 3s 间隔拉取，最多 10 次 |

### 重连要点
1. 检查 pendingDisconnects_（旧连接保存未完成）
2. 更新 connection 引用
3. 执行 `_pendingReconnectCallback`
4. `loadingUsers_` 防重复加载

### 断线保存
- 断线时必须保存到云数据库
- 最多重试 3 次
- 每次检查是否已重连（避免覆盖新数据）

## 3. 同步策略

### Local-First 架构
- 本地有存档 → 保留本地，仅补全缺失字段
- 本地无存档 → 使用服务器数据（新玩家）

### 关键标志位

| 标志 | 作用 | 错误后果 |
|------|------|---------|
| `initialSyncApplied_` | 防后续 Sync 覆盖本地 | 被旧数据覆盖 |
| `connected_` | 标记连接就绪 | 购买/存档全部失败 |
| `_clientSaveReceived` | 已收到客户端存档 | 云数据覆盖客户端 |
| `_loadError` | 云加载失败 | 错误标记为新玩家→数据清空 |

### 货币 Delta 合并
```
delta = serverAfter - serverBefore
localNew = localCurrent + delta
```
不能直接覆盖——客户端可能有未提交的本地变动。设置 `CURRENCY_SYNC_MAX_DELTA` 安全上限。

## 4. serverCloud 存储

### 模块化存储（推荐）
```lua
MODULE_KEYS = {"core","talent","gacha","tower","rogue","checkin","commerce","misc"}
serverCloud:BatchGet(userId, keys, callback)
serverCloud:BatchSet(userId, keyValuePairs, callback)
```

**⚠️ persistent_world 下 clientScore/clientCloud 永远不可用！** 所有持久化必须走 serverCloud。

## 5. 昵称解析（多级回退）

```
1. connection.identity → 2. SERVER_PLAYER_AUTH_INFOS 缓存
→ 3. serverCloud 云缓存 → 4. GetUserNickname API
→ 5. 定时重试队列（2s间隔，最多10次）
```

## 6. 玩家行为驱动的强制同步

**核心**: 不依赖定时保存——每次关键交互立即 `SavePlayerData()` + `SendToServer()`。

**货币三道防线**:
1. 购买后强制写 clientScore
2. 服务端同步后强制写 clientScore（`forceWriteCloudAfterSync()`）
3. ApplyServerSync 末尾无条件写入

**统一取值入口**:
```lua
local function GetCurrency(key)
    local cloud = clientScore.Get(key)
    if cloud and cloud > 0 then return cloud end
    return G_PlayerData[key] or 0
end
```

**安全上限**: 所有货币写入经过 `math.max(0, math.min(newVal, 99999999))` clamp。

## 7. 高频 Bug 速查

### 数据丢失类
| Bug | 根因 | 修复 |
|-----|------|------|
| 奖励重连后消失 | 只写 Tier 1 | 三层写入 |
| 购买成功道具未到 | 回调遗漏发放 | 购买回调 checklist |
| 月卡购买后未保存 | 未触发存档 | 购买后强制 save |

### 连接/超时类
| Bug | 根因 | 修复 |
|-----|------|------|
| 进入后功能全失效 | AssignPlayer 丢失 | SyncPlayerData 修复 connected_ |
| 重复加载 | ClientReady 重试 | loadingUsers_ 守卫 |

### 崩溃类
| Bug | 根因 | 修复 |
|-----|------|------|
| 除零崩溃 | 未校验分母 | 所有除法前检查 ≠0 |
| nil 访问 | 异步回调中对象已销毁 | 回调首行校验有效性 |
