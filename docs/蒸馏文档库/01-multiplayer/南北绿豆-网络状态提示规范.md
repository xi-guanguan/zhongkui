# 网络状态提示规范

> **这是什么**: 移动端联网游戏的网络状态分级展示、断线重连 UI 和弱网策略设计规范。
> **原作者**: 南北绿豆
> **推荐度**: L2（设计规范参考，需根据项目适配实现）
> **适用场景**: 联网游戏需要展示网络状态、断线重连、加载状态时

---

## 1. 网络状态分级

| 状态 | 判定条件 | UI 表现 |
|------|---------|---------|
| 正常 | RTT < 100ms | 无提示 |
| 延迟较高 | 100~300ms | 黄色信号图标 |
| 严重延迟 | ≥ 300ms | 红色信号图标 + "网络不佳" |
| 断线 | 连续 3s 无响应 | 全屏半透遮罩 + 重连 |

**状态切换规则**:
- RTT 用最近 5 次滑动平均值（抖动过滤）
- 恶化立即响应，恢复需持续 2s 才切换

## 2. 断线重连

**流程**: 检测断线 → 半透遮罩 → 自动重连（最多 5 次，指数退避 1/2/4/8/16s）→ 成功则 Toast "已恢复连接" / 全部失败则显示 [重试] [返回主菜单]

| 参数 | 值 |
|------|------|
| 最大重连次数 | 5 |
| 退避策略 | 1s → 2s → 4s → 8s → 16s |
| 遮罩 | (0,0,0,128)，Z-Order 500 |

## 3. 加载状态策略

| 预期时长 | UI 策略 |
|---------|---------|
| < 1s | 不显示 |
| 1-3s | 行内 Spinner |
| 3-10s | 全屏加载页 + 进度 |
| > 10s | 全屏 + 进度 + 趣味内容 |

## 4. 弱网交互策略

- **乐观更新**: 操作立即本地生效，后台同步，失败回滚
- **请求队列**: 弱网时排队，恢复后按序发送
- **超时提示**: 单次 > 5s → "请求超时，请重试"
- **离线降级**: 非核心功能显示缓存 + "离线模式"标签
