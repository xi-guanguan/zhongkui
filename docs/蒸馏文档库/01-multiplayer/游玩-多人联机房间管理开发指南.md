## 架构基础

### 服务端权威模式

所有游戏逻辑在服务端执行，客户端只负责渲染和输入上报：

```
客户端 A ──→ 输入上报 ──→ 服务端（权威）──→ 状态广播 ──→ 客户端 A
客户端 B ──→ 输入上报 ──→     ↓           ──→ 状态广播 ──→ 客户端 B
                          游戏逻辑计算
                          数据持久化
```

### 两种联机模式

| 模式 | 特点 | 典型游戏 | 构建配置 |
|------|------|---------|---------|
| **Match（房间制对局）** | 匹配 → 开房 → 对战 → 结算 | LOL、CS2、PUBG | `match_info` |
| **Persistent World（常驻世界）** | 服务器长期运行，玩家随进随出 | Minecraft、Roblox | `persistent_world` |

## 代码组织

### 入口分流

单个入口文件根据运行模式加载不同模块：

```lua
-- scripts/main.lua
local Module

function Start()
    if IsServerMode() then
        Module = require("network.Server")
    elseif IsNetworkMode() then
        Module = require("network.Client")
    else
        -- 单机模式（multiplayer.enabled = false 时）
        Module = require("network.Standalone")
    end
    Module.Start()
end

function Update(timeStep)
    if Module and Module.Update then
        Module.Update(timeStep)
    end
end
```

```
scripts/
├── main.lua                 # 入口，分流 Server/Client
├── config/
```

## 远程事件通信

### 注册与发送

```lua
-- Shared.lua —— 双端都要注册
local Events = {
    "PlaceBid",       -- 客户端 → 服务端：下单
    "BidResult",      -- 服务端 → 客户端：下单结果
    "GameStateSync",  -- 服务端 → 客户端：状态同步
}
for _, name in ipairs(Events) do
    RegisterRemoteEvent(name)
end
```

### 客户端发送

```lua
-- Client.lua
function SendBid(itemId, price)
    local data = VariantMap()
    data["action"] = "bid"
    data["itemId"] = itemId
    data["price"] = price
    network:GetServerConnection():SendRemoteEvent("PlaceBid", true, data)
    -- 第二个参数 true = 可靠传输（TCP 语义）
end
```

### 服务端接收与回复

```lua
-- Server.lua
SubscribeToEvent("PlaceBid", function(eventType, eventData)
    local conn = eventData["Connection"]:GetPtr("Connection")
    local action = eventData["action"]:GetString()
    local itemId = eventData["itemId"]:GetString()
    local price = eventData["price"]:GetInt()

    -- 处理逻辑...
    local result = ProcessBid(conn, itemId, price)

    -- 回复该客户端
    local reply = VariantMap()
    reply["success"] = result.success
    reply["message"] = result.message
    conn:SendRemoteEvent("BidResult", true, reply)
end)
```

### 使用 cjson 传输复杂数据

`cjson` 是内置全局模块（无需 require），适合传递复杂结构：

```lua
-- 发送端
local data = VariantMap()
data["json"] = cjson.encode({
    items = { {id = 1, price = 100}, {id = 2, price = 200} },
    timestamp = os.time(),
})
conn:SendRemoteEvent("GameStateSync", true, data)

-- 接收端
local json = eventData["json"]:GetString()
local state = cjson.decode(json)
for _, item in ipairs(state.items) do
    print(item.id, item.price)
end
```

## 连接管理生命周期

### 三个阶段

```
客户端连接
  ↓
① ClientConnected    — 物理连接建立（此时还不知道是谁）
  ↓
② ClientIdentity     — 身份验证完成（获得 user_id）
  ↓
③ ClientReady        — 场景加载完成（可以开始游戏）
```

```lua
-- Server.lua
SubscribeToEvent("ClientConnected", function(eventType, eventData)
    local conn = eventData["Connection"]:GetPtr("Connection")
    print("[Server] 新连接，等待身份验证...")
end)

SubscribeToEvent("ClientIdentity", function(eventType, eventData)
    local conn = eventData["Connection"]:GetPtr("Connection")
    local userId = eventData["UserId"]:GetString()
    print("[Server] 玩家身份确认: " .. userId)
    -- 加载玩家数据、分配房间等
    InitPlayer(conn, userId)
end)

SubscribeToEvent("ClientReady", function(eventType, eventData)
    local conn = eventData["Connection"]:GetPtr("Connection")
    print("[Server] 玩家场景就绪，开始同步游戏状态")
    SyncGameState(conn)
end)
```

### 断开处理

```lua
SubscribeToEvent("ClientDisconnected", function(eventType, eventData)
    local conn = eventData["Connection"]:GetPtr("Connection")
    -- 保存玩家数据
    SavePlayerData(conn)
    -- 从房间移除
    RemoveFromRoom(conn)
    -- 通知其他玩家
    BroadcastPlayerLeft(conn)
end)
```

## 房间系统

### 双层架构

```
QueueManager（匹配层）       RoomManager（游戏层）
├── 接收排队请求              ├── 创建/销毁房间
├── 按规则匹配               ├── 管理房间内逻辑
├── 匹配成功 → 创建房间        ├── 处理玩家操作
└── 通知玩家进入房间           └── 结算和清理
```

### 玩家状态流转

```
lobby（大厅）→ queuing（排队中）→ in_room（房间内）→ lobby
                  ↓ 取消排队
                lobby
```

### QueueManager 核心

```lua
-- server/QueueManager.lua
local QueueManager = {}
local waitingPlayers = {}

function QueueManager.JoinQueue(conn, userId, options)
    table.insert(waitingPlayers, {
        conn = conn,
        userId = userId,
        joinTime = os.time(),
        options = options,  -- 段位、模式等匹配条件
    })
    TryMatch()
end

function QueueManager.LeaveQueue(conn)
    for i, p in ipairs(waitingPlayers) do
        if p.conn == conn then
            table.remove(waitingPlayers, i)
            return true
        end
    end
    return false
end

local function TryMatch()
    -- 简单示例：凑够 N 人就开房
    local ROOM_SIZE = 4
    if #waitingPlayers >= ROOM_SIZE then
        local players = {}
        for i = 1, ROOM_SIZE do
            table.insert(players, table.remove(waitingPlayers, 1))
        end
        RoomManager.CreateRoom(players)
    end
end

return QueueManager
```

### RoomManager 核心

```lua
-- server/RoomManager.lua
local RoomManager = {}
local rooms = {}
local nextRoomId = 1

function RoomManager.CreateRoom(players)
    local roomId = nextRoomId
    nextRoomId = nextRoomId + 1

    local room = {
        id = roomId,
        players = {},
        state = "waiting",  -- waiting → playing → finished
        createdAt = os.time(),
    }

    for _, p in ipairs(players) do
        room.players[p.userId] = {
            conn = p.conn,
            score = 0,
            state = "active",
        }
        -- 通知客户端进入房间
        NotifyJoinRoom(p.conn, roomId)
    end

    rooms[roomId] = room
    return roomId
end

function RoomManager.HandleAction(roomId, userId, action)
    local room = rooms[roomId]
    if not room or room.state ~= "playing" then return end
    -- 处理游戏逻辑...
end

function RoomManager.DestroyRoom(roomId)
    local room = rooms[roomId]
    if room then
        for userId, p in pairs(room.players) do
            NotifyLeaveRoom(p.conn)
        end
        rooms[roomId] = nil
    end
end

return RoomManager
```

## serverCloud 数据持久化

### 基础 API

```lua
-- 读写单个 key
serverCloud:Get(key, function(success, value) end)
serverCloud:Set(key, value)
serverCloud:SetInt(key, intValue)
serverCloud:Add(key, delta)     -- 原子增减
serverCloud:Delete(key)

-- 批量操作（原子性）
local batch = serverCloud:BatchCommit()
batch:Set("player:1:gold", 100)
batch:Set("player:1:level", 5)
batch:Submit(function(success) end)

-- 批量读取
serverCloud:BatchGet({"key1", "key2", "key3"}, function(success, results) end)
```

### 子对象

```lua
-- 货币系统
serverCloud.money:Add("player:1", "gold", 100)
serverCloud.money:Get("player:1", "gold", function(success, amount) end)

-- 列表（排行榜数据）
serverCloud.list:Append("room:1:history", eventData)
serverCloud.list:GetRange("room:1:history", 0, 10, function(success, items) end)

-- 消息队列
serverCloud.message:Send("player:1", "inbox", messageData)
serverCloud.message:Receive("player:1", "inbox", function(success, messages) end)

-- 配额/限额
serverCloud.quota:Check("player:1", "dailyTrades", function(success, remaining) end)
```

## 9 个常见陷阱

### 1. 客户端 CreateChild 没指定 LOCAL

客户端创建的节点默认会被网络同步，导致冲突：

```lua
-- ❌ 会被同步到其他客户端
local node = scene_:CreateChild("LocalEffect")

-- ✅ 指定 LOCAL，仅本地可见
local node = scene_:CreateChild("LocalEffect", LOCAL)
```

### 2. 用 Remove() 而不是 Dispose()

网络节点需要用 `Dispose()` 正确清理：

```lua
-- ❌ 可能导致其他客户端残留
node:Remove()

-- ✅ 正确的网络节点移除
node:Dispose()
```

### 3. NodeAdded 时节点未就绪

```lua
-- ❌ NodeAdded 事件触发时，组件可能还没同步完
SubscribeToEvent("NodeAdded", function(eventType, eventData)
    local node = eventData["Node"]:GetPtr("Node")
    local model = node:GetComponent("StaticModel")  -- 可能是 nil！
end)

-- ✅ 使用 DelayedStart 或延迟一帧
function DelayedStart()
    -- 此时组件已同步完成
end
```

### 4. 忘记注册远程事件

```lua
-- ❌ 事件未注册，发了也收不到
conn:SendRemoteEvent("MyEvent", true, data)

-- ✅ 双端都要注册
RegisterRemoteEvent("MyEvent")
```

### 5. 不区分可靠/不可靠传输

```lua
-- 关键操作（交易、登录）→ 可靠传输（true）
conn:SendRemoteEvent("PlaceBid", true, data)

-- 高频状态同步（位置、旋转）→ 不可靠传输（false）更高效
conn:SendRemoteEvent("PosSync", false, data)
```

### 6. 服务端没做输入校验

```lua
-- ❌ 信任客户端数据
function OnPlaceBid(conn, price)
    SetPrice(price)  -- 客户端说多少就多少？
end

-- ✅ 服务端校验
function OnPlaceBid(conn, price)
    if price <= 0 or price > MAX_PRICE then return end
    if GetPlayerGold(conn) < price then return end
    SetPrice(price)
end
```

### 7. 关服时不保存数据

参考染墨轻尘的"关服生命周期重构"方案。

### 8. 在 ClientConnected 就开始发游戏数据

此时身份未验证，应该等 `ClientIdentity` 之后。

### 9. cjson 传输忘记处理 nil

```lua
-- ❌ nil 字段在 JSON 中会丢失
local data = { name = "test", score = nil }
local json = cjson.encode(data)  -- {"name":"test"} — score 丢了

-- ✅ 用 cjson.null 或默认值
local data = { name = "test", score = 0 }
```

## 开发顺序建议

1. **先跑通单机** → 确认游戏逻辑正确
2. **加入 Server/Client 分流** → `IsServerMode()` / `IsNetworkMode()`
3. **实现远程事件** → 一个简单的请求-响应
4. **添加连接管理** → ClientIdentity + ClientDisconnected
5. **实现房间系统** → QueueManager + RoomManager
6. **接入 serverCloud** → 数据持久化
7. **优化和防御** → 校验、断线重连、异常处理
