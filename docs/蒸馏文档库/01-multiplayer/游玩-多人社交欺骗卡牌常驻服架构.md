## 匹配系统

### FIFO 队列

```
玩家点击「开始匹配」
  → 客户端发 C2S_JOIN_QUEUE
  → 服务端 MatchQueue:Add(player)
  → 队列人数 >= 4？
    ├─ 是 → 取前 4 人创建 Room
    └─ 否 → 等待（15 秒超时后 AI 补位）
```

### AI 补位策略

| 模式 | 触发条件 | AI 数量 |
|------|---------|---------|
| 正常匹配 | 队列 ≥ 4 人 | 0 |
| 超时补位 | 等待 > 15 秒 | 补到 4 人 |
| 单人模式 | 玩家选择「单人游戏」 | 3 个 AI |

## 房间状态机

```
WAITING → DEALING → TURN_ACTIVE → RESOLVING → ROULETTE → GAME_OVER
   │                    ↑              │
   │                    └──────────────┘  (下一个玩家回合)
   │
   └─ 人数不足 → 直接 GAME_OVER
```

| 状态 | 说明 | 超时 |
|------|------|------|
| WAITING | 等待所有玩家确认就绪 | 30s |
| DEALING | 发牌动画（服务端发牌 → 广播 → 客户端播放） | 3s |
| TURN_ACTIVE | 当前玩家出牌/操作 | 30s（超时自动出牌） |
| RESOLVING | 判定出牌结果（真话/谎话/质疑） | 瞬时 |
| ROULETTE | 俄罗斯轮盘阶段（被抓说谎者开枪） | 5s |
| GAME_OVER | 结算（最后存活者获胜） | - |

## 游戏规则

### 牌组

| 牌面 | 数量 | 说明 |
|------|------|------|
| K | 6 | 普通牌 |
| Q | 6 | 普通牌 |
| A | 6 | 普通牌 |
| Joker | 2 | 万能牌（可代替任意牌面） |
| **合计** | **20** | 每人最多 5 张 |

### 出牌规则

```
当前玩家出牌
  ├─ 宣称「我出了 N 张 X」（可以说谎）
  ├─ 下家选择：
  │   ├─ 信任 → 接受，轮到下家
  │   └─ 质疑 → 翻牌验证
  │       ├─ 说谎 → 出牌者进入轮盘
  │       └─ 没说谎 → 质疑者进入轮盘
  └─ Joker 在翻牌验证时算作宣称的牌面
```

### 俄罗斯轮盘

```lua
-- 6 个弹槽，1 颗子弹
bulletPos = math.random(1, 6)   -- 随机子弹位置
shots = 0                       -- 已开枪次数

function PullTrigger()
    shots = shots + 1
    if shots == bulletPos then
        return true   -- 中弹 → 淘汰
    end
    return false      -- 空枪 → 安全
end
```

- 每次进入轮盘，`bulletPos` 重新随机
- 中弹 → 玩家淘汰（`alive = false`）
- 最后一个存活者获胜

## AI 决策架构（5 层）

```
StateLayer          → 状态快照（当前局面数据化）
  ↓
ProbabilityLayer    → 概率计算（对手手牌推断）
  ↓
StrategyLayer       → 策略选择（4 种人格 + 扰动）
  ↓
PsychologyLayer     → 心理模拟（延迟/犹豫/表情标记）
  ↓
MemoryLayer         → 跨轮记忆（stub，暂未实现）
```

### 各层职责

| 层 | 输入 | 输出 | 说明 |
|----|------|------|------|
| **StateLayer** | 房间状态、手牌、历史 | 结构化快照 | 将游戏状态转为 AI 可消费的数据 |
| **ProbabilityLayer** | 快照 | 各牌面概率、对手诚实概率 | 数学计算，无随机 |
| **StrategyLayer** | 概率 + 人格 | 决策（出什么牌/是否质疑/宣称什么） | 4 种人格 + 10% 随机扰动 + 8% 反转 |
| **PsychologyLayer** | 决策 | 延迟时间 + 行为标记 | 模拟思考时间（5~10 秒）和犹豫动作 |
| **MemoryLayer** | 历史记录 | 对手行为画像 | 当前为 stub，记录但不使用 |

### 4 种 AI 人格

| 人格 | 出牌风格 | 质疑倾向 | 说谎频率 |
|------|---------|---------|---------|
| **balanced**（均衡） | 中等 | 中等 | 中等 |
| **cautious**（谨慎） | 保守出牌 | 低（倾向信任） | 低 |
| **aggressive**（激进） | 大量出牌 | 高（频繁质疑） | 高 |
| **bluffer**（虚张声势） | 正常 | 中等 | 极高 |

**扰动机制**（防止 AI 行为过于可预测）：

```lua
-- 10% 概率随机扰动决策参数
if math.random() < 0.10 then
    -- 微调出牌数量或质疑阈值
end

-- 8% 概率完全反转决策
if math.random() < 0.08 then
    decision = (decision == "trust") and "challenge" or "trust"
end
```

## 断线处理

```
玩家断线
  │
  ├─ 在匹配队列中
  │   └─ MatchQueue:Remove(player) → 移出队列
  │
  ├─ 在房间游戏中
  │   ├─ player.alive = false → 标记为淘汰
  │   ├─ 广播 S2C_PLAYER_LEFT
  │   ├─ 如果是当前回合玩家 → 自动 NextTurn()
  │   └─ 如果仅剩 1 人 → GAME_OVER
  │
  └─ 在大厅
      └─ 仅清理 connections_ 表
```

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 常驻服 vs 房间匹配 | 常驻服 | 社交游戏需要大厅氛围，玩家可随进随出 |
| AI 补位 vs 等人 | 15 秒后补位 | 平衡等待体验和游戏质量 |
| 翻牌验证 | 服务端权威 | 防作弊，客户端不持有他人手牌 |
| 轮盘随机 | 服务端生成 | 客户端只收结果，无法预测 |
| AI 分层 | 5 层管线 | 解耦策略/表现/记忆，方便调参 |

## 踩坑备忘

| 问题 | 原因 | 解决 |
|------|------|------|
| QKA 卡牌被审核拒绝 | TapTap 疑似涉赌判定 | 改用非扑克牌面（如动物、元素、符号） |
| AI 出牌瞬间完成，缺乏真实感 | 没有 PsychologyLayer 延迟 | 加 5~10 秒随机延迟 + 犹豫标记 |
| 断线玩家回合卡住 | 没有超时自动跳过 | 断线 → 立即 NextTurn |
| 4 人房间 2 人断线后游戏无法结束 | 没有「仅剩 1 人」检查 | 每次 PlayerLeft 后检查存活人数 |
| AI 行为可预测 | 纯策略无随机 | 加 10% 扰动 + 8% 反转 |
