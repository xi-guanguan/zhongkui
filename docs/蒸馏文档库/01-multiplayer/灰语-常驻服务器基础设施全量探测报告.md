### 延迟

| 指标 | 值 |
|------|-----|
| 应用层 RTT (RemoteEvent) | P50=34ms, Avg=44.7ms, P95=67ms |
| 传输层 RTT (Connection.RTT) | ~17ms |

### 载荷上限

| 限制 | 值 |
|------|-----|
| WebSocket 单消息上限 | **65535 bytes (64KB-1)**，引擎硬编码 |
| 实测安全上限 | ~62KB（序列化开销 <1.5KB） |

100B~62KB 回声测试全部通过。大数据需分包。

### serverCloud API

25 项全部通过：CRUD/Batch/Rank/Money/List/Msg/Quota/Misc/MultiGet。Redis 后端完全可用。

## Lua 沙箱

### 标准库可用性

| 可用 | 受限 | 不可用 |
|------|------|--------|
| string/table/math/utf8/coroutine/package/debug | os（clock/time/date可用，getenv/execute被屏蔽） | io（完全移除）、bit32/jit |

### 关键约束

- **无法读写本地文件** — `io` 移除、`File.Write` 的 `FILE_WRITE` 常量为 nil、`fileSystem.FileExists` 始终 false
- **持久化只能用 serverCloud**
- **Lua 5.4 特性全可用** — 64位整数、原生位运算、math.type()、table.move()、utf8.len()
- **协程完整支持** — yield/resume/wrap，可用于异步逻辑/状态机/分帧处理

### C++ 层崩溃风险（pcall 无法捕获）

| 操作 | 规避 |
|------|------|
| 无限递归 | 控制深度 ≤50 |
| `Variant(StringHash("x"))` | 不要包装 StringHash |
| 嵌套 `Variant(VariantMap())` | 避免 VariantMap 嵌套 |
| 超大字符串 (1MB+) | 限制在 64KB |

## 设计约束速查

| 维度 | 约束 | 建议 |
|------|------|------|
| 网络延迟 | P50=34ms | 客户端预测+插值补偿 |
| 单消息 | 62KB 硬限制 | 大数据分包 |
| 持久化 | 仅 serverCloud | 文件 I/O 完全封锁 |
| 字符串 | 安全上限 64KB | 与 WebSocket 限制一致 |
| 递归 | ≤50 层 | 超过可能 C 栈溢出 |
| 重连 | Lobby 自动处理 | 脚本需处理 ServerConnected 重入 |
| JSON 同步 | ~500 实体/包 | 大世界需分区域 |
| 二进制同步 | ~3000 实体/包 | 位置同步用 VectorBuffer |
