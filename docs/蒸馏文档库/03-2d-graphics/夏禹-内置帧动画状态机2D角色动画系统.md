# 内置帧动画 + 状态机 2D 角色动画系统

> **这是什么**：不依赖外部动画编辑器（Spine/DragonBones），纯粹使用 UrhoX 内置组件（AnimatedSprite2D / StaticSprite2D + SpriteSheet2D）配合 Lua 状态机实现完整的 2D 角色动画方案，包含帧序列播放、状态切换、一次性动画锁定、翻转、受伤闪烁等实战模式。
>
> **原作者**：夏禹
> **推荐度**：L2（方案来自实际项目《星际歌姬》（横版动作 + 音游），8 套动作流畅过渡，已验证可用。纯内置方案无外部依赖，性能最优。但动画质量取决于帧数，不适合需要平滑骨骼变形的场景。）
> **适用场景**：像素风、卡通风、独立游戏等不需要骨骼变形的 2D 角色动画；角色数量多、性能敏感的场景；不想购买/学习 Spine 等外部工具的项目。
>
> **同类对比**：
> - **vs 糖心哥-spine-lua + NanoVG 方案**：糖心哥方案依赖外部 Spine 编辑器和 spine-lua runtime，动画质量极高（骨骼变形流畅），但纯 Lua 有性能上限（建议 10 角色以内）。夏禹方案纯内置组件，性能最优（原生渲染管线），但动画质量取决于帧数。
> - **vs 大雄-Spine自研Runtime方案**：大雄方案是自研 Spine runtime + CustomGeometry 渲染，功能全面但代码量大（~1600行）。夏禹方案代码简洁（状态机 + 帧控制约 100 行），无需理解骨骼变换/仿射矩阵。
> - **特色**：零外部依赖，纯代码控制，状态机模式可复用于任何帧动画项目。
> - **优点**：性能最优（原生 Sprite 组件）；工具链免费（只需像素画工具）；代码简洁可控；可与 Spine 方案混用。
> - **缺点**：动画平滑度取决于帧数（帧数少则不流畅）；不支持骨骼变形/IK/路径约束等高级效果；帧序列图集文件体积较大。

## 1. 方案概览

| 组件 | 用途 |
|------|------|
| `SpriteSheet2D` | 图集管理（合图 + 区域定义） |
| `StaticSprite2D` 或 `AnimatedSprite2D` | 帧序列播放 |
| Lua 状态机 | 动画切换逻辑（锁定、优先级、过渡） |

## 3. 帧动画数据定义

```lua
local spriteSheet = cache:GetResource("SpriteSheet2D", "Textures/Player/player_sheet.xml")
local sprite2D = characterNode:CreateComponent("StaticSprite2D")
sprite2D:SetSprite(spriteSheet:GetSprite("idle_01"))

-- 帧序列定义（Lua 数组，索引从 1 开始）
local animFrames = {
    idle   = {"idle_01", "idle_02", "idle_03", "idle_04"},
    run    = {"run_01", "run_02", "run_03", "run_04", "run_05", "run_06"},
    jump   = {"jump_01", "jump_02", "jump_03"},
    attack = {"atk_01", "atk_02", "atk_03", "atk_04", "atk_05"},
}

-- 每组动画的播放帧率
local animFPS = { idle = 6, run = 10, jump = 8, attack = 12 }

-- 帧控制变量
local frameTimer = 0
local currentFrame = 1
```

## 4. Lua 状态机

```lua
local AnimState = {
    current = "idle",
    locked = false,      -- 锁定状态（一次性动画播放期间不可切换）
    lockTimer = 0,
}

function AnimState:change(newState)
    if self.locked then return end
    if self.current == newState then return end

    self.current = newState
    currentFrame = 1
    frameTimer = 0

    -- 一次性动画（attack/hurt 等）：播完自动解锁回 idle
    if newState == "attack" or newState == "hurt" then
        self.locked = true
        local frames = animFrames[newState]
        local fps = animFPS[newState]
        self.lockTimer = #frames / fps  -- 动画总时长
    end
end

function AnimState:update(dt)
    if self.locked then
        self.lockTimer = self.lockTimer - dt
        if self.lockTimer <= 0 then
            self.locked = false
            self:change("idle")
        end
    end
end
```

**状态机要点**：
- `locked` 机制防止一次性动画被中断（攻击、受伤等必须播完）
- 锁定时长 = 帧数 / 帧率，自动计算
- 解锁后自动回到 idle

## 5. Update 中的完整逻辑

```lua
function HandleUpdate(eventType, eventData)
    local dt = eventData["TimeStep"]:GetFloat()

    -- 1. 更新状态机计时器
    AnimState:update(dt)

    -- 2. 根据输入/物理状态切换动画
    if input:GetKeyPress(KEY_J) then
        AnimState:change("attack")
    elseif not AnimState.locked then
        if not onGround then
            AnimState:change("jump")
        elseif math.abs(velocity.x) > 0.1 then
            AnimState:change("run")
        else
            AnimState:change("idle")
        end
    end

    -- 3. 更新帧动画
    local anim = AnimState.current
    local frames = animFrames[anim]
    local fps = animFPS[anim]

    frameTimer = frameTimer + dt
    if frameTimer >= 1.0 / fps then
        frameTimer = frameTimer - 1.0 / fps
        currentFrame = currentFrame + 1
        if currentFrame > #frames then
            currentFrame = 1  -- 循环（Lua 从 1 开始）
        end
    end

    sprite2D:SetSprite(spriteSheet:GetSprite(frames[currentFrame]))
end
```

## 6. 视觉效果

### 6.1 角色朝向翻转

```lua
if velocity.x < 0 then
    characterNode:SetScale2D(Vector2(-1, 1))  -- 水平翻转
elseif velocity.x > 0 then
    characterNode:SetScale2D(Vector2(1, 1))
end
```

### 6.2 受伤无敌闪烁

```lua
local blinkTimer = 0
local isInvincible = false

function startInvincible(duration)
    isInvincible = true
    blinkTimer = duration
end

function updateBlink(dt)
    if not isInvincible then return end
    blinkTimer = blinkTimer - dt
    if blinkTimer <= 0 then
        isInvincible = false
        sprite2D:SetColor(Color(1, 1, 1, 1))
        return
    end
    -- 10Hz 闪烁
    local visible = math.floor(blinkTimer * 10) % 2 == 0
    sprite2D:SetColor(visible and Color(1, 1, 1, 1) or Color(1, 1, 1, 0.2))
end
```

## 7. 方案选型指南

| 场景 | 推荐方案 |
|------|---------|
| 像素风 / 卡通风 / 独立游戏 | **内置帧动画**（本方案），简单够用 |
| 日系立绘 / 精细角色动画 | Spine（糖心哥方案或大雄方案），骨骼变形效果好 |
| 大量同屏角色（RTS/弹幕） | **内置帧动画**（本方案），性能最优 |
| 主角精细 + 小怪简单 | **混用**：主角用 Spine，小怪用帧动画 |

两种方案不冲突，可在同一项目中混用。
