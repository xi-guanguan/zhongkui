# 3D 骨骼自定义动作 — 可行性研究

> **这是什么**: 运行时骨骼编辑、权重绘制和关键帧动画的完整技术路线
> **原作者**: 点点
> **推荐度**: ★★★★ 精华帖，实测验证，含完整代码和 API 调研
> **适用场景**: AI 生成 3D 角色的自定义动作、换装系统、骨骼编辑器

---

## 背景

AI 生成的 3D 角色（Blue Archive 风格），自动绑骨后有 41 根标准人形骨骼，挂了 52 个动画。但遇到问题：
- 想让角色做比心手势 → 没有手指骨骼
- 想让头发飘动 → 没有头发物理骨骼
- 想做库里没有的自定义姿势 → 只能用现成动画

---

## 第一步：让骨骼动起来

### bone.animated — 最关键的一行代码

直觉上修改骨骼旋转很简单：

```lua
local skeleton = animModel:GetSkeleton()
local bone = skeleton:GetBone("Head")
bone.node.rotation = Quaternion(30, 0, 0)  -- 低头 30 度
```

但实际运行 **完全没效果**。动画系统每帧都会把骨骼重置回动画数据定义的姿势。

```lua
bone.animated = false  -- 告诉动画系统：这根骨骼我自己管
```

加上这一行，动画系统会跳过这根骨骼，手动设置的旋转就能保留。

### 旋转的正确姿势：增量叠加

直接设置绝对旋转会丢失骨骼的初始绑定姿势，导致模型变形：

```lua
bone.animated = false
-- 增量旋转（俯仰30度、偏航0度、翻滚0度）
local delta = Quaternion(30, 0, 0)
-- 叠加到初始姿势上
bone.node.rotation = delta * bone.initialRotation
```

`bone.initialRotation` 是模型在 T-Pose / A-Pose 时这根骨骼的旋转值。用增量乘以它 = "在原来姿势的基础上再转一点"。

### 做成交互式编辑器

给每根骨骼配三个滑块（俯仰、偏航、翻滚），实时拖拽调整：

```lua
function OnSliderChanged(boneName, axis, value)
    local bone = skeleton:GetBone(boneName)
    bone.animated = false
    rotations[boneName][axis] = value
    local r = rotations[boneName]
    local delta = Quaternion(r[1], r[2], r[3])
    bone.node.rotation = delta * bone.initialRotation
end
```

把 19 根主要骨骼分成 6 组（头部、躯干、左臂、右臂、左腿、右腿），每组点开选择具体骨骼。

## 第二步：缺少的骨骼怎么办

### 41 根骨骼清单

```
Root ─ Hip ─ Pelvis
              ├─ L_Thigh ─ L_Calf ─ L_Foot ─ L_ToeBase
              ├─ R_Thigh ─ R_Calf ─ R_Foot ─ R_ToeBase
              └─ Waist ─ Spine01 ─ Spine02
                          ├─ NeckTwist01 ─ NeckTwist02 ─ Head
                          ├─ L_Clavicle ─ L_Upperarm ─ L_ForearmTwist ─ L_Forearm ─ L_Hand
                          └─ R_Clavicle ─ R_Upperarm ─ R_ForearmTwist ─ R_Forearm ─ R_Hand
```

- **有**：头、颈、脊椎、肩、大臂、小臂、手掌、大腿、小腿、脚
- **没有**：手指（5×3=15 根）、头发链、眼球、下巴、舌头

### Skeleton 没有 AddBone()

引擎不能在运行时添加新骨骼。三个变通方案：

| 方案 | 思路 | 复杂度 |
|------|------|--------|
| A: 复用空闲骨骼 | 将 twist 骨骼重定位到手指关节位置，修改手指区域顶点的骨骼索引 | 中 |
| B: 顶点变形 | 直接读写 VertexBuffer 修改顶点位置 | 中 |
| C: 离线重建 | 编辑器中设计完整骨骼层级和权重，导出新 MDL 文件 | 高 |

**方案 B 的 API**：

```lua
local vb = geometry:GetVertexBuffer(0)
local data = vb:GetData()  -- 读取全部顶点数据
-- 修改特定顶点的位置...
vb:SetDataRange(modifiedData, startVertex, count)  -- 写回 GPU
```

引擎的 VertexBuffer 完整支持运行时读写，适合捏脸、表情等简单变形。

## 第三步：权重绘制

### 顶点权重的数据结构

每个顶点存储 4 组骨骼绑定数据：

```
顶点 #1234:
  BoneIndices: [12, 11, 0, 0]    -- 绑定到第 12、11 号骨骼
  BoneWeights: [0.7, 0.3, 0, 0]  -- 影响权重（合计 = 1.0）
```

这个顶点 70% 跟着第 12 号骨骼动，30% 跟着第 11 号骨骼动。

### 读写权重的 API

```lua
local vb = geometry:GetVertexBuffer(0)
if vb:HasElement(SEM_BLENDWEIGHTS) then
    local weightOffset = vb:GetElementOffset(SEM_BLENDWEIGHTS)
    local indexOffset = vb:GetElementOffset(SEM_BLENDINDICES)
    local data = vb:GetData()
    -- 逐顶点读取、修改、写回...
end
```

### 画刷选择顶点

引擎支持三角面级别的射线检测：

```lua
local ray = camera:GetScreenRay(mouseX / screenW, mouseY / screenH)
local result = octree:RaycastSingle(ray, RAY_TRIANGLE_UV, 100.0)
if result then
    local hitPos = result.position
    -- 找到命中点附近一定半径内的所有顶点
    -- 批量修改它们的权重
end
```

### 权重归一化

修改权重时必须保证每个顶点的 4 个权重之和 = 1.0：

```lua
function NormalizeWeights(w1, w2, w3, w4)
    local sum = w1 + w2 + w3 + w4
    if sum > 0 then
        return w1/sum, w2/sum, w3/sum, w4/sum
    end
    return 1, 0, 0, 0
end
```

## 第四步：时间线动画

### 关键帧数据结构

```lua
-- 一个关键帧 = 某个时间点所有骨骼的旋转状态
local keyframe = {
    time = 0.5,
    bones = {
        Head       = { pitch = 20,  yaw = 0,   roll = 0  },
        L_Upperarm = { pitch = -45, yaw = 30,  roll = 0  },
        R_Upperarm = { pitch = -45, yaw = -30, roll = 0  },
    }
}

-- 一段动画 = 多个关键帧的序列
local animation = {
    name = "挥手",
    duration = 2.0,
    loop = true,
    keyframes = {
        { time = 0.0,  bones = { ... } },  -- 起始姿势
        { time = 0.5,  bones = { ... } },  -- 手举到最高
        { time = 1.0,  bones = { ... } },  -- 手放回
        { time = 1.5,  bones = { ... } },  -- 再举起
        { time = 2.0,  bones = { ... } },  -- 回到起始（循环）
    }
}
```

### 关键帧插值

旋转使用四元数球面插值（Slerp），避免万向锁：

```lua
local q1 = Quaternion(rot1.pitch, rot1.yaw, rot1.roll)
local q2 = Quaternion(rot2.pitch, rot2.yaw, rot2.roll)
local qResult = q1:Slerp(q2, t)
bone.node.rotation = qResult * bone.initialRotation
```

### 时间线播放器

```lua
function Timeline:Update(dt)
    if not self.playing then return end
    self.time = self.time + dt
    if self.anim.loop then
        self.time = self.time % self.anim.duration
    elseif self.time > self.anim.duration then
        self.time = self.anim.duration
        self.playing = false
    end
    -- 找到当前时间所在的两个关键帧
    local kf1, kf2 = self:FindKeyframes(self.time)
    if kf1 and kf2 then
        local span = kf2.time - kf1.time
        local progress = span > 0 and (self.time - kf1.time) / span or 0
        local blended = LerpKeyframes(kf1, kf2, progress)
        for boneName, rot in pairs(blended) do
            ApplyBoneRotation(boneName, rot)
        end
    end
end

function Timeline:FindKeyframes(time)
    local kfs = self.anim.keyframes
    for i = 1, #kfs - 1 do
        if time >= kfs[i].time and time < kfs[i + 1].time then
            return kfs[i], kfs[i + 1]
        end
    end
    return kfs[#kfs - 1], kfs[#kfs]
end
```

## 第五步：3D→2D 动画导出

### 骨骼 2D 投影

正交相机下将 3D 世界坐标投影为 2D 屏幕坐标：

```lua
function ProjectBones(camera, skeleton)
    local frame = {}
    for i = 0, skeleton:GetNumBones() - 1 do
        local bone = skeleton:GetBone(i)
        if bone.node then
            local worldPos = bone.node.worldPosition
            local screenPos = camera:WorldToScreenPoint(worldPos)
            frame[bone.name] = {
                x = screenPos.x,
                y = screenPos.y,
                depth = screenPos.z,
            }
        end
    end
    return frame
end
```

### 逐帧录制

```lua
function RecordAnimation(timeline, camera, skeleton, fps)
    local frames = {}
    local dt = 1.0 / fps
    local time = 0
    while time <= timeline.anim.duration do
        timeline.time = time
        timeline:ApplyFrame()
        frames[#frames + 1] = {
            time = time,
            bones = ProjectBones(camera, skeleton),
        }
        time = time + dt
    end
    return frames
end
```

### 导出 JSON 格式

```json
{
  "name": "挥手",
  "fps": 30,
  "duration": 2.0,
  "boneCount": 41,
  "frames": [
    {
      "time": 0.0,
      "bones": {
        "Head":   { "x": 0.50, "y": 0.15, "depth": 0.42 },
        "L_Hand": { "x": 0.35, "y": 0.55, "depth": 0.38 },
        "R_Hand": { "x": 0.65, "y": 0.55, "depth": 0.38 }
      }
    }
  ]
}
```

2D 编辑器拿到这份数据后，可以把骨骼点映射到 2D 人偶控制点上，驱动 2D 变形动画 —— 2D 端可自由换装、换发型，不受 3D 模型限制。

## 技术总结

### 已验证可行

| 能力 | 关键 API | 复杂度 |
|------|---------|--------|
| 旋转现有骨骼 | `bone.animated = false` + `bone.node.rotation` | 低 |
| 实时预览 | 滑块 + 每帧应用旋转 | 低 |
| 读写顶点权重 | `VertexBuffer:GetData()` / `SetData()` | 中 |
| 射线拾取网格 | `Octree:RaycastSingle(ray, RAY_TRIANGLE_UV)` | 中 |
| 骨骼 2D 投影 | `camera:WorldToScreenPoint()` | 低 |

### 需要实现

| 功能 | 依赖 | 复杂度 |
|------|------|--------|
| 关键帧时间线 | 数据结构 + 四元数 Slerp 插值 | 中 |
| 时间线编辑器 UI | 可拖拽时间轴 + 关键帧标记 | 中高 |
| 权重画刷 | 射线拾取 + 顶点范围筛选 + 权重归一化 | 高 |
| 复用空闲骨骼做手指 | 骨骼重定位 + 权重修改 | 中 |
| 导出新 MDL 文件 | MDL 二进制格式写入 | 高 |

### 推荐路线

```
阶段 1 ── 骨骼旋转编辑器（已完成框架）
           用滑块实时调整 19 根骨骼的旋转
阶段 2 ── 关键帧时间线
           记录多个姿势 → 自动插值 → 播放自定义动画
阶段 3 ── 复用空闲骨骼
           借用 twist 骨骼做手指 → 修改顶点权重
阶段 4 ── 权重画刷编辑器
           射线选择网格区域 → 批量修改骨骼权重
阶段 5 ── 3D→2D 动画导出
           正交投影 → 逐帧录制 → JSON 导出给 Live2D 编辑器
```

**核心结论**：骨骼是数据，权重是数据，动画也是数据 —— 数据都能改。区别只在于改哪层数据、用什么工具改、以及愿意投入多少工程量。
