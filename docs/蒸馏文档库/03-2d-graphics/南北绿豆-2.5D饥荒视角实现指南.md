# 2.5D 饥荒视角实现指南（3D 场景 + 2D 精灵）

> **这是什么**: 在 UrhoX 中实现《饥荒》(Don't Starve) 风格 2.5D 视角的完整方案 —— 3D 世界承载地形和空间关系，角色/物件用 2D 精灵 Billboard 渲染，正交相机固定俯角，支持 Q/E 旋转视角和 8 方向精灵切换。
> **原作者**: 南北绿豆
> **推荐度**: L1（框架完整可直接使用，含正交相机、Billboard、方向系统、深度排序、伪阴影、视角旋转全套代码）
> **适用场景**: 饥荒/Hades/缺氧风格的 2.5D 游戏 —— 想要 3D 空间感但角色用手绘 2D 风格

---

## 一、什么是饥荒风格 2.5D

| 要素 | 实现方式 |
|------|---------|
| 世界/地形 | 真 3D 场景（Plane/Terrain） |
| 角色/物件 | 2D 精灵图（Billboard 始终面向相机） |
| 相机 | 正交投影 + 固定俯角（45°） |
| 深度遮挡 | 基于 Y+Z 坐标的排序 |
| 视角旋转 | Q/E 键 45° 步进旋转 |

**核心优势**: 3D 的空间感 + 2D 的美术风格，美术成本远低于全 3D。

## 二、正交相机设置

```lua
local cameraNode = scene_:CreateChild("Camera")
local camera = cameraNode:CreateComponent("Camera")

-- 正交投影（无近大远小）
camera.orthographic = true
camera.orthoSize = 20.0  -- 视野全高度 20 米

-- 固定 45° 俯角
cameraNode.rotation = Quaternion(45, Vector3.RIGHT)
-- 相机位置抬高，看向场景中心
cameraNode.position = Vector3(0, 30, -30)
```

**关键参数**:
- `orthoSize` 控制缩放级别，越小越近
- 俯角 45° 是饥荒经典角度，可根据需求调整（30°~60°）
- 正交相机下物体不会有近大远小的透视变形

## 三、Billboard 精灵（2D 角色在 3D 世界中）

使用 `BillboardSet` 组件让 2D 精灵始终面向相机：

```lua
function CreateSpriteCharacter(scene, position, spritePath)
    local node = scene:CreateChild("Character")
    node.position = position

    local bbSet = node:CreateComponent("BillboardSet")
    bbSet.numBillboards = 1
    bbSet.material = cache:GetResource("Material", "Materials/SpriteMaterial.xml")
    bbSet.sorted = true
    -- FaceCamera 模式：始终面向相机
    bbSet.faceCameraMode = FC_ROTATE_XYZ

    local bb = bbSet:GetBillboard(0)
    bb.position = Vector3(0, 0, 0)
    bb.size = Vector2(2.0, 2.0)  -- 精灵宽高（米）
    bb.enabled = true

    bbSet:Commit()
    return node
end
```

**精灵材质**（需要透明 + 无光照）:
```xml
<!-- Materials/SpriteMaterial.xml -->
<material>
    <technique name="Techniques/DiffAlpha.xml" />
    <texture unit="diffuse" name="Textures/character_idle_S.png" />
</material>
```

## 四、8 方向精灵切换

角色移动时根据移动方向 + 相机朝向，选择对应方向的精灵帧：

```lua
-- 8 方向枚举
local Direction = {
    S  = 1,  -- 面朝相机（正面）
    SE = 2,
    E  = 3,
    NE = 4,
    N  = 5,  -- 背对相机（背面）
    NW = 6,
    W  = 7,
    SW = 8,
}

--- 根据移动方向和相机 yaw 计算精灵方向
---@param moveDir Vector3 世界空间移动方向（归一化）
---@param cameraYaw number 相机当前 yaw 角度
---@return number 方向枚举值 1~8
function GetSpriteDirection(moveDir, cameraYaw)
    -- 移动方向的世界角度
    local moveAngle = math.deg(math.atan(moveDir.x, moveDir.z))
    -- 减去相机旋转，得到相对于屏幕的角度
    local relAngle = moveAngle - cameraYaw
    -- 归一化到 0~360
    relAngle = relAngle % 360

    -- 按 45° 扇区划分到 8 方向
    -- 0°=S, 45°=SE, 90°=E, ...
    local index = math.floor((relAngle + 22.5) / 45) % 8 + 1
    return index
end
```

**使用方式**:
```lua
-- 每帧更新精灵方向
local dir = GetSpriteDirection(moveDirection, currentCameraYaw)
local dirNames = {"S", "SE", "E", "NE", "N", "NW", "W", "SW"}
local texturePath = "Textures/character_walk_" .. dirNames[dir] .. ".png"
-- 更新 BillboardSet 的材质纹理
```

## 五、深度排序（解决遮挡关系）

2.5D 中最重要的视觉问题是遮挡 —— 前面的角色应该挡住后面的树。

### 方法：组合 Y+Z 坐标排序

```lua
--- 计算节点的绘制顺序值（值越大越靠前/越晚绘制）
function CalculateDrawOrder(node)
    local pos = node.position
    -- Y 坐标权重高（海拔差异），Z 坐标补充前后关系
    -- 负号：Y 越小（越靠屏幕下方）的物体越"前面"
    return -(pos.y * 1000 + pos.z)
end
```

**三种排序策略对比**:

| 策略 | 原理 | 适用场景 |
|------|------|---------|
| 纯 Y 坐标 | 屏幕下方的在前 | 纯俯视、无高度差 |
| 纯 3D 深度 | 离相机近的在前 | 透视相机 |
| **Y+Z 组合**（推荐） | 兼顾高度和前后 | **正交相机 + 有高度差** |

**实现**: 在 Update 中遍历所有精灵节点，按 drawOrder 设置 `renderOrder` 或直接排序 Z 值。

## 七、伪阴影（Blob Shadow）

饥荒风格不用实时阴影，用一个模糊圆形贴图贴在角色脚下：

```lua
function CreateBlobShadow(parentNode)
    local shadowNode = parentNode:CreateChild("BlobShadow")
    -- 贴在地面上，微抬避免 Z-fighting
    shadowNode.position = Vector3(0, 0.02, 0)
    -- 水平放置
    shadowNode.rotation = Quaternion(-90, Vector3.RIGHT)
    shadowNode.scale = Vector3(1.5, 1.5, 1.0)

    local bbSet = shadowNode:CreateComponent("BillboardSet")
    bbSet.numBillboards = 1

    -- 使用半透明材质
    local mat = Material:new()
    mat:SetTechnique(0, cache:GetResource("Technique", "Techniques/DiffAlpha.xml"))
    mat:SetTexture(TU_DIFFUSE, cache:GetResource("Texture2D", "Textures/blob_shadow.png"))
    bbSet.material = mat

    local bb = bbSet:GetBillboard(0)
    bb.size = Vector2(2.0, 2.0)
    bb.enabled = true
    bbSet.faceCameraMode = FC_NONE  -- 不面向相机，保持水平
    bbSet:Commit()

    return shadowNode
end
```

**blob_shadow.png**: 一张 64x64 的白色圆形渐变图（中心不透明，边缘全透明），运行时通过材质颜色调暗。

## 八、Q/E 视角旋转

饥荒标志性的 45° 步进旋转视角：

```lua
local currentYaw = 0
local targetYaw = 0
local ROTATE_STEP = 45  -- 每次旋转 45°

function HandleUpdate(eventType, eventData)
    local dt = eventData["TimeStep"]:GetFloat()

    -- Q/E 控制目标角度
    if input:GetKeyPress(KEY_Q) then
        targetYaw = targetYaw + ROTATE_STEP
    end
    if input:GetKeyPress(KEY_E) then
        targetYaw = targetYaw - ROTATE_STEP
    end

    -- 平滑插值到目标角度
    currentYaw = Lerp(currentYaw, targetYaw, 5.0 * dt)

    -- 更新相机绕 Y 轴旋转
    cameraNode.rotation = Quaternion(45, Vector3.RIGHT)
                        * Quaternion(currentYaw, Vector3.UP)

    -- 同时更新所有精灵的方向（因为相机转了）
    UpdateAllSpriteDirections(currentYaw)
end
```

**注意**: 相机旋转后，所有 Billboard 精灵的朝向贴图也要跟着切换（调用 `GetSpriteDirection` 时传入新的 `cameraYaw`）。

## 九、完整框架骨架

```lua
-- 全局变量
local cameraNode
local currentYaw = 0
local targetYaw = 0
local characters = {}  -- 所有角色节点

function Start()
    -- 1. 创建场景
    scene_ = Scene()
    scene_:CreateComponent("Octree")

    -- 2. 正交相机
    cameraNode = scene_:CreateChild("Camera")
    local camera = cameraNode:CreateComponent("Camera")
    camera.orthographic = true
    camera.orthoSize = 20.0
    cameraNode.rotation = Quaternion(45, Vector3.RIGHT)
    cameraNode.position = Vector3(0, 30, -30)
    renderer:SetViewport(0, Viewport:new(scene_, camera))

    -- 3. 光照（平行光，不投射实时阴影）
    local lightNode = scene_:CreateChild("Light")
    lightNode.rotation = Quaternion(60, Vector3.RIGHT)
    local light = lightNode:CreateComponent("Light")
    light.lightType = LIGHT_DIRECTIONAL
    light.castShadows = false

    -- 4. 地面
    CreateGround()

    -- 5. 创建角色（带伪阴影）
    local player = CreateSpriteCharacter(scene_, Vector3(0, 0, 0), "Textures/player_idle_S.png")
    CreateBlobShadow(player)
    table.insert(characters, player)

    -- 6. 订阅更新
    SubscribeToEvent("Update", "HandleUpdate")
end
```

## 十、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 精灵被地面吃掉 | Billboard Y 坐标太低 | 精灵中心抬高半个精灵高度 |
| 旋转视角后精灵方向不变 | 没有用相机 yaw 重算方向 | `GetSpriteDirection` 传入 `currentYaw` |
| 精灵之间互相穿插 | 缺少深度排序 | 用 `CalculateDrawOrder` 每帧排序 |
| 伪阴影浮在空中 | shadowNode 没有水平放置 | `rotation = Quaternion(-90, Vector3.RIGHT)` |
| 缩放后精灵模糊 | 纹理过滤模式 | 像素风格用 `FILTER_NEAREST` |
