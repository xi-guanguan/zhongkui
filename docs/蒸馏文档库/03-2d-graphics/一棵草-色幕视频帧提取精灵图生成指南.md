# 色幕视频帧提取 & 精灵图生成指南

> **这是什么**: 将角色素材通过 AI 视频生成 + 色幕抠图，自动产出透明背景动画序列帧和精灵图的完整工作流。
>
> **原作者**: 一棵草
>
> **推荐度**: L2（完整工作流参考，依赖视频生成工具和 Python 抠图脚本）
>
> **适用场景**: 需要从 AI 生成的视频中提取角色动画帧（行走/攻击/待机等），制作精灵图

---

## 工作流总览

```
素材图 → 选择色幕颜色 → 上传资产库 → AI 生成色幕视频
  → ⏸️ 用户确认 → 提取帧 → 去色幕 → 裁剪 → 精灵图
```

**关键约束**:
- 视频生成后**必须先展示给用户确认**再提取帧
- 同一账号同时只能运行 1 个视频生成任务，多个动画需依次排队

## 色幕颜色选择（重要）

| 角色主色调 | 推荐色幕 | 原因 |
|-----------|---------|------|
| 非绿色（默认） | 绿幕 `#00FF00` | 默认选择，效果最佳 |
| **绿色系**（绿衣/绿发） | **蓝幕** `#0000FF` | 绿幕会误抠角色绿色部分 |
| **蓝色系**（蓝衣/蓝发） | **绿幕** `#00FF00` | 蓝幕会误抠角色蓝色部分 |

> 经验教训: 绿色系角色用绿幕时，衣物/头发被错误抠掉。

## Prompt 组装规则（四段拼接）

**色幕前缀 + 镜头锁定 + 朝向 + 动作 + (可选)表情控制**

**① 色幕前缀**（按角色颜色选）:
- 绿幕: `绿幕背景(纯绿色#00FF00), 角色在画面中央, 全身可见,`
- 蓝幕: `蓝幕背景(纯蓝色#0000FF), 角色在画面中央, 全身可见,`

**② 镜头锁定**（必须加）: `固定机位(static camera, locked camera, no camera movement, no panning, no zooming),`

**③ 朝向**: 面朝右 `side view facing right` / 面朝左 / 正面 / 背面

**④ 动作**:
| 动作 | 关键词 |
|------|--------|
| 行走 | `原地行走动画, walk cycle in place, character stays centered` |
| 攻击 | `原地持武器连续挥砍, attack animation, slash combo` |
| 待机 | `待机呼吸动画, idle animation, breathing` |

**⑤ 表情控制**（可选）: 待机动画 AI 常自动生成嘴部运动，加 `嘴巴紧闭不动(mouth closed, no lip movement), 表情固定不变(fixed expression)` 可解决。

## 抠图工具

| 工具 | 适用场景 |
|------|---------|
| `greenscreen_sprite_tool.py` | 绿幕抠图 |
| `bluescreen_sprite_tool.py` | 蓝幕抠图（绿色角色） |

```bash
python3 scripts/greenscreen_sprite_tool.py 视频.mp4 \
  --copy-to assets/image/角色名_动作 \
  -n 20 --cols 5 --resize 512x512
```

**参数**: `-n` 帧数(默认20) / `--cols` 列数(默认5) / `--resize` 帧尺寸(默认512x512) / `--copy-to` 自动复制精灵图

**输出**: `frames_raw/` → `frames_transparent/` → `frames_trimmed/` → `frames_resized/` + `spritesheet.png`

## 常见问题速查

| 问题 | 解决 |
|------|------|
| 镜头旋转/平移 | 加强镜头锁定关键词 |
| 角色位移 | 动作关键词加 `in place` + `stays centered` |
| 嘴巴在动（待机） | 添加嘴巴不动 + 表情固定关键词 |
| 绿色角色被误抠 | 改用蓝幕 + bluescreen 工具 |
| 某些帧全透明 | 通常是首尾过渡帧，手动删除 |
