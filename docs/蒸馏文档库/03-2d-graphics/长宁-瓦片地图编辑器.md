# 等距场景编辑器 — 瓦片地图编辑完整指南

> **这是什么**：一个可嵌入任何 UrhoX 项目的完整等距瓦片地图编辑器，适用于 RPG、策略、模拟经营等类型游戏的关卡设计。提供六种编辑工具、双视图模式、多图层系统、行走规则、JSON 导出/导入、游玩测试等完整功能。
>
> **原作者**：长宁
> **推荐度**：L1（工具成熟、文档完整，部署即用）
> **适用场景**：需要在 UrhoX 项目中编辑 2D 瓦片地图时阅读此文档
>
> **声明**：该工具仅作为场景编辑以及序列化，交互等功能需要用户自行在项目中定义。
> **工具链接**：https://gitee.com/Nacchi/map-editor.git

## 1. 界面总览

编辑器采用三栏布局：

```
+--------------------------------------------------------------+
| 工具栏：[画笔][橡皮擦][填充][泼漆][选区][取色] [等距] [导出][导入][加载][清空] |
+--------------------------------------+-----------------------+
|                                      | 瓦片调色板              |
|                                      |-----------------------|
|          等距画布                      | 图层面板               |
|       （绘制和编辑区域）                |-----------------------|
|                                      | 图片资源               |
|                                      |-----------------------|
|                                      | 快捷键提示              |
+--------------------------------------+-----------------------+
| 状态栏：坐标 | 图层 | 工具 | 瓦片                               |
+--------------------------------------------------------------+
```

- **顶部工具栏**：编辑工具切换、视图模式切换、文件操作按钮
- **中间左侧**：等距画布（主编辑区域），支持缩放和平移
- **中间右侧**：侧边面板（250px），包含瓦片调色板、图层管理、图片资源
- **底部状态栏**：显示当前鼠标坐标、活跃图层、工具名称、选中瓦片

## 2. 六种编辑工具

六种工具覆盖了地图编辑的完整工作流——从逐格绘制到区域填充、从精确选取到批量操作。

| 工具 | 快捷键 | 用途 | 使用方式 |
|------|--------|------|---------|
| **画笔** | `B` | 逐格绘制瓦片 | 左键点击或拖拽，将选中的瓦片绘制到画布上。拖拽时连续绘制，松手后统一记录为一次撤销操作 |
| **橡皮擦** | `E` | 清除瓦片 | 左键点击或拖拽，将画布上的瓦片清除为空。悬停时显示红色高亮提示 |
| **填充** | `U` | 矩形区域批量填充 | 左键按住拖拽出一个矩形区域（拖拽过程中显示蓝色半透明预览），松手后整个矩形区域被填充为选中的瓦片。适合快速铺设大面积地面 |
| **泼漆** | `G` | 连通区域洪水填充 | 左键点击某个瓦片，自动将与它颜色相同且上下左右相连的整片区域替换为选中的瓦片（BFS 四向扩散）。类似画图软件的"油漆桶" |
| **选区** | `M` | 框选矩形区域 | 左键拖拽选出一个矩形区域（金色边框标记），配合 Ctrl+C/V 进行复制粘贴。详见"选区与复制粘贴"章节 |
| **取色** | `I` | 拾取画布上已有的瓦片 | 左键点击画布上任意瓦片，自动将其设为当前选中瓦片并切换到画笔工具。状态栏提示拾取了哪种瓦片。类似 Photoshop 的"吸管工具" |

每种工具在悬停时会显示不同颜色的高亮提示，方便识别当前处于哪种工具模式。

## 3. 双视图模式

编辑器支持两种不同视角的游戏场景：

| 视图模式 | 按钮文字 | 网格形状 | 适用游戏类型 |
|---------|---------|---------|------------|
| **等距视角** | "等距" | 菱形（钻石形） | RPG、策略、模拟经营等需要立体感的游戏 |
| **正视视角** | "正视" | 正方形方格 | 俯视角冒险、Roguelike、塔防等方格类游戏 |

工具栏中有一个**视图切换按钮**，点击即可切换。切换时相机会自动重置到初始位置。两种视角下所有编辑工具、图层操作、保存加载功能完全一致。

## 4. 瓦片调色板

调色板分为"颜色瓦片"和"图片瓦片"两类。颜色瓦片用于快速原型设计（不需要美术素材就能标注地形类型），图片瓦片用于正式美术素材的编辑。

### 颜色瓦片

编辑器内置 5 种预定义颜色瓦片，点击即可选中并自动切换到画笔工具：

| ID | 默认名称 | 颜色 | 典型用途 |
|----|---------|------|---------|
| 1 | 草地 | 绿色 | 可行走地面 |
| 2 | 水面 | 蓝色 | 不可通行水域 |
| 3 | 沙地 | 黄色 | 减速地形 |
| 4 | 石头 | 灰色 | 障碍物 |
| 5 | 泥土 | 棕色 | 道路或耕地 |

选中的瓦片会显示白色边框高亮。颜色瓦片的名称和标签都可以自定义修改（见下一节）。

### 图片瓦片

**如何添加图片素材**：通过对话向 AI 助手发送指令，AI 会自动完成以下操作：

1. 在 `assets/` 下创建图片文件夹（如 `assets/Tiles/`）
2. 将你提供的图片放入该文件夹
3. 自动生成 `manifest.json` 清单文件

示例对话：
- "帮我把这几张图片放到 Tiles 文件夹里作为瓦片素材"
- "用 AI 生成几张等距风格的地面瓦片，放到 assets/Ground/ 目录"

最终生成的目录结构如下：

```
assets/
└── Tiles/
    ├── manifest.json    ← AI 自动生成
    ├── grass.png
    ├── tree.png
    └── house.png
```

其中 `manifest.json` 内容为图片文件名列表：

```json
["grass.png", "tree.png", "house.png"]
```

**在编辑器中加载图片素材**：准备好后，在编辑器侧边面板的"图片资源"区域：

1. 在文本框中输入文件夹名称（如 `Tiles`）
2. 点击"扫描"按钮，编辑器读取 `manifest.json` 并加载所有图片
3. 加载成功后图片以缩略图网格展示，点击选中即可在画布上绘制

## 5. 瓦片命名与标签（Tag）

### 命名

默认的"草地""水面"等名称可能不符合具体项目需求。自定义命名让导出的数据更有语义——比如将"草地"改名为"森林地面"，在代码中读取地图数据时一目了然。

### 标签（Tag）

标签是一个可选的字符串，用于为瓦片附加游戏逻辑标记，而不改变瓦片的视觉表现。

标签的核心用途是配合"行走规则"系统实现碰撞检测——比如标记某些瓦片为 `wall`（不可行走）或 `slow`（减速区域）。标签也可以用于其他自定义用途，如标记刷怪点 `spawn`、宝箱位置 `chest` 等。

**使用方式**：在侧边面板选中一个瓦片后，下方的"瓦片属性"区域会显示：

- **名称输入框**：修改瓦片名称，按回车确认。所有使用该瓦片 ID 的格子都会更新名称
- **ID 标识**：显示瓦片的数字 ID（如 `ID: 1`，图片瓦片会标注 `(图片)`）

## 6. 图层系统

多图层系统是专业地图编辑器的标准功能。将地图内容分布在不同图层上，可以独立编辑地面、建筑物、装饰物等，互不干扰。

### 基本操作

| 操作 | 方式 |
|------|------|
| **选择图层** | 点击图层名称，该图层变为当前活跃编辑图层（蓝色高亮） |
| **重命名** | 双击图层名称，弹出重命名弹窗 |
| **切换可见性** | 点击 `V` 按钮，隐藏/显示该图层。隐藏的图层名称后会标注（隐） |
| **锁定** | 点击 `L` 按钮，锁定后无法在该图层上绘制。锁定按钮变为橙色 |
| **调整顺序** | 点击 `^`（上移）或 `v`（下移）调整图层渲染顺序。列表中靠上的图层先渲染（在底层） |
| **删除** | 点击 `x` 按钮删除图层。至少保留一个图层 |
| **新建** | 点击"+ 添加层"按钮，创建一个新的空白图层 |

### 透明度

选中某个图层后，在图层详情区域会出现**透明度滑块**（0% ~ 100%，步进 5%）：

- 透明度影响该图层的渲染不透明度
- 编辑模式下，非活跃图层会自动降低到 50% 透明度，让你更专注于当前编辑的图层
- 预览模式下，所有图层按各自设定的透明度完整渲染

### 图层标签

选中图层后可以设置**图层级标签**（如 `collision`、`decoration`）。图层标签与瓦片标签配合使用，可以实现更灵活的行走规则（详见"图层标签与行走规则"章节）。

## 7. 图层组

当地图层数较多时（如"地面""道路""建筑底部""建筑顶部""装饰""碰撞层"），逐个管理图层变得繁琐。图层组提供了一种**文件夹式的组织方式**——将相关图层归入同一组，可以一键折叠收起、批量切换可见性和锁定状态。

### 操作

| 操作 | 方式 |
|------|------|
| **新建组** | 点击"+ 新建组"按钮，创建一个新的图层组 |
| **重命名组** | 双击组名，弹出重命名弹窗 |
| **折叠/展开** | 点击组头的 `>` / `v` 按钮，或直接点击组名称。折叠后组内图层在面板中隐藏（但仍然参与渲染） |
| **批量可见性** | 点击组头的 `V` 按钮，一次性切换组内所有图层的可见性 |
| **批量锁定** | 点击组头的 `L` 按钮，一次性锁定/解锁组内所有图层 |
| **将图层加入组** | 未分组的图层会显示 `G+` 按钮，点击弹出组选择列表 |
| **将图层移出组** | 已分组的图层显示 `Gx` 按钮，点击即可脱离当前组 |
| **删除组** | 点击组头的 `x` 按钮，删除组时**不会删除组内图层**，图层变为未分组状态 |

### 典型用法

```
📁 地形组
  ├─ 地面层
  └─ 道路层
📁 建筑组
  ├─ 建筑底部
  └─ 建筑顶部
📁 逻辑组
  ├─ 碰撞层
  └─ 触发器层
```

编辑建筑时，折叠"地形组"和"逻辑组"，只展开"建筑组"，工作区域更整洁。需要测试碰撞时，展开"逻辑组"查看标记。

## 8. 图层标签与行走规则

地图编辑器不仅用于绘制视觉地图，还需要定义游戏逻辑。标签+行走规则系统让设计者可以**直接在编辑器内标注哪些区域可以行走、哪些区域是障碍**，并通过"游玩模式"立即测试效果。

### 工作原理

1. 给瓦片设置标签：如将"石头"瓦片的标签设为 `wall`
2. 给图层设置标签：如将整个"碰撞层"的标签设为 `block`
3. 行走规则判断（在游玩模式中生效）：
   - 检查目标格子上所有可见图层的瓦片
   - 如果图层有标签，优先使用图层标签判断
   - 否则使用瓦片自身的标签判断
   - 标签对应的规则为 `false` → 不可行走
   - 没有匹配规则 → 按默认规则（默认可行走）

### 标签与行走规则对照

| 标签 | 默认行走规则 | 典型用途 |
|------|------------|---------|
| `wall` | `false`（不可行走） | 墙壁、障碍物 |
| `water` | `false`（不可行走） | 水域 |
| `slow` | `true`（可行走，游戏中可自行实现减速） | 沼泽、泥地 |
| `teleport` | `true`（可行走，游戏中可自行实现传送） | 传送点 |

## 9. 选区与复制粘贴

地图中经常有重复的结构（如对称的建筑、重复的路段）。选区+复制粘贴可以避免重复劳动。

**使用方式**：

1. 按 `M` 切换到选区工具
2. 左键拖拽选出一个矩形区域（金色边框标记）
3. 按 `Ctrl+C` 复制选区内容（状态栏提示复制了多少格）
4. 按 `Ctrl+V` 进入粘贴模式——鼠标位置会显示一个浅蓝色半透明预览
5. 在目标位置左键点击，即可粘贴
6. 按 `Esc` 取消粘贴模式或清除选区

**注意事项**：

- 复制/粘贴操作只作用于当前活跃图层
- 粘贴操作会被记录为一次撤销步骤
- 如果目标图层已锁定，粘贴会被拒绝

## 10. 撤销与重做

支持最多 **100 步**撤销历史，且拖拽绘制、矩形填充等批量操作会被合并为一次撤销步骤，确保一键撤销就能回退整个操作。

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 撤销 | `Ctrl+Z` | 回退上一步操作 |
| 重做 | `Ctrl+Y` | 恢复上一步被撤销的操作 |

- 每次新操作会清空重做历史
- 调整地图尺寸后撤销历史会被清空（因为旧坐标可能越界）
- 状态栏会提示"撤销"或"无可撤销操作"

## 11. 地图尺寸调整

允许在开发过程中灵活调整地图大小——从小尺寸原型逐步扩展到最终尺寸。

**使用方式**：

1. 在侧边面板的图层区域下方，找到"地图 W×H"标签
2. 修改宽度或高度数值（范围 **2 ~ 100**）
3. 点击"应用"按钮
   - 扩大地图：原有数据完整保留，新增区域为空
   - 缩小地图：超出新尺寸范围的瓦片数据会被裁剪
4. 应用后相机自动重置，撤销历史清空

## 12. 保存

编辑器通过导出 JSON 来持久化地图数据。导出的 JSON 可以保存到本地，也可以通过导入功能重新加载到编辑器中继续编辑。

编辑器还支持退出时自动保存到临时存档，重新进入编辑器时会自动恢复上次的编辑状态（WASM 环境刷新页面后丢失）。

## 13. 导出与导入 JSON

### 导出

1. 点击工具栏"导出"按钮
2. 弹窗中显示 JSON 预览和操作提示
3. JSON 数据会自动输出到浏览器控制台
4. 按 F12 打开开发者工具 → Console 标签页
5. 找到 `===== MAP JSON START =====` 和 `===== MAP JSON END =====` 之间的内容复制

> 由于 WASM 平台限制，编辑器内的剪贴板无法直接与系统剪贴板交互，需要通过浏览器控制台中转。

### 导入

1. 将 JSON 内容保存为 `_import.json` 文件，放入 `assets/mapjson/` 目录
2. 点击工具栏"导入"按钮
3. 编辑器会自动读取并解析该文件，恢复所有数据（地图尺寸、图层、瓦片、组、标签等）

### JSON 数据格式（v4）

```json
{
  "version": 4,
  "width": 8,
  "height": 8,
  "layers": [
    {
      "name": "地面",
      "visible": true,
      "locked": false,
      "opacity": 1.0,
      "groupId": 1,
      "tag": "ground",
      "tiles": [
        { "x": 1, "y": 1, "id": 1 },
        { "x": 3, "y": 2, "id": 100, "path": "Tiles/tree.png", "tag": "obstacle" }
      ]
    }
  ],
  "groups": {
    "1": { "name": "地形组" }
  },
  "imageRegistry": [
    { "id": 100, "name": "tree", "imagePath": "Tiles/tree.png", "tag": "obstacle" }
  ],
  "tileCustomizations": [
    { "id": 1, "name": "森林地面", "tag": "walkable" }
  ],
  "walkRules": { "obstacle": false },
  "defaultWalkable": true
}
```

**约定**：

- **瓦片 ID**：`0` = 空，`1~5` = 颜色瓦片，`100+` = 图片瓦片
- **坐标**：`x`（列）和 `y`（行）均从 **1** 开始
- **稀疏存储**：`tiles` 数组只包含非空瓦片，解析时先创建全零网格再填入

## 14. 游玩测试模式

地图设计完成后，设计者需要亲身体验空间感——格子之间的距离是否合适？障碍物的摆放是否合理？通道是否太窄？

**使用方式**：

1. 按 `P` 或点击侧边面板的"游玩"按钮进入游玩模式
2. 编辑器自动切换到预览模式，并在地图中央生成一个角色
3. 使用 `WASD` 控制角色移动（等距视角下方向已映射为菱形方向）
4. 角色会被行走规则阻挡（需要提前设置瓦片/图层标签和行走规则）
5. 再次按 `P` 或 `Esc` 退出游玩模式

### WASD 方向映射

| 按键 | 等距视角方向 | 正视视角方向 |
|------|------------|------------|
| W | 左上（菱形上） | 上 |
| A | 左下（菱形左） | 左 |
| S | 右下（菱形下） | 下 |

## 17. 瓦片图片规范

### 格式要求

PNG，带透明通道（RGBA）

### 尺寸规范

| 类型 | 宽度 | 高度 | 说明 |
|------|------|------|------|
| 地面瓦片 | 128px（推荐） | 约 64~86px | 菱形比例约 2:1 |
| 物体/墙壁瓦片 | 128px（推荐） | 自由高度 | 底部宽度与地面菱形一致，高度自由延伸 |

### 两种视图模式的网格参数

| 参数 | 等距模式 (Isometric) | 正视模式 (Top-down) |
|------|---------------------|---------------------|
| 网格形状 | 菱形（钻石形） | 正方形 |
| tileWH | 40（半宽） | — |
| tileHH | 20（半高） | — |
| tdTileW | — | 40（格子宽） |
| tdTileH | — | 40（格子高） |
| 菱形宽高比 | 2:1 | 1:1 |
| 等效角度 | ~26.57°（arctan 0.5） | 90°（正俯视） |

### 推荐图片尺寸

**地面瓦片**：

| 参数 | 推荐值 |
|------|--------|
| 宽度 | 128px |
| 高度 | 64~96px |
| 形状 | 菱形区域（2:1 比例），四周留透明 |
| 角度 | 约 26.57°（像素等距标准） |

**物体/墙壁瓦片**：

| 参数 | 推荐值 |
|------|--------|
| 宽度 | 128px（与地面一致） |
| 高度 | 自由高度（按物体实际比例） |
| 底部 | 底部宽度与菱形一致 |
| 顶部 | 自由延伸 |

### 关键渲染规则

代码逻辑参考 `IsoCanvas.lua:190-210`：

- **宽度自适应**：图片会被缩放到菱形全宽（`tileWH * 2`），所以所有瓦片宽度建议统一（如 128px）
- **底部锚定**：图片的底边对齐到菱形的底顶点
- **透明度裁剪**：不做菱形遮罩裁剪，靠 PNG 透明度自然生效

### 同一张图两种模式共用

引擎用同一张 PNG 适配两种视图，缩放逻辑：

```
等距模式: drawW = tileWH * 2 (菱形全宽)，等比缩放高度
正视模式: drawW = tdTileW    (格子宽度)，等比缩放高度
```

所以一张图不需要做两个版本，引擎自动按宽度缩放。

### ⚠️ 外部生成素材的常见问题

外部工具生成等距素材常用 30° 标准等距（宽高比约 1.73:1），而本编辑器用 **2:1 像素等距**，角度差导致素材和网格错位。

**生成素材时的提示词建议**：

```
（英文）
2:1 pixel art isometric tile, 26.5 degree angle,
diamond shape with width:height ratio exactly 2:1,
transparent background, PNG format, 128px wide

（中文）
等距视角2.5D风格的{物体名称}，{物体描述}，竖向高大，
isometric游戏素材，菱形透视，像素画风格，物体居中，
边缘干净无残留，透明背景，无多余元素，干净简洁
```
