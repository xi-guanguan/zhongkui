# DragonBones 龙骨动画 Runtime（NanoVG 实现）

> **这是什么**：在 TapTap Maker（UrhoX）中实现 DragonBones 龙骨（骨骼）动画的播放，通过让 AI 参照其他语言的 runtime 代码来生成 NanoVG 版本的单文件 runtime。
>
> **原作者**：无争（《猎魔小队》制作人）
> **推荐度**：L3（思路可行并已跑通 demo，但属于"引导 AI 逐步实现"的方法，成品质量取决于 AI 当次表现和用户的调试耐心，无现成可复用代码）
> **适用场景**：需要在游戏中使用 DragonBones 骨骼动画（从 DragonBones 编辑器导出的 _ske.json + _tex.json + png 图集）

---

## 一、核心思路

引擎没有内置 DragonBones 支持，但可以：

1. 准备 DragonBones 官方的 **数据格式文档**（JSON 格式说明）
2. 准备一个**已有语言的 runtime 实现**作为参照（如 pixi.js 版本）
3. 让 AI **对照参照实现**，用 NanoVG 绘制 API 写一个 Lua 版 runtime

## 二、所需素材

| 文件 | 说明 |
|------|------|
| `DragonBones_5.0_data_format_zh.json` | 龙骨 5.0 数据格式说明文档 |
| `pixi_dragon.md`（或其他语言 runtime 源码） | 参照实现，AI 据此理解渲染逻辑 |
| `XXX_ske.json` | 龙骨导出的骨架数据 |
| `XXX_tex.json` | 龙骨导出的图集描述 |
| `XXX_tex.png` | 龙骨导出的图集纹理 |

## 三、分步引导 AI 实现

### 第 1 步：初次生成

向 AI 提供所有素材文件，并说明需求：

```
我上传了龙骨动画的相关文档：
- DragonBones_5.0_data_format_zh... 是 JSON 格式说明文档
- pixi_dragon.md 中有基于 pixi.js 的龙骨动画实现
- XXX_ske.json 和 XXX_tex.json 是龙骨动画导出的两个 JSON
- 素材库中有对应的 png 图片

请使用 NanoVG 实现一个 TapTap 制造的龙骨动画单文件 runtime，
同时制作一个 demo 展现出这个龙骨动画。
```

> 预期结果：第一次生成大概率会"歪七扭八"，这是正常的。

### 第 2 步：对照修正

```
请仔细阅读官方实现，然后修正或补充我们的新 runtime。
```

> 预期结果：骨骼结构和基本姿态基本正确，但细节可能仍有偏差。

### 第 3 步：细节调整

观察渲染结果，向 AI 指出具体的视觉问题。例如：

```
羊的小腿和蹄子之间有一些空隙，原版龙骨动画里没有这个空隙。
```

AI 会分析原因（如图集 trimming 导致的 frameX/frameY 偏移，pivot 计算用了裁切后尺寸而非原始帧尺寸）并修复。

> 关键：**用户需要有对照原版的能力**，能发现渲染差异并准确描述。

### 第 4 步：功能查漏补缺

```
你已经实现了哪些功能？还有哪些没实现？
```

AI 会列出已实现和未实现的功能清单，根据项目需求决定是否继续补全。

## 四、常见的细节陷阱

| 问题 | 原因 | 修复方向 |
|------|------|---------|
| 部件之间有空隙 | 图集 trimming 后的 frameX/frameY 偏移未正确处理 | pivot 计算应使用 frameWidth/frameHeight 而非裁切后的 width/height |
| 骨骼变形/错位 | 变换矩阵的乘法顺序或坐标系不一致 | 对照参照实现检查 SRT 的应用顺序 |
| 动画不流畅 | 关键帧插值实现不完整 | 检查缓动函数和补间逻辑 |
| 部件层级错乱 | 渲染顺序未按 zOrder 排序 | 按 slot 的 displayIndex 和 zOrder 排序后绘制 |

## 五、局限性

- **无现成可复用代码**：每次使用都需要 AI 重新生成，质量不稳定
- **依赖用户调试能力**：需要用户能对照原版发现问题并准确描述
- **功能可能不完整**：AI 会精简实现，网格变形、FFD、IK 约束等高级功能可能缺失
- **仅验证过简单骨骼**：复杂骨骼动画（多层嵌套、网格变形）未经测试
