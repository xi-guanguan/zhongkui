# 2D 等轴测方向系统工作原理

> **这是什么**: 等轴测游戏从鼠标点击到角色移动渲染的完整数据流六层拆解。
> **原作者**: 星海浩瀚
> **推荐度**: L2（概念深度参考，理解等轴测方向逻辑的必读材料）
> **适用场景**: 开发等轴测 RPG / SLG / 城建游戏时理解坐标变换和方向系统

---

## 核心问题

等轴测游戏有两套坐标系，方向逻辑的本质就是在两者之间做映射：

- **逻辑坐标系**（正交网格）: gx 向右，gy 向上
- **屏幕坐标系**（菱形）: gx+1 → 右下，gy+1 → 左下

## 六层数据流

### 第 1 层: 坐标变换

```
正向（网格→屏幕）:
  screenX = (gx - gy) × halfTileW
  screenY = (gx + gy) × halfTileH

逆向（屏幕→网格）:
  gx = (screenX/halfTileW + screenY/halfTileH) × 0.5
  gy = (screenY/halfTileH - screenX/halfTileW) × 0.5
```

### 第 2 层: 8 方向定义

| 屏幕方向 | 网格 dgx | 网格 dgy |
|---------|---------|---------|
| N（正上） | -1 | -1 |
| S（正下） | +1 | +1 |
| E（正右） | +1 | -1 |
| W（正左） | -1 | +1 |
| NE | 0 | -1 |
| NW | -1 | 0 |
| SE | +1 | 0 |
| SW | 0 | +1 |

### 第 3 层: 移动方向判定

用屏幕空间 `atan2(dy, dx)` 按 45° 扇区划分到 8 方向。

### 第 4 层: 动画帧选择

4 方向精灵 + 镜像 = 8 方向覆盖（详见「8方向走路动画」文档）。

### 第 5 层: 鼠标点击 → 目标格子

屏幕坐标 → 减相机偏移/除缩放 → 逆等轴测变换 → 取整 → 目标格子。

### 第 6 层: 通行性判定

边界检查 + 瓦片类型查询（建筑/水域不可走）。

## 完整数据流

```
鼠标点击 → 逆相机变换 → 逆等轴测变换 → 取整得目标格子
→ 通行性判定 → 计算屏幕位移 → atan2 求 8 方向
→ 选择精灵帧 → 逐帧插值移动 → 到达后更新逻辑坐标
```

每层独立可替换（如 8 方向改 16 方向，只改角度划分和精灵映射）。
