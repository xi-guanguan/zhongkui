# 逐帧入场动画防闪烁：手动插值替代引擎过渡

> **这是什么**：解决 UI 元素首次出现时的"闪烁"问题——元素在第一帧以最终状态渲染，下一帧才开始动画过渡，导致肉眼可见的跳变。方案是放弃引擎内置 transition，改用 Update() 中手动插值，从第一帧就控制元素状态。
>
> **原作者**：浠涫（《节拍前夜 Beat Eve》制作人）
> **推荐度**：L1（经过实战验证，部署即生效，效果稳定）
> **适用场景**：任何需要 UI 入场动画且不容忍首帧闪烁的场景——弹窗、卡片展开、列表项滑入、HUD 元素出现等

---

## 一、问题：为什么会闪烁

引擎内置的 transition/tween 系统存在一个时序问题：

```
第 0 帧：元素创建 → 以最终状态（完整大小、不透明、最终位置）渲染一次
第 1 帧：transition 开始 → 从初始状态过渡到最终状态
```

这个"第 0 帧的最终状态"就是闪烁的来源——用户看到元素突然出现在最终位置，然后又跳回起点开始动画。

## 二、解决方案：手动逐帧插值

**核心思路**：不依赖引擎 transition，在 `Update()` 中每帧手动计算元素的当前状态（位置/缩放/透明度），从创建的第一帧就开始动画。

### 2.1 基本模式

```lua
-- 动画状态
local anim = {
    progress = 0,       -- 动画进度 [0, 1]
    duration = 0.3,     -- 动画时长（秒）
    active = true,      -- 是否正在播放
}

-- 在 Update 中每帧更新
function HandleUpdate(eventType, eventData)
    local dt = eventData["TimeStep"]:GetFloat()
    
    if anim.active then
        anim.progress = math.min(1.0, anim.progress + dt / anim.duration)
        
        -- 应用缓动函数
        local t = easeOutCubic(anim.progress)
        
        -- 更新 UI 元素状态
        element.opacity = t                           -- 渐入
        element.scale = Vector2(t, t)                 -- 缩放入场
        element.position = lerpVec2(startPos, endPos, t)  -- 滑入
        
        if anim.progress >= 1.0 then
            anim.active = false
        end
    end
end
```

### 2.2 缓动函数库

```lua
-- 常用缓动函数（t 为 0~1 的线性进度，返回 0~1 的缓动值）

local function easeOutCubic(t)
    return 1 - (1 - t) ^ 3
end

local function easeOutBack(t)
    -- 略微过冲再回弹，适合弹窗/卡片入场
    local c1 = 1.70158
    local c3 = c1 + 1
    return 1 + c3 * (t - 1) ^ 3 + c1 * (t - 1) ^ 2
end

local function easeOutElastic(t)
    -- 弹性效果，适合游戏化场景（奖励、成就）
    if t == 0 or t == 1 then return t end
    local c4 = (2 * math.pi) / 3
    return 2 ^ (-10 * t) * math.sin((t * 10 - 0.75) * c4) + 1
end

local function easeInOutQuad(t)
    -- 平滑加减速，适合滑动/移动
    if t < 0.5 then
        return 2 * t * t
    else
        return 1 - (-2 * t + 2) ^ 2 / 2
    end
end

local function linear(t)
    return t
end
```

### 2.3 辅助插值函数

```lua
local function lerp(a, b, t)
    return a + (b - a) * t
end

local function lerpVec2(a, b, t)
    return Vector2(lerp(a.x, b.x, t), lerp(a.y, b.y, t))
end

local function lerpColor(a, b, t)
    return Color(
        lerp(a.r, b.r, t),
        lerp(a.g, b.g, t),
        lerp(a.b, b.b, t),
        lerp(a.a, b.a, t)
    )
end
```

## 三、不同 UI 类型的推荐参数

| UI 类型 | 动画属性 | 时长 | 缓动函数 | 说明 |
|---------|---------|------|---------|------|
| 弹窗/模态框 | scale + opacity | 0.25~0.35s | easeOutBack | 从 0 缩放到 1，略有过冲 |
| 卡片悬浮放大 | scale + y偏移 | 0.15~0.2s | easeOutCubic | 快速响应，无过冲 |
| 列表项滑入 | x偏移 + opacity | 0.2~0.3s | easeOutCubic | 从右侧滑入 |
| Toast/通知 | y偏移 + opacity | 0.3s | easeOutQuad | 从顶部滑下 |
| 数字跳动 | scale 脉冲 | 0.1~0.15s | easeOutElastic | 先放大再弹回 |

## 四、退出动画

入场动画的逆向操作——元素消失前播放反向动画，播完再真正移除：

```lua
-- 退出动画：progress 从 1 递减到 0
function startExitAnimation(element, onComplete)
    local exitAnim = {
        progress = 1.0,
        duration = 0.2,   -- 退出比入场稍快
        active = true,
        element = element,
        onComplete = onComplete,
    }
    -- 加入更新队列（在 Update 中处理）
    table.insert(activeAnimations, exitAnim)
end

-- Update 中处理退出动画
-- progress 递减，当 <= 0 时调用 onComplete（通常是移除元素）
```

## 五、注意事项

1. **首帧必须设置初始状态**：创建元素后、第一次 Update 前，就把元素设为动画起始状态（如 opacity=0, scale=0），否则仍会闪烁一帧
2. **动画队列管理**：多个元素同时入场时，用数组管理所有活跃动画，在 Update 中遍历更新
3. **避免每帧创建对象**：缓动函数中不要创建新的 Vector2/Color 等对象（垃圾回收压力），复用预分配的变量
4. **与 UI 组件库配合**：如果使用 `urhox-libs/UI`，可以通过修改组件的 style 属性（如 `opacity`、`transform`）来实现动画效果
5. **移动端考虑**：移动端帧率可能不稳定，用 `dt / duration` 做时间归一化保证动画时长一致
