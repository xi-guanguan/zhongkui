# 横版战斗角色序列帧制作方法

> **作者**: 云诺羲 | **级别**: L2-参考 | **来源**: TapTap 开发心得

---

## 核心观点

角色做起来好看的原因：**一帧一帧K的**。序列帧不是拼图，而是工作流。一张图可以只追求好看，但一套序列帧必须能裁、能播、能接、能打。

---

## 工作流总览

```
角色设计稿 → 拆动作为帧 → 逐帧生成（保持一致性）→ 检查清单 → 进游戏测试
```

---

## 1. 先拆动作，不直接生成图片

以"收伞前刺"为例拆为 9 帧：

| 帧 | 内容 |
|----|------|
| F01 | 战斗待机起点 |
| F02 | 身体下压，武器向后收 |
| F03 | 踏步准备前刺 |
| F04 | 武器刺出 |
| F05 | 命中停顿 |
| F06 | 回收武器 |
| F07 | 收招 |
| F08 | 可接下一段 |
| F09 | 回到待机 |

**动作有节奏**：准备 → 出手 → 命中 → 收招，不是九张姿势。

---

## 2. 每帧必须遵守的一致性规则

- same character — 角色不能变
- consistent proportions — 比例不能变
- consistent weapon size — 武器不能变
- consistent foot baseline — **脚底线不能乱**
- same canvas size — 每格画布一样大
- same pivot — 锚点一致

> 脚底线不一致 → 待机时上下抖、攻击时像在滑、落地后不稳。

---

## 3. 网格排列规则

| 格式 | 帧数 | 适用 |
|------|------|------|
| 3×3 | 9 帧 | 普攻、受击、小技能、短动作 |
| 4×4 | 16 帧 | 复杂待机、大动作、技能 |

排列固定：从左到右、从上到下。程序按固定网格裁图。

---

## 4. 每格之间必须留缝

- 角色可以缩小一点
- 武器和特效不要贴边
- 不要串行串列

> 序列帧不是海报，不能追求填满画面。它首先要能裁。

---

## 5. 动作和特效分层制作

**不要把角色动作和特效画死在一起**：
- 动作对了特效太大 → 改不了
- 命中点偏了特效挪不了
- 程序想调透明度也调不了
- 打击感不够无法单独加强命中特效

**正确做法**：角色动作和 VFX 分开生成，后期叠加。

---

## 6. 命中帧必须三对齐

真正打到敌人的那一帧，必须同时满足：
1. 角色动作最用力
2. 攻击特效最明显
3. 判定盒最有效

> 如果角色 F05 才打出去，但特效 F03 就爆完了，玩家会觉得攻击是飘的。

---

## 7. 武器稳定性（以伞武器为例）

提示词必须包含：
```
the parasol handle must be connected to the hand
the shaft must align with the hand and canopy center
no floating umbrella / no detached handle
```

武器状态要清晰可辨（收/半开/全开 = 不同功能）。

---

## 8. 绿幕工作流

用纯绿幕背景不是为了好看，而是检查：
- 边缘干不干净
- 有没有脏色/半透明污染
- 特效是否和背景混了
- 相邻格子有没有串图

---

## 9. Sprite Sheet 检查清单

生成后必须逐项检查：

- [ ] 帧数对不对
- [ ] 排列顺序对不对
- [ ] 画布比例对不对
- [ ] 每格间距够不够
- [ ] 脚底线是否一致
- [ ] 角色比例是否一致
- [ ] 武器大小是否一致
- [ ] 武器是否连着手
- [ ] 动作是否能连续播放
- [ ] 命中帧是否清楚
- [ ] 特效有没有遮住角色
- [ ] 程序能否按固定网格裁切

---

## 10. 多段普攻设计示例

| 段 | 动作 | 武器状态 | 作用 |
|----|------|----------|------|
| A1 | 刺 | 收起 | 快速前刺，起手试探 |
| A2 | 扫 | 半开 | 横扫，扩大攻击范围 |
| A3 | 爆 | 全开 | 终结冲击，击退或削韧 |

- 每段命中帧和 VFX 不同
- 三段尾帧要能接下一段或回到待机
- A1 快 → A2 范围大 → A3 力度最强

---

## 总结

序列帧核心就几句话：
1. 先拆动作，再画帧
2. 先做角色动作，再做特效
3. 脚底线、比例、武器、画布必须稳定
4. 命中帧、特效帧、判定帧必须对齐
5. 生成完要检查，最后必须进游戏测试
