# NanoVG 像素格地图 → 精灵贴图场景改造指南

> **这是什么**: 将 NanoVG 纯色方块渲染的 2D 俯视地图改造为类似《环世界》风格精灵贴图地图的完整方案。
> **原作者**: 相信未来
> **推荐度**: L2（方案参考，含可直接发给 AI 的改造话术模板）
> **适用场景**: NanoVG 渲染的格子地图需要视觉升级时

---

## 核心思路

数据层（grid 数组、cell 坐标）完全不动，只改渲染层 —— 从单循环画纯色矩形，重构为 6 层渲染管线。

## 6 层渲染管线（从底到顶）

| Pass | 层 | 作用 |
|------|------|------|
| 0 | 基底纹理 | 泥土纹理平铺，填充缝隙（露底） |
| 1 | 地形纹理 | 每种地形一张 tileable 纹理 + **不规则边缘多边形** |
| 2 | 区域边界 | 检测 regionId 差异，画描边线 |
| 3 | 网格线 | 可选，高缩放时淡显 |
| 4 | 装饰精灵 | 松树/灌木/岩石等，确定性哈希定位 |
| 5 | 建筑精灵 | 宗门/城镇等 + 名称标签 |

## 关键技术要点

1. **世界空间纹理坐标**: `nvgImagePattern` 的 ox/oy 必须基于世界原点，否则相邻格子纹理不连续
2. **确定性哈希**: 装饰位置用 `x*374761393 + y*668265263` 等哈希，不用 `math.random`（否则闪烁）
3. **纹理加载标志**: `NVG_IMAGE_REPEATX | NVG_IMAGE_REPEATY | NVG_IMAGE_NEAREST`
4. **不规则边缘是视觉核心**: 地形交界处沿边取 7 个采样点，用哈希向内偏移 0~40%，连成不规则多边形

## 常见误区

| 误区 | 表现 | 正确做法 |
|------|------|---------|
| 只换纹理不加不规则边缘 | "纹理马赛克" | 加 Pass 1 锯齿路径 |
| 纹理坐标用每格局部空间 | 相邻格子纹理不连续 | 用世界空间原点 |
| 装饰物用 math.random | 每帧闪烁 | 确定性哈希 |
| 没有基底纹理层 | 边缘露黑 | 先铺泥土底 |
