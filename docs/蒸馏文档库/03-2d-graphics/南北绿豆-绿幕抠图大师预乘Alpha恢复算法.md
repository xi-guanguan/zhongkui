# 绿幕抠图大师（Chroma Key V2 预乘 Alpha 恢复算法）

> **这是什么**: 基于广播级预乘 Alpha 恢复算法的绿幕素材抠图工具完整文档，包含 AI 生图绿幕规范、算法原理、脚本用法和调参指南。能保留半透明区域（云雾/光晕/毛发边缘）。

## 绿幕背景色标准

**必须使用 RGB(0, 250, 0) / `#00FA00`**

| 项目 | 值 |
|------|---|
| 标准背景色 | RGB(0, 250, 0) |
| 可接受范围 | R≤10, G∈[230,255], B≤10 |

> 为什么 250 而非 255: AI 模型处理通道极值 255 时易产生色带(banding)。G=250 满足算法 `G - max(R,B) > 200` 要求，且背景更均匀。

## AI 生图 Prompt 模板

**通用结构**: `{主体描述}，明亮的荧光绿纯色背景，背景颜色为RGB(0,250,0)，不要暗绿色，背景完全均匀无纹理无渐变，{补充要求}`

**关键**: AI 对 RGB 数值不敏感，必须同时加**视觉描述词**（"荧光绿""不要暗绿色"）才能命中目标色。

**generate_image 参数**: `transparent` 必须为 `false`（需要绿色填充背景，非透明背景）。

**生成后质量验收**:
- [ ] 四角和中心背景色一致，无渐变/杂色
- [ ] 主体轮廓与背景有明确分界
- [ ] 主体无大面积绿色
- [ ] 背景无阴影/地面/装饰

## 算法原理（简要）

```
观察颜色 = 真实颜色 × alpha + 绿幕颜色 × (1 - alpha)

alpha = 1 - clamp((G - max(R,B)) / (Gk - max(Rk,Bk)), 0, 1)
真实颜色 = (观察颜色 - 绿幕颜色 × (1-alpha)) / alpha   ← 反预乘
绿色溢出抑制: spill = max(0, G_true - max(R_true, B_true))
边缘精炼: alpha 边缘区 sigma=0.5 高斯模糊消除锯齿
```

## 脚本用法

```bash
source /home/Maker/rembg-env/bin/activate
python /home/Maker/chroma_key_v2.py <输入> <输出> [gain] [cutoff] [despill]
```

| 参数 | 默认 | 说明 |
|------|------|------|
| gain | 1.0 | Alpha 增益，>1 更不透明，<1 更透明 |
| cutoff | 0.02 | 低于此值强制全透明，消除残留噪点 |
| despill | 1.0 | 绿色溢出抑制强度，0=不抑制 |

**常用场景**:
```bash
# 标准（默认参数）
python /home/Maker/chroma_key_v2.py input.png output.png

# 边缘太透 → 加 gain
python /home/Maker/chroma_key_v2.py input.png output.png 1.3

# 云雾素材要更多半透明 → 降 gain + 放宽 cutoff
python /home/Maker/chroma_key_v2.py input.png output.png 0.9 0.01
```

**批量处理**:
```bash
for f in assets/image/*_green_*.png; do
    out=$(echo "$f" | sed 's/_green_[0-9]*//')
    python /home/Maker/chroma_key_v2.py "$f" "$out"
done
```

## 调参速查

| 问题 | 调整 |
|------|------|
| 边缘残留绿色 | 增大 despill（1.5~2.0） |
| 半透明太透 | 增大 gain（1.2~1.5） |
| 背景残留噪点 | 增大 cutoff（0.05~0.1） |
| 主体边缘被裁切 | 减小 gain + cutoff |
| 主体含绿色元素 | 改用蓝幕（需修改脚本检测逻辑） |
