- **这是什么**: 杀戮尖塔风格的竖向分叉网状地图的完整实现思路，覆盖数据定义、五步生成算法、无交叉连线、NanoVG 渲染、节点状态管理、种子确定性等核心环节。四文件解耦架构，约 2300 行代码
- **原作者**: 浠涫（《节拍前夜 Beat Eve》开发者）
- **推荐度**: L1（拿来就能用——算法思路清晰完整，五步流水线可直接复现，核心去交叉算法附代码）
- **适用场景**: Roguelike / 卡牌肉鸽类游戏的分叉地图系统

## 五步生成流水线

### Step 1：确定每层节点数

查表获取范围，随机取值。起点层 3-4 个（多起点），Boss 层固定 1 个。

### Step 2：分配节点类型（多重硬约束）

纯概率分配会产生连续战斗、前期刷精英、商店扎堆等问题，需叠加约束：

| 约束 | 规则 |
|------|------|
| Layer 1-3 | 强制全战斗（入门保护） |
| Layer 17 | 强制全录音房（Boss 前喘息） |
| Layer 18 | 强制 Boss |
| Layer 1-7 | 精英权重归零（前期保护期） |
| Layer 4 | 战斗权重归零（强制玩法多样性） |
| 商店间隔 | 两个商店至少隔 2 层 |
| 同层多样性 | 3 个节点禁止全同类型 |

### Step 3：连线算法（核心难点）

分 ABCDE 五个子步骤：

**A. 分配自然目标**：按位置比例映射上层节点，左连左、右连右，保证基础路径规整。

**B. 额外连线**：60% 节点连 1 个、30% 连 2 个、10% 连 3 个，仅相邻节点扩展。

**C. 去交叉（核心）**：边排序后保留 to 值单调不递减，一趟扫描解决：

```lua
local maxTo = 0
for _, edge in ipairs(allEdges) do
    if edge.to >= maxTo then
        validEdges[#validEdges + 1] = edge
        maxTo = edge.to
    end
end
```

**D. 保证出边**：修复无出边的死路节点。

**E. 保证入边**：修复无入边的孤岛节点，确保全图连通。

### Step 4：渲染（NanoVG）

- 连线用贝塞尔曲线（控制点居中），已走=实线+发光，当前可选=虚线脉冲，锁定=灰色细虚线
- 节点：圆形+Emoji 图标，紫色=可选，灰绿=已完成，暗灰=锁定，红金=Boss
- 当前可选节点添加 sin 波脉冲呼吸效果
- 虚线渲染优化：初期 O(n²)，改为单调前进指针后性能稳定

### Step 5：入场动画（灯串逐层点亮）

```lua
function MapGraphWidget:_GetLayerAlpha(layer)
    local revealTime = (layer - 1) * 0.03  -- 每层间隔 30ms
    local fadeIn = 0.18                     -- 单节点 0.18s 淡入
    local elapsed = self.entranceElapsed_ - revealTime
    if elapsed < 0 then return 0.0 end
    if elapsed >= fadeIn then return 1.0 end
    return elapsed / fadeIn
end
```

配合 ScrollView 从 Boss 端缓缓滚到当前层的 2 秒动画。

## 节点状态管理

四种状态，与杀戮尖塔规则一致：

| 状态 | 视觉 | 可交互 |
|------|------|--------|
| completed | 灰绿色+勾 | 否 |
| current | 紫色脉冲 | 是（可点击） |
| locked | 暗灰色 | 否 |
| boss | 红金发光 | 终局解锁后可点击 |

计算规则：已走路径内 → completed，最后节点的后继 → current，初始为空则 Layer1 全为 current，其余 → locked。

## 种子确定性

Roguelike 必须保证同种子生成相同地图。核心问题：**Lua `pairs` 遍历无序**。

解决方案：权重随机前先对键排序，保证遍历顺序固定：

```lua
local function WeightedRandom(weights)
    local sorted = {}
    for k, w in pairs(weights) do
        sorted[#sorted + 1] = k
    end
    table.sort(sorted)  -- 关键：排序后遍历顺序确定
    local r = SeedManager.Random("map") * total
    local acc = 0
    for _, k in ipairs(sorted) do
        acc = acc + weights[k]
        if r <= acc then return k end
    end
end
```

用 SeedManager 隔离随机流：地图、事件、战利品各用独立 key，互不干扰。

## 踩坑避坑

| 坑 | 问题 | 解决 |
|----|------|------|
| 层数规划 | 15 层节奏仓促 | 增到 18 层，加 3 层入门区 |
| 去交叉 | 交换目标修复法会产生新交叉 | 改用单调递增过滤，一趟扫描搞定 |
| 节点抖动 | 固定抖动值导致重叠 | 按间距 35% 比例抖动 |
| 奇遇节点 | 纯随机可玩性低 | 新增惊喜变异（15% 变战斗、10% 变商店） |

## 优先级清单

| 优先级 | 内容 |
|--------|------|
| **P0 必须有** | 节点类型+概率权重、逐层连线+不交叉、节点状态计算、种子确定性 |
| **P1 体验提升** | 硬约束（入门保护/Boss前休息/精英保护期）、多样性保障、贝塞尔曲线连线+虚实线区分、滚动定位+入场动画 |
| **P2 锦上添花** | 装饰花纹、灯串入场、品质递增、惊喜变异 |

**核心总结**：每层 2-4 个节点，逐层连线不交叉，硬约束控分布，状态驱动渲染。
