# AI 视频转游戏序列帧完整工作流

> **这是什么**: 将 AI 生成的视频转换为游戏可用序列帧精灵图的完整 5 步流水线，涵盖预生产确认、参考图准备、AI 视频生成、帧提取、背景移除和精灵图组装。
> **原作者**: 南北绿豆
> **推荐度**: L2（参考学习价值 — 流水线完整，适合需要从 AI 视频产出游戏动画素材的团队）
> **适用场景**: 角色攻击/技能动画、特效动画、NPC 行为动画等需要 AI 视频生成序列帧的场景

---

## 一、5 步流水线总览

```
步骤1: 准备参考图（拍照/截图/AI生成）
  ↓
步骤2: AI 生成视频（Seedance/create_video_task）
  ↓
步骤3: 提取帧（ffmpeg）
  ↓
步骤4: 移除背景（flood fill 或 rembg AI）
  ↓
步骤5: 组装精灵图（统一包围盒 + 紧凑网格）
```

---

## 二、预生产确认（4 轮对话）

在动手之前，必须与用户确认 4 轮信息：

| 轮次 | 确认内容 | 示例 |
|------|---------|------|
| 第 1 轮 | 动作类型 | 攻击/跑步/待机/技能释放 |
| 第 2 轮 | 具体风格 | 横斩/竖劈/旋风斩/刺击 |
| 第 3 轮 | 朝向和配色 | 右向/左向、红色系/蓝色系 |
| 第 4 轮 | 尺寸和画风 | 128x128 像素风 / 256x256 手绘风 |

**自动推导**: 第 4 轮确认后，自动推导下游参数（帧率、总帧数、视频时长、背景移除方法）。

## 三、步骤 1: 准备参考图

### 参考图要求

- 角色/对象占画面约 **60%**（居中）
- 纯色或简单背景（便于后续抠图）
- 清晰展示动作起始姿态

### 自动预处理

如果原图占比不足 60%，自动添加 padding：

```python
# 检测占比 → 计算目标画布尺寸 → 居中粘贴 → 输出
occupancy = (bbox_area / total_area)
if occupancy < 0.6:
    target_size = calc_padded_size(bbox, 0.6)
    padded = Image.new("RGBA", target_size, bg_color)
    padded.paste(img, center_offset)
```

## 四、步骤 2: AI 生成视频

### 两种生成模式

| 模式 | API 参数 | 适用场景 |
|------|---------|---------|
| **multi_modal_reference** | `mode="multi_modal_reference"`, images=[参考图] | 有参考图，动作自由度高 |
| **first_frame** | `mode="first_frame"`, images=[起始帧] | 精确控制起始姿态 |

### Prompt 模板

```
[角色描述]，[动作描述]，[风格要求]，
plain [背景颜色] background，centered composition，
smooth motion，game sprite animation
```

### 关键参数

| 参数 | 推荐值 |
|------|--------|
| `duration` | 4-6 秒（短动作 4s，长动作 6s） |
| `ratio` | "1:1"（精灵图通常正方形） |
| `resolution` | "720p" |

### 用户确认点

视频生成后，**必须让用户确认视频质量**再进入下一步。

## 五、步骤 3: 提取帧

### ffmpeg 命令

```bash
ffmpeg -i input.mp4 -vf "fps=10" -qscale:v 2 frames/frame_%04d.png
```

| 参数 | 说明 |
|------|------|
| `fps=10` | 推荐 10 FPS（平衡帧数和流畅度） |
| `-qscale:v 2` | 高质量输出 |

### 帧率选择参考

| 动作类型 | 推荐 FPS | 理由 |
|---------|---------|------|
| 快速攻击 | 12-15 | 需要更多中间帧保证流畅 |
| 跑步/行走 | 8-10 | 循环动画，帧数适中 |
| 待机/呼吸 | 6-8 | 缓慢动作，少帧即可 |

## 八、动画时序参考

### 攻击动画三阶段

| 阶段 | 含义 | 帧占比 |
|------|------|--------|
| **蓄力 (Wind-up)** | 准备动作 | 20-30% |
| **打击 (Strike)** | 攻击瞬间 | 10-20% |
| **恢复 (Recovery)** | 收招复位 | 50-60% |

### 帧数建议

| 动作类型 | 推荐总帧数 |
|---------|-----------|
| 快速攻击 | 8-12 帧 |
| 重击/技能 | 15-24 帧 |
| 跑步循环 | 8-12 帧 |
| 待机呼吸 | 6-10 帧 |
| 死亡倒地 | 12-18 帧 |

## 九、常见问题

| 问题 | 解决方案 |
|------|---------|
| AI 视频中角色偏移/飘动 | 重新生成，prompt 加 "centered, stable position" |
| 背景移除后边缘有残留 | 增大 flood fill 容差，或改用 rembg |
| 精灵图中角色大小不一致 | 使用统一包围盒（步骤 5） |
| 动画播放不流畅 | 提高提取帧率（fps=12-15） |
| 动画循环不衔接 | 使用 first_last_frame 模式，首尾帧相同 |
