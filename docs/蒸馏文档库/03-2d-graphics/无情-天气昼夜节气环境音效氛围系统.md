# 天气/昼夜/节气/环境音效氛围系统

> **这是什么**：挂机 RPG 的完整世界氛围系统——12 时辰昼夜渐变、7 种天气双层加权随机、24 节气联动、8 层环境音效管线，四个系统通过查询接口解耦，链式反应驱动"活着的世界"。
>
> **原作者**：无情
> **推荐度**：L2（设计思路完整、系统联动精巧；原文偏高层设计解析而非逐行代码，需根据项目自行实现细节）
> **适用场景**：挂机/放置类游戏需要低成本氛围感；任何需要天气/昼夜/环境音效系统的 2D 游戏

---

## 一、总体架构

四个系统通过查询接口解耦，共享同一份状态表（引用传递，零 GC）：

```
WeatherSystem(状态) → WeatherRenderer(渲染) → WeatherParticles(粒子)
       ↓ 查询
AmbientSoundSystem(每帧查天气+时辰→决定播什么)
```

**核心原则**：数据驱动、平滑过渡、系统解耦、提前返回、零 GC。

## 二、昼夜：12 时辰细粒度渐变

### 2.1 为什么不用 4 段式

常见 4 段（白天→黄昏→夜晚→黎明）过渡生硬。用 **12 时辰**，每个时辰独立 RGBA 色调：

| 时辰 | 色调示例 |
|------|---------|
| 深夜 | 暗蓝 `{5,5,25,168}` |
| 清晨 | 暖橘 `{255,180,110,22}` |
| 上午 | 通透 `{0,0,0,0}` |
| 傍晚 | 橘红 `{255,105,35,42}` |

### 2.2 平滑过渡

相邻时辰**边缘 25% 线性插值**：中间 50% 纯色保证辨识度，首尾各 25% 过渡不跳变。

### 2.3 真实时间同步

进入地图读 `os.date()` 映射到游戏内 2 小时循环。晚上 10 点打开就是夜晚，早上 7 点就是暖橘清晨。

### 2.4 视觉分层

**白天**：7 个径向渐变阳光光斑 + 漂浮粒子（正午 1.6 倍增强）

**夜晚**：双层暗角 + 40 颗正弦波闪烁星星（大星星带十字光芒）+ 月光渐变 + 3 条斜向光束

每层提前返回，不需要画的直接 `return`。

## 三、天气：双层加权随机 + 平滑过渡

### 3.1 七种天气

晴 / 多云 / 雨 / 暴雨 / 雪 / 风 / 雾

### 3.2 双层权重叠加

**第一层：地图主题权重**（决定天气基调）

| 地图 | 权重示例 |
|------|---------|
| 亡灵 | `{cloudy=35, fog=25}` |
| 沙漠 | `{clear=50, wind=35}` |
| 冰原 | `{snow=40}` |

**第二层：节气修正系数**

- 谷雨：雨概率 ×2.5
- 大雪：雪概率 ×2.5
- 另有禁用表：大暑禁雪、大雪禁雨

最终加权随机抽取，通过 `transitionT`(0→1) 平滑插值切换。

### 3.3 粒子三层景深

以雪为例：

| 层 | 尺寸 | 速度 | 密度 | 特效 |
|---|---|---|---|---|
| 近景 | 大雪花 | 慢 | 疏 | 有旋转 |
| 中景 | 标准 | 中 | 中 | 标准 |
| 远景 | 小 | 快 | 密 | — |

各层带正弦波左右摆动。粒子不用多——雨 80、雪 60、雾 30 颗足够。

### 3.4 闪电：生产者-消费者模式

天气系统设 `lightningFlash=true` + 白屏衰减，音频系统读后即清同帧播雷声，视听同步。

## 四、节气：真实时间→世界气质

### 4.1 时间映射

**24 小时 = 24 节气**，每小时轮换。挂机单次游玩 10-30 分钟，两三次打开大概率看到不同节气。

### 4.2 节气影响

| 维度 | 影响 |
|------|------|
| 天气权重 | 修正系数 + 禁用表 |
| 季节色调 | 春绿 / 夏暖黄 / 秋橘 / 冬冷蓝 |
| 地面覆盖 | 积雪 / 落叶 / 绿意 |
| 特殊效果 | 惊蛰闪电、白露晨雾、冬至极光 |

### 4.3 八种氛围粒子自动切换

根据节气/时段/天气/地图四维判断：

| 粒子 | 触发条件 |
|------|---------|
| 萤火虫 | 夏夜（双层 glow 纹理 + 正弦脉动） |
| 落叶 | 秋风 |
| 花瓣 | 春晴 |
| 冰晶 | 冬季 |
| 晨露 | 白露 |
| 沙尘 | 沙漠地图 |
| 霜粒 | 霜降 |
| 符文 | 魔法地图 |

### 4.4 三层色调混合

时辰×权重1 + 天气×权重2 + 季节×权重3，权重由各层 Alpha 决定，总 Alpha 限 195。简单叠加会糊，要加权混合。

## 五、环境音效：8 层管线 + Crossfade

### 5.1 分层

| 层 | 内容 | 控制 |
|---|---|---|
| 地貌底音 | 地图基调 | 独立音量 |
| 天气叠加 | 雨声/风声 | 独立音量 |
| 生灵之声 | 12 时辰 + 天气感知 | 独立音量 |
| 时辰提示 | 鸡鸣/晚钟 | 独立音量 |
| 闪电雷鸣 | 闪电同帧触发 | 独立音量 |
| 副本音效 | 副本专属 | 独立音量 |

### 5.2 循环接缝解决方案：手动 Crossfade

核心难题：循环接缝有截断声。

```
A 距结尾 2s → 创建 B(音量0)
A 渐出, B 渐入
A 播完 → B 接替
循环往复
```

### 5.3 天气感知生灵之声

同一时辰不同天气不同生物：

| 时辰 | 晴天 | 雨天 | 雪天 |
|------|------|------|------|
| 深夜 | 虫鸣蛙声 | 蛙鸣+雨打树叶 | 呼啸风雪 |

先查天气专属表，没有退回时辰默认表。同场景 6 种变化。

### 5.4 副本分层

进副本：天气/生灵压到 **20%**（像隔一扇门），副本底音 40%，副本音乐 45%。

完全关掉失去世界感，全音量太嘈杂，20% 是测出的平衡点。

另有每 2 秒健康检查，检测意外停播的音轨并重启。

## 六、系统联动（链式反应）

```
节气「谷雨」→ 雨概率×2.5 → 抽到雨
  → 色调灰蓝 + 雨粒子
  → 音效切雨声 + 生灵切蛙鸣
  → 完整谷雨氛围

进入亡灵地图 → 多阴多雾 → 节气修正 → 全部跟随
```

## 七、实现路线图

**推荐顺序**（渐进式，每步可独立交付）：

| 步骤 | 内容 | 预期效果 |
|------|------|---------|
| 1 | 昼夜（timer + 色调表 + 矩形覆盖） | 基本昼夜循环 |
| 2 | 天气（权重随机 + 粒子） | 偶尔下雨/下雪 |
| 3 | 音效（一轨循环起步） | 有环境音 |
| 4 | 节气（加一层修正系数） | 天气随时节变化 |

**30 分钟 MVP**：4 段色调插值 + 一轨环境音循环 + 偶尔下雨（30 个圆点往下掉）——加起来就比 90% 的挂机游戏有氛围了。

## 八、关键设计要点

| 要点 | 说明 |
|------|------|
| 零 GC | 所有子模块共享同一份状态表，引用传递 |
| 提前返回 | 每层渲染先判断是否需要画，不需要直接 `return` |
| 数据驱动 | 色调表、权重表、生灵表全是 table 配置，不硬编码逻辑 |
| 平滑过渡 | 天气切换用 `transitionT`(0→1)，色调用边缘 25% 插值 |
| 生产者-消费者 | 闪电：天气系统设标记 → 音频系统读后即清 → 视听同步 |
