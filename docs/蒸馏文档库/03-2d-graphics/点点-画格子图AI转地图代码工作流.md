# 画格子图 AI 转地图代码工作流

> **这是什么**：用绘图工具画格子图（黑白网格），发给 AI 视觉识别后自动转换为游戏地图数据（Lua 代码），并通过概率门变体和旋转翻转将 38 张手绘图扩展为 900+ 种运行时布局。
>
> **原作者**：点点
> **推荐度**：L1（经实战验证的完整工作流，从 38 张手绘图生成 114 个模板、~6900 行代码，直接用在了已发布的魔塔游戏中。流程清晰、零工具链依赖、可直接复用。）
> **适用场景**：魔塔/Roguelike 地牢、推箱子/解谜关卡、塔防路径、棋盘类布局、像素画场景——任何基于离散网格的 2D 地图设计。

---

## 1. 整体流程

```
手绘格子图 → 发给 AI 视觉识别 → AI 输出文本地图 → 概率门变体扩展 → 写入 Lua 代码文件
```

| 步骤 | 输入 | 输出 | 耗时 |
|------|------|------|------|
| 1. 画格子图 | 绘图工具 | PNG 图片 | ~1 小时（38 张） |
| 2. 发给 AI | PNG 图片 | — | 几秒 |
| 3. AI 识别 | 图片 | 文本符号矩阵 | ~10 分钟（38 张） |
| 4. 变体扩展 | 基础地图 | 3 个概率门变体/图 | AI 自动 |
| 5. 写入代码 | 变体数据 | Lua 数据文件 | AI 自动 |

## 2. 画图规则

### 2.1 颜色约定

| 颜色 | 含义 | 符号 |
|------|------|------|
| 黑色/深色 | 墙壁（不可通行） | `#` |
| 绿色/浅色 | 空地（可通行） | `.` |

### 2.2 网格尺寸

以魔塔 13x13 为例：
- **外圈**（第 1 行/列和第 13 行/列）：固定为墙壁，形成地图边界
- **内部 11x11**：自由设计的可玩空间

> 网格尺寸可根据游戏调整（9x9、16x16 等），关键是事先约定好。

### 2.3 设计原则

| 原则 | 说明 |
|------|------|
| 保证连通性 | 所有空地区域必须相互连通，不能有封闭孤岛 |
| 留出走廊和咽喉 | 狭窄通道是放置概率门的理想位置 |
| 考虑对称性 | 对称布局更美观，也更容易让 AI 准确识别 |
| 墙壁数量适中 | 内部墙壁建议占 30%~50%，太少缺策略性，太多行走空间不足 |

### 2.4 工具推荐

任何支持网格绘制的工具都可以：
- **推荐 SAI**：油桶工具直接填色，效率最高
- 像素画工具、Excel 也可以
- 关键是格子边界分明、颜色对比明显

## 3. 发给 AI 识别

### 3.1 操作方式

将画好的格子图截图或导出为 PNG，直接发到 AI 对话中。

**示例指令**：

```
这些是我设计的 13×13 魔塔地形图，请读取每张图的墙壁布局，
用 # 表示墙壁、. 表示空地，输出为文本格式。
```

### 3.2 注意事项

| 要求 | 说明 |
|------|------|
| 图片要清晰 | 格子边界分明，黑白/深浅对比明显 |
| 一张图一个地图 | 不要把多个地图画在同一张图里 |
| 网格要对齐 | 格子大小一致，不要歪斜 |
| 可以批量发 | AI 能同时处理多张图片，一次发完更高效 |

### 3.3 为什么识别准确率极高

- 格子图本身就是离散网格结构，不存在模糊边界
- 只有两种状态（墙/空），判断逻辑简单
- 对比度足够高（黑色 vs 绿色），不会混淆

## 4. 概率门变体扩展

### 4.1 什么是概率门

概率门是放置在空地上的特殊元素，用符号 `i` 表示。运行时随机决定结果：

| 结果 | 概率 | 效果 |
|------|------|------|
| 消失（空地） | 50% | 路径畅通，不消耗资源 |
| 黄色门 | 43% | 需要 1 把黄钥匙开启 |
| 蓝色门 | 6.5% | 需要 1 把蓝钥匙开启 |
| 红色门 | 0.5% | 需要 1 把红钥匙开启 |

### 4.2 放置规则

每张基础地图生成 3 个变体，AI 按以下规则放置概率门：

| 规则 | 说明 |
|------|------|
| 每个变体恰好 7 个概率门 | 数量固定 |
| 优先放在咽喉位置 | 走廊中间、两面是墙的通道口 |
| 禁止放在死胡同 | 三面是墙的位置没有通行意义 |
| 3 个变体的门位置互不相同 | 确保每次游戏体验不同 |

### 4.3 指令示例

```
请基于每张基础地图生成 3 个概率门变体：
- 将空地 . 替换为概率门 i，每个变体恰好 7 个
- 优先放在咽喉位置（两面是墙的通道）
- 不要放在死胡同（三面是墙的格子）
- 3 个变体的门位置互不相同
```

## 5. 写入 Lua 代码

### 5.1 符号到瓦片映射

```lua
local SYMBOL_TO_TILE = {
    ["#"] = TILE.WALL,       -- 墙壁
    ["."] = nil,             -- 空地
    ["i"] = "RANDOM_DOOR",   -- 概率门（运行时掷骰决定）
}
```

### 5.2 模板数据格式

```lua
{
    id = "normal_001",
    object = {
        "# # # # # # # # # # # # #",
        "# . # . # . # . # . # . #",
        "# . # . # . # . # . # . #",
        "# i # . # . # . # . # . #",
        "# . # . # . # . # . # . #",
        "# i # . # i # . # . # . #",
        "# . . . . . . . . . i . #",
        "# . # i # . # i # . # . #",
        "# . # . # . # . # . # . #",
        "# . # . # . # . # . # . #",
        "# i # . # . # . # . # . #",
        "# . # . # . # . # . # . #",
        "# # # # # # # # # # # # #",
    },
}
```

### 5.3 运行时解析

游戏加载时遍历每行符号：
- `#` → 创建墙壁瓦片
- `.` → 空地（不创建）
- `i` → 标记为概率门，玩家进入该层时掷骰决定最终结果

### 5.4 运行时变换

除了 114 个静态模板，游戏还对抽到的模板做随机变换：
- **水平翻转**：左右镜像
- **旋转**：90/180/270 度

## 7. 方法优势总结

| 优势 | 说明 |
|------|------|
| 直觉设计 | 画格子比写代码直观得多，所见即所得 |
| 批量高效 | 38 张图一次性发给 AI，几分钟内全部转换 |
| 零误差 | AI 对离散网格的识别准确率极高 |
| 自动扩展 | 概率门 + 旋转翻转让内容量翻 20+ 倍 |
| 无需工具链 | 不需要地图编辑器、导出脚本、中间格式 |
