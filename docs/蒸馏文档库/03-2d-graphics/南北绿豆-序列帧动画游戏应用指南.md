# 序列帧动画游戏应用指南（焚绝工作流 → 游戏集成）

> **这是什么**: 从 AI 视频生成序列帧精灵图，并在 UrhoX 游戏中通过 SpriteAnimator 类播放动画的完整实战指南。涵盖"焚绝"工作流（AI 视频 → 提帧 → 去背景 → 拼精灵图）和游戏端集成代码。
> **原作者**: 南北绿豆
> **推荐度**: L1（工作流经过实战验证，SpriteAnimator 类可直接复用，含真实案例数据）
> **适用场景**: 需要快速产出角色动画素材（攻击/技能/怪物行为）并集成到 UrhoX 游戏中

---

## 一、"焚绝"工作流总览

"焚绝"是团队内部对 AI 视频→序列帧→游戏精灵图管线的代号。

```
准备参考图 → AI 生成动作视频 → FFmpeg 提帧
  → 去除背景 → 组装精灵图 → SpriteAnimator 播放
```

**实战数据**: 12 个怪物 × 4 个动作 = 48 套动画，3 天完成（传统手绘需要数月）。

---

## 二、准备参考图

AI 视频生成需要一张角色参考图作为起点：

- **要求**: 角色居中、背景干净（纯色最佳）、姿势自然（T-pose 或待机姿势）
- **尺寸**: 建议 512×512 或 1024×1024
- **来源**: AI 生图、手绘稿、3D 渲染截图均可

**提示**: 参考图四周留 20% 以上空白，给动作预留运动空间。

## 三、AI 视频生成

使用嗒啦啦（Talala / create_video_task）生成动作视频：

```
模式选择:
- first_frame    — 只给首帧参考图（适合简单动作）
- first_last_frame — 给首尾帧（适合需要回到初始姿势的循环动画）
- multi_modal_reference — 给多张参考图（适合复杂动作）
```

**Prompt 模板**:
```
攻击动画: "角色挥刀攻击，动作干净利落，保持角色朝向不变，纯色背景"
待机动画: "角色原地待机呼吸，微微晃动，保持位置不变，纯色背景"
行走动画: "角色原地踏步行走循环，不要位移，纯色背景"
受击动画: "角色被击中后仰，短暂停顿后恢复，纯色背景"
```

**关键词**: "纯色背景" + "保持位置不变" + "原地"，避免角色跑出画面。

## 四、FFmpeg 提帧

从视频中按固定帧率提取 PNG 序列：

```bash
# 提取为 10fps，缩放到 128×128（像素风格用 neighbor 插值）
ffmpeg -i attack.mp4 \
  -vf "fps=10,scale=128:128:flags=neighbor" \
  frames/attack_%03d.png

# 提取为 12fps，保持原始分辨率（高清风格）
ffmpeg -i attack.mp4 \
  -vf "fps=12" \
  frames/attack_%03d.png
```

**帧率选择参考**:

| 动作类型 | 推荐帧率 | 总帧数（4秒视频） |
|---------|---------|----------------|
| 待机/呼吸 | 8 fps | 32 帧 |
| 行走循环 | 10 fps | 40 帧 |
| 攻击动作 | 12 fps | 48 帧 |
| 快速技能 | 15 fps | 60 帧 |

**注意**: 帧数越多精灵图越大，移动端建议单张精灵图不超过 2048×2048。

## 五、去除背景

### 方法 A: Flood Fill（纯色背景推荐）

背景是纯色时最高效：

```bash
# ImageMagick flood fill 去除绿色背景
convert frame.png -fuzz 15% -transparent "#00FF00" frame_transparent.png

# 批量处理
for f in frames/attack_*.png; do
    convert "$f" -fuzz 15% -transparent "#00FF00" "${f%.png}_t.png"
done
```

`-fuzz 15%` 控制颜色容差，背景不够干净时适当调大（最大 25%）。

### 方法 B: rembg AI 去背景（复杂背景）

```bash
pip install rembg
rembg p frames/ frames_transparent/
```

AI 去背景效果更好但速度慢，适合背景不规则的情况。

## 六、组装精灵图（Sprite Sheet）

将所有帧拼成一张大图：

```bash
# ImageMagick 水平拼接（单行）
montage frames/attack_*_t.png \
  -tile x1 \
  -geometry 128x128+0+0 \
  -background none \
  spritesheet_attack.png

# 网格拼接（多行，适合帧数多的动画）
montage frames/attack_*_t.png \
  -tile 8x \
  -geometry 128x128+0+0 \
  -background none \
  spritesheet_attack.png
```

**输出示例**（12 帧，单行）:
```
┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
```

## 七、SpriteAnimator —— UrhoX 游戏端播放

这是在 UrhoX 中播放精灵图动画的核心类，通过 UV 窗口滑动实现帧切换：

```lua
---@class SpriteAnimator
local SpriteAnimator = {}
SpriteAnimator.__index = SpriteAnimator

--- 创建精灵动画器
---@param spriteSheet string 精灵图路径（如 "Textures/attack.png"）
---@param frameCount number 总帧数
---@param columns number 精灵图列数（单行则 = frameCount）
---@param fps number 播放帧率
function SpriteAnimator:new(spriteSheet, frameCount, columns, fps)
    local o = setmetatable({}, self)
    o.frameCount = frameCount
    o.columns = columns
    o.rows = math.ceil(frameCount / columns)
    o.fps = fps or 10
    o.frameTime = 1.0 / o.fps
    o.elapsed = 0
    o.currentFrame = 0
    o.playing = false
    o.loop = false
    o.spriteSheet = spriteSheet

    -- 每帧的 UV 尺寸
    o.uWidth = 1.0 / o.columns
    o.uHeight = 1.0 / o.rows

    return o
end

--- 开始播放
---@param loop boolean 是否循环
function SpriteAnimator:Play(loop)
    self.playing = true
    self.loop = loop or false
    self.currentFrame = 0
    self.elapsed = 0
end

--- 停止播放
function SpriteAnimator:Stop()
    self.playing = false
end

--- 每帧更新，返回当前帧的 UV 矩形
---@param dt number 帧间隔（秒）
---@return number u, number v, number w, number h UV 矩形
function SpriteAnimator:Update(dt)
    if self.playing then
        self.elapsed = self.elapsed + dt
        if self.elapsed >= self.frameTime then
            self.elapsed = self.elapsed - self.frameTime
            self.currentFrame = self.currentFrame + 1

            if self.currentFrame >= self.frameCount then
                if self.loop then
                    self.currentFrame = 0
                else
                    self.currentFrame = self.frameCount - 1
                    self.playing = false
                end
            end
        end
    end

    -- 计算当前帧的 UV 坐标
    local col = self.currentFrame % self.columns
    local row = math.floor(self.currentFrame / self.columns)
    local u = col * self.uWidth
    local v = row * self.uHeight

    return u, v, self.uWidth, self.uHeight
end

return SpriteAnimator
```

## 八、在游戏中使用 SpriteAnimator

### NanoVG 渲染方式

```lua
local SpriteAnimator = require("scripts.SpriteAnimator")

local attackAnim  -- 动画器实例
local attackImage -- NanoVG 图片句柄

function Start()
    -- 创建动画器：12 帧，单行，10fps
    attackAnim = SpriteAnimator:new("Textures/spritesheet_attack.png", 12, 12, 10)

    -- 加载精灵图到 NanoVG
    attackImage = nvgCreateImage(vg, "Textures/spritesheet_attack.png", 0)

    SubscribeToEvent("Update", "HandleUpdate")
    SubscribeToEvent("NanoVGRender", "HandleNanoVGRender")
end

function HandleUpdate(eventType, eventData)
    local dt = eventData["TimeStep"]:GetFloat()
    attackAnim:Update(dt)
end

function HandleNanoVGRender(eventType, eventData)
    local w, h = graphics:GetWidth(), graphics:GetHeight()
    nvgBeginFrame(vg, w, h, 1.0)

    -- 获取当前帧 UV
    local u, v, uw, uh = attackAnim:Update(0)  -- dt=0，只取 UV 不推进

    -- 绘制当前帧（128×128 显示在屏幕中央）
    local drawSize = 128
    local dx = (w - drawSize) / 2
    local dy = (h - drawSize) / 2

    -- 用 UV 窗口只显示精灵图的一帧
    local imgW, imgH = nvgImageSize(vg, attackImage)
    local pattern = nvgImagePattern(vg,
        dx - u * imgW, dy - v * imgH,  -- 偏移让目标帧对准绘制区域
        imgW, imgH, 0, attackImage, 1.0)

    nvgBeginPath(vg)
    nvgRect(vg, dx, dy, drawSize, drawSize)
    nvgFillPaint(vg, pattern)
    nvgFill(vg)

    nvgEndFrame(vg)
end
```

### 3D Billboard 方式

```lua
-- 在 3D 场景中播放精灵动画
-- 通过动态修改材质的 UV 偏移实现

function UpdateSpriteMaterial(material, animator)
    local u, v, uw, uh = animator:Update(0)
    -- 设置 UV 变换：偏移 + 缩放
    material:SetShaderParameter("UOffset", Variant(Vector4(uw, 0, u, 0)))
    material:SetShaderParameter("VOffset", Variant(Vector4(0, uh, 0, v)))
end
```

## 九、多方向动画

为同一角色准备多套方向的精灵图，运行时按方向切换：

```lua
-- 每个方向一张精灵图
local animators = {}
local directions = {"S", "SE", "E", "NE", "N", "NW", "W", "SW"}

for _, dir in ipairs(directions) do
    animators[dir] = SpriteAnimator:new(
        "Textures/monster_attack_" .. dir .. ".png",
        12, 12, 10
    )
end

-- 运行时根据朝向选择对应动画器
local currentDir = "S"
local currentAnim = animators[currentDir]
currentAnim:Play(true)
```

**镜像优化**: SE/S/SW 可以镜像复用 NE/N/NW 的精灵图（水平翻转），只需准备 5 个方向就够。

## 十、循环动画处理

AI 生成的动画首尾往往不能完美衔接。解决方案：

| 方法 | 做法 | 适用 |
|------|------|------|
| **首尾帧模式** | `create_video_task` 用 `first_last_frame` 模式，首尾给同一张图 | 最佳方案 |
| **交叉淡入淡出** | 最后 3 帧与前 3 帧做 alpha 混合 | 后期补救 |
| **截断循环段** | 人工观察找到最佳循环点，丢弃多余帧 | 简单粗暴 |

## 十二、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 动画播放速度不对 | fps 参数不匹配 | 调整 SpriteAnimator 的 fps |
| 精灵图显示错位 | columns/rows 计算错误 | 确认拼图时的 `-tile` 参数 |
| 角色动画有位移 | AI 视频中角色移动了 | Prompt 加"原地"+"保持位置不变" |
| 背景去不干净 | fuzz 值太小或背景不纯 | 提高 fuzz 值或改用 rembg |
| 精灵图太大 | 帧数多 + 分辨率高 | 降低帧率或缩小帧尺寸 |
| 循环动画有跳帧 | 首尾帧不一致 | 用 `first_last_frame` 模式重新生成 |
