# 序列帧动画完整指南：从素材整理到皮肤系统

> **这是什么**：序列帧（Sprite Frame）动画的完整实现方案——从素材分类、命名规范、方向扇区计算、帧推进逻辑，到镜像复用决策和皮肤系统设计。适用于 RPG、塔防、动作游戏等所有需要序列帧动画的场景，思路引擎无关。
>
> **原作者**：糖心哥
> **推荐度**：L2（覆盖面完整，从素材到代码到皮肤全链路讲清楚了；代码示例为伪代码/通用 Lua，需要适配到具体引擎的纹理加载 API）
> **适用场景**：做序列帧动画的游戏（等距视角角色、2D RPG、塔防怪物、技能特效等），尤其是刚拿到素材包不知道怎么组织的阶段

## 一、素材类型识别

**拿到素材包第一件事：判断它是哪种结构，后续处理方式完全不同。**

### 类型 A：精灵图集（Sprite Sheet）

所有帧拼在一张大图上，需要按坐标裁切。

```
┌──────────────────────────────┐
│ [帧0] [帧1] [帧2] [帧3] ... │  ← idle 动作
│ [帧0] [帧1] [帧2] [帧3] ... │  ← walk 动作
│ [帧0] [帧1] [帧2] [帧3] ... │  ← attack 动作
└──────────────────────────────┘
```

- **优点**：单文件，加载快
- **缺点**：需要配套描述文件（.json / .xml）记录每帧坐标和尺寸

### 类型 B：平铺帧序列（Flat Frame Sequence）

每帧是单独的图片文件，按编号命名。

```
walk_00.png  walk_01.png  walk_02.png  ...
```

- **优点**：直观，方便单帧替换
- **缺点**：文件数量多，IO 次数多

### 类型 C：带方向的帧序列

在平铺基础上加方向维度，常见于等距视角游戏。

```
d0_walk_00.png  ← 方向0（S）walk 第0帧
d4_walk_00.png  ← 方向4（N）walk 第0帧
d6_walk_00.png  ← 方向6（E）walk 第0帧
```

或更隐式的：所有帧平铺成一个序列，方向按固定间隔排列（如每隔 10 帧换一个方向）。

> **关键**：拿到素材第一件事——确认是哪种类型，搞清方向和动作的组织方式，再动代码。

## 二、素材命名规范

### 2.1 方向定义

等距视角常用 8 方向，俯视角常用 4 方向。**方向编号没有统一标准**，常见两种：

```
方案一：从北顺时针（Unity/Unreal 常见）    方案二：从东逆时针（数学角度约定）

         N(0)                                      N
   NW(7)      NE(1)                          NW       NE
 W(6)            E(2)                      W               E(0°)
   SW(5)      SE(3)                          SW       SE
         S(4)                                      S
```

> ⚠️ **关键**：收到外购素材时，**必须问清楚或自己测试**哪个编号对应哪个方向。不要假设，美术的"南"和你代码里的"南"很可能不是同一个方向。

### 2.2 建议命名格式

```
{角色ID}_{动作}_{方向}_{帧编号}.png

示例：
hero_walk_d0_00.png
hero_walk_d0_01.png
hero_attack_d2_00.png
boss_idle_d6_00.png
```

或用目录区分：

```
assets/characters/
  hero/
    idle/  d0_00.png  d0_01.png  d4_00.png ...
    walk/  d0_00.png  d0_01.png  ...
    attack/ ...
  enemy_elite/
    ...
```

规范命名让代码用循环批量加载，省去大量硬编码。

## 三、动画系统核心设计

### 3.1 最小数据结构：AnimClip

每套动画（一个方向的一个动作）需要记录：

```lua
{
    frames = {"d0_walk_00", "d0_walk_01", "d0_walk_02"},  -- 帧名列表
    fps = 8,          -- 播放速率
    loop = true,      -- 是否循环
    faceRight = true, -- 该方向的帧是否朝右（用于镜像判断）
}
```

### 3.2 注册表（Registry）

把所有角色的所有方向、所有动作预先索引好：

```lua
REGISTRY["hero_walk_d0"] = { frames = {...}, fps = 8, loop = true }
REGISTRY["hero_walk_d4"] = { frames = {...}, fps = 8, loop = true }
-- ...
```

### 3.3 方向扇区计算

拿到移动向量 `(dx, dy)` 后，计算角度，映射到最近的方向扇区：

```lua
-- 屏幕坐标：X 右为正，Y 下为正
local function getDirectionSector(dx, dy)
    if math.abs(dx) < 0.01 and math.abs(dy) < 0.01 then
        return nil  -- 静止，保持当前方向
    end
    local angle = math.atan(dy, dx) * (180 / math.pi)
    if angle < 0 then angle = angle + 360 end
    -- 每个扇区 45°，加 22.5° 使边界对齐方向中心
    local sector = math.floor((angle + 22.5) / 45) % 8
    return sector
end
```

然后用**映射表**把 sector 转成素材里的方向编号：

```lua
-- 按你的素材实际标注填写，不要猜
local SECTOR_TO_DIR = {
    [0] = 6,  -- 向右(E) → 素材 dir6
    [1] = 7,  -- 右下(SE) → 素材 dir7
    [2] = 0,  -- 向下(S) → 素材 dir0
    [3] = 1,  -- 左下(SW) → 素材 dir1
    [4] = 2,  -- 向左(W) → 素材 dir2
    [5] = 3,  -- 左上(NW) → 素材 dir3
    [6] = 4,  -- 向上(N) → 素材 dir4
    [7] = 5,  -- 右上(NE) → 素材 dir5
}
```

> **要点**：映射集中放在一张表里，发现方向对不上只改这里，不用到处找代码。

### 3.4 帧推进逻辑

```lua
function updateAnimation(anim, dt)
    anim.timer = anim.timer + dt
    local frameDuration = 1.0 / anim.clip.fps

    while anim.timer >= frameDuration do
        anim.timer = anim.timer - frameDuration
        anim.frameIndex = anim.frameIndex + 1
        if anim.frameIndex > #anim.clip.frames then
            if anim.clip.loop then
                anim.frameIndex = 1
            else
                anim.frameIndex = #anim.clip.frames
                anim.finished = true
            end
        end
    end
    return anim.clip.frames[anim.frameIndex]
end
```

> ⚠️ **必须用 dt 累积时间**控制帧切换，不要用帧计数（否则播放速率会随帧率波动）。

## 四、镜像复用决策

### 什么时候可以用镜像

- 角色完全左右对称（圆形小球、无武器的简单怪物）
- 素材预算不够做全 8 方向

### 什么时候不该用镜像

- 等距视角角色（阴影、肩宽、武器握法都不对称）
- 角色身上有文字或图案
- 外购素材已有原生全方向帧（没必要用镜像降质量）

### 素材完整性验算

收到素材包后做简单除法：

```
总帧数 ÷ 动作数 ÷ 每个动作平均帧数 = 方向数

例：360 帧 ÷ 3 个动作 ÷ 15 帧/动作 = 8（完整 8 方向，不需要镜像）
```

如果方向数能整除 8，基本就是完整 8 方向素材。

## 五、皮肤系统设计

**皮肤的本质**：同一套动画逻辑，换一套帧图。

### 方案一：注册表替换（推荐，简单直接）

每套皮肤预先注册一套 REGISTRY，切换时替换使用的注册表：

```lua
-- 默认皮肤
SKINS["default"]["hero_walk_d0"] = { frames = {"hero_walk_d0_00", ...} }
-- 黄金皮肤
SKINS["gold"]["hero_walk_d0"]    = { frames = {"hero_gold_walk_d0_00", ...} }

-- 切换皮肤
function setSkin(character, skinName)
    character.registry = SKINS[skinName]
end
```

### 方案二：纹理替换（帧数完全一致时）

适合同一套骨架、只换贴图颜色/材质的皮肤：

```lua
character.texture = loadTexture("skins/" .. skinName .. "/walk_sheet.png")
-- 帧坐标不变，只换纹理引用
```

### 方案三：染色/调色（Shader 方案，进阶优化）

适合纯颜色变体皮肤：

```lua
character.colorMatrix = HUE_ROTATE(120)  -- 红→绿
```

> **建议**：先用方案一，简单直接，出问题好排查。方案三是进阶优化，先把游戏做出来再考虑。

> ⚠️ **切换皮肤时必须重置 `frameIndex` 和 `timer`**，否则会乱帧。

## 六、常见坑速查

| 现象 | 最可能的原因 | 排查方法 |
|------|------------|---------|
| 所有方向都走背面 | 翻转逻辑 bug（falsy 值穿透） | 逐个 print 翻转值，检查分支是否走对 |
| 某几个方向走背面 | 方向映射表填错 | 强制固定方向测试，逐个确认素材对应关系 |
| 方向对但镜像方向别扭 | 用了镜像但素材有原生帧 | 检查素材完整性，补提取缺失方向 |
| 纵向 S/N 对调 | 素材方向标注与代码约定不一致 | 在映射表里交换 S/N 的值 |
| 动画速率忽快忽慢 | 帧推进没用 dt 累积，而是每帧固定推进 | 改用累积时间控制帧切换 |
| 切换皮肤后动画乱帧 | 切换时没有重置帧计数器 | setSkin 时同时 reset frameIndex 和 timer |

---

## 七、总结

做好序列帧动画，核心三件事：

1. **搞清楚素材结构**：方向怎么排，动作怎么分，每个方向几帧
2. **映射表集中管理**：方向扇区 → 素材编号的映射放在一个地方，发现问题只改一处
3. **皮肤和逻辑分离**：动画逻辑不依赖具体图片路径，换皮肤只换数据
