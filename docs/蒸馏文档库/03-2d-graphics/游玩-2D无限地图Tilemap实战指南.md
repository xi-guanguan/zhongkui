# 2D 无限地图 Tilemap 实战指南

- **这是什么**: 用 AI 生图 + Tile 平铺 + 装饰散布，搭建可无限漫游的 2D 游戏地图的完整工作流。覆盖地砖贴图生成、相机跟随渲染、装饰素材生成、基于区块的确定性伪随机散布四个环节
- **原作者**: 游玩
- **推荐度**: L1（拿来就能用——四步工作流清晰，AI 生图提示词模板可直接复制，适合吸血鬼幸存者/弹幕天国类大地图游戏）
- **适用场景**: 需要无限大地图自由移动的 2D 游戏（生存类、弹幕类、开放世界类）

---

## 核心思路

用一小块地砖（Tile）反复平铺覆盖整个世界，就像现实中的地砖。一块 128x128 像素的贴图就能铺出无限大地面。每帧只渲染相机可见区域的十几块 tile，性能与世界大小无关。

---

## 四步工作流

```
第1步：生成地砖贴图（AI 生图）
    ↓
第2步：实现 Tile 平铺渲染（相机跟随 + 仅渲染可见区域）
    ↓
第3步：生成装饰素材（草、石头等）
    ↓
第4步：随机散布装饰物（基于区块的伪随机）
```

---

## 第 1 步：生成地砖贴图

### 关键原则

地砖要无缝平铺，不能有明显图案/边缘/方向感，否则拼接会产生强烈的"壁纸效应"。

### AI 生图提示词要素

| 要素 | 说明 |
|------|------|
| 色调统一 | 整体一个色系，不要太花 |
| 细微变化 | 有轻微噪点/色差，不要纯色块 |
| 无明显图案 | 不要裂缝、碎石、花纹等强特征 |
| 无边缘 | 不能有明显边界线 |
| 平坦 | 俯视角度，无立体感 |

### 提示词模板

```
[色调]地面纹理，细微颗粒噪点，轻微色差变化，
无裂缝无碎石无图案无边缘，简洁平坦，游戏地面贴图
```

**实际案例**（暗黑修仙风格）：
```
深灰紫色地面纹理，细微颗粒噪点，轻微色差变化，
无裂缝无碎石无图案无边缘，简洁平坦，游戏地面贴图
```

**尺寸**：128x128（太小会模糊，太大浪费内存）

### 迭代调整

| 问题 | 解决 |
|------|------|
| 太复杂（裂缝、碎石） | 加"无裂缝无碎石"，去掉"废墟"等词 |
| 太单调（纯色块） | 加"细微颗粒噪点，轻微色差变化" |
| 有方向感（斜纹） | 加"无图案无边缘" |
| 颜色不搭 | 先截图看游戏现有风格，调整色调描述 |

一般 2-3 轮即可找到满意效果。

## 第 2 步：Tile 平铺渲染

### 核心思路

不需要创建巨大地图图片。每帧只需：
1. 算出相机当前能看到哪些 tile（根据相机位置和屏幕大小）
2. 只渲染这些 tile（通常十几块）

无论世界多大，每帧渲染的 tile 数量固定，性能无问题。

### 相机跟随

```
屏幕坐标 = 世界坐标 - 相机坐标
```

玩家移动 → 相机跟着走 → 所有元素（敌人/道具/特效）的世界坐标减去相机坐标 = 屏幕显示位置。

### 接缝消除

先铺一层与 tile 颜色接近的底色，再让每块 tile 渲染时扩大 1-2 像素（接缝补偿）。两者结合完全消除网格线。

## 第 3 步：生成装饰素材

### 素材要求

- **透明背景**：PNG 透明底，叠在地砖上
- **俯视角度**：与地面视角一致
- **尺寸小**：64x64 够用，渲染时可缩放
- **风格统一**：与地砖/游戏整体搭配
- **3-5 种变化**：避免单调

### 提示词模板

**草类**：
```
俯视角度，一小簇[色调]杂草，[风格]风格，
深色调，透明背景，游戏2D装饰素材
```

**石头类**：
```
俯视角度，[数量]块[色调]碎石/石碑残片，
[风格描述]，透明背景，游戏2D地面装饰素材
```

### 实际案例（暗黑修仙风格 4 种）

1. 暗色杂草：`俯视角度，一小簇暗绿色杂草，枯黄边缘，修仙废墟风格，深色调，透明背景，游戏2D装饰素材`
2. 灵气苔藓：`俯视角度，一小片深紫绿色苔藓地衣，微微发光边缘，暗黑仙侠风格，透明背景，游戏2D地面装饰`
3. 碎石块：`俯视角度，两三块深灰色碎石，棱角分明，表面粗糙，暗色调，透明背景，游戏2D地面装饰素材`
4. 残碑断石：`俯视角度，一块破损的暗灰色石碑残片，上面隐约有古老符文，修仙废墟风格，透明背景，游戏2D装饰`

## 第 4 步：随机散布装饰物

### 核心需求

"看起来随机，但每次经过同一位置看到的东西一样" = **确定性伪随机**。

### 区块系统

把世界分成固定大小的区块（如 256x256 像素），每个区块根据坐标算出固定随机种子，用该种子生成装饰物位置和类型。

**优势**：
- **无需存储**：位置是算出来的，不用保存到内存
- **确定性**：同一区块永远生成相同装饰，来回走不会变
- **仅渲染可见**：只计算屏幕范围内的区块

**关键**：不要用 `math.random()`，要用基于坐标的确定性伪随机算法（如 LCG）。

### 每个装饰物的随机属性

- 位置：区块内 X/Y 偏移
- 图片类型：从素材列表随机选
- 透明度：0.4~0.8 随机，增加自然感

### 密度控制

| 密度 | 每区块数量 | 效果 |
|------|-----------|------|
| 稀疏 | 1-2 | 荒野、沙漠感 |
| 适中 | 3-4 | 自然、平衡 |
| 密集 | 5-8 | 森林、草地感 |

## 参数速查表

| 参数 | 推荐值 |
|------|--------|
| Tile 尺寸 | 128x128 px |
| 装饰素材尺寸 | 64x64 px |
| 装饰渲染尺寸 | 32x32 px（可缩放） |
| 区块大小 | 256x256 px |
| 每区块装饰数 | 3（按风格调整） |
| 接缝补偿 | 1px（每边扩大） |

## 常见问题

| 问题 | 解答 |
|------|------|
| Tile 拼接有网格线？ | 铺底色 + 每块扩大 1-2px |
| 装饰物每次位置变了？ | 用基于区块坐标的确定性伪随机，不要用 math.random() |
| 走到很远会卡吗？ | 不会，每帧只渲染可见范围，与世界大小无关 |
| AI 生成 tile 重复感强？ | 去掉所有强特征，只保留色调+轻微噪点 |
| 不同区域不同地面？ | 按区域切换 tile 和装饰素材即可，架构不需改 |

---

## 适用场景总结

无需地图编辑器，无需预制大图，生成的世界可无限大，性能开销极小。适合吸血鬼幸存者、弹幕天国这类需要大地图自由移动的游戏。
