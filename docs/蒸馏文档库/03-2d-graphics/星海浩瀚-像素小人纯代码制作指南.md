# 像素小人纯代码制作指南

## 这是什么

不用图片资源，纯代码在 NanoVG 上绘制像素角色的完整方案。用 8×8 数字矩阵作为"画布"，每个数字映射一种颜色，渲染时逐格画矩形色块。覆盖 4 方向设计、2 帧行走动画、HP 动态变色、Player↔SpriteRenderer 解耦架构。

- **原作者**: 星海浩瀚
- **推荐度**: L2（参考学习）—— 方案简洁优雅，但仅适用于极简像素风格，精度受限于 8×8 矩阵
- **适用场景**: 像素风格游戏原型快速开发，无需美术资源的角色渲染，NanoVG 矢量绘制学习

---

## 核心原理

```
8×8 数字矩阵 → 6 种颜色编码 → NanoVG 逐格绘制矩形色块
瓦片 64×64px, 小人占 70% 高度 ≈ 44.8px, 每色块 ≈ 5.6px
```

### 颜色编码

| 编码 | 部位 | 颜色 | RGBA |
|------|------|------|------|
| 0 | 空（透明） | — | — |
| 1 | 头盔 | 动态（随HP） | 见下方 |
| 2 | 面罩 | 半透明天蓝 | (100,180,255,200) |
| 3 | 身体 | 宇航服蓝 | (60,130,200,240) |
| 4 | 背包 | 金黄 | (180,140,60,220) |
| 5 | 靴子 | 深灰蓝 | (80,80,100,240) |
| 6 | 手臂 | 同身体更透明 | (60,130,200,200) |

### 矩阵示例（朝下站立）

```
0 0 1 1 1 1 0 0    ← 头盔顶部
0 1 2 2 2 1 0 0    ← 头盔+面罩
0 1 2 2 2 1 0 0    ← 头盔+面罩
0 0 1 1 1 0 0 0    ← 脖子
0 6 3 3 3 6 0 0    ← 手臂+身体
0 0 3 4 3 0 0 0    ← 身体+背包
0 0 3 0 3 0 0 0    ← 双腿
0 5 5 0 5 5 0 0    ← 靴子
```

## 4 方向设计

| 方向 | 头盔特征 | 身体特征 |
|------|---------|---------|
| 下 | 面罩可见（编码2） | 背包在后方 |
| 上 | 无面罩（全编码1） | 背包可见 |
| 右 | 面罩在右半 | 背包在左侧 |
| 左 | **= 右的水平镜像** | 不单独定义 |

**朝下 vs 朝上**：仅第 2-3 行差异 — 编码 2 替换为 1（面罩消失）。

**镜像实现**：

```lua
if mirror then
    px = baseX + (8 - col) * cellSize  -- 镜像X
else
    px = baseX + (col - 1) * cellSize  -- 正常X
end
```

## 2 帧行走动画

只改腿部（第 7-8 行），上半身不变：

```
站立帧 (frame 0)          行走帧 (frame 1)
0 0 3 0 3 0 0 0  行7     0 3 0 0 0 3 0 0  ← 腿张开
0 5 5 0 5 5 0 0  行8     5 5 0 0 0 5 5 0  ← 脚外移
```

### 切换逻辑

```lua
self.animTimer = self.animTimer + dt
if self.animTimer >= 0.25 then       -- 每0.25s切换
    self.animTimer = self.animTimer - 0.25
    self.animFrame = 1 - self.animFrame  -- 0↔1
end
-- 停止时重置为 0（站立帧）
```

## HP 动态变色

头盔是唯一动态调色项：

| HP范围 | 颜色 | 视觉 |
|--------|------|------|
| > 50 | (200,220,240) | 亮白银 — 正常 |
| 20~50 | (220,180,100) | 暗黄 — 受伤警告 |
| < 20 | (200,100,100) | 暗红 — 危险 |

玩家通过小人颜色直觉感知生命值，不需要看 HUD。

## 朝向判定

```lua
if math.abs(dx) >= math.abs(dy) then
    facing = dx > 0 and "right" or "left"
else
    facing = dy > 0 and "down" or "up"
end
-- 停止时保持最后朝向
```

优先级：对角移动取绝对值更大轴，相等时优先水平。

## 渲染管线

```
1. 查表: SPRITES[facing][animFrame+1]  (left→查right+mirror)
2. 尺寸: spriteH = TILE_SIZE*0.7, cellSize = spriteH/8
3. 定位: baseX = wx - spriteW/2, baseY = wy - spriteH*0.55
4. 阴影: nvgEllipse 脚底半透明椭圆 (alpha=50)
5. 逐像素: 8×8 双循环, 跳过编码0, 查调色板, nvgRect
6. O2条: <100%时显示, >30%蓝色, ≤30%红色
```

## 架构解耦

```
Player.lua（逻辑）        SpriteRenderer.lua（视觉）
facing   ──────────→      选择精灵矩阵
animFrame ─────────→      选择帧
wx, wy   ──────────→      定位
hp       ──────────→      头盔变色
o2       ──────────→      O2指示条
```

Player 不知道自己长什么样，SpriteRenderer 不知道移动逻辑。后续替换为精灵图时只需改 `_drawPlayer()` 内部实现，接口不变。

---

## 扩展指南

- **新动作帧**：定义新矩阵（如双手举起），Player 新增状态映射到新 animFrame 范围
- **新颜色部位**：选编码 7+，在矩阵中填入，在 `getPalette()` 添加 RGBA
- **替换精灵图**：矩阵逐像素绘制 → `nvgImagePattern`，Player 端字段不变
