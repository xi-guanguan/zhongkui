# 纸娃娃骨骼链换装系统（Paper Doll Bone Chain）

> **这是什么**：一套完整的 2D 纸娃娃换装 + 骨骼动画系统，支持四方向（正/背/左/右）、独立部件换装（Loadout 字典）、骨骼链变换继承、关键帧动画、梯形变形柔体效果。含离线 Python 合成和运行时 Lua/NanoVG 渲染两套等价实现。
>
> **原作者**：嗒啦啦（系统设计与文字说明）、点点/林六（代码实现与技术文档）
> **推荐度**：L2（架构完整、数学模型清晰，但部件定义数据量大，需根据自身美术风格调整骨骼树和素材）
> **适用场景**：2D 角色换装游戏（RPG 装备展示、头像定制、纸娃娃系统）；需要骨骼动画但不想用 Spine/DragonBones 的轻量场景

## 一、系统概览

### 核心能力

| 能力 | 说明 |
|------|------|
| **骨骼链变换** | 部件通过父子关系形成树状骨骼，子部件继承父部件的旋转、位移、缩放 |
| **独立换装** | 眼睛、发型、衣服、鞋子等可独立替换，通过 Loadout 字典控制 |
| **四方向支持** | 正面 / 背面 / 右侧 / 左侧，每个方向有独立的部件定义和素材 |
| **关键帧动画** | 待机、行走等动画，支持 smoothstep 插值 |
| **梯形变形** | 行走时头发飘动、裙摆摆动等柔体效果 |
| **双端一致** | 离线 Python 合成与运行时 Lua 渲染使用相同的数学模型 |

### 两套运行环境

## 四、骨骼链变换

### 4.1 变换公式

子部件的最终变换 = 所有祖先的变换依次累积：

```
M_final = M_root × M_parent × ... × M_self
```

每个节点的单次变换矩阵：

```
M = T(pivot + offset) × R(rotation) × S(scaleX, scaleY) × T(-pivot)
```

其中：
- `T(pivot + offset)` — 平移到旋转中心 + 动画位移
- `R(rotation)` — 旋转
- `S(scaleX, scaleY)` — 缩放
- `T(-pivot)` — 平移回原点

### 4.2 Python 离线实现

```python
def compute_part_transform(part_def, parts_dict, pose):
    M = np.eye(3)                    # 单位矩阵
    
    # 1. 从根到父，依次累乘祖先变换
    chain = get_ancestor_chain(part_def["name"], parts_dict)
    for ancestor in chain:
        a_pose = pose.get(ancestor["name"], {})
        M = M @ make_transform_matrix(
            a_pose.tx, a_pose.ty, a_pose.rotation,
            a_pose.scaleX, a_pose.scaleY,
            ancestor.pivot_x, ancestor.pivot_y
        )
    
    # 2. 最后乘以自身变换
    p_pose = pose.get(part_def["name"], {})
    M = M @ make_transform_matrix(
        p_pose.tx, p_pose.ty, p_pose.rotation,
        p_pose.scaleX, p_pose.scaleY,
        part_def.pivot_x, part_def.pivot_y
    )
    return M
```

### 4.3 Lua/NanoVG 运行时等价实现

Lua 运行时通过 NanoVG 的变换栈实现相同效果：

```lua
-- 对每个祖先部件：
nvgTranslate(vg, ancestor.pivot.x + ancestor.tx, ancestor.pivot.y + ancestor.ty)
nvgRotate(vg, math.rad(ancestor.rotation))
nvgScale(vg, ancestor.scaleX, ancestor.scaleY)
nvgTranslate(vg, -ancestor.pivot.x, -ancestor.pivot.y)

-- 自身变换：
nvgTranslate(vg, part.pivot.x + part.tx, part.pivot.y + part.ty)
nvgRotate(vg, math.rad(part.rotation))
nvgScale(vg, part.scaleX, part.scaleY)
nvgTranslate(vg, -part.pivot.x, -part.pivot.y)
```

**数学等价关系**：NanoVG 变换栈操作 = 矩阵乘法 `T(p+t) × R × S × T(-p)`，两端完全一致。

## 六、换装系统（Loadout）

### 6.1 核心概念

换装通过 **Loadout 字典** 控制，每个键是一个装备槽位（slot），值是素材变体名称：

```python
LOADOUT_塔拉拉 = {
    "eyes":      "塔拉拉",    # 眼睛 → 塔拉拉版
    "blush":     "塔拉拉",    # 腮红 → 塔拉拉版
    "backhair":  "塔拉拉",    # 后发 → 塔拉拉版
    "fronthair": "塔拉拉",    # 前发 → 塔拉拉版
    "coat":      None,        # 外套 → 不穿
    "pants":     None,        # 裤子 → 不穿
    "shoes":     None,        # 鞋子 → 不穿
    "shirt":     "塔拉拉",    # 衬衫 → 塔拉拉版
    "skirt":     "塔拉拉",    # 裙子 → 塔拉拉版
    "socks":     "塔拉拉",    # 袜子 → 塔拉拉版
}
```

- 值为 `None` 表示该槽位不穿戴（隐藏）
- 同一槽位可以有多个变体（不同角色/风格的素材）

### 6.2 素材库结构（PART_ASSETS）

```python
PART_ASSETS = {
    "槽位名": {
        "变体名": {
            "front": { "部件槽": "图片路径", ... },
            "back":  { ... },
            "right": { ... },
            "left":  { ... },
            "bh_peak": 0.06,  # 可选：后发梯形变形峰值
        },
    },
}
```

### 6.3 换装流程

```
1. 读取 Loadout 字典
2. 对每个槽位：
   a. 如果值为 None → 跳过（不渲染该部件）
   b. 否则 → 从 PART_ASSETS[槽位][变体名][方向] 加载素材
3. 与 BASE_BODY 合并 → 调用 build_image_map() 组装图片映射
4. 渲染时按图层排序绘制
```

关键帧之间使用 smoothstep 插值：

```python
def smoothstep(t):
    return t * t * (3 - 2 * t)

# 在两个关键帧之间：
t = (current_time - frame_a.time) / (frame_b.time - frame_a.time)
t = smoothstep(t)
value = frame_a.value + (frame_b.value - frame_a.value) * t
```

## 八、梯形变形（Trapezoid Deformation）

### 8.1 概述

梯形变形是一种**柔体模拟效果**，用于让头发、裙摆、外套下摆等在行走时产生自然的飘动感。与骨骼旋转不同，它作用于**部件内部像素**，改变形状而非整体位置。

### 8.2 全梯形变形（trapezoidScale）

**原理**：对 bbox 区域内的每一行做不同的水平缩放。顶部保持不变，底部缩放最大。

```
scale > 0:      scale < 0:
┌──────┐        ┌──────┐
│      │        │      │
│      │        │      │
└────────┘      └────┘
 底边更宽        底边更窄
```

```python
def apply_trapezoid(img, bbox, scale):
    for row in range(bbox_height):
        ratio = row / (bbox_height - 1)    # 0(顶) ~ 1(底)
        sx = 1.0 + scale * ratio           # 缩放因子
        # 对该行水平缩放，以 bbox 水平中心为锚点
```

### 8.3 半梯形变形（halfTrapezoidScale）

**原理**：在全梯形基础上加入水平方向的权重渐变。将 bbox 分为 5 列，变形量按正弦曲线从不变形侧到变形侧平滑过渡。

```
正值（右侧膨胀）：     负值（左侧膨胀）：
┌──────┐               ┌──────┐
│     /│               │\     │
│    / │               │ \    │
└───/──┘               └──\───┘
左侧不变,右侧鼓出      右侧不变,左侧鼓出
```

**应用场景**：裙摆和外套下摆在行走时的不对称飘动。

### 8.4 梯形变形应用位置

| 部件 | 变形类型 | 场景 |
|------|---------|------|
| backhair（后发） | trapezoidScale | 行走时后发飘动 |
| skirt（裙子） | halfTrapezoidScale | 行走时裙摆不对称摆动 |
| coat（外套） | halfTrapezoidScale | 行走时下摆飘动 |
| fronthair（前发） | trapezoidScale | 轻微飘动 |

## 九、实现要点与注意事项

### 坐标系

- 画布尺寸：512×512（工作区），最终输出缩放至 256×256
- pivot 坐标在 512×512 空间内定义
- 所有位移 (tx, ty) 单位为像素

### 性能考量

- **离线合成**：Python 端可预计算所有帧导出为精灵图，加载时零计算开销
- **运行时渲染**：NanoVG 变换栈操作轻量，适合实时渲染（移动端友好）
- **素材管理**：每个部件每个方向一张 PNG，按需加载

### 扩展方向

1. **新角色**：新增 Loadout 配置 + 对应 PART_ASSETS 素材即可
2. **新装备**：新增槽位变体，不影响已有配置
3. **新动画**：新增关键帧序列，骨骼树不变
4. **新方向**：理论可扩展到 8 方向（需额外部件定义）

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 部件渲染顺序错误 | layer 值设置不当 | 检查图层排序 + 方向覆盖规则 |
| 左方向素材反了 | 忘记 flipH | 左方向复用右侧定义，整体水平翻转 |
| 骨骼变换抖动 | 祖先链计算错误 | 确认 `get_ancestor_chain` 从根到当前节点顺序正确 |
| 梯形变形锯齿 | bbox 区域太小 | 在 512 工作区操作，最后再缩放到 256 |
| 换装后部件缺失 | PART_ASSETS 缺少对应方向的素材 | 确认四个方向都有素材，或设置可见性规则隐藏 |
