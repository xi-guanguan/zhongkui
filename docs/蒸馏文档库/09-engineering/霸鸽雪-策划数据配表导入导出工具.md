# 策划数据配表导入导出工具

> **这是什么**: 将游戏数据抽取为 JSON 配置文件，配合外部 JSON↔Excel 转换工具实现低积分消耗的数据迭代工作流。  
> **原作者**: 霸鸽雪  
> **推荐度**: L3（需额外获取 — 核心的 JSON↔Excel 转换工具在官方 QQ 测试交流群群文件中，帖子未提供下载链接）  
> **适用场景**: 游戏策划需要大量数值配置（课程表、关卡数据、物品属性等）的数据驱动型项目

---

## 核心思路

### 问题

游戏中大量数值（关卡配置、物品属性、任务数据等）硬编码在 Lua 代码中，每次修改都要消耗积分让 AI 改代码。

### 解决方案

将数据从代码中抽离为 JSON 配置文件，游戏逻辑改为读取配置文件驱动：

```
代码中硬编码数据
  ↓ （让 AI 抽取）
JSON 配置文件
  ↓ （外部工具转换）
Excel 表格（策划友好）
  ↓ （修改后转回）
JSON 配置文件
  ↓ （终端指令覆盖进项目）
游戏读取新配置
```

## 优势

| 方面 | 传统方式 | 配表方式 |
|------|---------|---------|
| 修改数据 | 每次找 AI 改代码，消耗积分 | 自己改 Excel，0 积分 |
| 数据可读性 | 散落在代码各处 | 集中在配置文件，表格化 |
| 协作 | 只有会写代码的人能改 | 策划可直接编辑 Excel |
| 版本管理 | 代码 diff 难看 | JSON diff 清晰 |

---

## 适用边界

- **适合**：数据量大的游戏（RPG 物品表、关卡配置、任务链等）
- **不适合**：数据量很小的休闲游戏（直接写在代码里更简单）
- **限制**：JSON↔Excel 工具需要从 QQ 群获取，帖子未提供独立下载
