# 游戏 Bug 排查通用方法论——五类高频诡异 Bug 的定位与修复清单

> **这是什么**：从独立游戏开发实战中提炼的 5 类"看起来灵异，实际有规律"的 Bug 模式及其排查思路，覆盖物理抖动、音频叠层、异步时序、输入去重、渲染前置条件缺失。每个 Bug 都附带根因分析和通用修复策略，可作为排查清单使用。
>
> **原作者**：半醉半醒
> **推荐度**：L2（五个 Bug 均来自真实项目《像素酒馆》的实战经验，每个都是 UrhoX 开发者大概率会踩的坑。部分修复方案与引擎文档已有记录重合，但作为"症状→根因→修复"的排查清单仍有独立价值。）
> **适用场景**：遇到"现象诡异、难以复现、查了半天不知道哪错了"的 Bug 时，先对照此清单排查。

## Bug 模式清单

### #1 物理碰撞体边界抖动

**症状**：角色走到特定瓦片（通常是地图边缘）时疯狂抖动/抽搐。

**根因**：Box2D 碰撞体的 `center` 偏移使用了整数值（如 `Vector2(0, -0.5)`），底部检测射线恰好落在两个瓦片的精确交界线上，导致每帧在"碰到/没碰到"之间反复跳变。

**修复策略**：碰撞体偏移量避开整数边界，留出微小 margin。

```lua
-- 错误：精确对齐瓦片边界
center = Vector2(0, -0.5)

-- 正确：留 0.01~0.02 的 margin
center = Vector2(0, -0.48)
```

**通用规律**：Box2D 的碰撞检测在精确边界上行为不稳定。所有碰撞体的偏移/尺寸都应避免与瓦片网格精确对齐。

### #2 音频叠层播放

**症状**：进入某个场景后，BGM 同时播放了两层（音量翻倍、有相位差）。

**根因**：场景切换时旧的 `SoundSource` 节点未被销毁。UrhoX 的场景切换**不会自动清理所有节点**，特别是挂在 scene 外部的全局节点。新场景又创建了一个新的音频节点，导致两层叠加。

**修复策略**（二选一）：
1. 在 `Stop()` 中手动 `bgmNode:Remove()` 销毁旧音频节点
2. 使用全局单例模式管理音频节点，创建前检查是否已存在

**通用规律**：任何需要跨场景存在的资源（音频、UI、全局状态），都必须有明确的生命周期管理——要么保证唯一性（单例），要么保证清理（Stop/析构）。

### #3 异步 API 时序竞争

**症状**：`clientScore.Get` 回调返回的分数永远是 0。

**根因**：在 `Start()` 中立即调用云变量的 `Get` 操作，但此时云端服务尚未初始化完成，返回了默认值 0。

**修复策略**：
1. 延迟调用：加 1 秒延时后再请求
2. 事件驱动：监听初始化完成事件后再发起请求
3. 超时兜底：设置超时计时器，超时后重试或使用本地缓存值

**通用规律**：所有云端/网络/异步操作都不能在 `Start()` 中直接调用后立即使用结果。必须通过回调、延时或事件驱动来处理时序依赖。

> 另见：BUG-72c 超时兜底方案（项目级避雷清单）。

### #4 移动端输入事件重复触发

**症状**：玩家在触屏设备上轻点一下按钮，回调被触发了 2~3 次。

**根因**：触屏同时产生了 `TouchBegin` 事件和模拟的 `MouseButtonDown` 事件，导致一次物理操作触发多次逻辑响应。

**修复策略**：使用 `InputManager` 统一处理输入，它内部已实现触控/鼠标事件的去重。

```lua
local InputManager = require "urhox-libs.Platform.InputManager"
-- InputManager 会自动处理触控与鼠标事件的去重
```

**通用规律**：移动端开发**必须**使用 `InputManager`，不要直接监听原始输入事件。原始事件在移动端会产生重复，在不同平台上行为也不一致。

### #5 NanoVG 渲染前置条件缺失

**症状**：`nvgText()` 调用了，代码没报错，但屏幕上什么都不显示。

**根因**：忘记在 `Start()` 中调用 `nvgCreateFont()`。NanoVG **没有默认字体**，不创建字体就调用文本绘制 API 会静默失败。

**修复策略**：NanoVG 文字渲染必须完成"三步曲"，缺任何一步都不显示：

| 步骤 | 调用时机 | API |
|------|---------|-----|
| 1. 创建字体 | `Start()` 中，只调一次 | `nvgCreateFont(vg, "sans", "Fonts/MiSans-Regular.ttf")` |
| 2. 设置字体 | 每帧渲染时 | `nvgFontFace(vg, "sans")` + `nvgFontSize(vg, 24)` |
| 3. 绘制文本 | 每帧渲染时 | `nvgText(vg, x, y, "文字")` |

**通用规律**：NanoVG 所有渲染操作都有隐含的前置条件（字体、画笔颜色等），静默失败不会报错。遇到"代码执行了但什么都不显示"的情况，优先检查前置条件是否满足。

## 排查思维框架

从以上 5 个 Bug 中提炼的通用排查思路：

| 症状模式 | 优先排查方向 |
|---------|-------------|
| 物理抖动/跳变 | 碰撞体边界是否精确对齐了网格 |
| 资源叠加/重复 | 场景切换时的生命周期管理 |
| 异步结果为空/默认值 | 是否在初始化完成前就使用了结果 |
| 事件触发次数异常 | 是否有多个事件源映射到同一逻辑（尤其移动端） |
| 渲染调用无效果 | 渲染 API 的前置条件是否全部满足 |
