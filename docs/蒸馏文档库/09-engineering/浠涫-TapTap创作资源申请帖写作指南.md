# 【创作资源申请】帖撰写指南

> 基于 9 个真实案例 + 2026-04-28 新政策蒸馏。目标受众：编辑（天一），不是玩家。

---

## §0 与【作品自荐】的区别

| 维度 | 作品自荐 | 创作资源申请 |
|------|---------|-------------|
| 目的 | 首页流量分发 | 积分资源升级/延续 |
| 受众 | 玩家 + 编辑 + 开发者 | **主要是编辑（天一）** |
| 语言 | 感性、场景化、让人想玩 | **理性、证明投入和规划** |
| 重点 | 突破性 + 体验代入 + 灵魂 | **完成度 + 技术力 + 持续迭代 + 资源需求合理性** |
| 长度 | 800-2500 字 | 可以更长，需要详细展示 |
| 必须有 | 游戏链接、截图 | 游戏链接、开发进度、后续计划、资源用途 |

**核心差异**：自荐帖要让人想玩，资源申请帖要让编辑相信你值得投资。

## §1 积分三档速查

| 档位 | 积分额度 | 周期 | 对象 | 申请难度 |
|------|---------|------|------|---------|
| 计划一 | 30,000 | 一次性 | 新用户 | 低（默认） |
| 计划二 | 120,000 | 28天 | 4.28前已激活开发者 | 中 |
| 计划三 | 600,000 | 28天 | 历史入选编辑推荐的开发者 | 高 |

**官方通过率数据（2026-04-28 首批）**：

| 结果 | 比例 | 说明 |
|------|------|------|
| 计划三（60万积分） | **51%** | 符合进一步支持条件 |
| 计划二（12万积分） | **41.5%** | 暂保持/5.6后变更，后续观察完成度/质量/更新 |
| 无法处理 | 少量 | 没附游戏链接 |

**解读**：超过半数通过了计划三，说明门槛不是高不可攀。被卡在计划二的 41.5% 主要问题是"粗糙"和"同质化"——这两个是可以通过帖子策略和游戏改进来解决的。

**续期规则**：计划二/三进入最后一周时，需通过【创作资源申请】延续，否则自动降级。

**档位匹配建议**：
- 新游戏、内容量一般 -> 先申请计划二，用成绩证明后升计划三
- 内容量大、技术力强、品类独特 -> 可直接申请计划三
- 不要在帖子里写"申请计划三"但论据只够计划二 -> 编辑会觉得不匹配

## §2 硬门槛自检（四道门）

在写帖子之前，先过这四道门。**任何一道不过，帖子写得再好也会被拒或降级**。

### 门 1：游戏链接

**必须有线上可体验的游戏链接**（TapTap 链接、测试链接、分享链接均可）。

- 9 个案例中，4 个因"无链接"被直接拒绝；官方确认仍有少量帖子因无链接无法处理
- 编辑原话："游戏未带有线上游戏链接、测试链接或分享链接，不具备创作资源申请的条件"
- **没有链接 = 编辑无法体验 = 直接出局，无论帖子多好**

**如果游戏还没上线**：
- 至少上传一个可体验的测试版本
- 或提供分享链接让编辑能打开玩

### 门 2：同质化风险评估

**编辑会独立判断同质化**，即使有链接也可能因同质化被拒。

**官方原话**："修仙挂机是重灾区，同质化非常严重，建议想做这个品类的开发者，一定要做出让人一眼看到差异的内容。"

**高危品类（红海）**：
- 修仙文字 RPG（5 个相关案例，全部失败或受限；官方点名重灾区）
- 传奇类
- 武侠文字养成
- 挂机放置类（无差异化时）

**自检问题**：
1. 在 TapTap 搜索你的品类关键词，同类游戏超过 20 款？-> 高危
2. 你的游戏用一句话能和同类区分开吗？-> 不能则需要强化差异化论证
3. 去掉美术和名字，玩法和同类游戏有本质区别吗？

**红海品类不是必死**：案例 5（末日危城）也是"文字 RPG"，但通过"合成策略 + 词缀 Build + AI 拍卖行"论证了差异化，拿到计划三。关键在于 §4 的差异化论证。

### 门 3：制作品质（粗糙度）

**官方原话**："没有比较优秀的 UI 也没有强烈风格化的表达，建议多参考比较成熟游戏的美术风格。"

**"粗糙"是独立于同质化的拒绝因素**。即使品类不同质，如果游戏看起来粗糙，也会被卡在计划二。

**粗糙的表现**：
- UI 布局杂乱、控件大小不一、无统一色系
- 美术素材风格混搭（像素 + 写实 + 卡通混用）
- 文字排版随意（字号混乱、颜色刺眼、无层次）
- 无加载画面、无过渡动画、界面切换突兀

**如何应对**：
- **有美术能力**：在帖子截图中展示最佳画面，首图选最有冲击力的
- **无美术能力但有风格**：强调"风格化表达"（像素风、手绘风、极简风都行，关键是统一）
- **确实粗糙**：坦诚承认 + P0 美术升级计划（"当前为程序员美术，已参考 XX 游戏风格，美术重制列入 P0"）

**帖子策略**：
- 截图选最好看的系统/画面（不是随便截）
- 有 GIF/视频更好——动态效果能弥补静态截图的粗糙感
- 如果 UI 确实不好看，文字中提前承认并给出改进计划，比让编辑自己发现要好

### 门 4：档位匹配

不要过高申请。如果你的投入证据和游戏完成度只够计划二，就申请计划二。

## §3 状态判断与帖子结构

### 两种游戏状态

| 状态 | 特征 | 帖子侧重 |
|------|------|---------|
| **开发中/未发布** | 有原型或测试版，尚未正式上线 | 展示潜力、创意独特性、开发规划 |
| **已发布/维护中** | 已上线运营，有用户数据 | 展示成绩、持续迭代、数据增长 |

### 六段式结构模板

以下是通用结构，括号内标注两种状态的侧重差异。

---

**【段 1】基本信息**（100-200 字）

必须包含：
- 游戏名称 + 品类定位
- 一句话核心卖点（"这是一个___的___游戏，核心特色是___"）
- 当前开发状态（开发中/已上线 vX.X）
- 游戏链接
- 开发者身份（个人独立 / 小团队）

> **开发中**：强调创意来源和设计愿景
> **已发布**：补充上线时间、当前版本号、用户量级

---

**【段 2】已完成内容 / 当前进展**（300-800 字）

这是编辑评估"你做了多少"的核心段。

**写法要求**：
- 按系统/模块分类列出，不是流水账
- 每个系统带具体数字（"77 种合成配方"而非"丰富的合成系统"）
- 突出技术难点（"客户端/服务端分离"、"1800 人同服"）

**量化维度**（尽量覆盖）：
- 代码量（行数）
- 功能模块数
- 内容量（关卡数、物品数、配方数等）
- 版本迭代次数
- 开发文档/策划文档数量
- 开发周期

> **开发中**：重点展示已完成的核心循环和原型
> **已发布**：可补充用户数据（DAU、留存、评分）

**截图插入**：此段插入 3-5 张核心玩法截图，展示主要系统的实际画面。
**截图策略**：选最好看的画面，能体现 UI 品质和视觉风格（应对"粗糙"门槛）。

---

**【段 3】创作想法 / 差异化论证**（200-500 字）

**这是扬长避短的核心段**，详见 §5。

> **开发中**：侧重设计理念、品类交叉创新
> **已发布**：侧重"和同类游戏的实际差异"、玩家反馈验证

**GIF/视频插入**：此段插入 1-2 个核心机制的动态演示（GIF 或短视频），直接证明差异化。

---

**【段 4】后续计划**（200-400 字）

**写法要求**：
- 分短期（1-2 周）/ 中期（1-3 月）/ 长期
- 标注优先级（P0 / P1 / P2）
- 坦诚列出已知问题和改进计划

> **开发中**：重点是"什么时候能上线"、"上线前还缺什么"
> **已发布**：重点是"下一个大版本做什么"、"用户反馈如何回应"

**可选**：此段可插入开发路线图/里程碑图表。

---

**【段 5】资源需求说明**（100-200 字）

说清楚：
- 申请哪个档位的积分
- 积分主要用在哪里（构建/托管/算力？）
- 为什么当前积分不够用

> **不要写**："积分不够用所以来申请"（态度问题，案例 8 的教训）
> **要写**："当前游戏包体 XXM，日均构建 X 次，月消耗积分约 XXX，已超出计划一额度"

---

**【段 6】总结**（50-100 字）

简短收尾，重申核心价值和对平台的承诺。

---

## §4 平台风向感知——写帖前必做

### 为什么要先看风向？

官方已明确两大拒因：**粗糙**和**同质化**。这两个判断都是**相对于平台当前生态**的——你的游戏是否同质化，取决于平台上现在有什么；你的游戏是否粗糙，取决于同品类的平均水准。

**所以，写帖子之前必须先了解 TapTap 制造当前的生态风向**。

### AI 执行步骤（撰写前自动执行）

**第 1 步：扫描同品类竞品**

提醒用户：
> "在写申请帖之前，建议先去 TapTap 制造的排行榜/最新发布页面，看一下你的品类目前有多少同类游戏。
> 请告诉我：
> 1. 你搜索你的品类关键词（如"修仙""塔防""跑酷"），排行榜前 20 里有几款类似的？
> 2. 这些同类游戏的美术/UI 水准大概是什么档次？
> 3. 你觉得你的游戏和它们最大的不同是什么？"

**第 2 步：风向判断**

根据用户反馈判断：

| 同品类数量 | 同品类质量 | 风向判断 | 帖子策略 |
|-----------|----------|---------|---------|
| < 5 款 | 任意 | 蓝海，品类稀缺 | 简述品类定位即可，差异化天然成立 |
| 5-15 款 | 一般 | 黄海，有竞争 | 需要 1-2 个差异化锚点 |
| 5-15 款 | 较高 | 黄海偏红 | 需要差异化 + 品质超越证明 |
| > 15 款 | 任意 | 红海 | 必须有强差异化论证（§5），否则建议不申请或降档 |

**第 3 步：品质水位对比**

如果同品类有美术/UI 做得好的标杆游戏：
- **你的品质 >= 标杆**：截图/视频直接展示，自信比较
- **你的品质 < 标杆但有风格**：强调"风格化表达"，用统一的视觉语言弥补精度不足
- **你的品质 < 标杆且无风格**：坦诚 + 美术升级P0计划，帖子中着重展示玩法/技术力而非画面

**第 4 步：写入帖子**

在段 3（差异化论证）中，可以主动提及平台现状：

> "目前 TapTap 制造上的 [品类] 游戏以 [共性特征] 为主，本作与之不同的是 [差异化锚点]..."

这种写法告诉编辑：你做过功课，你知道自己在竞争什么，你的差异是有意为之而非无知。

## §5 差异化论证——扬长避短的核心

### 为什么差异化论证如此重要？

从 9 个案例 + 官方通报看：
- 5 个失败案例中，4 个被提到"同质化"
- 案例 9（凡人修仙路）有链接但仍因同质化被拒
- 案例 5（末日危城）是"文字 RPG"但通过差异化论证拿到计划三
- 官方通报："同质化"是无法获得计划三的两大原因之一，修仙挂机被点名重灾区

**结论**：同质化品类要过审，差异化论证是**必须的**，不是可选的。

### 三步差异化论证法

**第一步：定位你的品类风险等级**

| 风险等级 | 品类举例 | 论证强度要求 |
|---------|---------|-------------|
| 低风险 | Roguelike 酒馆经营、体素世界、音游 | 简要说明即可 |
| 中风险 | 塔防、跑酷、模拟经营 | 需要明确差异点 |
| 高风险 | 修仙/传奇/武侠文字 RPG、挂机放置 | 必须有强论证 |

**第二步：找到你的差异化锚点**

差异化不是说"我的画面更好"或"我的数值更多"。有效的差异化锚点：

| 锚点类型 | 示例（成功案例） | 说服力 |
|---------|----------------|--------|
| **品类交叉** | Roguelike x 酒馆经营 x 迷宫探索（迷宫酒馆） | 极强 |
| **核心机制创新** | 合成系统 + 词缀 Build + AI 拍卖行（末日危城） | 强 |
| **技术突破** | 1800 人同服、客户端/服务端分离（无尽战争 2） | 中强 |
| **场景/叙事创新** | "开了家酒馆，开在迷宫门口"（迷宫酒馆） | 中强 |
| **强烈风格化表达** | 统一的像素风/手绘风/赛博风，一眼可辨 | 中（应对粗糙门槛） |
| **数值/美术风格** | "画面更好"、"数值更平衡" | 弱（基本无效） |

**第三步：写差异化段落**

结构：
```
1. 点明平台现状："目前 TapTap 上的 [品类] 游戏以 [共性] 为主"
2. 提出差异："本作的核心差异在于 [锚点1] + [锚点2]"
3. 展开论证：每个锚点用 2-3 句话解释为什么不同
4. 举证支撑：截图/GIF/视频 直接展示差异点
```

### 弱点转化技巧

**坦诚 > 掩饰**。编辑会实际玩你的游戏、看你的代码。

| 弱点 | 错误做法 | 正确做法（案例来源） |
|------|---------|-------------------|
| 新手引导差 | 不提 | "新手引导完成度 0%，已列入 P0 计划，预计 2 周内完成"（末日危城） |
| 功能还没做完 | "功能丰富" | "核心循环已跑通，XX 系统在开发中，计划 X 月上线" |
| 美术粗糙 | 放精修截图 | "当前为程序员美术，已参考 XX 风格，美术重制列入 P0" |
| UI 不好看 | 不提 | "UI 正在参考 [标杆游戏] 重制，下版本上线" |
| 用户量少 | 不提 | "上线 X 天，自然量 XXX，留存 XX%，说明核心玩法有吸引力" |

**关键案例对比**：
- 末日危城：坦诚"新手引导 0%"+ P0 计划 -> 计划三
- 凡人修仙路：48 个版本但不提任何问题 -> 拒绝
- 无尽战争 2：编辑主动指出新手体验差 -> 维持计划二（不是拒绝）

## §6 量化投入证据

### 编辑在看什么

从案例中逆推，编辑关注的投入证据维度：

| 维度 | 弱（减分） | 中 | 强（加分） |
|------|----------|---|----------|
| 代码量 | 不提 | "数千行代码" | "34,000 行 Lua，17 个独立模块"（末日危城） |
| 迭代频率 | 一次性提交 | "定期更新" | "日更节奏，已迭代 48 个版本" |
| 策划文档 | 无 | "有设计文档" | "200+ 份策划文档"（赛博修仙） |
| 开发周期 | 不提 | "开发了几个月" | "2026 年 3 月下旬启动，7 个里程碑"（赛博修仙） |
| 技术架构 | 不提 | "使用 XX 引擎" | "C/S 分离，1800 人同服"（无尽战争 2） |
| 内容量 | "有很多关卡" | "50 个关卡" | "77 种配方、52 种材料、70 层深渊"（末日危城） |

### 包装技巧

- **数字优于形容词**："34,000 行代码" > "大量代码"
- **分类优于堆砌**：按"核心循环 -> 促活留存 -> 运营基础设施"分层展示（赛博修仙的做法）
- **里程碑优于流水账**：用图表展示开发历程的关键节点
- **有代码量的就写代码量**：编辑看源码（无尽战争 2 的回复证明了这点）

## §7 媒体编排指南

### 媒体类型与用途

| 类型 | 用途 | 插入位置 | 说服力 |
|------|------|---------|--------|
| **截图** | 展示画面、UI、系统 | 段 2（已完成内容） | 中强 |
| **GIF** | 展示动态玩法、核心机制 | 段 3（差异化论证） | 强 |
| **短视频** | 展示完整游戏体验 | 段 3 或独立段 | 极强 |
| **开发图表** | 展示里程碑、路线图 | 段 4（后续计划） | 中强 |
| **对比图** | "我的 vs 同类" 的差异 | 段 3 | 强 |

### 数量建议

| 状态 | 截图 | GIF/视频 | 图表 |
|------|------|---------|------|
| 开发中 | 3-5 张 | 1-2 个（核心机制） | 0-1 个 |
| 已发布 | 5-8 张 | 1-3 个（亮点玩法） | 1 个（开发路线图） |

### 媒体编排原则

1. **首图要有冲击力**：第一张截图决定编辑是否继续看下去。选 UI 最好看、画面最精致的那张（应对"粗糙"门槛）
2. **GIF > 截图**：动态展示比静态截图更有说服力（迷宫酒馆的 3D 实机视频是杀手锏）
3. **每段至少一张图**：纯文字段落容易被跳过
4. **图片有说明文字**：每张图下方标注"这是什么系统/功能"
5. **不要放无关图片**：概念图、网上找的参考图不要放，只放实机截图
6. **截图体现统一风格**：即使美术不顶尖，只要风格统一就不会被判"粗糙"

## §8 编辑决策树（从 9 个案例 + 官方通报逆推）

理解编辑的评估逻辑，反向优化你的帖子。

```
收到帖子
  |
(1) 有没有游戏链接？
  |-- 无 -> 直接搁置，无法处理（案例 1/6/7/8）
  +-- 有
       |
(2) 品类同质化程度？
  |-- 极高（修仙/传奇/武侠文字 RPG）
  |    +-- 有没有强差异化论证？
  |         |-- 无 -> 拒（案例 9·凡人修仙路）
  |         +-- 有 -> 继续评估（案例 5·末日危城 -> 计划三）
  |-- 中等 -> 继续评估
  +-- 低/独特 -> 加分（案例 4·迷宫酒馆 -> 计划三）
       |
(2.5) 制作品质？
  |-- 粗糙（UI差 + 无风格化）-> 计划二，建议提升美术
  +-- 有品质或有风格 -> 继续
       |
(3) 编辑实际体验游戏
  |-- 核心循环跑通、可玩 -> 继续
  +-- 新手体验差 -> 维持/降级（案例 2·无尽战争 2 -> 维持计划二）
       |
(4) 开发投入证据
  |-- 强（10 万行 + 日更 + 里程碑）-> 计划三（案例 3·赛博修仙）
  |-- 中（34K 行 + 量化数据 + 差异化）-> 计划三（案例 5·末日危城）
  +-- 弱（无数据/只列功能）-> 降级或拒
       |
(5) 后续计划可执行性
  |-- 有优先级 + 具体方案 -> 加分
  +-- 空洞承诺 -> 减分
```

**官方确认的两大计划二/拒绝原因**：
1. **粗糙**——UI 差 + 无风格化表达
2. **同质化**——修仙挂机重灾区

## §10 AI 工作流

### 标准流程

```
用户说"帮我写创作资源申请帖"
  |
1. 收集信息
   - 游戏名称、品类、当前状态（开发中/已发布）
   - 游戏链接（没有则提醒必须先准备）
   - 核心玩法和特色
   - 开发投入数据（代码量、版本数、开发周期等）
   - 已知问题和后续计划
   |
2. 平台风向感知（§4 — 必做）
   - 提醒用户去 TapTap 制造排行榜/搜索同品类
   - 确认：同品类数量、质量水位、差异点
   - 判断风向：蓝海/黄海/红海
   |
3. 风险评估（§2 四道门）
   - 检查链接 -> 无则警告
   - 评估品类同质化风险 -> 高危则启动差异化论证
   - 评估制作品质 -> 粗糙则启动弱点转化
   - 档位匹配检查
   |
4. 确定结构（§3）
   - 判断游戏状态 -> 选择侧重方向
   - 选择六段式结构
   |
5. 撰写帖子
   - 按六段式逐段撰写
   - 差异化论证（§5）重点打磨
   - 量化投入（§6）数字化包装
   - 媒体编排（§7）标注插图位置
   |
6. 自检清单
```

### 撰写前自检清单

- [ ] 游戏链接已准备好？
- [ ] 已扫描 TapTap 同品类竞品数量和质量水位？
- [ ] 品类同质化风险已评估？高危品类有差异化论证？
- [ ] 制作品质通过门槛？粗糙则有改进计划？
- [ ] 申请档位与投入证据匹配？
- [ ] 有足够的量化数据（代码量/版本数/内容量）？
- [ ] 已知问题坦诚列出并配有改进计划？
- [ ] 截图/GIF/视频已准备？首图是最佳画面？

### 撰写后自检清单

- [ ] 帖子包含游戏链接？
- [ ] 每个系统/模块有具体数字而非形容词？
- [ ] 差异化论证段（段 3）有实质内容而非空话？
- [ ] 差异化论证提到了平台现状（"目前 TapTap 上的同类游戏..."）？
- [ ] 后续计划有优先级（P0/P1/P2）而非空洞承诺？
- [ ] 资源需求说明具体（积分用在哪、为什么不够）而非"积分不够用"？
- [ ] 媒体编排合理（截图在段 2、GIF/视频在段 3、图表在段 4）？
- [ ] 首图展示了最佳画面和 UI 品质？
- [ ] 弱点已坦诚列出并配有改进计划？
- [ ] 语言理性克制，没有营销话术？
