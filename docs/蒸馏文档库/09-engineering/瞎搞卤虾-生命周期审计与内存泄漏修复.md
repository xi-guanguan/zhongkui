# 生命周期审计与内存泄漏修复

> **作者**: 瞎搞卤虾 | **级别**: L3-注意 | **标签**: `生命周期` `内存泄漏` `事件订阅` `模块清理`

---

## 审计方法

对项目所有 Lua 文件逐一检查：
1. 有无 Init/Start 但缺 Destroy/Cleanup 配对
2. SubscribeToEvent 是否有对应 Unsubscribe
3. 音频/定时器/网络连接是否在切屏时释放

---

## 常见问题分类

| 严重度 | 问题 | 后果 |
|--------|------|------|
| 🔴 高 | 全局 SubscribeToEvent 从未 Unsubscribe | 切屏后旧逻辑仍执行，逻辑冲突 |
| 🔴 高 | 模块无 Destroy/Cleanup 函数 | 累积对象无法 GC |
| 🔴 高 | AudioManager 重复 Init 不检查 | 音频句柄泄漏 |
| 🟡 中 | 定时器回调引用已销毁对象 | 偶发 nil 报错 |
| 🟡 中 | UI 元素创建后未在切屏时移除 | 层叠残留 |

## 修复模式

### 1. SwitchScreen 统一切换

```lua
function SwitchScreen(newScreen)
    if currentScreen and currentScreen.Teardown then
        currentScreen.Teardown()  -- 清理旧屏幕
    end
    currentScreen = newScreen
    currentScreen.Init()
end
```

### 2. Module.Teardown 模板

```lua
local M = {}

function M.Init()
    M.sub1 = SubscribeToEvent("Update", M.HandleUpdate)
    M.node = scene:CreateChild("MyNode")
end

function M.Teardown()
    if M.sub1 then UnsubscribeFromEvent("Update") end
    if M.node then M.node:Remove() end
    M.sub1 = nil
    M.node = nil
end

return M
```

### 3. AudioManager 防重入

```lua
function AudioManager.Init()
    if AudioManager.initialized then return end
    -- 初始化逻辑...
    AudioManager.initialized = true
end

function AudioManager.Shutdown()
    -- 释放所有音频资源
    AudioManager.initialized = false
end
```

## 检查清单

- [ ] 每个模块都有 Init/Teardown 配对
- [ ] SubscribeToEvent 在 Teardown 中 Unsubscribe
- [ ] 单例模块有防重入保护
- [ ] 切屏时调用旧屏幕的 Teardown
- [ ] 定时器回调前检查目标对象是否存活
