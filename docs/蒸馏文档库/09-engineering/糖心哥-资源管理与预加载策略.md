# 资源管理与预加载策略（DWP 体系）

> **这是什么**：UrhoX 引擎的资源分类、构建引用策略、预下载策略、手动下载 API 的完整指南。覆盖从"哪些资源进包"到"什么时候下载"的全流程。
>
> **原作者**：糖心哥
> **推荐度**：L3（内容详实但 DWP 机制是引擎内部行为，部分 API 细节未经充分实战验证，需谨慎对照官方文档使用）
> **适用场景**：游戏包体优化、启动速度优化、多关卡预下载、Loading 界面实现

---

## 第一层：资源的三种分类

引擎按加载行为将资源分为三类：

| 类型 | 文件格式 | 加载行为 | 类比 |
|------|---------|---------|------|
| **阻塞资源** | `.lua` `.json` `.xml` `.material` `.prefab` `.effect` `.fsm` `.blendspace` | 启动前全部就绪，直接可用 | 网页的 CSS/JS |
| **DWP 媒体资源** | `.png` `.jpg` `.mdl` `.ani` `.ogg` `.mp3` `.wav` `.ttf` `.otf` | 先占位（默认贴图/T-pose/静音），后台下载后自动替换 | 网页的图片懒加载 |
| **其他资源** | `.bin` `.dat` `.csv` 等 | 无占位机制，必须手动下载后再读取 | 需要 AJAX 请求的数据 |

> **注意**：角色一开始 T-pose 过一会儿才播放正确动画 — 这不是 BUG，是 DWP 在工作。

## 第二层：构建引用策略 — 决定哪些资源进包

配置文件：`.project/resources.json` 的 `groups` 字段。

| 策略 | groups 值 | 原理 | 适用 |
|------|----------|------|------|
| **全量引用** | `**` | 所有本地资源全入包 | 资源量小（几十 MB）的小游戏 |
| **增强引用** | `scripts/**` | 从脚本出发追踪引用链，可达的才打包 | 资源量大的项目 |

### 增强引用的路径拼接陷阱

增强引用靠扫描代码中的**字符串字面量**找资源路径。路径拼接会导致追踪失败：

```lua
-- ❌ 引擎扫不到完整路径
local path = "Textures/" .. characterName .. ".png"

-- ✅ 用完整字面量映射表
local textures = {
    warrior = "Textures/warrior.png",
    mage    = "Textures/mage.png",
    archer  = "Textures/archer.png",
}
local path = textures[characterName]
```

## 第三层：预下载策略 — 决定启动时下载什么

配置文件：`.project/resources.json` 的 `preload_groups` 字段。

| 策略 | preload_groups 值 | 效果 | 适用 |
|------|-------------------|------|------|
| **全预下载** | `["default"]` | 启动时下载所有入包资源，进游戏即就绪 | 资源少、要求画质 |
| **全 DWP** | `[]`（空数组） | 启动不预下载，全部边玩边下 | 资源多、要求速度（**默认**） |

### 四种组合速查

| 组合 | groups | preload_groups | 特点 |
|------|--------|---------------|------|
| 全量 + 全预下载 | `**` | `["default"]` | 最简单，包大启动慢 |
| 全量 + 全 DWP | `**` | `[]` | 新版默认配置 |
| 增强 + 全预下载 | `scripts/**` | `["default"]` | 包小，启动稍慢 |
| 增强 + 全 DWP | `scripts/**` | `[]` | 包最小启动最快，**推荐大项目** |

## 第四层：手动下载 API

### 场景一：Loading 遮罩消除占位跳变

用 `ObserveDownloads` 监听所有后台下载进度，不需要知道具体资源列表：

```lua
cache:ObserveDownloads({
    onProgress = function(downloaded, total)
        -- 更新进度条
    end,
    onComplete = function()
        -- 隐藏 Loading，开始游戏
    end,
})
```

### 场景二：等待特定资源

```lua
cache:GetResourceAsync("Texture2D", "Textures/hero.png", function(resource)
    if resource then
        sprite:SetTexture(resource)
    end
end)
```

### 场景三：关卡预下载

```lua
-- 获取 prefab 的所有依赖资源（递归）
local refs = cache:GetResRefs("Level2/Scene.prefab", true)
-- 批量下载
cache:DownloadResources(refs, function(success)
    if success then
        -- 切换关卡
    end
end)
```

### 场景四：手动下载非 DWP 资源

```lua
cache:DownloadResource("Data/level_data.bin", function(success)
    if success then
        local res = cache:GetResource("File", "Data/level_data.bin")
    end
end)
```

## 最大的坑：负缓存

对还没下载的非 DWP 资源调用 `GetResource`，引擎会缓存"不存在"（负缓存）。**之后即使下载完了，再调 `GetResource` 还是返回 nil。**

**铁律**：先下载 → 确认成功 → 再 `GetResource`。

---

## 资源路径规范

`scripts/` 和 `assets/` 都是资源根目录，引用时**直接从下一级开始**：

```lua
-- ✅ 正确
cache:GetResource("Texture2D", "Textures/player.png")

-- ❌ 错误：多了目录前缀
cache:GetResource("Texture2D", "assets/Textures/player.png")
```

辅助 API：
- `cache:Exists("Textures/hero.png")` — 检查本地是否就绪（不触发下载）
- `cache:GetResInfo("Textures/hero.png")` — 返回 uuid/大小/来源，nil 表示不在包里

## 常见坑速查

| 坑 | 原因 | 解决 |
|----|------|------|
| `Could not find resource` | 增强引用 + 路径拼接，追踪不到 | 用完整字面量或把目录加到 groups |
| `GetResource` 返回 nil | 路径错误 / 非 DWP 资源未下载 / 负缓存 | 检查路径、先下载再读取 |
| 刚进游戏贴图灰色 | DWP 正常行为 | 等几秒自动替换，或加 Loading 遮罩 |
| 启动太慢 | 全预下载 + 资源多 | 改全 DWP，清空 preload_groups |
| 包体太大 | 全量引用 | 改增强引用 `scripts/**` |
| 下载回调嵌套深 | 多个资源逐个下载 | 用 `DownloadResources` 批量下载 |
