## 第一章：理解推荐机制——编辑在找什么

### 1.1 Token 奖励机制运作方式

TapTap制造每周发布「Token奖励作品」周报，选出的游戏获得：

| 层级 | 标签 | 奖励 | 含义 |
|------|------|------|------|
| **第一梯队** | 「边界突破」 | 2万Token + 额外分发权重 | 在某个维度实现了平台级突破 |
| **第二梯队** | 「亮点游戏」 | 2万Token | 有明显亮点，完成度好 |
| **连续入选** | 叠加奖励 | 额外加码 | 持续产出高质量更新 |

**关键认知**：编辑推荐不只是"好游戏"的表彰——它是**平台能力的证明**。编辑在找的是"能证明TapTap制造这个平台能做出什么"的案例。

### 1.2 编辑评选的5个真实维度

从7期周报中提炼的实际评选标准（按权重排序）：

| 维度 | 编辑怎么看 | 周报原文举例 |
|------|-----------|-------------|
| **① 突破性** | 是否在某个维度打破了"TapTap制造不可能做到X"的认知 | "打破了轻量开发工具在3D赛车品类的上限认知" |
| **② 品类拓展** | 是否拓展了平台品类版图——做了别人没做过的类型 | "拓展了平台横版RPG品类""填补了驾驶赛车这一品类空白" |
| **③ 完成度** | 内容量、打磨程度、系统完备性 | "游戏完成度相当之高""7个可用角色、12种武器" |
| **④ 技术力** | 美术、音效、操作手感、性能表现 | "3D场景搭建能力、光照与氛围渲染""手感相当之好" |
| **⑤ 设计深度** | 数值设计、系统交互、玩家选择的有意义程度 | "3492轮对话、不同路线不同结局""策略性和数据驱动的优化" |

**核心洞察**：编辑不说"这个游戏好玩"，而是说"这个游戏**证明了**XXX"。自荐帖要翻译成这种语言。

### 1.3 「边界突破」标签分类法

编辑使用的突破标签有固定模式。从7期周报中提取的标签体系：

**品类突破类**（最高频）：
- "XX品类突破" — 做了平台上没有或很少的品类
- "XX品类拓展" — 在已有品类基础上做了显著延伸
- "XX品类填补" — 填补了平台某个品类空白

**能力突破类**：
- "技术力突破" — 展示了平台开发工具的技术上限
- "叙事表达突破" — 故事/对话/分支系统的深度
- "美术表现力突破" — 视觉品质超越平台平均水平

**体验突破类**：
- "沉浸感突破" — 完整的世界观+高完成度+沉浸式UI
- "操作手感突破" — 操控反馈超越同类产品
- "内容量突破" — 内容丰富度远超预期

**你的自荐帖必须暗示编辑：你的作品该贴哪个标签。**

### 1.4 CEO的新分发政策——自荐帖=入场券

TapTap制造负责人的公开声明重点：

1. **发帖即视为申请**——论坛自荐帖现在是编辑初筛的正式入口
2. **取消"发布即进首页冷启动池"**——不再自动获得首页曝光，必须经编辑推荐
3. **编辑主动巡视+用户自荐**——双通道，但自荐帖让你从被动等待变为主动出击
4. **鼓励半成品交流**——"即使还未做成完整版本，也鼓励早点来交流"

**底线**：

> "可以借鉴别人的玩法、美术、叙事结构——但不能'整体照搬'、'逻辑照搬'或'简单翻皮'。"

## 第二章：定位包装术——"把臭的说成香的"

> **核心理念**：每个作品都有它的"突破角度"，问题不是作品好不好，而是你有没有找到那个角度。
>
> 不这样做的后果：**作品差 → 宣传差 → 没积分 → 没曝光 → 作品更差**——斩杀线循环。

### 2.1 定位的本质：选择参照系

同一个作品，换一个参照系，就是完全不同的故事：

| 你的实际情况 | ❌ 自杀式表述 | ✅ 突破式表述 |
|-------------|-------------|-------------|
| 抄了 Monkey Mart | "参考了Monkey Mart的玩法" | "在放置经营品类基础上，融入了XX机制/XX题材，拓展了平台经营模拟的可能性" |
| 画面简陋 | "美术是我一个人做的，比较粗糙" | "采用极简/像素/Low-poly风格，将开发精力聚焦在XX核心体验上" |
| 内容量少 | "目前只有3关" | "核心关卡精心打磨，单关可重复游玩性高；后续将持续更新" |
| 是个缝合怪 | "融合了A+B+C的玩法" | "探索了A×B的品类交叉空间，这是平台上第一个将A的XX和B的XX结合的尝试" |
| 没有原创美术 | "用了AI生成的美术" | "聚焦设计创新，以XX独特机制为核心卖点。美术风格统一，服务于XX主题氛围" |
| Bug还比较多 | "正在修Bug" | "核心体验已完整可玩，持续迭代优化中。我们更重视玩法验证，欢迎社区反馈" |
| 就是个练手作 | "这是我第一个游戏" | "作为开发者的首部作品，在XX方面做了大胆尝试，证明了XX的可行性" |

### 2.2 突破角度挖掘法（5步）

**任何作品都能找到突破角度，关键是系统性地搜索。**

```
步骤1: 列出你做了什么（客观事实清单）
步骤2: 对每个事实，问"在TapTap制造平台上，这算不算少见？"
步骤3: 对"少见"的点，问"编辑能给它贴什么突破标签？"
步骤4: 选最有说服力的1-2个角度作为主打
步骤5: 围绕这个角度重组叙事
```

**实战示例：纯抄 Monkey Mart 的情况**

```
步骤1 事实清单:
- 放置经营玩法
- 用了中国食材主题
- 加了一个食材搭配系统
- 有排行榜
- 画面是像素风

步骤2 少见检查:
- 放置经营 → 平台上不少 → ❌
- 中国食材主题 → 做美食的很多但中式食材百科型的少 → ⚠️
- 食材搭配系统 → 这是自己加的原创机制 → ✅
- 排行榜 → 平台标配 → ❌
- 像素风 → 常见但如果和中式主题结合有独特感 → ⚠️

步骤3 标签匹配:
- "食材搭配系统" → 可以打"玩法创新"标签
- "中式食材×像素风" → 可以打"美术差异化"标签
- 整体 → "经营模拟品类拓展"

步骤4 主打角度:
→ "在经典放置经营的框架下，加入了原创的食材搭配系统——
   这不只是摆摊卖货，而是一本可以玩的中式食谱百科"

步骤5 重组叙事:
→ 全文围绕"可玩的食谱百科"这个核心身份展开，
   Monkey Mart 只是品类参照，不是你的身份
```

### 2.3 竞品借鉴的正确表述法

编辑明确说过："可以借鉴别人的玩法、美术、叙事结构"，底线是"不能整体照搬"。

**表述策略**：不否认借鉴，但**把借鉴转化为"站在巨人肩膀上的创新"**：

| 策略 | 公式 | 示例 |
|------|------|------|
| **致敬+超越** | "致敬了A的XX，在此基础上加入了YY" | "致敬了Monkey Mart的经营核心循环，在此基础上加入了食材搭配和图鉴收集系统" |
| **品类对标** | "A×B的品类交叉" | "将Monkey Mart式的放置经营与料理模拟器的深度搭配结合" |
| **问题驱动** | "发现A的XX不够好，所以做了改进" | "喜欢放置经营但觉得同类产品缺少策略深度，于是加入了食材属性互克系统" |

**绝对红线**：
- ❌ "完整复刻了XXX" — 等于承认换皮
- ❌ "和XXX一样" — 等于说你没有灵魂
- ❌ 只字不提原作直接抄 — 编辑看得出来，还不如大方承认借鉴

### 2.4 弱项转化话术

**完成度低 → 强调验证阶段**
> "核心玩法验证版本——我们想先确认XX机制是否成立，再投入全面制作。诚邀社区玩家参与打磨。"

**画面普通 → 强调设计聚焦**
> "有限的开发资源下，我们选择将精力聚焦在XX系统的深度打磨上。视觉风格服务于XX主题，追求[统一/克制/功能性]而非华丽。"

**玩法简单 → 强调纯粹体验**
> "做减法的设计理念——去掉所有不服务于核心体验的系统，让玩家专注于XX本身的乐趣。"

**个人作品 → 强调独立精神**
> "一人开发的独立作品，从设计到实现都体现了对XX的个人理解。虽然体量有限，但每个细节都是有意为之。"

### 2.5 定位包装的底线

**可以做**：
- 选择最有利的角度呈现事实
- 用编辑的语言重新表述你的特点
- 把"缺点"重构为"设计选择"
- 对标竞品时突出差异化

**不能做**：
- 虚构不存在的功能
- 捏造评价或数据
- 隐瞒致命缺陷（比如根本不能运行）
- 把100%照搬说成"原创"

**原则**：包装是选择最佳角度展示真实的自己，不是捏造另一个自己。

## 第三章：编辑推荐文风——像编辑一样说话

### 3.1 编辑推荐语的语言模式

从7期周报中提取的编辑实际用语模式：

**模式A：突破性声明（开场常用）**
```
"这是一款在XX维度实现了突破的作品"
"[游戏名]打破了XX的上限认知"  
"填补了平台在XX品类的空白"
```

**模式B：能力证明（中段常用）**
```
"证明了[平台/工具]在XX方面的可能性"
"展示了出色的XX能力"
"在XX方面做到了XX水准"
```

**模式C：量化背书（数据段常用）**
```
"XX种角色、XX种武器、XX个关卡，内容量远超预期"
"3492轮对话、不同路线不同结局，叙事深度令人印象深刻"
"光照、粒子效果、3D场景搭建——技术完成度相当之高"
```

**模式D：竞品对标（定位常用）**
```
"完美复刻了《XX》的核心体验"
"在《XX》的基础上融入了YY元素"
"达到了同品类头部产品的水准"
```

### 3.2 你的帖子该怎么借用这种语言

**不是让你模仿编辑的语气写帖子**（那样读起来很假），而是：

1. **在灵魂声明段暗埋突破标签**——让编辑读完后自然联想到他可以给你贴什么标签
2. **用编辑的评价维度来组织记忆点**——不是"我觉得好的地方"，而是"在突破性/品类拓展/技术力/完成度上的亮点"
3. **提供可被引用的金句**——编辑写周报时需要给每个游戏写2-4句评语，如果你的帖子里有一句话恰好可以直接拿来用，你的入选率会更高

**提供"编辑可引用句"的技巧**：

```
❌ 你写："我的游戏画面还不错"
✅ 你写："在光照和场景氛围上做了重点打磨——城堡火把的暖光和走廊深处的冷色调形成对比，
         每个场景都有自己的色彩叙事。"

编辑可能引用为："光照与氛围渲染能力突出，场景色彩叙事有想法"
```

### 3.3 关键文风差异对比

| 维度 | ❌ 旧指南风格（茶馆聊天） | ✅ 新风格（提案书） |
|------|--------------------------|---------------------|
| 定位 | "我做了一个XX游戏" | "这是一个在XX维度做了突破尝试的作品" |
| 特色 | "有一个很有趣的系统" | "在XX方面实现了XX水准的表现" |
| 动机 | "我自己想玩所以做了" | "发现平台缺少XX类型的体验，于是做了这个尝试" |
| 数据 | "有很多内容" | "XX种角色、XX种武器、XX小时流程——内容完成度XX" |
| 弱项 | "这个做得不太好" | （不提弱项，或转化为设计选择） |
| 收尾 | "希望大家喜欢" | "期待社区反馈，共同打磨这个XX品类的新尝试" |

## 第四章：自荐帖结构框架（编辑推荐适配版）

### 4.1 黄金结构（7段式·v2）

```
+-----------------------------------------------------+
| 段1 钩子+身份声明                        [必要]      |
|    > 3秒抓注意力 + 1句话说清"这是什么"               |
|    > 暗含突破标签                                    |
|                                                      |
| 段2 灵魂声明 / 突破宣言                  [必要]      |
|    > 为什么做 + 这个作品的突破点在哪                  |
|    > ★ 编辑判断"有没有灵魂"的依据                    |
|                                                      |
| 段3 核心体验（场景化）                   [必要]      |
|    > 让人"看见自己在玩"                              |
|    > 用30秒循环→5分钟循环→单局弧描述                  |
|                                                      |
| 段4 突破亮点（2-4个，编辑维度对齐）      [必要]      |
|    > 每个亮点对应一个编辑评价维度                     |
|    > 提供"可被编辑引用"的金句                        |
|                                                      |
| 段5 完成度证据（克制列举）               [推荐]      |
|    > 数字+场景锚点                                   |
|    > 证明"不只是demo，是完整作品"                    |
|                                                      |
| 段6 社会证明 / 迭代承诺                  [可选]      |
|    > 已有评价 或 更新计划                            |
|    > 开发中作品用"迭代路线图"替代                    |
|                                                      |
| 段7 收束（行动号召 + 链接）              [必要]      |
|    > 情感共鸣 + 具体链接/试玩入口                    |
|    > "TapTap搜索XXX即可游玩"                        |
+-----------------------------------------------------+
```

### 4.2 各段详细写法

#### 段1：钩子+身份声明

**目标**：3秒抓住注意力，同时植入"突破标签"。

**公式**：`场景/矛盾/反差` + `一句话核心身份` + `(暗示突破方向)`

**示例**：

```
[场景钩子]
"城堡的火把噼啪作响，走廊尽头的影子在移动——
 你手里没有剑，只有一张人格测试卷。"

[身份声明]
一款将MBTI人格系统与塔防策略深度融合的对战游戏。

→ 编辑读到"MBTI×塔防"自然会想到"品类交叉突破"
```

```
[矛盾钩子]
"所有赛车游戏都需要3D引擎——真的吗？"

[身份声明]
在纯2D画面中实现了完整赛车物理的竞速游戏。

→ 编辑读到"纯2D实现赛车"自然会想到"技术力突破"
```

**三种有效钩子**：

| 类型 | 模式 | 暗示的突破标签 |
|------|------|---------------|
| 场景沉浸 | 具体细节+情绪张力 | 沉浸感/叙事突破 |
| 认知反差 | "XX真的需要YY吗？" | 品类/技术突破 |
| 身份定义 | "第一个将A和B结合的游戏" | 品类交叉突破 |

**禁忌**：
- 用品类名开头："这是一款休闲放置游戏" → 零信息量
- 用自贬开头："虽然做得不太好但是" → 直接劝退编辑
- 用技术参数开头："采用UrhoX引擎开发" → 编辑不关心

#### 段2：灵魂声明/突破宣言

**目标**：30秒内让编辑判断"这个作品有灵魂"，并明确你的突破方向。

**v2升级**：不只是"为什么做"（动机），还要说"**做出了什么突破**"（价值主张）。

**五种灵魂声明模式**：

| 模式 | 模板 | 适用场景 |
|------|------|---------|
| **空白填充** | "平台/市场上缺少XX类型的体验，我来填补这个空白" | 做了平台没有的品类 |
| **痛点驱动** | "XX类游戏的XX问题一直没人解决，我的方案是YY" | 改进了现有品类 |
| **交叉创新** | "我想看看A和B结合会产生什么化学反应" | 品类交叉 |
| **极致追求** | "在XX这个方向上，我想做到极致" | 技术力/美术力突破 |
| **文化表达** | "我想用游戏这个载体来表达XX" | 叙事/题材突破 |

**双层结构（动机+价值）**：

```
[动机层] 为什么做
"我发现放置经营游戏很少有真正有策略深度的——大部分就是点点点等等等。"

[价值层] 做出了什么
"于是我在经典放置框架中加入了食材属性互克系统，让每次摆摊都是一道策略题。
 这可能是平台上第一个让放置经营有'构筑乐趣'的尝试。"

→ 动机真诚 + 突破明确 + 暗示标签"品类拓展"
```

#### 段3：核心体验（场景化）

**目标**：让编辑和读者在脑中"看见自己在玩"。

**写法**：描述一次典型游玩过程，用**动词**而非**名词**。

```
❌ 名词堆砌："游戏包含食材系统、搭配系统、摊位管理系统、图鉴收集系统"
✅ 动词驱动："你先从菜市场挑选当日食材——注意看属性标签，辣味食材搭配
   酸味会触发'开胃组合'加成。摆好摊位后客人自动上门，但高级客人只认
   精品搭配。一天结束，你拿着营业额决定：继续升级摊位，还是解锁新菜谱？"
```

**核心循环描述的三个层级**：

| 层级 | 描述内容 | 字数 |
|------|---------|------|
| 即时反馈（30秒） | 核心操作的手感/即时反馈 | 1-2句 |
| 短循环（5分钟） | "再来一局"的动力 | 2-3句 |
| 长弧线（单局/赛季） | 成长感和目标感 | 1-2句 |

#### 段4：突破亮点（对齐编辑维度）

**目标**：给编辑2-4个能写入周报的亮点。**每个亮点对应一个编辑评价维度。**

**亮点写作的三段式**：

```
[亮点标题] ← 用信息密度高的短句
[场景描写] ← 让人体验到这个亮点
[编辑可引用句] ← ★ 一句话总结，编辑可以直接用

示例：
### 食材互克——放置经营里的"构筑"乐趣
辣×酸=开胃、甜×苦=回甘、鲜×咸=极味——不是简单的1+1，
每种搭配都会触发独特的客人反应和营业额加成。
你不是在摆摊，你是在解构一本活的食谱。

→ 编辑可引用："食材互克系统为放置经营注入了构筑乐趣，
   每一次搭配都是有意义的策略选择"
```

**亮点与编辑维度的对齐**：

| 编辑维度 | 你的亮点要回答 | 举例 |
|---------|---------------|------|
| 突破性 | "平台上有没有人做过类似的？" | "首次将XX和YY结合" |
| 品类拓展 | "这丰富了平台什么品类？" | "为平台的经营类增加了策略深度" |
| 技术力 | "技术上有什么值得一提的？" | "在2D框架下实现了物理级赛车手感" |
| 完成度 | "内容量和打磨程度如何？" | "7个角色×12种武器×5个章节" |
| 设计深度 | "玩家的选择有意义吗？" | "每个搭配影响后续3轮的客流" |

#### 段5：完成度证据

**目标**：用数字和事实证明这不是一个PPT作品。

**两种格式**：

**A. 标签流**（适合系统丰富的游戏）
```
【角色】7个可用 · 各有3条天赋线 | 【武器】12种 · 4大类
【关卡】24个主线 + 8个隐藏 | 【剧情】3个结局 · 5h+流程
```

**B. 对比式**（适合需要强调超预期的游戏）
```
用TapTap制造开发——但实现了：
完整的3D物理碰撞 | 动态光影 | 7段式BGM系统 | 60FPS稳定运行
```

**注意**：数字要精确（"39种"而非"很多种"），但不要让数字淹没故事。

#### 段6：社会证明/迭代承诺

**如果已发布**：
- 玩家评价引用（真实评论 > 评分数字）
- 社区讨论截图
- 数据表现（"发布X天，评分X分"）

**如果开发中**：
- 迭代路线图（近期具体 > 远期模糊）
- "目前版本包含XX，下一版本将加入YY"
- 当前可体验的核心内容范围

**CEO原话**："即使还未做成完整版本，也鼓励早点来交流。"——半成品也能发帖，但要坦诚说明状态。

#### 段7：收束

**三要素**：情感共鸣 + 游戏链接 + 状态说明

```
[情感收束 - 回扣开头]
"城堡走廊尽头的影子——那是你的第一个敌人，也是你的人格镜像。"

[行动引导]
TapTap搜索「SBTI大战MBTI」即可游玩，无需下载。

[状态说明]
当前版本：v0.9 · 核心战斗+人格系统+排行榜已完整
持续更新中 · 欢迎社区反馈
```

## 第五章：截图与视觉素材

### 5.1 截图的核心作用

在编辑视角中，截图是**完成度和技术力的直接证据**。

| 截图类型 | 证明什么 | 建议数量 |
|---------|---------|---------|
| **核心玩法截图** | 让编辑1秒看懂品类 | 1-2张 |
| **UI/界面截图** | 完成度+美术力 | 1-2张 |
| **特色系统截图** | 配合突破亮点 | 每个亮点1张 |
| **对比截图** | 与竞品/同类的差异 | 0-1张 |
| **评价截图** | 社会证明 | 0-2张 |

### 5.2 截图放置策略

```
段1 钩子+身份    → 可放1张最有冲击力的全景截图
段2 灵魂声明     → 不放或放1张氛围图
段3 核心体验     → 放1-2张核心玩法截图     ★
段4 突破亮点     → 每个亮点配1张           ★
段5 完成度证据   → 可放1张系统汇总截图
段6 社会证明     → 评价截图
段7 收束         → 不放截图（文字收束更有力）
```

### 5.3 截图质量标准

- 分辨率清晰、不模糊
- 展示实际游戏画面（非概念图/草稿）
- GIF/短视频 > 静态截图（展示动态效果）
- 有UI文字的截图确保文字可读
- **截图是你最好的状态**——选最能展示技术力和完成度的画面

### 5.4 素材清单

写帖前准备：

| 素材 | 用途 | 数量 |
|------|------|------|
| 核心玩法截图 | 段3配图 | 2-3张 |
| 特色系统截图 | 段4配图 | 2-4张 |
| 游戏ICON | 帖子头图 | 1张 |
| 评价截图（如有） | 段6配图 | 1-2张 |
| GIF/短视频 | 动态展示 | 0-2个 |
| 游戏链接/试玩链接 | 段7 | **必须1个** |

## 第六章：写作风格指南

### 6.1 调性光谱

```
旧指南推荐          本指南推荐              避免
"茶馆聊天"    →    "提案书"           →  "广告文案"
真诚但无章法        真诚+有策略+有结构      空洞+夸张+无实质
```

**最佳语感**：像一个创业者在路演——
- 对自己的产品有信仰
- 知道评委关心什么
- 能用精炼的语言传达价值
- 弱点不回避但也不放大

### 6.2 人称与视角

- **钩子段**：第二人称"你"或无人称场景 → 代入感
- **灵魂声明段**：第一人称"我/我们" → 真诚感
- **体验描述段**：第二人称"你" → 让读者代入
- **亮点段**：混合使用 → 灵活
- **数据段**：无人称客观陈述 → 专业感

### 6.3 篇幅控制

| 游戏体量 | 建议字数 | 说明 |
|---------|---------|------|
| 小型休闲 | 800-1200字 | 精炼，每句话都有信息量 |
| 中型独立 | 1200-2000字 | 标准篇幅，该有的都有 |
| 大型复杂 | 2000-2500字 | 上限，超过必然有废话 |

**超过2500字大概率信息过载。** 如果删不下来，说明你在堆功能而非讲故事。

### 6.4 标题写法

**公式**：`[自荐]` + `核心突破点/身份` + `——《游戏名》`

**好标题**：
- "[自荐] 在TapTap制造上做了一款3D赛车——《XXX》"（暗示技术突破）
- "[自荐] 当MBTI人格测试变成塔防——《SBTI大战MBTI》"（暗示品类交叉）
- "[自荐] 一个让放置经营有策略深度的尝试——《XXX》"（暗示品类拓展）

**坏标题**：
- "[自荐] 我做的游戏"（零信息量）
- "[自荐] 超好玩的全新RPG"（空洞夸张）
- "[自荐] 史上最强塔防游戏"（标题党）

## 第七章：AI 写作工作流

### 7.1 写作前——信息采集+定位

**阶段A：事实采集**

| 必须了解 | 来源 | 用途 |
|---------|------|------|
| 游戏一句话定义 | 用户/GDD | 钩子+身份声明 |
| 为什么做这个游戏 | 用户口述 | 灵魂声明 |
| 核心循环 | GDD/代码 | 核心体验段 |
| 2-4个最独特的设计点 | GDD/用户选择 | 突破亮点段 |
| 游戏当前状态 | 用户确认 | 措辞基调 |
| 竞品参考 | 用户/市场 | 定位策略 |
| 游戏链接 | 用户提供 | **必须** |
| 截图素材 | 用户提供 | 配图 |

**阶段B：定位决策**（★ 新增关键步骤）

```
1. 运行"突破角度挖掘法"（第二章2.2节）
2. 确定主打突破标签（品类突破/技术力/叙事/完成度...）
3. 确定竞品对标策略（致敬+超越 / 品类交叉 / 问题驱动）
4. 识别弱项，准备转化话术
5. 向用户确认定位方向

输出物：一段话的"定位共识"
例："主打'品类交叉突破'——MBTI人格系统×塔防策略的融合，
    平台上第一个将心理测试和战斗系统结合的游戏。
    弱项（美术粗糙）转化为'低多边形风格的设计选择'。"
```

### 7.2 写作中——逐段推进

按7段结构逐一推进。**每段产出后给用户确认**。

**各段的质量自检**：

| 段落 | 核心检查项 |
|------|-----------|
| 段1钩子 | 3秒内能不能让人停下滑动？有没有暗示突破标签？ |
| 段2灵魂 | 编辑读完能不能说出"这个作品的突破在XX"？ |
| 段3体验 | 读者能不能"看见自己在玩"？用的是动词还是名词？ |
| 段4亮点 | 每个亮点有没有"编辑可引用句"？ |
| 段5完成度 | 数字精确吗？有场景锚点吗？ |
| 段6证明 | 是真实的还是编造的？ |
| 段7收束 | 有没有链接？有没有情感闭合？ |

### 7.3 写作后——终稿检查

**编辑推荐适配检查（8项）**：

```
[ ] 1. 突破标签：编辑能给你贴什么标签？帖子暗示了吗？
[ ] 2. 可引用句：有没有编辑能直接拿来写周报的金句？
[ ] 3. 灵魂判断：30秒内能判断"有灵魂"吗？
[ ] 4. 体验代入：能想象自己在玩吗？
[ ] 5. 完成度证据：有数字支撑吗？
[ ] 6. 视觉证据：有截图吗？截图质量如何？
[ ] 7. 弱项处理：弱项被转化为设计选择了吗？（而非暴露或回避）
[ ] 8. 行动转化：有链接、有明确下一步吗？
```

**基础检查（5项）**：
```
[ ] 篇幅800-2500字
[ ] 标题有信息量
[ ] 不含编辑红线内容（商业目的/换皮暗示/刷量/虚构数据）
[ ] 读感像人在说话，不像AI生成
[ ] 竞品引用是"致敬+超越"而非"和XX一样"
```

### 7.4 常见场景处理

**Q: 游戏是高度借鉴某个游戏的，怎么办？**
A: 用第二章2.3节的竞品表述法。不否认借鉴，但围绕"你的差异化"组织叙事。关键是找到你的"10%原创"并把它放到聚光灯下。

**Q: 游戏还没做完，很粗糙，能发帖吗？**
A: 能。CEO明确说"鼓励早期交流"。用"验证阶段"的定位：强调核心玩法已可体验，诚邀社区参与打磨。把"未完成"变成"开放式开发"的叙事。

**Q: 游戏是GameJam作品/练手项目，怎么写？**
A: 两个角度——(1) 时间约束本身就是亮点（"48小时完成"），(2) 聚焦创意和灵魂而非完成度。编辑在找的是"灵魂"，不是"完美"。

**Q: 游戏品类在平台上很多人做了，没有品类稀缺性怎么办？**
A: 从"品类突破"转向其他维度——技术力（"同品类中做到了XX"）、设计深度（"在经典XX玩法上加入了有意义的YY选择"）、完成度（"内容量远超同类"）。

**Q: 用了AI生成的美术/音乐，要说吗？**
A: 不需要主动强调，但如果被问到不要隐瞒。帖子焦点放在设计和创意上——"AI是工具，灵魂是人给的"。如果AI生成的内容风格统一、服务于主题，这本身就是设计能力的体现。

**Q: 游戏评分不高或者评价两极分化怎么办？**
A: 选择性引用正面评价，或不放社会证明段（完全可以跳过）。如果差评里有建设性反馈，可以转化为"基于社区反馈持续迭代"的叙事。

## 第八章：品质维度与自荐帖对应

自荐帖本质上是在向编辑展示你的游戏在关键品质维度上的表现。

| 品质维度 | 对应段落 | 怎么展示 | 编辑评价维度 |
|---------|---------|---------|-------------|
| 核心玩法趣味性 | 段3+段4 | 场景描写传达 | 设计深度 |
| 操作手感 | 段4亮点 | 具体操作反馈描述 | 技术力 |
| 游戏节奏与心流 | 段3体验 | "再来一局"的冲动来源 | 设计深度 |
| 数值平衡 | 段4亮点 | 有意义的选择和后果 | 设计深度 |
| UI/UX设计 | 截图 | 让画面说话 | 技术力/完成度 |
| 视觉风格 | 截图+段4 | 风格描述+截图证据 | 技术力 |
| 音频体验 | 段4亮点 | 如果是卖点就展示 | 技术力 |
| 内容丰富度 | 段5完成度 | 克制用数字 | 完成度 |
| 叙事与世界观 | 段2灵魂+段4 | 用故事展示 | 突破性/设计深度 |
| 社交与分享性 | 段5中简短提及 | 如果有排行榜/对战 | 品类拓展 |
| 商业化与留存 | **不在帖子中提及** | 编辑反感内购导向 | ❌ |

**记住**：挑你最强的3-5个维度深入展示，不要试图覆盖全部。

## 第九章：反面教材与红线

### 9.1 编辑雷区（触碰即出局）

| 红线 | 为什么致命 | 案例 |
|------|-----------|------|
| 自荐帖中暗示商业目的 | 直接证明"没有灵魂" | "本意是为了内购" |
| 承认整体照搬 | CEO原话"不能整体照搬" | "完整复刻了XX的全部玩法" |
| 标题党/过度夸张 | 破坏信任 | "史上最强XXX游戏" |
| 虚构评价/数据 | 一旦发现永久失信 | 编造用户评论 |
| 求赞/求投票 | 破坏社区生态 | "帮忙点赞投票" |
| 贬低其他开发者 | 社区氛围破坏 | "比XX做得好多了" |

### 9.2 常见减分点

| 减分点 | 改进方向 |
|--------|---------|
| 功能列举超过40%篇幅 | 场景描写替代 |
| 没有截图 | 至少3张 |
| 超过2500字 | 砍掉非核心内容 |
| 没有游戏链接 | 必须加 |
| 读感像AI生成的 | 加入个性口语化表达 |
| 只有"我有XX"没有"你会体验到XX" | 切换到第二人称 |
| 通篇"我觉得很好" | 用数据和场景替代主观评价 |
| 弱项直白暴露 | 转化为设计选择 |
